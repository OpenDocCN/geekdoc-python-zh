##### 什么是协程

协程是一种可以被挂起和恢复的函数。

它通常被定义为一种广义的子程序。

子程序是一种可以从一个点开始执行并在另一个点结束的函数。而协程可以在执行后被挂起，并在最终终止前多次恢复。

具体来说，协程可以控制何时挂起其执行。

这可能涉及使用特定的表达式，例如await表达式，或者像生成器中的yield表达式。

协程可能因多种原因挂起，例如执行另一个协程（例如等待另一个任务），或等待某些外部资源，如套接字连接或进程返回数据。

协程用于并发。

可以同时创建和执行多个协程。它们控制何时挂起和恢复，从而允许它们协作决定何时执行并发任务。

这被称为协作式多任务处理，与线程通常使用的抢占式多任务处理不同。

抢占式多任务处理涉及操作系统选择挂起和恢复哪些线程以及何时执行，而协作式多任务处理则由任务本身决定。

现在我们对协程有了一些概念，让我们通过将它们与其他熟悉的编程结构进行比较来加深理解。

##### 协程与例程和子程序

在现代编程中，*例程*和*子程序*通常指同一事物。

或许更准确地说，例程是一个程序，而子程序是程序中的一个函数。

例程包含子程序。

它是一个离散的表达式模块，被赋予一个名称，可以接受参数并可能返回值。

- **子程序：** 一个可以根据需要执行的指令模块，通常有名称，可以接受参数并返回值。也称为函数。

子程序被执行，运行表达式，并以某种方式返回。通常，子程序由另一个子程序调用。

协程是子程序的扩展。这意味着子程序是协程的一种特殊类型。

协程在许多方面类似于子程序，例如：

- 它们都是离散的命名表达式模块。
- 它们都可以接受参数，或者不接受。
- 它们都可以返回值，或者不返回。

主要区别在于协程可以选择在返回和退出之前多次挂起和恢复其执行。

协程和子程序都可以调用其他实例。子程序可以调用其他子程序。协程执行其他协程。然而，协程也可以执行其他子程序。

当协程执行另一个协程时，它必须挂起其执行并允许另一个协程运行。

这使得协程调用另一个协程比子程序调用另一个子程序更强大。这是协程促进的协作式多任务处理的核心。

接下来，让我们看看协程与生成器的关系。

##### 协程与生成器

生成器是一种可以挂起其执行的特殊函数。

生成器函数可以像普通函数一样定义，尽管它在挂起执行并返回值的点使用yield表达式。

生成器函数将返回一个生成器迭代器对象，该对象可以被遍历，例如通过for循环。每次执行生成器时，它从上次挂起的点运行到下一个yield表达式。

协程可以使用await表达式挂起或让出给另一个协程。一旦被等待的协程完成，它将从该点恢复。

我们可以将生成器视为一种特殊类型的协程和用于循环的协作式多任务处理。

在协程开发之前，生成器被扩展，以便可以在我们的程序中像协程一样使用。

这需要大量关于生成器的技术知识和自定义任务调度器的开发。

这通过生成器的更改和yield from表达式的引入成为可能。

这些后来被弃用，转而使用现代的async/await表达式。

接下来，让我们看看协程与线程的不同之处。

##### 协程与线程

线程是指计算机程序中的执行线程。

每个程序都是一个进程，并且至少有一个线程执行该进程的指令。

当我们运行脚本时，它会启动一个解释器实例，在主线程中运行我们的代码。主线程是进程的默认线程。

底层操作系统控制如何创建新线程、何时执行线程以及哪个CPU核心执行它们。

协程比线程更轻量级。

协程被定义为一个函数，而线程是由底层操作系统创建和管理的对象，表示为threading.Thread对象。

- **线程：** 由操作系统管理的并发单元，由对象表示。属于一个进程，可以执行多个协程。

这意味着协程通常创建和开始执行更快，并且占用更少的内存。相反，线程比协程创建和开始执行更慢，并且占用更多内存。

协程在一个线程内执行，因此单个线程可以执行多个协程。

现在我们已经熟悉了协程，让我们看看异步编程。

#### 什么是异步编程

在本节中，我们将探讨异步编程及其与异步 I/O 的关系。

在深入异步编程之前，让我们先理解 *异步* 的含义。

##### 什么是异步

异步意味着不是同时发生，与同步或同时发生相对。

在编程中，异步意味着操作被请求，但在请求时并未执行。它会在稍后执行。

例如，我们可以进行一次异步函数调用。

这将发出函数调用的请求，并且不会等待调用完成。我们可以选择稍后检查函数调用的状态或结果。

- **异步函数调用**：请求在某个时间以某种方式调用一个函数，允许调用者恢复并执行其他活动。

函数调用将在后台以某种方式在某个时间发生，程序可以执行其他任务或响应其他事件。

这一点很关键。我们无法控制请求如何或何时被处理，只希望它在程序执行其他任务时被处理。

发出异步函数调用通常会产生一个请求句柄，调用者可以使用它来检查调用状态或获取结果。这通常被称为 future。

- **Future**：异步函数调用的句柄，允许检查调用状态并检索结果。

异步函数调用和 future 的组合通常被称为异步任务。这是因为它比函数调用更复杂，例如允许取消请求等。

- **异步任务**：用于指代异步函数调用和产生的 future 的聚合。

发出异步任务和进行异步函数调用被称为异步编程。

- **异步编程**：使用异步技术，例如发出异步任务或函数调用。

我们现在有了考虑异步 I/O 的基础。

#### 什么是异步 I/O

异步编程主要用于非阻塞 I/O，例如从与其他进程或系统的套接字连接中读写。

非阻塞 I/O 是一种执行 I/O 的方式，其中读写操作被请求，但异步执行。调用者无需等待操作完成即可返回。

读写操作以某种方式执行（例如由底层操作系统或构建在其上的系统执行），操作的状态和/或数据由调用者稍后在可用时或调用者准备好时检索。

- **非阻塞 I/O**：通过异步请求和响应执行 I/O 操作，而不是等待操作完成。

因此，我们可以看到非阻塞 I/O 与异步编程的关系。实际上，我们通过异步编程使用非阻塞 I/O。

非阻塞 I/O 与异步编程的结合非常普遍，以至于它被简称为异步 I/O。

- **异步 I/O**：一个简写，指将异步编程与非阻塞 I/O 结合起来。

接下来，让我们考虑 Python 中的异步编程支持。

#### Python 中的异步编程

广义上，Python 中的异步编程指的是发出请求并阻塞等待它们完成。

我们可以通过多种方式在 Python 中实现异步编程。

第一个也是显而易见的例子是 asyncio 模块。

该模块直接提供了一个异步编程环境，使用 async/await 语法以及与套接字和子进程的非阻塞 I/O。

它使用在事件循环中运行的协程实现，事件循环本身在单个线程中运行。

- **Asyncio**：通过 asyncio 模块在 Python 中提供的异步编程环境。

我们将在下一节中了解更多关于此模块的内容。

更广泛地说，Python 提供了可以异步执行任务的线程和进程。

例如，一个线程可以启动第二个线程来执行函数调用并恢复其他活动。操作系统将在某个时间调度并执行第二个线程，第一个线程可能会也可能不会手动检查任务的状态。

更具体地说，Python 在 `ThreadPoolExecutor` 和 `ProcessPoolExecutor` 类中提供了基于 Executor 的线程池和进程池。

这些类使用相同的接口，并通过 `submit()` 方法支持异步任务，该方法返回一个 `Future` 对象。

`multiprocessing` 模块还使用 `Pool` 和 `ThreadPool` 类中的进程和线程提供了工作池，这些是 `ThreadPoolExecutor` 和 `ProcessPoolExecutor` 类的前身。

这些类的功能描述为工作异步执行任务。它们显式地提供了每个方法的同步（阻塞）和异步（非阻塞）版本来执行任务。

例如，可以通过 `apply()` 方法同步发出一次性函数调用，或通过 `apply_async()` 方法异步发出。

这突出了尽管我们专注于使用协程和 asyncio 模块进行异步编程，但 Python 提供了开发异步程序的替代方式。

现在我们已经熟悉了协程和异步编程，让我们最终看看 asyncio。

#### 欢迎使用 Asyncio

Python 将协程作为一等对象提供，asyncio 模块支持异步编程。

使用协程和 asyncio 模块的 Python 异步编程通常简称为 *asyncio*。

然而，至少在最初，将这些关注点分开是有帮助的，这样我们可以更好地理解它们。

##### Python 中的协程

Python 通过语言扩展直接支持协程，包括新的表达式和类型。

协程可以通过 `async def` 表达式定义。

这是定义子程序的 `def` 表达式的扩展。

它定义了一个可以创建并返回协程对象的协程。

例如：

```
# define a coroutine
async def custom_coro():
    # ...
```

使用 `async def` 表达式定义的协程被称为 *协程函数*。

调用协程函数将返回一个协程对象，它是 `coroutine` 类的实例，这是 Python 语言中提供的一种类型。

然后协程可以在其内部使用协程特定的表达式，例如 `await`、`async for` 和 `async with`。

await 表达式将挂起调用协程并调度指定的协程执行。调用者将不会恢复，直到指定的协程完成。

例如：

```
# define a coroutine
async def custom_coro():
    # await another coroutine
    await another_coro()
```

这只是 Python 中协程的一小部分，我们将在 *第 02 课：协程和任务* 中学习更多关于如何定义、创建和运行协程的内容。

接下来，让我们了解更多关于 asyncio 模块的内容。

##### asyncio 模块

Python 通过 asyncio 模块直接支持异步编程。

该模块提供实用函数和类，以协助创建和管理异步任务，以及与套接字和子进程执行非阻塞 I/O。

它还提供实用函数，使用线程池和进程池在底层模拟非阻塞任务与阻塞 I/O 和 CPU 密集型任务。

关键的是，它提供了执行协程所需的事件循环。

协程可以被定义和创建，但它们只能在事件循环中执行。执行协程的事件循环管理协程之间的协作多任务。它还负责执行回调函数和管理非阻塞网络 I/O。

在 asyncio 应用程序中，我们不需要直接与 asyncio 事件循环交互，除了启动它。然而，有一个低级 API 可以访问事件循环对象以及与之交互的方法。

启动 asyncio 事件循环的方式是通过 `asyncio.run()` 函数。

该函数接受一个协程并返回该协程的值。提供的协程可以作为基于协程的程序的入口点。我喜欢称它为*主协程*，类似于其他并发形式中的主线程和主进程。

例如：

```
...
# 启动一个 asyncio 程序
asyncio.run(main())
```

除了事件循环基础设施外，asyncio 模块还提供了许多用于使用我们异步程序的高级功能，例如：

- 用于创建和调度异步任务的实用函数。
- 用于打开和管理非阻塞 TCP 套接字连接的函数和类。
- 用于启动和管理非阻塞子进程的函数和类。
- 用于促进协程安全的同步原语。
- 用于协程间消息传递的协程安全队列。

既然我们已经了解了 Python 如何支持协程以及 asyncio 模块如何支持开发基于协程的异步编程，让我们来看一个实际的例子。

#### Asyncio Hello World 示例

我们用新编程语言编写的第一个程序是 *hello world* 程序。

asyncio 模块与大多数其他模块不同。

使用它就像用一种新的编程语言编写代码。它与*正常*编程不同。

这就是为什么许多开发者兴奋地使用它，而另一些人则害怕开始的原因。

进入 asyncio 的第一步是编写一个 hello world。第二步是理解它做了什么。

##### Asyncio Hello World

让我们为 asyncio 编写一个 hello world。

完整的示例如下所示。

输入示例并运行它，或者复制粘贴它。

这是我们在 asyncio 旅程上的第一步。

```python
# SuperFastPython.com
# asyncio 的 hello world 程序示例
import asyncio

# 定义一个协程
async def custom_coroutine():
    # 报告一条消息
    print('Hello world')

# 执行协程
asyncio.run(custom_coroutine())
```

运行该示例会报告 *hello world* 消息。

```
Hello world
```

充满疑问？很好！

现在，让我们放慢一切，理解这个示例做了什么。

#### Asyncio Hello World 详解

我们知道如何输入和运行 asyncio 的 hello world，现在让我们理解它做了什么（至少从鸟瞰的角度看）。

asyncio 的 hello world 程序应该做什么？

如果 asyncio 用于基于协程的并发，我们应该创建并运行一个协程。

如果我们正在探索 threading 模块中的基于线程的并发，我们将创建并运行一个线程。如果我们正在探索 multiprocessing 模块中的基于进程的并发，情况也是如此。

回想一下，子程序是一个函数。例如，我们可以定义一个打印 hello world 的函数，如下所示：

```python
# 自定义子程序
def custom_routine():
    # 报告一条消息
    print('Hello world')
```

协程是一个可以被挂起的函数。

我们可以像定义普通函数一样定义协程，只是它在 `def` 关键字前添加了 `async` 关键字。

例如：

```python
# 定义一个协程
async def custom_coroutine():
    # 报告一条消息
    print('Hello world')
```

到目前为止，一切顺利。

我们不能像执行子程序那样执行协程。

如果我们直接调用我们的 `custom_coroutine`，我们会得到看起来像错误的警告消息。

例如：

```python
# SuperFastPython.com
# 直接调用协程的示例

# 定义一个协程
async def custom_coroutine():
    # 报告一条消息
    print('Hello world')

# 直接调用协程
custom_coroutine() # 引发警告
```

运行该示例会导致看起来很吓人的警告消息。

第一个说我们从未*等待*（awaited）我们的协程。

第二个则完全没有帮助。

```
...
RuntimeWarning: coroutine '...' was never awaited
  custom_coroutine()
RuntimeWarning: Enable tracemalloc to get the object allocation traceback
```

我们不能直接调用协程。

相反，我们必须让 asyncio 运行时（称为事件循环）为我们调用该协程。

这可以通过调用 `asyncio.run()` 模块函数来实现。

该函数将在当前线程中启动 asyncio 事件循环，并且我们一次只能在一个线程中运行一个这样的事件循环。

它接受一个协程作为参数。

不是协程的名称，就像 `threading.Thread` 或 `multiprocessing.Process` 的 `target` 参数那样，而是接受一个协程的实例。

我们可以像创建对象一样创建一个协程的实例，看起来就像我们在调用协程。

例如：

```python
...
# 执行协程
asyncio.run(custom_coroutine())
```

这将启动事件循环并执行协程并打印消息。

但是，让我们进一步分解。

如果我们正在创建一个协程的实例并将其传递给 `asyncio.run()` 模块函数来执行，为什么不先赋值呢？

例如：

```python
...
# 创建协程并将其赋值给一个变量
coro = custom_coroutine()
# 执行协程
asyncio.run(coro)
```

现在这更有道理了。

我们可以清楚地看到协程的创建，并且它被传递给 `asyncio.run()` 函数执行。

还有一件事。

如果我们不执行协程的实例，我们会得到一个警告。

这意味着如果我们创建并赋值了我们自定义协程的实例，但没有将其传递给 `asyncio.run()`，就会报告一条警告消息。

例如：

```python
# SuperFastPython.com
# 创建但不等待协程的示例

# 定义一个协程
async def custom_coroutine():
    # 报告一条消息
    print('Hello world')

# 创建协程并将其赋值给一个变量
coro = custom_coroutine() # 引发警告
```

运行该示例会创建协程，但不会对其执行任何操作。

然后运行时会将其报告为警告消息，类似于我们直接调用协程时看到的情况。

```
sys:1: RuntimeWarning: coroutine '...' was never awaited
```

所以现在我们知道我们的 hello world 示例做了什么。我们将在后面的课程中更详细地介绍。

既然我们已经看到了我们的第一个协程和 asyncio 程序，让我们看看何时应该使用 asyncio，何时不应该。

#### 何时使用 Asyncio

在项目中采用 asyncio 可能有三个主要原因，它们是：

- 需要与子进程或套接字连接进行非阻塞 I/O。
- 协程的好处大于线程和进程的好处。
- 异步编程范式是首选或必需的。

##### 需要非阻塞 I/O

Asyncio 专门设计用于与子进程和 TCP 套接字连接进行非阻塞 I/O。

如果应用程序需要大量并发的套接字或子进程任务，那么 asyncio 是一个显而易见的选择。

典型的例子包括：

- 在系统上执行和检查许多命令的结果。
- 建立和管理许多套接字连接。
- 为许多客户端套接字连接提供访问服务。

尽管这是 asyncio 模块的重点，但它并不是在项目中采用它的唯一原因。

##### 需要基于协程的并发

协程和 asyncio 为使用线程或进程进行并发提供了替代方案。

线程、进程和协程各有其优点和局限性。协程和 asyncio 模块共同为并发编程提供了一种替代方案。

通常，协程是轻量级的，它们只是一种函数类型。与由底层操作系统管理的线程和进程相比，它们使用更少的内存，并且启动、挂起和恢复的速度更快。

一个 asyncio 程序可能拥有比线程程序能拥有的并发线程或 multiprocessing 程序能拥有的并发进程多出几个数量级的并发协程。

如果一个程序可能有成千上万的并发任务，可能就需要使用 asyncio。

##### 需要异步编程

异步编程是一种新的或现代的编程类型。

它在其他编程语言（如 JavaScript）中已被证明非常流行。

协程和 asyncio 在 Python 解释器和标准库中解锁了这种能力，无需第三方库。

如果异步编程范式是首选，那么为项目采用 asyncio 可能是合适的。有时编程范式是预先选择的，使其成为事实上的要求。

使用 asyncio 的异步编程可以替代或补充过程式编程、面向对象编程或函数式编程范式。

#### 何时不使用 Asyncio

Asyncio 不应总是被使用。

Python 提供了强大且功能齐全的基于线程的并发和基于进程的并发。对于 I/O 密集型任务和 CPU 密集型任务，这些选项应分别用于 Python 中的大多数并发编程情况。

现代 `concurrent.futures` 模块中的线程池支持使用线程和进程来执行异步临时任务。这提供了一种替代方式来引入异步编程元素，而无需采用协程。

一个常见的误解是认为 `asyncio` 会使程序比线程或进程更快，或者认为协程不会遭遇竞态条件或死锁。这两种观念都是错误的。

使用并发线程下载网页或查询开放端口，与在一个线程中使用并发协程一样快。

当使用基于协程的并发时，代码必须被设计为协程安全的，就像使用线程时必须确保线程安全，使用基于进程的并发时必须确保进程安全一样。在使用并发编程时，无法回避这些概念。

另一个误解是认为 `asyncio` 使并发编程变得容易。

无论并发单元或编程范式如何，并发编程总是困难的。

协程可以使代码更易于阅读。协程的定义看起来和读起来都像函数。但是，正如我们之前所描述的，良好设计和并发编程安全性的挑战，并不会仅仅通过改变并发单元就消失。

事实上，采用异步编程可能会引入新的、不同的挑战。

### 第 02 课：协程与任务

在本课中，我们将探讨如何在 asyncio 程序中创建和使用协程与任务。这些是 asyncio 程序中的主要并发单元，可以被挂起、恢复、取消等。

- 如何查询 asyncio 任务的状态、获取结果以及检查异常。
- 如何取消 asyncio 任务并添加完成回调函数。

让我们开始吧。

#### 如何创建和运行协程

协程是一种可以被挂起和恢复的函数。

Python 为并发提供了协程。

它们主要通过两种方式提供：

- 通过语言的特定扩展，例如 `async` 和 `await` 表达式。
- 通过标准库中的特定模块，例如 `asyncio` 模块。

让我们探讨如何创建和使用协程。

#### 如何定义协程

协程可以通过 `async def` 表达式来定义。

这是对定义子程序的 `def` 表达式的扩展。

它定义了一个可以被创建并返回协程对象的协程。

例如：

```
# 定义一个协程
async def custom_coro():
    # ...
```

使用 `async def` 表达式定义的协程被称为协程函数。

协程函数可以像普通函数一样接受参数并返回值。

另一个关键区别是它可以使用特殊的协程表达式，例如 `await`、`async for` 和 `async with`。

我们将在接下来的部分中很快看到 `await` 表达式。我们将在 *第 04 课：迭代器、生成器和上下文管理器* 中学习 `async for` 和 `async with` 表达式。

接下来，让我们看看如何创建协程。

#### 如何创建协程

一旦定义了协程，就可以创建它。

这看起来像调用一个子程序。

例如：

```
...
# 创建一个协程
coro = custom_coro()
```

这并不会执行协程。

它返回一个协程对象。

协程对象有方法，例如 `send()` 和 `close()`。它是一种类型。

我们可以通过创建一个协程实例并调用内置函数 `type()` 来报告其类型来演示这一点。

例如：

```
# SuperFastPython.com
# 检查协程类型的示例

# 定义一个协程
async def custom_coro():
    # 什么都不做
    pass

# 创建协程
coro = custom_coro()
# 检查协程的类型
print(type(coro))
```

运行该示例会报告创建的协程是一个协程类。

协程对象是可等待的。

这意味着它是一种实现了 `__await__()` 方法的类型。

我们还会得到一个 `RuntimeError`，因为协程被创建但从未执行，我们将在下一节中探讨这一点。

```
<class 'coroutine'>
sys:1: RuntimeWarning: coroutine '...' was never awaited
```

接下来，让我们看看如何运行协程。

#### 如何运行协程

协程可以被定义和创建，但它们只能在事件循环中执行。

执行协程的事件循环管理着协程之间的协作式多任务。

启动协程事件循环的典型方式是通过 `asyncio.run()` 函数。

该函数接受一个协程并返回协程的值。提供的协程可以用作基于协程程序的入口点。

例如：

```
...
# 创建协程
coro = custom_coroutine()
# 启动事件循环并执行协程
asyncio.run(coro)
```

执行协程的这种方法通常在一行中完成。

例如：

```
...
# 启动事件循环并执行协程
asyncio.run(custom_coroutine())
```

这将启动 asyncio 运行时（称为事件循环）并执行协程。

这是启动 asyncio 应用程序的主要方式。

例如：

```
# SuperFastPython.com
# 运行协程的示例
import asyncio

# 主协程
async def main():
    # 报告一条消息
    print('Hello from a coroutine')

# 启动协程程序
asyncio.run(main())
```

运行该示例会创建 `main()` 协程并将协程对象传递给 `run()` 函数。

然后 `run()` 函数启动 asyncio 事件循环并调度 `main()` 协程。

`main()` 协程执行并报告一条消息。

协程终止。没有更多的协程需要执行，因此事件循环终止。

```
Hello from a coroutine
```

接下来，让我们看看如何等待一个协程。

#### 如何等待协程

运行协程的另一种方式是从一个正在运行的协程中等待它。

这需要使用 `await` 表达式。

`await` 表达式接受一个可等待对象并挂起调用者。如果需要，它会调度提供的可等待对象。只有当提供的可等待对象完成后，调用者才会恢复。

例如：

```
...
# 等待一个可等待对象
await coro
```

可等待对象是一个可以使用 `await` 表达式等待的对象。

它是一个实现了 `__await__()` 方法的对象。

协程是可等待对象，但 `asyncio.Task` 和 `asyncio.Future` 对象也是。

我们可以通过直接从另一个协程中等待它来执行一个协程。

例如：

```
# 定义一个自定义协程
def another_coroutine():
    ...
    # 创建协程
    coro = custom_coroutine()
    # 执行并等待协程完成
    await coro
```

这通常在一行中完成。

例如：

```
...
# 执行并等待协程完成
await custom_coroutine()
```

asyncio API 提供协程函数，这些函数返回可以被等待的协程对象。

一个例子是 asyncio.sleep()。这个函数接受一个以整数或浮点值表示的秒数，并返回一个可以被等待的协程。

例如：

```
...
# 执行并等待协程完成
await asyncio.sleep(1)
```

这会挂起调用者并执行一个休眠给定秒数的协程，即它也被挂起。

这是一个有用的函数，我们将经常使用它来模拟协程执行一些工作。

下面的例子执行一个协程，该协程又执行并等待 asyncio.sleep() 协程。

```
# SuperFastPython.com
# 从协程运行协程的示例
import asyncio

# 主协程
async def main():
    # 报告一条消息
    print('Hello from a coroutine')
    # 休眠片刻
    await asyncio.sleep(1)

# 启动协程程序
asyncio.run(main())
```

运行该示例会创建 main() 协程，并将协程对象传递给 `run()` 函数。

然后 `run()` 函数启动 asyncio 事件循环并调度 `main()` 协程。

`main()` 协程执行并报告一条消息。然后它创建 `sleep()` 协程并调度其执行，将其挂起直到完成。

`sleep()` 协程运行并挂起自身固定秒数。

`sleep()` 协程终止，然后 `main()` 协程恢复并终止。

没有更多的协程需要执行，因此事件循环终止。

```
Hello from a coroutine
```

我们现在知道如何在 asyncio 程序中创建和运行协程。

接下来，让我们探索 asyncio 任务。

#### 如何创建和运行任务

Asyncio 提供任务，提供了一种独立包装和执行协程的方式。

好处是任务提供了对协程的句柄，可以查询该句柄以获取结果，并提供了一种取消正在运行的协程的方法。

在本节中，我们将探讨如何在 asyncio 程序中创建和运行任务。

##### 什么是 Asyncio 任务

`asyncio.Task` 是一个调度并独立运行 asyncio 协程的对象。

它提供了一个对已调度协程的句柄，asyncio 程序可以查询该句柄并与协程交互。

任务是从协程创建的。它需要一个协程对象，包装该协程，调度其执行，并提供与之交互的方式。

任务是独立执行的。这意味着它被调度在 asyncio 事件循环中，并将执行，无论创建它的协程中发生什么其他事情。这与直接执行协程不同，后者调用者必须等待其完成。

`asyncio.Task` 类扩展了 `asyncio.Future` 类，其实例是可等待的。

`Future` 是一个更底层的类，表示一个最终会到达的结果。

扩展 `Future` 类的类通常被称为 Future 类。

因为 `Task` 是可等待的，这意味着协程可以使用 `await` 表达式等待任务完成。

例如：

```
...
# 等待任务完成
await task
```

现在我们知道什么是 asyncio 任务了，让我们看看如何创建一个。

##### 如何创建任务

任务是使用提供的协程实例创建的。

事实上，任务只能在协程内创建和调度。

可以使用 asyncio.create_task() 函数创建任务。

asyncio.create_task() 函数接受一个协程实例和一个可选的任务名称，并返回一个 asyncio.Task 实例。

例如：

```
...
# 创建一个协程
coro = task_coroutine()
# 从协程创建一个任务
task = asyncio.create_task(coro)
```

这可以通过在一行中使用复合语句来实现。

例如：

```
...
# 从协程创建一个任务
task = asyncio.create_task(task_coroutine())
```

这将做几件事：

1. 将协程包装在 Task 实例中。
2. 在事件循环中调度任务执行。
3. 返回一个 Task 实例。

任务实例可以被丢弃，通过方法交互，并由协程等待。

还有其他方法可以使用低级 API 创建任务，但这是在 asyncio 程序中从协程创建 Task 的首选方式。

接下来，让我们看看任务究竟何时运行。

##### 任务何时运行

创建任务后的一个常见问题是它何时运行？

这是一个好问题。

虽然我们可以使用 create_task() 函数将协程调度为独立运行的任务，但它可能不会立即运行。

事实上，在事件循环有机会执行任务之前，任务不会执行。

这不会发生，直到所有其他协程都不在运行，并且轮到该任务运行。

例如，如果我们有一个 asyncio 程序，其中有一个协程创建并调度了一个任务，那么调度的任务将不会运行，直到创建该任务的调用协程被挂起。

如果调用协程选择休眠、选择等待另一个协程或任务，或者选择等待新调度的任务，就会发生这种情况。

例如：

```
...
# 从协程创建一个任务
task = asyncio.create_task(task_coroutine())
# 等待任务，允许其运行
await task
```

现在我们知道什么是任务以及如何创建它，让我们看一个实际的例子。

##### 创建和运行任务的示例

我们可以探讨一个在 asyncio 程序中从协程创建 Task 的示例。

这是创建和使用任务的典型用例。

在这个例子中，我们将定义一个协程，我们将把它包装在一个任务中。然后我们将定义主协程，它将作为程序的入口点。将从我们的任务协程创建一个任务，然后等待任务完成。

完整的例子如下所示。

```
# SuperFastPython.com
# 创建和等待 asyncio 任务的示例
import asyncio

# 为任务定义一个协程
async def task_coroutine():
    # 报告一条消息
    print('executing the task')
    # 挂起片刻
    await asyncio.sleep(1)

# 自定义协程
async def main():
    # 报告一条消息
    print('main coroutine')
    # 创建并调度任务
    task = asyncio.create_task(task_coroutine())
    # 等待任务完成
    await task

# 启动 asyncio 程序
asyncio.run(main())
```

运行该示例首先创建 `main()` 协程，并将其用作 asyncio 程序的入口点。

然后 `main()` 协程报告一条消息。然后它创建 `task_coroutine()` 协程的一个实例，并将其传递给 `create_task()` 函数，以便将其包装在 Task 实例中并返回该实例。

这会在能够执行时立即调度 Task 包装的协程执行。

然后 `main()` 协程继续执行，然后挂起执行并等待任务完成。

这给了任务执行的机会。它报告一条消息，然后挂起，休眠一秒钟。

此时，`main()` 协程和 Task 都被挂起。

Task 恢复然后终止。

然后 `main()` 协程继续执行并终止，这会关闭 asyncio 程序。

```
main coroutine
executing the task
```

现在我们知道如何创建任务了，让我们看看如何与它们交互。

#### 如何使用 Asyncio 任务

`asyncio.Task` 对象提供了一个对已调度协程的句柄。

它提供了许多方法，我们可以在 asyncio 程序中使用这些方法来查询和与协程交互。

在本节中，我们将探讨一旦创建了 `asyncio.Task` 对象后如何使用它们。

##### 如何检查任务状态

任务创建后，我们可以检查其状态。

我们可以检查两种状态，它们是：

-   任务是否已完成。
-   任务是否已被取消。

让我们依次仔细看看每一种。

##### 检查任务是否完成

我们可以通过 `done()` 方法检查任务是否已完成。

如果任务已完成，该方法返回 `True`，否则返回 `False`。

例如：

```
...
# 检查任务是否已完成
if task.done():
    # ...
```

如果任务有机会运行并且现在不再运行，则该任务已完成。

已调度的任务尚未完成。同样，正在运行的任务也尚未完成。

任务在以下情况下被视为已完成：

-   协程正常完成。
-   协程显式返回。
-   协程中引发意外错误或异常。
-   任务被取消。

##### 检查任务是否已被取消

我们可以通过 `cancelled()` 方法检查任务是否已被取消。

如果任务已被取消，该方法返回 `True`，否则返回 `False`。

例如：

```
...
##### 检查任务是否已被取消
if task.cancelled():
    # ...
```

如果在任务上调用了 `cancel()` 方法并成功完成（例如 `cancel()` 返回 `True`），则该任务被视为已取消。

如果未调用 `cancel()` 方法，或者调用了 `cancel()` 方法但未能取消任务，则该任务未被取消。

接下来，让我们看看如何从任务中获取结果。

##### 如何获取任务的结果

我们可以通过简单地等待任务来获取由任务包装的协程的返回值。

例如：

```
...
# 从包装的协程获取返回值
value = await task
```

另一种获取任务结果的方法是通过 `result()` 方法。

这将返回由任务包装的协程的返回值，如果包装的协程没有显式返回值，则返回 `None`。

例如：

```
...
# 从包装的协程获取返回值
value = task.result()
```

如果协程引发未处理的错误或异常，则在调用 `result()` 方法时会重新引发该错误或异常，可能需要处理。

例如：

```
...
try:
    # 从包装的协程获取返回值
    value = task.result()
except Exception:
    # 任务失败，没有结果
```

如果任务已被取消，则在调用 `result()` 方法时会引发 `CancelledError` 异常，可能需要处理。

例如：

```
...
try:
    # 从包装的协程获取返回值
    value = task.result()
except asyncio.CancelledError:
    # 任务已被取消
```

因此，最好先检查任务是否已被取消。

例如：

```
...
# 检查任务是否未被取消
if not task.cancelled():
    # 从包装的协程获取返回值
    value = task.result()
else:
    # 任务已被取消
```

如果任务尚未完成，则在调用 `result()` 方法时会引发 `InvalidStateError` 异常，可能需要处理。

例如：

```
...
try:
    # 从包装的协程获取返回值
    value = task.result()
except asyncio.InvalidStateError:
    # 任务尚未完成
```

因此，最好先检查任务是否已完成。

例如：

```
...
# 检查任务是否未完成
if not task.done():
    await task
# 从包装的协程获取返回值
value = task.result()
```

接下来，让我们看看如何获取任务引发的未处理异常。

##### 如何获取任务的异常

由任务包装的协程可能引发未处理的异常。

这实际上会导致任务*失败*。

我们可以通过 `exception()` 方法检索由任务包装的协程中的未处理异常。

例如：

```
...
# 获取任务引发的异常
exception = task.exception()
```

如果包装的协程中未引发未处理的异常，则返回 `None` 值。

如果任务已被取消，则在调用 `exception()` 方法时会引发 `CancelledError` 异常，可能需要处理。

同样，如果任务尚未完成，则在调用 `exception()` 方法时会引发 `InvalidStateError` 异常，可能需要处理。

接下来，让我们看看如何取消正在运行的任务。

##### 如何取消任务

我们可以通过 `cancel()` 方法取消已调度的任务。

如果任务已被取消，`cancel` 方法返回 `True`，否则返回 `False`。

例如：

```
...
# 取消任务
was_cancelled = task.cancel()
```

如果任务已完成，则无法取消，`cancel()` 方法将返回 `False`，并且任务不会具有已取消的状态。

如果取消请求成功，下次任务有机会运行时，它将引发 `CancelledError` 异常。

如果包装的协程中未处理 `CancelledError` 异常，则任务将被取消。

否则，如果包装的协程中处理了 `CancelledError` 异常，则任务将不会被取消。

`cancel()` 方法还可以接受一个消息参数，该参数将用于 `CancelledError` 的内容中。

我们可以探讨如何取消任务。

在此示例中，我们将像往常一样创建并调度一个任务。然后调用者将等待片刻，让任务开始执行。

然后它将取消任务并检查取消请求是否成功。

调用者将再等待片刻让任务被取消，然后报告任务的状态以确认其已标记为已取消。

完整示例如下。

```
# SuperFastPython.com
# 取消 asyncio 任务的示例
import asyncio

# 定义任务的协程
async def task_coroutine():
    # 报告消息
    print('executing the task')
    # 暂停片刻
    await asyncio.sleep(1)

# 自定义协程
async def main():
    # 报告消息
    print('main coroutine')
    # 创建并调度任务
    task = asyncio.create_task(task_coroutine())
    # 等待片刻
    await asyncio.sleep(0.5)
    # 取消任务
    was_cancelled = task.cancel()
    print(f'>was canceled: {was_cancelled}')
    # 等待片刻
    await asyncio.sleep(0.1)
    # 报告状态
    print(f'>canceled: {task.cancelled()}')

# 启动 asyncio 程序
asyncio.run(main())
```

运行示例首先创建 `main()` 协程并将其用作 asyncio 程序的入口点。

然后 `main()` 协程报告一条消息。它接着创建 `task_coroutine()` 协程的一个实例，并将其传递给 `create_task()` 方法，以便将其包装在 `Task` 实例中并返回该实例。

然后 `main()` 协程暂停执行并暂停半秒。

这给了任务执行的机会。任务报告一条消息，然后暂停，休眠一秒。

`main()` 协程恢复并取消新任务。然后它报告取消任务的请求是否成功。成功是因为我们知道任务尚未完成。`main()` 协程然后暂停一小会儿。

这给了任务另一次执行的机会，在这种情况下，包装的协程中会引发 `CancelledError` 异常，从而取消任务。

然后 `main()` 协程恢复并检查任务的取消状态，确认它确实已完成并被取消。

```
main coroutine
executing the task
>was canceled: True
>canceled: True
```

接下来，让我们看看如何向任务添加完成回调函数。

##### 如何在任务中使用回调

我们可以通过 `add_done_callback()` 方法向任务添加完成回调函数。

此方法接受一个函数名，该函数在任务完成时被调用。

回调函数必须将 `Task` 实例作为参数。

例如：

```
# 完成回调函数
def handle(task):
    print(task)

...
# 注册完成回调函数
task.add_done_callback(handle)
```

回想一下，当包装的协程正常完成、返回、引发未处理的异常或任务被取消时，任务可能已完成。

`add_done_callback()` 方法可用于添加或注册任意数量的完成回调函数。

我们还可以通过 `remove_done_callback()` 函数移除或取消注册回调函数。

例如：

```
...
# 移除完成回调函数
task.remove_done_callback(handle)
```

我们可以探讨如何在任务上使用完成回调函数。

在此示例中，我们将定义一个完成回调函数，该函数将报告任务是否已完成。

该函数将在任务创建后注册到任务上。

完整的示例如下所示。

```python
# SuperFastPython.com
# example of adding a done callback function to a task
import asyncio

# custom done callback function
def handle(task):
    print(f'Task callback done: {task.done()}')

# define a coroutine for a task
async def task_coroutine():
    # report a message
    print('executing the task')
    # suspend for a moment
    await asyncio.sleep(1)

# custom coroutine
async def main():
    # report a message
    print('main coroutine')
    # create and schedule the task
    task = asyncio.create_task(task_coroutine())
    # add a done callback function
    task.add_done_callback(handle)
    # wait for the task to complete
    await task

# start the asyncio program
asyncio.run(main())
```

运行该示例首先会创建 `main()` 协程，并将其作为 asyncio 程序的入口点。

然后，`main()` 协程会报告一条消息。接着，它创建一个 `task_coroutine()` 协程的实例，并将其传递给 `create_task()` 方法，以便将其包装在一个 Task 实例中并返回该实例。

随后，`main()` 协程在任务上注册完成回调函数。然后它挂起执行，并等待任务完成。

这给了任务执行的机会。它报告一条消息，然后挂起，休眠一秒钟。之后它恢复执行并终止。

这会触发 asyncio 基础设施调用回调函数，并将 Task 实例传递给它。

回调函数被执行并报告一条消息，确认任务确实已被标记为完成。

```
main coroutine
executing the task
Task callback done: True
```

#### 如何将多个任务作为一组运行

`asyncio.gather()` 函数允许调用者将多个可等待对象组合在一起，并让它们并发执行。

在我们预先创建许多任务或协程，然后希望一次性执行它们并等待它们全部完成再继续的情况下，我们可能会使用 `asyncio.gather()` 函数。

这是一种常见的情况，即需要从许多相似的任务中获取结果，例如，相同任务或协程但使用不同的数据。

可等待对象可以并发执行，返回结果，主程序可以利用这些结果恢复执行。

既然我们知道了 `asyncio.gather()` 函数是什么，让我们看看如何使用它。

#### 如何使用 asyncio.gather()

`asyncio.gather()` 函数接受一个或多个可等待对象作为参数。

回想一下，可等待对象可以是协程、`Future` 或 `Task`。因此，我们可以使用多个任务、多个协程或任务和协程的混合来调用 `gather()` 函数。

例如：

```python
...
# execute multiple coroutines
asyncio.gather(coro1(), coro2())
```

如果向 `gather()` 提供了 `Task` 对象，它们将已经在运行，因为任务在创建时就被调度了。

`asyncio.gather()` 函数将可等待对象作为位置参数。

我们不能创建一个可等待对象的列表或集合并将其提供给 gather，因为这会导致错误。

例如：

```python
...
# create a list of coroutines
coros = [coro1(), coro2()]
# cannot provide a list of awaitables directly
asyncio.gather(coros) # error
```

如果首先使用星号运算符 (*)（也称为星号运算符）将可等待对象列表解包为单独的表达式，则可以提供该列表。

此运算符专门将可迭代对象（如列表）解包为单独的表达式。它通常被称为可迭代解包运算符。

例如：

```python
...
# create a list of coroutines
coros = [coro1(), coro2()]
# gather with an unpacked list of awaitables
asyncio.gather(*coros)
```

如果向 `gather()` 提供了协程，它们会自动包装在 Task 对象中。

`gather()` 函数不会直接挂起。相反，它返回一个表示可等待对象组的 `asyncio.Future` 对象。

例如：

```python
...
# get a future that represents multiple awaitables
group = asyncio.gather(coro1(), coro2())
```

一旦创建了 Future 对象，它就会在事件循环中自动调度。

这意味着，如果调用者没有执行其他操作，那么被调度的可等待对象组将会运行（假设调用者挂起）。我们可以通过 `Future` 对象与该组进行交互，就像与单个任务交互一样，例如添加完成回调函数、取消该组以及检查该组的状态。

然而，`asyncio.gather()` 最常见的用法是直接等待返回的 `Future`。

这将收集协程和任务的返回值，并将它们作为可迭代对象返回。

例如：

```
...
# 执行协程并获取返回值
values = await asyncio.gather(coro1(), coro2())
```

既然我们知道了如何使用 `asyncio.gather()`，让我们来看一个实际的例子。

#### 在列表中运行多个协程的示例

通常会预先创建多个协程，然后再将它们收集起来。

这允许程序准备要并发执行的任务，然后一次性触发它们的并发执行并等待它们完成。

我们可以手动或使用列表推导式将多个协程收集到一个列表中。

例如：

```
...
# 创建多个协程
coros = [task_coro(i) for i in range(10)]
```

然后我们可以用列表中的所有协程调用 `gather()`。

协程列表不能直接提供给 `gather()` 函数，因为这会导致错误。

相反，`gather()` 函数要求每个可等待对象作为单独的位置参数提供。

这可以通过将列表解包为单独的表达式并传递给 `gather()` 函数来实现。星号运算符 (`*`) 将为我们执行此操作。

例如：

```
...
# 运行任务
await asyncio.gather(*coros)
```

将这些内容整合在一起，使用 `gather()` 运行预先准备好的协程列表的完整示例如下所示。

```
# SuperFastPython.com
# 在列表中运行多个协程的 gather 示例
import asyncio

# 用于任务的协程
async def task_coro(value):
    # 报告一条消息
    print(f'>任务 {value} 正在执行')
    # 休眠片刻
    await asyncio.sleep(1)

# 用作入口点的协程
async def main():
    # 报告一条消息
    print('main 开始')
    # 创建多个协程
    coros = [task_coro(i) for i in range(10)]
    # 运行任务
    await asyncio.gather(*coros)
    # 报告一条消息
    print('main 完成')

# 启动 asyncio 程序
asyncio.run(main())
```

运行该示例会将 `main()` 协程作为程序的入口点执行。

然后 `main()` 协程使用列表推导式创建一个包含 10 个协程对象的列表。

然后将该列表提供给 `gather()` 函数，并使用星号运算符解包为 10 个单独的表达式。

然后 `main()` 协程等待从 `gather()` 调用返回的 `Future` 对象，挂起并等待所有已调度的协程完成执行。

协程在能够运行时立即运行，报告其唯一的消息，休眠后终止。

只有在组中的所有协程都完成后，`main()` 协程才会恢复并报告其最终消息。

这突出了我们如何准备一组协程并将它们作为单独的表达式提供给 `gather()` 函数。

```
main 开始
>任务 0 正在执行
>任务 1 正在执行
>任务 2 正在执行
>任务 3 正在执行
>任务 4 正在执行
>任务 5 正在执行
>任务 6 正在执行
>任务 7 正在执行
>任务 8 正在执行
>任务 9 正在执行
main 完成
```

接下来，让我们看看如何在 asyncio 程序中等待多个任务。

#### 如何等待多个任务

`asyncio.wait()` 函数可用于等待一组 asyncio 任务完成。

回想一下，asyncio 任务是 `asyncio.Task` 类的一个实例，它包装了一个协程。它允许协程被独立调度和执行，而 Task 实例提供了一个用于查询状态和获取结果的句柄。

`wait()` 函数允许我们等待一组任务完成。

对 `wait` 的调用可以配置为等待不同的条件，例如所有任务完成、第一个任务完成以及第一个任务因错误失败。

接下来，让我们看看如何使用 `wait()` 函数。

#### 如何使用 asyncio.wait()

`asyncio.wait()` 函数接受一组可等待对象，通常是 Task 对象。

这可以是我们创建的任务对象的 `list`、`dict` 或 `set`，例如通过在列表推导式中调用 `asyncio.create_task()` 任务函数。

例如：

```
...
# 创建多个任务
tasks = [asyncio.create_task(task(i)) for i in range(10)]
```

`asyncio.wait()` 在满足任务集合的某些条件之前不会返回。

默认情况下，条件是所有任务都已完成。

`wait()` 函数返回一个包含两个集合的元组。第一个集合包含满足条件的所有任务对象，第二个集合包含尚未满足条件的所有其他任务对象。

这些集合被称为 *已完成* 集合和 *待处理* 集合。

例如：

```
...
# 等待所有任务完成
done, pending = await asyncio.wait(tasks)
```

从技术上讲，`asyncio.wait()` 是一个返回协程的协程函数。

然后我们可以等待这个协程，它将返回集合的元组。

例如：

```
...
# 创建等待协程
wait_coro = asyncio.wait(tasks)
# 等待等待协程
t = await wait_coro
```

等待的条件可以通过 `return_when` 参数指定，默认设置为 `asyncio.ALL_COMPLETED`。

例如：

```
...
# 等待所有任务完成
done, pending = await asyncio.wait(tasks,
    return_when=asyncio.ALL_COMPLETED)
```

我们可以通过将 `return_when` 设置为 `FIRST_COMPLETED` 来等待第一个任务完成。

例如：

```
...
# 等待第一个任务完成
done, pending = await asyncio.wait(tasks,
    return_when=asyncio.FIRST_COMPLETED)
```

当第一个任务完成并返回到已完成集合中时，剩余的任务不会被取消，并继续并发执行。

我们可以通过将 `return_when` 设置为 `FIRST_EXCEPTION` 来等待第一个任务因异常失败。

例如：

```
...
# 等待第一个任务失败
done, pending = await asyncio.wait(tasks,
    return_when=asyncio.FIRST_EXCEPTION)
```

在这种情况下，已完成集合将包含第一个因异常失败的任务。如果没有任务因异常失败，已完成集合将包含所有任务，并且 `wait()` 将在所有任务完成后才返回。

我们可以通过 `timeout` 参数（以秒为单位）指定我们愿意等待给定条件多长时间。

如果在满足条件之前超时，则返回任务元组，其中包含当时满足条件的任何任务子集，例如，如果等待所有任务完成，则返回已完成的任务子集。

例如：

```
...
# 等待所有任务完成，设置超时
done, pending = await asyncio.wait(tasks, timeout=3)
```

如果在满足条件之前达到超时，则不会引发异常，剩余的任务也不会被取消。

既然我们知道了如何使用 `asyncio.wait()` 函数，让我们来看一个实际的例子。

#### 等待所有任务的示例

我们可以探索如何使用 `asyncio.wait()` 等待所有任务。

在这个例子中，我们将定义一个简单的任务协程，该协程生成一个随机值，休眠一小段时间，然后报告一条包含生成值的消息。

然后主协程将在列表推导式中使用该协程创建多个任务，然后等待所有任务完成。


任务开始执行。每个任务生成一个随机值，休眠片刻，然后报告其生成的值。

所有任务完成后，`main()` 协程恢复执行，并报告一条最终消息。

此示例重点展示了如何使用 `wait()` 函数来等待一组任务完成。

接下来，让我们看看如何等待一个协程在固定超时时间内完成。

#### 如何为任务设置超时等待

`asyncio.wait_for()` 函数允许调用者等待一个 asyncio 任务或协程在超时时间内完成。

如果未指定超时时间，`wait_for()` 函数将一直等待直到任务完成。

如果指定了超时时间，并且在任务完成前超时时间已到，则该任务将被取消。

这允许调用者既设定他们愿意等待任务完成的预期时间，又能在超时时间到达时通过取消任务来强制执行超时。

现在我们了解了 `asyncio.wait_for()` 函数是什么，接下来看看如何使用它。

#### 如何使用 asyncio.wait_for()

`asyncio.wait_for()` 函数接受一个可等待对象和一个超时时间。

可等待对象可以是协程或任务。

必须指定超时时间，可以是 `None`（表示无超时）、整数或浮点数（表示秒数）。

`wait_for()` 函数返回一个协程，该协程在被显式等待或作为任务调度之前不会执行。

例如：

```
...
# 等待任务完成
await asyncio.wait_for(coro, timeout=10)
```

如果提供的是协程，它将在 `wait_for()` 协程执行时被转换为任务。

如果在任务完成前超时时间已到，任务将被取消，并引发 `asyncio.TimeoutError` 异常，这可能需要处理。

例如：

```
...
# 执行一个带超时的任务
try:
    # 等待任务完成
    await asyncio.wait_for(coro, timeout=1)
except asyncio.TimeoutError:
    # ...
```

如果等待的任务因未处理的异常而失败，该异常将传播回等待 `wait_for()` 协程的调用者，在这种情况下可能需要处理。

例如：

```
...
# 执行一个可能失败的任务
try:
    # 等待任务完成
    await asyncio.wait_for(coro, timeout=1)
except asyncio.TimeoutError:
    # ...
except Exception:
    # ...
```

现在我们了解了 `wait_for()` 函数的工作原理，接下来看一个实际示例。

#### 等待任务超时的示例

我们可以探索如何等待一个协程，其超时时间在任务完成前就已到期。

在此示例中，我们执行一个协程，但调用者等待固定的超时时间 0.2 秒或 200 毫秒。

回想一下，一秒等于 1,000 毫秒。

任务协程被修改为休眠超过一秒，确保超时时间总是在任务完成前到期。

完整示例如下。

```
# SuperFastPython.com
# 等待协程超时的示例
from random import random
import asyncio

# 在新任务中执行的协程
async def task_coro():
    # 生成一个 1 到 2 之间的随机值
    value = 1 + random()
    # 报告消息
    print(f'>task got {value}')
    # 休眠片刻
    await asyncio.sleep(value)
    # 报告全部完成
    print('>task done')

# 主协程
async def main():
    # 创建一个任务
    task = task_coro()
    # 执行并等待任务，不设超时
    try:
        await asyncio.wait_for(task, timeout=0.2)
    except asyncio.TimeoutError:
        print('Gave up waiting, task canceled')

# 启动 asyncio 程序
asyncio.run(main())
```

运行示例首先会创建 `main()` 协程，并将其作为 asyncio 程序的入口点。

`main()` 协程创建任务协程。然后调用 `wait_for()`，传递任务协程并将超时时间设置为 0.2 秒。

`main()` 协程被挂起，`task_coro()` 开始执行。它报告一条消息并休眠片刻。

超时时间到期后，`main()` 协程恢复执行。`wait_for()` 协程取消了 `task_coro()` 协程，`main()` 协程被挂起。

`task_coro()` 再次运行并响应终止请求。它引发 `TimeoutError` 异常并终止。

`main()` 协程恢复执行并处理 `task_coro()` 引发的 `TimeoutError`。

这重点展示了如何调用 `wait_for()` 函数并设置超时时间，以及如何在任务未在超时时间内完成时取消任务。

**注意：** 由于使用了随机数，每次运行程序的结果都会有所不同。

```
>task got 1.4231068884240894
Gave up waiting, task canceled
```

接下来，让我们看看如何按完成顺序处理协程结果。

#### 如何按完成顺序处理任务

`asyncio.as_completed()` 函数将并发运行一组任务和协程。

更重要的是，它返回一个可迭代对象，我们可以用它来按完成顺序检索可等待对象。

它允许我们并发执行许多协程或任务，并在任务完成时获取结果，而不是按照我们发出任务的顺序。

现在我们了解了 `as_completed()` 是什么，接下来看看如何使用它。

#### 如何使用 asyncio.as_completed()

`asyncio.as_completed()` 函数通过一个可等待对象集合来调用。

这可以是列表、字典或集合，可以包含 `asyncio.Task` 对象、协程或其他可等待对象。

任何提供给 `as_completed()` 的协程都将被包装在 `Task` 对象中以独立执行。

它返回一个可迭代对象，遍历时将按提供列表的顺序生成可等待对象。调用者可以按顺序等待这些对象，以便按任务完成的顺序获取结果，例如从下一个完成的任务获取结果。

例如：

```
...
# 遍历可等待对象
for task in asyncio.as_completed(tasks):
    # 获取下一个结果
    result = await task
```

`as_completed()` 函数还接受一个 `timeout` 参数。

这指定了调用者愿意等待所有可等待对象完成的时间。

例如：

```
...
# 带超时遍历可等待对象
for task in asyncio.as_completed(tasks, timeout=10):
    # 获取下一个结果
    result = await task
```

如果在所有可等待对象完成前超时时间已到，将引发 `asyncio.TimeoutError` 异常，这可能需要处理。

例如，我们可以在循环内处理它：

```
...
# 带超时遍历可等待对象
for task in asyncio.as_completed(tasks, timeout=10):
    # 处理超时
    try:
        # 获取下一个结果
        result = await task
    except asyncio.TimeoutError:
        # ...
```

这并不理想，因为一旦超时时间到期，每次在生成器上调用 `next()` 时都会引发 `asyncio.TimeoutError`。

因此，最好将整个循环包装在 try-except 表达式中。

例如：

```
...
# 处理超时
try:
    # 带超时遍历可等待对象
    for task in asyncio.as_completed(tasks, timeout=10):
        # 获取下一个结果
        result = await task
```

#### asyncio.as_completed() 的工作原理

该函数通过提供一个生成器来工作，该生成器产出协程，每个协程将返回一个提供的可等待对象的结果。

asyncio.as_completed() 函数不会挂起，而是返回一个生成器。

例如：

```
...
# 获取一个按完成顺序产出可等待对象的生成器
generator = asyncio.as_completed(tasks)
```

对生成器调用内置函数 `next()` 不会挂起，而是产出一个协程。

返回的协程并非提供的可等待对象之一，而是 `as_completed()` 函数的一个内部协程，它负责管理和监控哪个已发出的任务将下一个返回结果。

例如：

```
...
# 获取下一个协程
coro = next(generator)
```

只有当返回的协程之一被等待时，调用者才会挂起。

例如：

```
...
# 从下一个完成的任务获取结果
result = await coro
```

现在我们已经了解了如何使用 `as_completed()` 函数及其工作原理，让我们来看一个实际示例。

#### 动态处理任务结果的示例

我们可以探索如何使用 `asyncio.as_completed()` 函数并发执行协程，并在任务完成时获取协程结果。

在这个示例中，我们将定义一个简单的协程任务，它接受一个整数参数，生成一个随机值，休眠一小段时间，然后返回整数参数乘以生成的值。

任务协程的列表被创建并传递给 `as_completed()` 函数。该函数返回一个生成器，使用 for 循环遍历。

在循环的每次迭代中，从生成器产出一个协程，然后在循环体中等待它。返回下一个完成的协程的结果，并报告该结果。

完整示例如下。

```
# SuperFastPython.com
# 获取协程完成时结果的示例
from random import random
import asyncio

# 在新任务中执行的协程
async def task_coro(arg):
    # 生成一个介于 0 和 1 之间的随机值
    value = random()
    # 暂停片刻
    await asyncio.sleep(value)
    # 返回结果
    return arg * value

# 主协程
async def main():
    # 创建多个协程
    coros = [task_coro(i) for i in range(10)]
    # 在协程完成时获取结果
    for coro in asyncio.as_completed(coros):
        # 从下一个完成的任务获取结果
        result = await coro
        # 报告结果
        print(f'>got {result}')

# 启动 asyncio 程序
asyncio.run(main())
```

运行该示例首先创建 `main()` 协程，然后将其用作 asyncio 程序的入口点。

`main()` 协程运行并创建一个协程列表。

然后将此列表传递给 `as_completed()` 函数，该函数返回一个生成器。生成器在 for 循环中遍历，每次迭代产出一个协程。

协程被等待。这会挂起 `main()` 协程。

任务开始执行，生成一个随机值，并休眠。一个任务完成并返回一个值。

`main()` 协程恢复，接收返回值，并报告它。

循环重复，另一个协程被产出，`main()` 协程等待它并挂起，然后返回另一个结果。

此过程持续进行，直到提供的列表中的所有协程完成。

此示例突出显示，我们可以遍历协程集合，并按任务完成的顺序获取和使用结果。

**注意：** 由于使用了随机数，每次运行程序的结果都会有所不同。

```
>got 0.07236962530232949
>got 0.5171864910147306
>got 0.7153626682953872
>got 2.54812333824902
>got 0.5960648114598495
>got 5.051883987489034
>got 0.0
>got 2.842043340472799
>got 6.343694133393031
>got 4.903128525746293
```

接下来，让我们看看如何在不阻塞 asyncio 事件循环的情况下调用阻塞函数。

#### 如何运行阻塞任务

我们可以使用 `asyncio.to_thread()` 函数在 asyncio 中执行线程阻塞的函数调用。

它将接受一个函数调用并在一个新线程中执行它，该线程与执行 asyncio 事件循环的线程分开。

它允许 asyncio 事件循环将阻塞函数调用视为协程，并使用基于线程的并发性异步执行，而不是基于协程的并发性。

`asyncio.to_thread()` 函数专门设计用于执行阻塞 I/O 函数，而不是也可能阻塞 asyncio 事件循环的 CPU 密集型函数。

当我们有一个 asyncio 程序需要同时执行非阻塞 I/O（例如使用套接字）和阻塞 I/O（例如使用文件或遗留 API）时，这是一个有用的函数。

现在我们知道如何在 asyncio 中异步执行阻塞函数，让我们看看如何使用 `to_thread()` 函数。

#### 如何使用 asyncio.to_thread()

`to_thread()` 函数接受要执行的阻塞函数的名称以及该函数的任何参数。

然后返回一个协程，可以等待它以获取函数的返回值（如果有）。

例如：

```
...
# 为阻塞函数创建一个协程
blocking_coro = asyncio.to_thread(blocking, arg1, arg2)
# 等待协程并获取返回值
result = await blocking_coro
```

阻塞函数在被等待或独立执行之前不会在新线程中执行。

协程可以包装在 `asyncio.Task` 中，以独立执行阻塞函数调用。

例如：

```
...
# 为阻塞函数创建一个协程
blocking_coro = asyncio.to_thread(blocking)
# 独立执行阻塞函数
task = asyncio.create_task(blocking_coro)
```

这允许阻塞函数调用像任何其他 asyncio Task 一样使用。

现在我们知道如何使用 `asyncio.to_thread()` 函数，让我们来看一个实际示例。

#### 在线程中运行阻塞函数的示例

我们可以探索如何在新线程中执行阻塞调用而不停止事件循环。

在下面的示例中，我们调用 `asyncio.to_thread()` 为调用 `blocking_task()` 函数创建一个协程。

然后等待此协程，允许主协程挂起，并在新线程中执行阻塞函数。

完整示例如下。

```
# SuperFastPython.com
# 在新线程中运行阻塞调用的示例
import time
import asyncio

# 阻塞函数
def blocking_task():
    # 报告消息
    print('task is running')
    # 阻塞线程
    time.sleep(2)
    # 报告消息
    print('task is done')

# 后台协程任务
async def background():
    # 无限循环
    while True:
        # 报告消息
        print('>background task running')
        # 暂停片刻
        await asyncio.sleep(0.5)

# 主协程
async def main():
    # 运行后台任务
    _ = asyncio.create_task(background())
    # 为阻塞函数调用创建一个协程
    coro = asyncio.to_thread(blocking_task)
    # 在新线程中进行调用并等待结果
    await coro

# 启动 asyncio 程序
asyncio.run(main())
```

运行该示例首先创建 `main()` 协程，并将其用作 asyncio 程序的入口点。

`main()` 协程运行。它创建后台协程并安排它尽快执行。

然后 `main()` 协程创建一个协程以在新线程中运行后台任务，然后等待此协程。

这做了几件事。

首先，它挂起 `main()` 协程，允许事件循环中的任何其他协程运行，例如在新线程中执行阻塞函数的新协程。

新协程运行并启动一个新线程，并在新线程中执行阻塞函数。此协程也被挂起。

事件循环空闲，后台协程有机会运行，循环并报告其消息。

另一个线程中的阻塞调用完成，挂起后台任务，恢复主线程，并终止程序。

这突出了在新线程中运行阻塞调用不会阻塞事件循环，允许其他协程在阻塞调用执行时运行，同时挂起除主事件循环线程之外的某些线程。

```
task is running
>background task running
>background task running
>background task running
>background task running
task is done
```

#### 如何使用异步迭代器

异步迭代器是实现了 `__aiter__()` 和 `__anext__()` 方法的对象。

在我们仔细研究异步迭代器之前，让我们回顾一下经典迭代器，看看它们与异步迭代器的比较。

#### 什么是经典迭代器和异步迭代器

迭代器是实现了特定接口的对象。

具体来说，`__iter__()` 方法返回迭代器的实例，`__next__()` 方法将迭代器推进一个周期并返回一个值。

迭代器可以使用内置函数 `next()` 进行步进，或使用 for 循环进行遍历。

许多对象是可迭代的，最著名的是容器，如列表。

异步迭代器是实现了特定接口的对象，具体来说是 `__aiter__()` 和 `__anext__()` 方法。

`__aiter__()` 方法必须返回迭代器的实例。`__anext__()` 方法必须返回一个可等待对象，该对象推进迭代器。

异步迭代器只能在 asyncio 程序中步进或遍历，例如在协程中。

异步迭代器可以使用内置函数 `anext()` 进行步进，该函数返回一个可等待对象，该对象执行迭代器的一步，例如调用一次 `__anext__()` 方法。

异步迭代器可以使用 `async for` 表达式进行遍历，该表达式将在每次迭代时自动调用 `anext()` 并等待返回的可等待对象以获取返回值。我们稍后将更详细地了解 `async for` 表达式。

接下来，让我们看看如何定义异步迭代器。

#### 如何定义异步迭代器

我们可以通过定义一个实现了 `__aiter__()` 和 `__anext__()` 方法的类来定义异步迭代器。

这些方法按照常规方式在对象上定义。

重要的是，因为 `__anext__()` 函数必须返回一个可等待对象，所以它必须使用 `async def` 表达式定义。

当迭代完成时，`__anext__()` 方法必须引发 `StopAsyncIteration` 异常。

例如：

```python
# 定义一个异步迭代器
class AsyncIterator():
    # 构造函数，定义一些状态
    def __init__(self):
        self.counter = 0

    # 创建迭代器的实例
    def __aiter__(self):
        return self

    # 返回下一个可等待对象
    async def __anext__(self):
        # 检查是否没有更多项
        if self.counter >= 10:
            raise StopAsyncIteration
        # 递增计数器
        self.counter += 1
        # 返回计数器值
        return self.counter
```

因为异步迭代器是一个协程，并且每个迭代器返回一个在 asyncio 事件循环中调度和执行的可等待对象，所以我们可以在迭代器的主体内执行和等待可等待对象。

例如：

```python
...
# 返回下一个可等待对象
async def __anext__(self):
    # 检查是否没有更多项
    if self.counter >= 10:
        raise StopAsyncIteration
    # 递增计数器
    self.counter += 1
    # 模拟工作
    await asyncio.sleep(1)
    # 返回计数器值
    return self.counter
```

接下来，让我们看看如何使用异步迭代器。

#### 如何创建异步迭代器

要使用异步迭代器，我们必须创建迭代器。

这涉及按照常规方式创建对象。

例如：

```python
...
# 创建迭代器
it = AsyncIterator()
```

这返回一个*异步可迭代对象*，它是*异步迭代器*的实例。

接下来，让我们看看如何步进异步迭代器。

#### 如何步进异步迭代器

迭代器的一步可以使用内置函数 `anext()` 进行遍历，就像经典迭代器可以使用 `next()` 函数进行遍历一样。

结果是一个可等待对象，需要被等待。

例如：

```python
...
# 获取迭代器一步的可等待对象
awaitable = anext(it)
# 执行这一步并获取结果
result = await awaitable
```

这可以在一行中完成。

例如：

```python
...
# 步进异步迭代器
result = await anext(it)
```

接下来，让我们看看如何遍历异步迭代器直到完成。

#### 如何遍历异步迭代器

异步迭代器也可以使用 `async for` 表达式在循环中遍历，该表达式将自动等待循环的每次迭代。

例如：

```python
...
# 遍历异步迭代器
async for result in AsyncIterator():
    print(result)
```

这不会并行执行 for 循环。相反，这是一个异步 for 循环。

区别在于执行 for 循环的协程将挂起并在内部等待每个可等待的迭代。

在幕后，这可能需要调度和等待协程，或等待任务。

异步迭代器不能使用 `for` 表达式遍历，传统的迭代器无法使用 `async for` 表达式进行遍历。每种表达式都要求被迭代的对象上存在不同的方法。

我们也可以将异步列表推导式与 `async for` 表达式结合使用，以收集迭代器的结果。

例如：

```
# 使用异步迭代器的异步列表推导式
results = [item async for item in AsyncIterator()]
```

这被称为异步列表推导式。我们也可以在异步字典和集合推导式中使用 `async for` 表达式。

现在我们已经熟悉了如何创建和使用异步迭代器，让我们来看一个实际的例子。

#### 使用异步迭代器的示例

我们可以探索如何使用 `async for` 表达式来遍历异步迭代器。

在这个例子中，我们将使用 `async for` 循环来创建并完整遍历异步迭代器。

这个循环将自动等待从迭代器返回的每个可等待对象，检索返回的值，并使其在循环体内可用，以便在本例中可以将其打印出来。

这可能是异步迭代器最常见的使用模式。

完整的示例如下所示。

```
# SuperFastPython.com
# 使用异步 for 循环的异步迭代器示例
import asyncio

# 定义一个异步迭代器
class AsyncIterator():
    # 构造函数，定义一些状态
    def __init__(self):
        self.counter = 0

    # 创建迭代器的实例
    def __aiter__(self):
        return self

    # 返回下一个可等待对象
    async def __anext__(self):
        # 检查是否没有更多项
        if self.counter >= 10:
            raise StopAsyncIteration
        # 递增计数器
        self.counter += 1
        # 模拟工作
        await asyncio.sleep(1)
        # 返回计数器值
        return self.counter

# 主协程
async def main():
    # 使用异步 for 循环遍历异步迭代器
    async for item in AsyncIterator():
        print(item)

# 执行 asyncio 程序
asyncio.run(main())
```

运行该示例首先会创建 `main()` 协程，并将其作为 asyncio 程序的入口点。

`main()` 协程运行并启动 for 循环。

创建一个异步迭代器的实例，循环会自动使用 `anext()` 函数来步进它，以返回一个可等待对象。然后循环等待该可等待对象并检索一个值，该值在循环体内可用，以便在本例中可以打印出来。

然后重复此过程，挂起 `main()` 协程，执行迭代器的一步并挂起，然后恢复 `main()` 协程，直到迭代器耗尽。

一旦迭代器的内部计数器达到 10，就会引发 `StopAsyncIteration` 异常。这不会终止程序。相反，这是预期的行为，由 `async for` 表达式处理并中断循环。

这突出了如何使用 `async for` 表达式来遍历异步迭代器。

```

2
3
4
5
6
7
8
9
10
```

接下来，让我们看看如何在 asyncio 程序中使用异步生成器。

#### 如何使用异步生成器

异步生成器是使用 `yield` 表达式的协程。

在深入探讨异步生成器的细节之前，让我们先回顾一下经典生成器，并看看它们与异步生成器的比较。

##### 什么是经典生成器和异步生成器

生成器是通过 `yield` 表达式返回值的函数。

例如：

```
# 定义一个生成器
def generator():
    for i in range(10):
        yield i
```

生成器执行到 `yield` 表达式，然后返回一个值。这会在该点挂起生成器。下次执行生成器时，它会从挂起的点恢复，并运行到下一个 `yield` 表达式。

从技术上讲，生成器函数创建并返回一个生成器迭代器。生成器迭代器执行生成器函数的内容，根据需要进行 `yield` 和恢复。

生成器可以使用内置函数 `next()` 来分步执行。

例如：

```
# 创建生成器
gen = generator()
# 步进生成器
result = next(gen)
```

不过，更常见的是将生成器迭代到完成，例如使用 for 循环或列表推导式。

例如：

```
# 遍历生成器并收集结果
results = [item for item in generator()]
```

异步生成器是使用 `yield` 表达式的协程。

与函数生成器不同，协程可以调度和等待其他协程和任务。

与经典生成器类似，异步生成器函数可用于创建异步生成器迭代器，该迭代器可以使用内置的 `anext()` 函数（而不是 `next()` 函数）进行遍历。

这意味着异步生成器迭代器实现了 `__anext__()` 方法，可以与 `async for` 表达式一起使用。

生成器的每次迭代都被调度并作为可等待对象执行。`async for` 表达式将调度并执行生成器的每次迭代，挂起调用协程并等待结果。

接下来，让我们探讨如何定义异步生成器。

##### 如何定义异步生成器

我们可以通过定义一个至少包含一个 `yield` 表达式的协程来定义异步生成器。

这意味着该函数是使用 `async def` 表达式定义的。

例如：

```
# 定义一个异步生成器
async def async_generator():
    for i in range(10):
        yield i
```

因为异步生成器是一个协程，并且每次迭代都返回一个在 asyncio 事件循环中调度和执行的可等待对象，所以我们可以在生成器体内执行和等待可等待对象。

例如：

```
# 定义一个等待的异步生成器
async def async_generator():
    for i in range(10):
        # 挂起并休眠片刻
        await asyncio.sleep(1)
        # 向调用者 yield 一个值
        yield i
```

接下来，让我们看看如何创建异步生成器。

##### 如何创建异步生成器

要使用异步生成器，我们必须创建生成器。

这看起来像是调用生成器，但实际上创建并返回一个迭代器对象，称为可迭代对象。

例如：

```
# 创建迭代器
it = async_generator()
```

这会返回一种称为异步生成器迭代器的异步迭代器。

接下来，让我们看看如何步进异步生成器。

##### 如何步进异步生成器

可以使用内置函数 `anext()` 来步进生成器，就像经典生成器使用 `next()` 函数一样。

结果是一个可等待对象，可以被等待。

例如：

```
# 获取生成器一步的可等待对象
awaitable = anext(gen)
# 执行生成器的这一步并获取结果
result = await awaitable
```

这可以在一步中完成。

例如：

```
# 步进异步生成器
result = await anext(gen)
```

接下来，让我们看看如何遍历异步生成器。

##### 如何遍历异步生成器

异步生成器也可以使用 `async for` 表达式在循环中遍历，该表达式将自动等待循环的每次迭代。

例如：

```
# 遍历异步生成器
async for result in async_generator():
    print(result)
```

我们也可以将异步列表推导式与 `async for` 表达式结合使用，以收集生成器的结果。

例如：

```
# 使用异步生成器的异步列表推导式
results = [item async for item in async_generator()]
```

接下来，让我们看看如何在 asyncio 程序中使用异步上下文管理器。

#### 如何使用异步上下文管理器

异步上下文管理器是一个实现了 `__aenter__()` 和 `__aexit__()` 方法的对象。

在深入了解异步上下文管理器的细节之前，让我们回顾一下经典上下文管理器，并看看它们与异步上下文管理器的比较。

#### 什么是经典和异步上下文管理器

上下文管理器是一个实现了 `__enter__()` 和 `__exit__()` 方法的对象。

`__enter__()` 方法定义了在代码块开始时发生的事情，例如打开或准备资源，如文件、套接字或线程池。

`__exit__()` 方法定义了在退出代码块时发生的事情，例如关闭已准备好的资源。

通常，上下文管理器对象在 `with` 表达式的开头创建，并自动调用 `__enter__()` 方法。内容的主体通过命名的上下文管理器对象使用资源，然后在代码块正常退出或通过异常退出时，自动调用 `__exit__()` 方法。

例如：

```
...
# 打开一个上下文管理器
with ContextManager() as manager:
    # ...
# 自动关闭
```

这类似于 try-finally 表达式。

例如：

```
...
# 创建对象
manager = ContextManager()
try:
    manager.__enter__()
    # ...
finally:
    manager.__exit__()
```

异步上下文管理器提供了一个可以在进入和退出时挂起的上下文管理器。

`__aenter__()` 和 `__aexit__()` 方法被定义为协程，并由调用者等待。

这是通过 `async with` 表达式实现的，我们稍后会详细学习。

因此，异步上下文管理器只能在 asyncio 程序中使用，例如在调用协程中。

接下来，让我们更仔细地看看如何定义异步上下文管理器。

#### 如何定义异步上下文管理器

我们可以将异步上下文管理器定义为一个实现了 `__aenter__()` 和 `__aexit__()` 方法的对象。

重要的是，这两个方法都必须使用 `async def` 表达式定义为协程，因此必须返回可等待对象。

例如：

```
# 定义一个异步上下文管理器
class AsyncContextManager:
    # 进入异步上下文管理器
    async def __aenter__(self):
        # 报告消息
        print('>进入上下文管理器')

    # 退出异步上下文管理器
    async def __aexit__(self, exc_type, exc, tb):
        # 报告消息
        print('>退出上下文管理器')
```

因为每个方法都是协程，它们本身可以等待协程或任务。

例如：

```
# 定义一个异步上下文管理器
class AsyncContextManager:
    # 进入异步上下文管理器
    async def __aenter__(self):
        # 报告消息
        print('>进入上下文管理器')
        # 挂起片刻
        await asyncio.sleep(0.5)

    # 退出异步上下文管理器
    async def __aexit__(self, exc_type, exc, tb):
        # 报告消息
        print('>退出上下文管理器')
        # 挂起片刻
        await asyncio.sleep(0.5)
```

接下来，让我们看看如何创建和使用异步上下文管理器。

#### 如何使用异步上下文管理器

异步上下文管理器通过 `async with` 表达式使用。

这是 `with` 表达式在协程中的扩展。

它会自动等待进入和退出协程，并根据需要挂起调用协程。

例如：

```
...
# 使用异步上下文管理器
async with AsyncContextManager() as manager:
    # ...
```

这等同于类似这样的代码：

```
...
# 创建或进入异步上下文管理器
manager = await AsyncContextManager()
try:
    # ...
finally:
    # 关闭或退出上下文管理器
    await manager.close()
```

异步上下文管理器不能通过 `with` 表达式使用，经典上下文管理器也不能通过 `async with` 表达式使用。每个表达式期望目标对象上存在不同的方法。

注意，我们实现的模式与传统上下文管理器非常相似，只是创建和关闭上下文管理器涉及等待协程。

这会挂起当前协程的执行，调度一个新的协程，并等待其完成。

因此，异步上下文管理器必须实现 `__aenter__()` 和 `__aexit__()` 方法，这些方法必须通过 `async def` 表达式定义。这使得它们本身成为协程，也可以等待。

现在我们知道如何使用异步上下文管理器了，让我们看一个实际的例子。

#### 使用异步上下文管理器的示例

我们可以通过 `async with` 表达式来探索如何使用异步上下文管理器。

在这个例子中，我们将以正常方式创建和使用上下文管理器。

我们将使用 `async with` 表达式，在一行中创建并进入上下文管理器。这将自动等待进入方法。

然后我们可以在内部代码块中使用管理器。在这种情况下，我们只报告一条消息。

退出内部代码块将自动等待上下文管理器的退出方法。

完整的示例如下。

```
# SuperFastPython.com
# 通过 async with 使用异步上下文管理器的示例
import asyncio

# 定义一个异步上下文管理器
class AsyncContextManager:
    # 进入异步上下文管理器
    async def __aenter__(self):
        # 报告消息
        print('>进入上下文管理器')
        # 挂起片刻
        await asyncio.sleep(0.5)

    # 退出异步上下文管理器
    async def __aexit__(self, exc_type, exc, tb):
        # 报告消息
        print('>退出上下文管理器')
        # 挂起片刻
        await asyncio.sleep(0.5)

# 定义一个简单的协程
async def custom_coroutine():
    # 创建并使用异步上下文管理器
    async with AsyncContextManager() as manager:
        # 报告结果
        print('在管理器内部')

# 启动 asyncio 程序
asyncio.run(custom_coroutine())
```

运行示例首先会创建 `main()` 协程，并将其作为 asyncio 程序的入口点。

`main()` 协程开始运行，并在 `async with` 表达式中创建 `AsyncContextManager` 类的一个实例。

该表达式自动调用进入方法并等待协程。报告一条消息，协程挂起片刻。

`main()` 协程恢复并执行上下文管理器的主体，打印一条消息。

退出代码块，上下文管理器的退出方法被自动等待，报告一条消息并休眠片刻。

这突出了异步上下文管理器在 asyncio 程序中的正常使用模式。

```
>进入上下文管理器
在管理器内部
>退出上下文管理器
```

#### 什么是协程安全

线程安全是指可以由多个线程并发执行而不会出现并发错误的程序代码。

它主要指代码没有竞态条件。

竞态条件是并发编程中的一个错误。它是一种失败情况，其中程序的行为取决于两个或多个线程的执行顺序。这意味着程序的行为将是不可预测的，每次运行时可能都会发生变化。

进程安全是指可以由多个进程并发执行而不会出现并发错误的程序代码。它是*线程安全*概念应用于进程，其中进程是并发单元而不是线程。

因此，协程安全是*线程安全*和*进程安全*概念应用于协程的思想。

尽管两个或多个协程不能在事件循环中同时执行，但程序状态和资源仍可能通过并发执行而被破坏或变得不一致。

协程安全意味着代码或程序可以与多个协程并发执行，而不会导致诸如竞态条件之类的并发失败模式。

asyncio 模块提供了许多协程安全的并发原语。

本课将重点介绍在协程之间安全地共享数据、保护数据和协调行为的方法。

#### 如何使用队列在协程之间共享数据

队列可用于在协程之间共享数据。

队列是一种协程安全的数据结构，可用于在协程之间共享数据而不会发生竞态条件。

asyncio 模块提供了用于通用目的的 asyncio.Queue 类，但也通过 asyncio.LifoQueue 类提供了后进先出（LIFO）队列，以及通过 asyncio.PriorityQueue 类提供了优先级队列。

队列是一种数据结构，可以通过调用 put() 添加项目，也可以通过调用 get() 检索项目。

协程安全意味着它可以被多个协程并发地 put 和 get 项目而不会发生竞态条件。

Queue 类提供了一个先进先出（FIFO）队列，这意味着项目按照它们被添加的顺序从队列中检索。首先添加到队列的项目将首先被检索。

接下来，让我们看看如何在 asyncio 程序中使用队列。

#### 如何使用 Asyncio 队列

asyncio.Queue 可以通过首先创建该类的实例来使用。这将默认创建一个无界队列，即没有大小限制的队列。

例如：

```
...
# 创建一个无界队列
queue = asyncio.Queue()
```

可以通过 maxsize 构造函数参数为新队列设置最大容量。

例如：

```
...
# 创建一个具有最大容量的队列
queue = asyncio.Queue(maxsize=100)
```

可以通过调用 put() 方法将项目添加到队列中。这是一个必须被 await 的协程，它会挂起调用者，直到项目成功放置在队列上。例如：

```
...
# 向队列添加一个项目
await queue.put(item)
```

如果一个大小受限的队列已满，则无法添加新项目，并且对 put() 的调用将挂起，直到有可用空间。

可以通过调用 get() 从队列中检索项目。这也是一个协程，必须被 await，它会挂起调用者，直到成功从队列中检索到一个项目。

例如：

```
...
# 从队列中获取一个项目
item = await queue.get()
```

#### 使用队列的生产者和消费者任务示例

我们可以探讨一个使用队列在两个协程之间共享数据的示例。

在这个例子中，我们将定义一个自定义任务函数，该函数接受队列作为参数，生成一些数据，并将该数据放入队列。它是一个生产者协程。

主协程将创建队列，将其与新协程共享，然后挂起，等待数据到达队列。主协程将是消费者协程。

使用队列在协程之间共享数据的完整示例如下所示。

```
# SuperFastPython.com
# 通过队列连接的生产者/消费者示例
from random import random
import asyncio

# 用于生成工作的协程
async def producer(queue):
    print('Producer: Running')
    # 生成工作
    for _ in range(10):
        # 生成一个值
        value = random()
        # 挂起以模拟工作
        await asyncio.sleep(value)
        # 添加到队列
        await queue.put(value)
    # 发送全部完成信号
    await queue.put(None)
    print('Producer: Done')

# 用于消费工作的协程
async def consumer(queue):
    print('Consumer: Running')
    # 消费工作
    while True:
        # 获取一个工作单元
        item = await queue.get()
        # 检查停止信号
        if item is None:
            break
        # 报告
        print(f'>got {item}')
    # 全部完成
    print('Consumer: Done')

# 入口点协程
async def main():
    # 创建共享队列
    queue = asyncio.Queue()
    # 运行生产者和消费者
    await asyncio.gather(
        producer(queue), consumer(queue))

# 启动 asyncio 程序
asyncio.run(main())
```

运行该示例首先创建 `main()` 协程，并将其用作 asyncio 程序的入口点。

`main()` 协程运行，然后创建共享的 `asyncio.Queue` 对象。

接下来，创建生产者协程并传递队列实例。然后启动消费者协程，主协程挂起，直到两个协程都终止。

生产者协程在任务的每次迭代中生成一个新的随机值，挂起，并将其添加到队列。消费者协程在队列上等待项目到达，然后一次消费一个项目，并报告其值。

最后，生产者任务完成，一个 `None` 值被放入队列，协程终止。消费者协程获取 `None` 值，中断其循环，也终止。

这突出了如何使用 `asyncio.Queue` 轻松地在生产者和消费者协程之间共享数据。

#### 如何使用互斥锁保护临界区

互斥锁，简称 mutex 锁，是一种并发原语，旨在防止竞态条件。

竞态条件是一种并发失败情况，当两个协程运行相同的代码并访问或更新相同的资源（例如数据变量、流等）时，会导致资源处于未知且不一致的状态。

竞态条件通常会导致程序行为异常和/或数据损坏。

这些可以被多个协程并发执行并可能导致竞态条件的敏感代码部分被称为临界区。临界区可能指单个代码块，但也指从多个函数对同一数据变量或资源的多次访问。

接下来，让我们看看如何使用互斥锁。

#### 如何使用 asyncio 锁

Python 通过 `asyncio.Lock` 类为协程提供了互斥锁。

可以创建一个 Lock 类的实例，然后在协程访问临界区之前获取它，并在退出临界区后释放它。

`acquire()` 方法用于获取锁。它是一个协程，必须被 await，这会挂起调用它的协程。锁可以稍后通过 `release()` 方法再次释放。

例如：

```
...
# 创建一个锁
lock = asyncio.Lock()
# 获取锁
await lock.acquire()
# ...
# 释放锁
lock.release()
```

在任何时刻，只有一个协程可以持有锁。如果一个协程没有释放已获取的锁，它就不能再次获取锁。

尝试获取锁的协程将被挂起，直到锁被获取，例如，如果另一个协程当前持有锁然后释放了它。

我们也可以通过上下文管理器接口，使用 `async with` 表达式来使用锁，这允许临界区是上下文管理器内的一个代码块，并且一旦代码块退出（无论是正常还是异常退出），锁都会自动释放。

例如：

```
...
# 创建一个锁
lock = asyncio.Lock()
# 获取锁
async with lock:
    # ...
```

这是使用锁的首选方式，因为它清楚地指明了受保护代码的开始和结束位置，并确保锁总是被释放，即使在临界区内发生异常或错误。

现在我们知道了如何使用互斥锁，让我们看一个实际的例子。

#### 使用 asyncio 锁的示例

我们可以开发一个示例来演示如何使用互斥锁。

在这个例子中，我们将定义一个目标任务，该任务接受一个锁作为参数，并使用该锁来保护一个临界区，在这个例子中，临界区将打印一条消息并休眠片刻。

完整的示例如下所示。

```
# SuperFastPython.com
# asyncio 互斥锁示例
from random import random
import asyncio

# 带有临界区的任务协程
async def task(lock, num, value):
    # 获取锁以保护临界区
    async with lock:
        # 报告一条消息
        print(f'>{num} 获得锁，休眠 {value}')
        # 挂起片刻
        await asyncio.sleep(value)

# 入口点
async def main():
    # 创建一个共享锁
    lock = asyncio.Lock()
    # 创建许多并发协程
    coros = [task(lock, i, random()) for i in range(10)]
    # 执行并等待任务完成
    await asyncio.gather(*coros)

# 运行 asyncio 程序
asyncio.run(main())
```

运行该示例首先创建 `main()` 协程，并将其用作 asyncio 程序的入口点。

`main()` 协程运行，首先创建共享锁。

然后它创建一个协程列表，每个协程都传递共享锁、一个唯一的整数和一个随机浮点值。

协程列表被传递给 `gather()` 函数，`main()` 协程挂起，直到所有协程完成。

一个任务协程执行，获取锁，报告一条消息，然后等待休眠，挂起。

另一个协程恢复。它尝试获取锁并被挂起，同时它在等待。这个过程在许多（如果不是全部）协程中重复。

第一个协程恢复，退出代码块，并通过异步上下文管理器自动释放锁。

第一个等待锁的协程恢复，获取锁，报告一条消息，然后休眠。

这个过程重复，直到所有协程都有机会获取锁、执行临界区并终止。

一旦所有任务终止，`main()` 协程恢复并终止，关闭程序。

#### 如何使用信号量限制对资源的访问

信号量是一种并发原语，它允许限制可以获取保护临界区或资源的锁的协程数量。

它是互斥锁的扩展，增加了一个计数器，用于在额外协程挂起之前可以获取锁的协程数量。一旦达到上限，新的协程只能在现有持有信号量的协程释放访问权限后才能获取信号量。

在内部，信号量维护一个由互斥锁保护的计数器，每次获取信号量时计数器递减，每次释放时递增。创建信号量时，会设置计数器的上限。如果设置为 1，那么信号量将像互斥锁一样工作。

接下来，让我们看看如何创建和使用信号量。

#### 如何使用 asyncio 信号量

Python 通过 `asyncio.Semaphore` 类为协程提供了信号量。

Semaphore 对象在创建时必须配置，以设置内部计数器的限制。这个限制将与可以持有信号量的并发协程数量相匹配。

例如，我们可以将其设置为 100：

```
...
# 创建一个限制为 100 的信号量
semaphore = asyncio.Semaphore(100)
```

每次获取信号量时，内部计数器递减。每次释放信号量时，内部计数器递增。

如果信号量没有可用位置（例如计数为零），则无法获取信号量，在这种情况下，尝试获取它的线程必须挂起，直到有位置可用。

可以通过调用 `acquire()` 方法来获取信号量，该方法必须被 await。

例如：

```
...
# 获取信号量
await semaphore.acquire()
```

默认情况下，调用的协程将挂起，直到信号量上有可用的访问权限。

一旦获取，信号量可以通过调用 `release()` 方法再次释放。

例如：

```
...
# 释放信号量
semaphore.release()
```

Semaphore 类支持通过上下文管理器使用，它将自动为我们获取和释放信号量。因此，它是在程序中使用信号量的首选方式。

例如：

```
...
# 获取信号量
async with semaphore:
    # ...
```

#### 使用 asyncio 信号量的示例

我们可以通过一个实际示例来探讨如何使用 `asyncio.Semaphore`。

在这个示例中，我们将启动一组协程，但限制同时执行操作的数量。我们将使用信号量来限制可并发执行的任务数量，该数量将小于协程总数，从而允许一些协程挂起、等待访问权限，然后被通知并获取访问权限。

完整示例如下：

```python
# SuperFastPython.com
#### 使用 asyncio 信号量的示例
from random import random
import asyncio

# 任务协程
async def task(semaphore, number):
    # 获取信号量
    async with semaphore:
        # 生成一个 0 到 1 之间的随机值
        value = random()
        # 挂起片刻
        await asyncio.sleep(value)
        # 报告消息
        print(f'Task {number} got {value}')

# 主协程
async def main():
    # 创建共享的信号量
    semaphore = asyncio.Semaphore(2)
    # 创建并调度任务
    tasks = [asyncio.create_task(task(semaphore, i))
        for i in range(10)]
    # 等待所有任务完成
    _ = await asyncio.wait(tasks)

# 启动 asyncio 程序
asyncio.run(main())
```

运行该示例首先会创建 `main()` 协程，它作为 asyncio 程序的入口点。

`main()` 协程运行并首先创建一个初始计数器值为 2 的共享信号量，这意味着一次可以有两个协程持有该信号量。

然后，`main()` 协程创建并调度 10 个任务来执行我们的 `task()` 协程，传递共享信号量和一个 0 到 9 之间的唯一编号。

接着，`main()` 协程挂起并等待所有任务完成。

任务一次运行一个。

每个任务首先尝试获取信号量。如果有空位，它就继续执行；否则，它会等待空位出现。

一旦获取到信号量，任务会生成一个随机值，挂起片刻，然后报告生成的值。随后，它释放信号量并终止。在任务挂起于 `asyncio.sleep()` 调用期间，信号量不会被释放。

信号量上下文管理器的主体一次仅限于两个信号量。

这突显了我们如何限制并发执行代码块的协程数量。

#### 如何使用事件在协程之间发送信号

事件是一个协程安全的布尔标志，可用于在两个或多个协程之间发送信号。

它对于协调许多协程的行为非常有用，这些协程可以检查标志的状态，例如开始处理，或停止处理并退出。

接下来，让我们看看如何创建和使用事件。

#### 如何使用 Asyncio 事件

Python 通过 `asyncio.Event` 类为协程提供了事件对象。

`Event` 类封装了一个布尔变量，该变量可以被设置（True）或未设置（False）。共享 `Event` 对象的协程可以检查事件是否已设置、设置事件、清除事件（使其未设置）或等待事件被设置。

事件提供了一种简单的方式在协程之间共享布尔变量，该变量可以作为操作的触发器。

首先，必须创建一个 `Event` 对象，事件将处于未设置状态。

```python
...
# 创建一个事件实例
event = asyncio.Event()
```

创建后，我们可以通过 `is_set()` 方法检查事件是否已设置，如果事件已设置则返回 `True`，否则返回 `False`。

例如：

```python
...
# 检查事件是否已设置
if event.is_set():
    # 执行某些操作...
```

事件可以通过 `set()` 方法进行设置。任何等待事件设置的协程都将被通知。

例如：

```python
...
# 设置事件
event.set()
```

最后，协程可以通过 `wait()` 方法等待事件被设置，该方法必须被 `await`。调用此方法将挂起，直到事件被标记为已设置（例如，另一个协程调用 `set()` 方法）。如果事件已经设置，`wait()` 方法将立即返回。

```python
...
# 等待事件被设置
await event.wait()
```

现在我们知道了如何使用事件，让我们看一个实际示例。

#### 使用 Asyncio 事件的示例

我们可以探讨如何使用 `asyncio.Event` 对象。

在这个示例中，我们将创建一组协程，每个协程将执行一些工作并报告消息。所有协程都将使用一个事件，在开始工作前等待其被设置。主协程将设置事件并触发新协程开始工作。

```python
# SuperFastPython.com
# 使用 asyncio 事件对象的示例
from random import random
import asyncio

# 任务协程
async def task(event, number):
    # 等待事件被设置
    await event.wait()
    # 生成一个 0 到 1 之间的随机值
    value = random()
    # 挂起片刻
    await asyncio.sleep(value)
    # 报告消息
    print(f'Task {number} got {value}')

# 主协程
async def main():
    # 创建一个共享的事件对象
    event = asyncio.Event()
    # 创建并运行任务
    tasks = [asyncio.create_task(task(event, i))
        for i in range(5)]
    # 允许任务开始
    print('Main suspending...')
    await asyncio.sleep(0)
    # 在所有任务中开始处理
    print('Main setting the event')
    event.set()
    # 等待所有任务终止
    _ = await asyncio.wait(tasks)

# 运行 asyncio 程序
asyncio.run(main())
```

运行该示例首先会创建 `main()` 协程，并将其用作 asyncio 程序的入口点。

`main()` 协程运行并创建并调度五个任务协程。

然后它休眠，挂起并允许任务运行并开始等待事件。

主协程恢复，报告消息，然后将事件设置为 `True`。接着它挂起并等待所有已发出的任务完成。

这触发了所有五个协程。它们依次恢复，执行处理并报告消息。

这突显了协程如何等待事件被设置，以及我们如何使用事件通知协程。

```

Task 1 got 0.4852630342496812
```

接下来，让我们看看如何在 asyncio 程序中使用条件变量。

#### 如何使用条件变量进行协调

条件变量，也称为监视器，允许多个协程等待并接收关于某些结果的通知。

协程可以获取条件，然后等待另一个协程通知某些内容已更改。在等待期间，协程被挂起并释放条件上的锁，以便其他协程可以获取。

然后，另一个协程可以获取条件，在程序中进行更改，并通知一个、所有或等待条件的协程子集某些内容已更改。

等待的协程随后可以恢复，重新获取条件，对任何更改的状态进行检查并执行所需的操作。

接下来，让我们看看如何创建和使用条件变量。

#### 如何使用 Asyncio 条件

Python 通过 `asyncio.Condition` 类提供条件变量。

例如：

```python
...
# 创建一个新的条件变量
condition = asyncio.Condition()
```

为了使协程能够使用 `Condition`，它必须获取并释放它，就像互斥锁一样。

这可以通过 `acquire()` 和 `release()` 方法手动实现。`acquire()` 方法是一个协程，必须被 `await`。

例如，我们可以获取 `Condition`，然后等待条件被通知，最后释放条件，如下所示：

## 获取条件
await condition.acquire()
# 等待通知
await condition.wait()
# 释放条件
condition.release()

`wait()` 方法也是一个协程，必须被 await。

除了直接调用 `acquire()` 和 `release()` 方法外，另一种选择是使用上下文管理器，它会自动为我们执行获取和释放操作，例如：

```
# 获取条件
async with condition:
    # 等待通知
    await condition.wait()
```

如果我们希望通知等待的协程，也必须在协程中获取条件。这同样可以通过直接调用获取和释放方法，或通过上下文管理器来实现。

我们可以通过 `notify()` 方法通知单个等待的协程。

例如：

```
# 获取条件
with condition:
    # 通知一个等待的协程
    condition.notify()
```

被通知的协程将尽快停止等待，一旦它能重新获取条件。这将作为其调用 `wait()` 的一部分自动尝试，我们不需要做任何额外的事情。

我们可以通过 `notify_all()` 方法通知所有在条件上等待的协程。

```
# 获取条件
with condition:
    # 通知所有在条件上等待的协程
    condition.notify_all()
```

现在我们知道了如何使用条件变量，让我们来看一个实际的例子。

#### 使用 Asyncio 条件的示例

在这个例子中，我们将探索使用 asyncio.Condition 来通知等待的协程某事已经发生。

我们将使用一个任务来准备一些数据并通知一个等待的协程。在主协程中，我们将创建并调度新任务，并使用条件来等待工作完成。

完整的示例如下所示。

```
# SuperFastPython.com
# 使用 asyncio 条件进行等待/通知的示例
import asyncio

# 任务协程
async def task(condition, work_list):
    # 暂停片刻
    await asyncio.sleep(1)
    # 将数据添加到工作列表
    work_list.append(33)
    # 通知等待的协程工作已完成
    print('Task sending notification...')
    async with condition:
        condition.notify()

# 主协程
async def main():
    # 创建一个条件
    condition = asyncio.Condition()
    # 准备工作列表
    work_list = []
    # 等待通知数据已准备好
    print('Main waiting for data...')
    async with condition:
        # 创建并启动任务
        _ = asyncio.create_task(
            task(condition, work_list))
        # 等待通知
        await condition.wait()
    # 我们知道数据已准备好
    print(f'Got data: {work_list}')

# 运行 asyncio 程序
asyncio.run(main())
```

运行该示例首先创建 main() 协程，它被用作 asyncio 程序的入口点。

main() 协程运行并创建共享的条件和工作列表。

然后 main() 协程获取条件。创建并调度一个新任务，提供共享的条件和工作列表。

然后 main() 协程等待被通知，暂停并调用新调度的任务运行。

task() 协程运行。它首先暂停片刻以模拟工作，然后向共享列表添加工作。获取条件并通知等待的协程，然后自动释放条件。任务终止。

main() 协程恢复并报告最终消息，显示共享列表的更新内容。

这突出了我们如何使用条件变量在协程之间使用等待-通知模式。

```
Main waiting for data...
Task sending notification...
Got data: [33]
```

在asyncio中，有两种方式可以将外部程序作为子进程执行：

#### 如何直接运行命令

可以调用`asyncio.create_subprocess_exec()`函数在子进程中执行命令。

`create_subprocess_exec()`函数是一个协程，必须被await。它将挂起调用者，直到子进程启动（而不是完成）。

```
...
```

例如：

```
...

process = await asyncio.create_subprocess_exec(
    'echo', 'Hello World',
    stdout=asyncio.subprocess.PIPE)
```

子进程启动后，会报告子进程的详细信息。

接下来，让我们看看如何等待子进程完成。

#### 如何等待子进程

我们可以通过`wait()`方法等待子进程完成。

这是一个必须被await的协程。

调用者将被挂起，直到子进程终止，无论是正常终止还是其他方式。

如果子进程期望输入，并且配置为通过管道从我们的asyncio程序接收输入，那么调用`wait()`可能会导致死锁，因为如果调用者被挂起，它就无法向子进程提供输入。

例如：

```
...
# 在子进程中运行一个命令
process = await asyncio.create_subprocess_shell(
    'sleep 3')
# 等待子进程终止
await process.wait()
```

接下来，让我们看看如何从子进程读取和写入数据。

#### 如何与子进程进行数据读写

我们可以通过`communicate()`方法从asyncio中的子进程读取数据。

从子进程读取数据要求`create_subprocess_shell()`或`create_subprocess_exec()`函数的`stdout`或`stderr`参数被设置为`PIPE`常量。

该方法不提供参数，并返回一个包含来自`stdout`和`stderr`输入的元组。数据将被读取，直到收到文件结束符（EOF）。

`communicate()`方法是一个协程，必须被await。

例如：

```
...
# 在子进程中运行一个命令
process = await asyncio.create_subprocess_shell(
    'echo Hello World', stdout=asyncio.subprocess.PIPE)
# 从子进程读取数据
data, _ = await process.communicate()
```

如果没有数据可读，调用将挂起，直到子进程终止。

我们也可以通过`communicate()`方法从asyncio协程向子进程写入数据。数据通过`input`参数以字节形式提供。

通过`communicate()`方法向子进程写入数据，要求`create_subprocess_shell()`或`create_subprocess_exec()`函数中的`stdin`参数被设置为`PIPE`常量。

例如：

```
...
# 在子进程中运行一个命令
process = await asyncio.create_subprocess_exec(
    'cat', stdin=asyncio.subprocess.PIPE)
# 向子进程写入数据
_ = await process.communicate(b'Hello World\n')
```

#### 如何终止和杀死子进程

我们可以通过`terminate()`方法停止子进程。

在大多数平台上，这会向子进程发送`SIGTERM`信号并立即终止它。

例如：

```
...
# 终止子进程
process.terminate()
```

我们也可以通过`kill()`方法杀死子进程。

在大多数平台上，这会向子进程发送`SIGKILL`信号以立即停止它。

#### 如何使用非阻塞 I/O 流

Asyncio 提供了非阻塞的 I/O 套接字编程。

这通过流来实现。

可以打开套接字以提供对流写入器和流读取器的访问。

然后可以使用协程从流中写入和读取数据，并在适当时挂起。

完成后，可以关闭套接字。

asyncio 的流功能属于底层功能，这意味着任何所需的协议都必须手动实现。

这可以包括常见的基于 ASCII 的 Web 协议，例如：

- 用于与 Web 服务器交互的 HTTP 或 HTTPS。
- 用于与电子邮件服务器交互的 SMTP。
- 用于与文件服务器交互的 FTP。

流也可以用于创建服务器，以使用标准协议处理请求，或开发我们自己的特定于应用程序的协议。

#### 如何打开套接字连接

可以使用 `asyncio.open_connection()` 函数打开一个 asyncio TCP 客户端套接字连接。

这是一个必须被等待的协程，并在套接字连接打开后返回。

该函数返回一个 `StreamReader` 和一个 `StreamWriter` 对象，用于与套接字交互。

例如：

```
...
# 打开一个连接
reader, writer = await asyncio.open_connection(...)
```

asyncio.open_connection() 函数接受许多参数来配置套接字连接。

两个必需的参数是主机和端口。

主机是一个字符串，指定要连接的服务器，例如域名或 IP 地址。

端口是套接字端口号，例如 HTTP 服务器的 80，HTTPS 服务器的 443，SMTP 的 23 等等。

例如：

```
...
# 打开一个到 http 服务器的连接
reader, writer = await asyncio.open_connection(
    'www.google.com', 80)
```

支持通过 SSL 协议建立加密的套接字连接。

最常见的例子是正在取代 HTTP 的 HTTPS。

这可以通过将 ssl 参数设置为 True 来实现。

例如：

```
...
# 打开一个到 https 服务器的连接
reader, writer = await asyncio.open_connection(
    'www.google.com', 443, ssl=True)
```

```
# 启动一个服务器来接收 http 连接
server = await asyncio.start_server(
    handler, '127.0.0.1', 80)
```

接下来，让我们看看如何向流中写入数据。

#### 如何使用 StreamWriter 写入数据

我们可以使用 asyncio.StreamWriter 向套接字写入数据。

数据以字节形式写入。

可以使用 `write()` 方法将字节数据写入套接字。

例如：

```
...
# 写入字节数据
writer.write(byte_data)
```

或者，可以使用 `writelines()` 方法将组织成列表或可迭代对象的多行字节数据写入。

例如：

```
...
# 写入多行字节数据
writer.writelines(byte_lines)
```

这两种写入数据的方法都不会挂起调用协程。

写入字节数据后，最好通过 `drain()` 方法排空套接字。

这是一个协程，它将挂起调用者，直到字节已传输且套接字准备就绪。

例如：

```
...
# 写入字节数据
writer.write(byte_data)
# 等待数据传输
await writer.drain()
```

字符串数据可以通过编码转换为字节数据进行传输。这可以通过在字符串上使用 `encode()` 方法来实现，该方法将返回使用默认 UTF8 编码的字节数据，准备传输。

例如：

```
...
# 将字符串数据编码为字节数据以进行传输
byte_data = string_data.encode()
```

接下来，让我们看看如何从流中读取数据。

#### 如何使用 StreamReader 读取数据

我们可以使用 asyncio.StreamReader 从套接字读取数据。

数据以字节格式读取，因此字符串在使用前可能需要编码。

所有读取方法都是必须被等待的协程。

可以通过 read() 方法读取任意数量的字节，该方法将读取直到文件结束（EOF）。

```
...
# 读取字节数据
byte_data = await reader.read()
```

此外，可以通过 n 参数指定要读取的字节数。

如果我们知道下一个响应预期的字节数，这可能会有所帮助。

例如：

```
...
# 读取字节数据
byte_data = await reader.read(n=100)
```

可以使用 readline() 方法读取单行数据。

这将返回字节，直到遇到换行符 '\n' 或 EOF。

这在读取使用文本行的标准协议时很有帮助。

```
...
# 读取一行数据
byte_line = await reader.readline()
```

此外，还有一个 `readexactly()` 方法用于读取精确数量的字节，否则引发异常，以及一个 `readuntil()` 方法，它将读取字节直到读取到指定的字节形式的字符。

从流中读取的数据可以使用 `decode()` 方法和默认的 UTF8 编码从字节解码为字符串数据。

例如：

```
...
# 将字节数据解码为字符串数据
string_data = byte_data.decode()
```

#### 如何关闭套接字连接

可以通过 `asyncio.StreamWriter` 关闭套接字。

可以调用 `close()` 方法来关闭套接字。此方法不会挂起。

例如：

```
...
# 关闭套接字
writer.close()
```

虽然 `close()` 方法不会挂起，但我们可以等待套接字完全关闭后再继续。

这可以通过 `wait_closed()` 方法实现。

这是一个可以被等待的协程。

例如：

```
...
# 关闭套接字
writer.close()
# 等待套接字关闭
await writer.wait_closed()
```

我们可以通过 `is_closing()` 方法检查套接字是否已关闭或正在关闭过程中。

例如：

```
...
# 检查套接字是否已关闭或正在关闭
if writer.is_closing():
    # ...
```

#### 检查网页状态的示例

我们可以开发一个示例，使用非阻塞 I/O 并发检查多个网页的 HTTP 状态。

这可以通过向每个网页发出 HTTP GET 请求并读取包含网页状态的第一行响应来实现。

这需要首先使用 SSL 在端口 443 上打开到 HTTPS 服务器的套接字连接。然后我们必须制定包含所需 URL 和主机名的 HTTP GET 请求。字符串请求必须在传输前编码为字节数据。

然后我们可以从服务器读取响应的第一行，解码它并将其作为服务器的 HTTP 状态返回。然后可以关闭 TCP 套接字。这假设服务器存在并且我们可以连接到它。

这个过程可以包装成一个协程，并为我们要查询的每个网站 URL 并发执行。

将这些结合起来，完整的示例如下。

```

    # 读取单行响应
    response = await reader.readline()
    # 关闭连接
    writer.close()
    # 解码并去除空白
    status = response.decode().strip()
    # 返回响应
    return status
```

#### 开发一个 Asyncio 端口扫描器

我们可以通过打开套接字来连接到其他计算机，这称为套接字编程。

打开套接字需要服务器的名称或 IP 地址以及要连接的端口号。

例如，当我们的网络浏览器在 python.org 上打开一个网页时，它正在打开一个到该服务器端口 80 或 443 的套接字连接，然后使用 HTTP 协议请求并下载（GET）一个 HTML 文件。

套接字编程或网络编程非常有趣。

一个好的入门套接字编程项目是开发一个端口扫描器。这是一个报告给定服务器上所有打开的套接字的程序。

实现端口扫描器的一种简单方法是循环遍历所有要测试的端口，并尝试在每个端口上建立套接字连接。如果可以建立连接，我们立即断开连接并报告服务器上的端口是打开的。

历史上，服务器上有很多开放端口是一种安全风险，因此通常会锁定面向公众的服务器并关闭所有非必要的端口以阻止外部流量。这意味着扫描公共服务器在最好的情况下可能只会发现很少的开放端口，或者在最坏的情况下，如果服务器认为我们试图入侵，可能会拒绝未来的访问。

因此，尽管开发端口扫描器是一个有趣的套接字编程练习，但我们必须小心使用它以及扫描哪些服务器。

接下来，让我们看看如何在单个端口上打开套接字连接。

#### 如何在端口上打开套接字连接

我们可以使用 asyncio.open_connection() 函数在 asyncio 中打开套接字连接。

该函数接受主机和端口号，并返回一个 StreamReader 和 StreamWriter，用于通过套接字与服务器交互。

asyncio.open_connection() 函数是一个协程，必须被等待。它将在连接打开后返回。

例如：

```python
...
# 打开一个套接字连接
reader, writer = asyncio.open_connection(
    'python.org', 80)
```

如果可以建立连接，则端口是打开的。否则，如果无法建立连接，则端口未打开。

问题是，我们如何知道无法建立连接？

如果端口未打开，调用可能会等待很长时间才会放弃。我们需要一种在时间限制后放弃的方法。

这可以使用 asyncio.wait_for() 函数来实现。

这是一个协程，它将执行一个可等待对象，并在固定间隔（以秒为单位）后放弃并引发 asyncio.TimeoutError 异常。

我们可以为 asyncio.open_connection() 函数创建协程，并将其传递给 wait_for() 协程。

这将允许我们在给定端口上尝试建立套接字连接一段固定时间，例如一秒钟或三秒钟。

例如：

```python
...
# 创建用于打开连接的协程
coro = asyncio.open_connection('python.org', 80)
# 使用超时执行协程
try:
    # 打开连接并等待片刻
    _ = await asyncio.wait_for(coro, 1.0)
    # ...
except asyncio.TimeoutError:
    # ...
```

如果可以在时间限制内建立连接，我们就可以关闭连接。

这可以通过调用 asyncio.open_connection() 返回的 StreamWriter 对象的 close() 方法来实现。

例如：

```python
...
# 连接打开后关闭连接
writer.close()
```

否则，如果引发 asyncio.TimeoutError 异常，我们可以假设端口可能未打开。

我们可以将所有这些整合到一个协程函数中，该函数测试一个主机上的一个端口，如果端口打开则返回 True，否则返回 False。

下面的 test_port_number() 协程函数实现了这一点。

```python
# 如果可以建立连接则返回 True
async def test_port_number(host, port, timeout=3):
    # 创建用于打开连接的协程
    coro = asyncio.open_connection(host, port)
    # 使用超时执行协程
    try:
        # 打开连接并等待片刻
        _, writer = await asyncio.wait_for(coro, timeout)
        # 连接打开后关闭连接
        writer.close()
```

#### 如何扫描服务器上的端口范围（慢速方法）

我们可以扫描给定主机上的一个端口范围。

许多常见的互联网服务运行在0到1,024之间的端口上。

端口的有效范围是0到65,535，在POSIX系统的`/etc/services`文件中，我们可以查看最常用的端口号及其对应的服务列表。

我们可以通过反复调用上一节开发的协程来扫描一个端口范围，并将任何允许连接的端口报告为*开放*状态。

下面的`main()`协程函数实现了这一功能，报告所有发现的开放端口。

最后，我们可以调用此函数并指定主机和端口范围。

在这个例子中，我们将对python.org进行端口扫描（出于对Python的热爱，而非恶意意图）。

```python
...
```

#### 如何并发扫描端口（*快速方法*）

我们可以使用asyncio并发扫描端口。

每个端口可以在一个单独的协程中并发测试。

打开连接会挂起调用者，允许其他协程运行。那些尝试连接到未开放端口的协程将保持挂起状态，直到超时时间过去，从而允许其他协程运行。

这可以通过为每个要扫描的端口创建一个协程来实现，然后并发执行所有协程并等待它们完成。这可以使用`asyncio.gather()`函数来实现。

它首先需要创建协程。每个端口一个协程，这将是一个包含超过1,000个协程的集合。我们可以使用列表推导式来实现这一点。

例如：

```python
...
# create all coroutines
coros = [test_port_number(host, port) for port in ports]
```

接下来，我们可以使用`asyncio.gather()`函数并发执行所有这些协程。

此函数接受可等待对象作为参数，并且在可等待对象完成之前不会返回。它不接受可等待对象的列表，因此我们必须使用星号（*）运算符将列表展开为单独的表达式。

例如：

```python
...
# execute all coroutines concurrently
results = await asyncio.gather(*coros)
```

这将并发执行所有协程，并按提供的顺序返回每个协程返回值的可迭代对象。

然后，我们可以遍历返回值列表和端口列表，并报告结果。

回想一下，我们可以使用内置的`zip()`函数一起遍历两个或多个可迭代对象。

例如：

```python
...
# report results
for port, result in zip(ports, results):
    if result:
        print(f'> {host}:{port} [OPEN]')
```

将以上内容整合，完整示例如下。

```python
# SuperFastPython.com
# example of a concurrent port scanner using gather
import asyncio

# returns True if a connection can be made
async def test_port_number(host, port, timeout=3):
    # create coroutine for opening a connection
    coro = asyncio.open_connection(host, port)
    # execute the coroutine with a timeout
    try:
        # open the connection and wait for a moment
        _, writer = await asyncio.wait_for(coro, timeout)
        # close connection once opened
        writer.close()
        # indicate the connection can be opened
        return True
    except asyncio.TimeoutError:
        # indicate the connection cannot be opened
        return False

# main coroutine
async def main(host, ports):
    # report a status message
    print(f'Scanning {host}...')
    # create all coroutines
    coros = [test_port_number(host, port) for port in ports]
    # execute all coroutines concurrently
    results = await asyncio.gather(*coros)
    # report results
    for port, result in zip(ports, results):
        if result:
            print(f'> {host}:{port} [OPEN]')

# define a host and ports to scan
host = 'python.org'
ports = range(1, 1024)
# start the asyncio program
asyncio.run(main(host, ports))
```

运行此示例会执行`main()`协程作为我们asyncio程序的入口点。

首先创建一个协程列表。

然后使用`asyncio.gather()`并发执行所有协程。

这会挂起`main()`协程，直到所有协程完成。每个协程测试一个端口，尝试打开连接并挂起，直到连接打开或超时时间过去。

一旦所有任务完成，`main()`协程恢复执行，并报告所有结果。

报告了两个开放端口，与之前相同。

主要区别在于执行速度。在这种情况下，它大约需要3.1秒，而之前的示例需要超过50分钟。

这大约快了3,063秒，即速度提升了989倍，也就是将近1000倍。

```
Scanning python.org...
> python.org:80 [OPEN]
> python.org:443 [OPEN]
```

接下来，让我们看看如何在结果可用时立即报告，而不是在所有协程完成后才报告。

#### 如何动态报告扫描结果

在前面的示例中，我们并发执行了协程，并在所有任务完成后报告结果。

另一种方法是在任务完成时报告结果。

这将使程序响应更及时，并在结果可用时向用户展示。

我们可以通过让`test_port_number()`协程直接报告其结果来实现这一点。

另一种方法是按照协程完成的顺序遍历它们。

这可以使用`asyncio.as_completed()`函数来实现。

此函数接受一个可等待对象的集合。如果它们是协程，它们将被作为任务发出。

然后，该函数返回一个可迭代对象，其中的协程按它们完成的顺序产出。

我们可以直接遍历这个可迭代对象，不需要使用为异步可迭代对象保留的`async for`表达式。

例如：

```python
...
# execute coroutines and handle results as they complete
for coro in asyncio.as_completed(coros):
    # check the return value from the coroutine
    # ...
```

缺点是我们没有简单的方法将协程与测试的端口关联起来。因此，我们可以更新我们的`test_port_number()`协程，使其返回端口是否开放以及测试的端口号。

已测试。

例如：

```python
# 如果连接可以建立则返回 True
async def test_port_number(host, port, timeout=3):
    # 创建用于打开连接的协程
    coro = asyncio.open_connection(host, port)
    # 带超时执行协程
    try:
        # 打开连接并等待片刻
        _, writer = await asyncio.wait_for(coro, timeout)
        # 连接打开后关闭它
        writer.close()
        # 表明连接可以打开
        return True, port
    except asyncio.TimeoutError:
        # 表明连接无法打开
        return False, port
```

然后我们可以按照协程完成的顺序遍历它们，并从每个协程获取端口详情及其是否开放，然后进行报告。

例如：

```python
...
# 执行协程并在它们完成时处理结果
for coro in asyncio.as_completed(coros):
    # 检查协程的返回值
    result, port = await coro
    if result:
        print(f'> {host}:{port} [OPEN]')
```

这将并发执行所有协程，并在发现开放端口时立即报告，而不是等到最后才报告。

将以上内容整合，完整的示例如下。

```python
# SuperFastPython.com
# 使用 as_completed 的并发端口扫描器示例
import asyncio

# 如果连接可以建立则返回 True
async def test_port_number(host, port, timeout=3):
    # 创建用于打开连接的协程
    coro = asyncio.open_connection(host, port)
    # 带超时执行协程
    try:
        # 打开连接并等待片刻
        _, writer = await asyncio.wait_for(coro, timeout)
        # 连接打开后关闭它
        writer.close()
        # 表明连接可以打开
        return True, port
    except asyncio.TimeoutError:
        # 表明连接无法打开
        return False, port

# 主协程
async def main(host, ports):
    # 报告状态消息
    print(f'Scanning {host}...')
    # 创建所有协程
    coros = [test_port_number(host, port)
        for port in ports]
    # 执行协程并动态处理结果
    for coro in asyncio.as_completed(coros):
        # 检查协程的返回值
        result, port = await coro
        if result:
            print(f'> {host}:{port} [OPEN]')

# 定义要扫描的主机和端口
host = 'python.org'
ports = range(1, 1024)
# 启动 asyncio 程序
asyncio.run(main(host, ports))
```

运行该示例会执行 `main()` 协程作为 asyncio 程序的入口点。

首先创建一个协程列表。

然后将这些协程传递给 `asyncio.as_completed()` 函数。

该函数将每个协程包装在另一个协程中，并并发且独立地执行它们。

它立即返回一个协程的可迭代对象。

在内部，它等待协程完成并逐个产生结果。

检索每个协程的返回值，并在结果可用时立即报告。

该示例显示的开放端口与之前的并发示例相同，执行时间也大致相同，但程序的响应性更好。

端口几乎立即显示为开放状态，而不是等到范围内的所有端口都被检查并超时后才显示。
