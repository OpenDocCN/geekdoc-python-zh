访问 python.org 并下载最新版本的Python（它是免费的）。像安装任何其他程序一样将其安装到你的计算机上。

在访问网站时，你可能想浏览在线文档。点击几下就能到达 docs.python.org/3/，从那里点击库参考链接，你会看到一个长长的模块列表，每个模块都包含许多有用的方法。

在你的Python下载中，你会找到一个简单的*IDE（集成开发环境）*，名为IDLE。你有四个选择：

- 你可以花五分钟学习如何使用IDLE。如果你赶时间，我推荐这样做。

## 2 ■ 快速入门 Python 3

- 如果你已经熟悉专业的IDE，如Eclipse、IntelliJ IDEA、NetBeans或Jupyter Notebook，会有多种Python插件可供选择。你可以花费无限的时间来选择、下载、安装和配置你的新插件。我建议在你对Python有了一些熟悉之后再这样做。

### 1.2 IDLE

当你运行IDLE时，你会得到一个“Shell”窗口，你可以在>>>提示符下输入表达式或语句。

- 如果你输入一个表达式，如2+2，IDLE会响应4和一个新的提示符。
- 如果你输入一个语句，如x=2+2，IDLE只响应一个新的提示符。（你可以通过输入表达式x来查看赋的值。）

这是测试少量代码的好方法。

要编写完整的程序，选择File > New File并在那里输入你的代码。你必须在运行之前保存文件。扩展名.py会自动添加到文件名中，或者你可以自己添加。通过选择Run > Run Module或按F5来运行你的程序。

> **专业提示：** 每次你对程序进行更改时，都会在再次运行之前提示你保存。你可以通过转到Preferences > General，在At Start of Run (F5)处选择No Prompt来跳过此步骤。

花几分钟浏览一下菜单；里面有一些可以节省你时间的东西。请注意，菜单会根据你是在shell窗口还是文件窗口中而有所不同。

### 1.3 变量

变量没有类型，也不需要声明。当你给变量赋值时，变量就存在了。

**示例：** best_value = 36

**约定：** 变量名以小写字母开头，或在特殊情况下以下划线开头。对于多词变量名，大多数程序员使用下划线分隔单词（如上），但有些人使用“camelCase”（有时称为“studlyCaps”），如bestValue。

**注意：** Python的内置类型名和函数名不是保留字，很容易意外覆盖它们。例如，如果你使用list或abs作为变量名，你就不能再使用这些名称来创建列表或求绝对值。

### 1.4 重要的数据类型

基本数据类型与你已经习惯的类似：

- int（整数）：73，-15，12345678901234567890。整数可以任意大。
- float（实数）：3.1416，6.022e+23。
- bool（逻辑值，或布尔值）：True和False。注意这些是大写的。
- str（字符串）："Hello World"，'Goodbye'。你可以使用单引号或双引号。你可以在双引号字符串中插入单引号，反之亦然。没有单独的“字符”类型。
- list（类数组列表）：[1, 2, "Buckle my shoe"]。列表不完全是数组，但你可以这样对待它们。列表是零索引的，所以如果my_list保存上述列表，my_list[0]是1。

还有一个简单但不寻常的类型：

- NoneType，具有单个值None。None是一个实际的值，可以赋给变量；它*不是*尚未赋值的变量的默认值。在Python中，每个函数都返回一个值，None最常见于没有显式返回值的函数的结果。

其他数据类型（元组、集合和字典）将在后面讨论。

### 1.5 基本算术

- **加法**（+）、**减法**（-）、**乘法**（*）和**乘方**（**）。如果两个操作数都是整数，结果就是整数，否则结果是浮点数。
- **除法**（/）。使用/进行除法的结果始终是浮点数。
- **整数除法**（//）。两个整数相除的结果是结果的整数部分，向下取整。如果//的一个或两个操作数是浮点数，结果是浮点数，向下舍入到最接近的整数值。因此，10 // 4是2，而10.0 // 4是2.0。
- **取模**（%）。两个整数“取模”的结果是除法的余数。因此，20 % 7是6，因为7进入20两次，剩下6。如果%的一个或两个操作数是浮点数，结果是浮点数。*%运算符与百分比无关。*
- **运算符优先级：** 在复杂表达式中，
  - 乘方首先执行，然后是乘法和除法（包括取模），然后是加法和减法。
  - 多个乘方从右到左执行；也就是说，2**3**4意味着2**(3**4)。
  - 其他相同优先级的操作从左到右执行，所以10-5-2意味着(10-5)-2。
  - 如果有疑问，使用括号来控制运算顺序。

除了/（总是产生浮点数）的情况外，如果两个操作数都是整数，操作的结果将是整数，否则将是浮点数。

> **注意：** 不同语言*对于使用//或%运算符与负数时的结果并不一致*！避免这种情况，或者如果无法避免，请进行实验直到你确定Python中的规则。然后在你的代码中添加适当的注释。

> **风格：** 在所有算术和比较运算符（包括赋值运算符=）周围放置空格。这使你的代码更易于阅读。

类型之间的转换可以使用`float`、`int`、`round`和`str`函数进行。例如：

- `float(123)`、`float("123")`和`float("123.0")`都返回浮点数123.0。
- round(123.45678, 3)返回浮点数123.457，四舍五入到小数点后三位。这对于更整洁地打印数字很有用。
- int(56.78)和int("56")都返回整数56。从浮点数转换时，小数点后的数字被丢弃。
- round(56.78)返回57。浮点数四舍五入到最接近的整数。
- str(arg)将返回其参数的字符串版本。几乎所有东西，而不仅仅是数字，都可以转换为字符串表示。

### 1.6 比较

比较：

- 等于（==），不等于（!=），小于（<），小于或等于（<=），大于（>），大于或等于（>=）。
- 链式比较：与大多数语言不同，Python允许你链式比较，例如，2 < 3 < 4 > 2。所有比较都是成对执行的，如果每个单独的比较都是True，则结果为True。

### 1.7 布尔值

两个布尔（逻辑）值是True和False。

Python具有布尔运算符not、and和or，具有通常的含义。and和or运算符是短路的；也就是说，只有在必要时才评估第二个操作数。

- 在2 == 2 or f(x)中，第一部分（2 == 2）使整个表达式为True（因为True或任何东西都是True），所以第二部分f(x)永远不会被评估。
- 类似地，在2 == 5 and f(x)中，第一部分（2 == 5）使整个表达式为False，所以第二部分f(x)永远不会被评估。

比较运算符（<, <=, ==, !=, >=, >）也适用于布尔值；True大于False。

在数值表达式中，True的值为1，False的值为0。在布尔表达式中，所有数值零值为假，所有非零值为真。使用这些事实会使你的代码不够明确。

特殊值None表示“无值”，被视为假。

**if表达式**根据给定的**条件**是否为真，产生两个值之一。语法是：

```
valueIfTrue if condition else valueIfFalse
```

### 1.8 列表

**列表**写为方括号中的一系列值。

列表可以像数组一样对待。如果my_list是一个包含100个值的列表，第一个元素是my_list[0]，最后一个元素是my_list[99]。你可以使用括号表示法来访问列表元素并更改其值；例如，

```
my_list[5] = my_list[4] + my_list[3]
```

方便的是，列表可以从另一端访问，使用负数：my_list[-1]与my_list[99]相同，my_list[-2]与my_list[98]相同，依此类推。

列表的长度由函数 `len` 给出，因此 `len(my_list)` 的值为 100。

### 1.9 字符串

*字符串* 是用引号括起来的一系列表示字符的序列。字符串可以使用 `+` 运算符进行拼接（相加）；`"Hello" + "World"` 的结果是 `"HelloWorld"`。你也可以省略字面量（带引号的）字符串之间的 `+`，只用空格或制表符分隔它们。（如果字符串被括在圆括号内，也可以使用换行符）。

在 Python 中，你可以用单引号（`'...'`）、双引号（`"..."`）、三单引号（`'''...'''`）或三双引号（`"""..."""`）来括起字符串。你可以在双引号字符串内插入单引号，在单引号字符串内插入双引号，或者在三引号字符串内插入任意一种。三引号字符串可以跨越多行。Python 在打印结果时通常使用单引号。

没有单独的“字符”类型。

特殊字符可以通过“转义”（在它们前面加上反斜杠）的方式包含在字符串中。

- `\n` 是一个“换行”字符，具体编码取决于你的计算机系统。
- `\t` 是一个制表符。它是一个字符，但它所代表的空格数量取决于查看代码的应用程序中制表位的设置。
- `\"` 是一个双引号字符（在双引号字符串内很有用）。
- `\'` 是一个单引号字符（在单引号字符串内很有用）。
- `\\` 是一个反斜杠字符。
- `\uxxxx` 或 `\uxxxxxxxx` 是一个 Unicode 字符（四个或八个十六进制数字）。

字符串可以像列表一样被索引。`'HelloWorld'[0]` 是 `'H'`。

### 1.10 注释

**注释** 以井号 `#` 开头，并持续到行尾。

### 1.11 语句类型

**缩进很重要。** 同一级别的语句必须缩进完全相同的量。嵌套语句，例如循环体内的语句，必须相对于循环的开始进行缩进。在 Java 和 C 系列语言使用花括号的地方，Python 使用缩进。

标准缩进是四个空格。**你 *不能* 混用空格和制表符。** 然而，任何 IDE 和任何好的文本编辑器都可以设置为在你按下键盘上的 Tab 键时插入四个空格。

程序的第一行不能缩进。

每条语句单独写在一行上，但如果该行包含未闭合的 `(`、`[` 或 `{`，则可以在下一行继续。

如果你想把两条语句放在同一行，可以用分号 `;` 分隔它们。如果你习惯于每条语句都以分号结尾的语言，这在 Python 中也能用，只是在经验丰富的 Python 程序员看来会很奇怪。

如果你使用的是 IDLE，它会在任何以冒号结尾的行之后自动缩进。如果它没有缩进，那意味着你忘记了冒号。

注意。虽然我已尽力保持代码行简短，但有些语句可能会换行到下一行，尤其是在窄屏幕上阅读时。不要被这个误导。

注意。从本书的电子版复制粘贴代码可能无法正常工作。所有代码都经过测试，但在某些情况下，为了保留缩进，添加了不可见的格式代码。

与其抽象地谈论语法，不如直接看每种语句类型的示例更容易理解。

#### 1.11.1 赋值语句

**赋值语句** 由一个 *变量*、一个 *等号* 和一个 *表达式* 组成。例如，

```
x = 1
```

赋值快捷方式使用一个运算符和一个单独的 `=` 号。

- `x += y` 是 `x = x + y` 的快捷方式
- `x -= y` 是 `x = x - y` 的快捷方式
- ...对于其他二元运算符也类似。

“海象运算符” `:=` 是一个可以在表达式中使用的赋值运算符，例如，`x = (n := 3 * n) + 1` 会将 `n` 的当前值乘以 3，将结果赋回 `n`，然后加 1 并将结果放入 `x`。如果 `n` 初始值为 10，结果将是 `n` 被设置为 30，`x` 被设置为 31。

#### 1.11.2 打印“语句”

**打印“语句”** 实际上是对名为 `print` 的函数的调用，但每个人都把它们当作语句使用。你可以给 `print` 任意数量的参数，它们都会被打印在同一行，中间用空格分隔。

例如，如果 `x` 的值为 5，那么函数调用 `print('x =', x)` 将打印 `x = 5`。

每个打印的行都以换行符结尾。如果你希望接下来的打印语句继续在同一行，请使用 `end=""` 作为最后一个参数。

`print()` 将打印一个空行。

#### 1.11.3 If 语句

**If 语句** 使用关键字 `if` 和（可选的）`elif` 和 `else`。

```
if n == 1:
    print("One")
elif n == 2:
    print("Two")
elif n == 3:
    print("Three")
else:
    print("Many")
    print("or maybe none at all")
```

你可以使用任意多个 `elif`，每个都有一个关联的条件，并且可以有一个最终的 `else`。每个 `if`、`elif` 或 `else` 行都以冒号（`:`）结尾。每个分支下的语句必须缩进。

#### 1.11.4 While 循环

**While 循环** 在条件为真时持续执行。例如，

```
n = 5
while n > 0:
    print(n)
    n -= 1
print("Blast off!")
```

海象运算符可以在 while 循环中使用。上面的代码等价于

```
n = 6
while (n := n - 1) > 0:
    print(n)
print("Blast off!")
```

#### 1.11.5 For 循环

**For 循环** 对列表、字符串、元组或其他可迭代类型的每个元素执行一次其循环体。

```
for i in [5, 4, 3, 2, 1]:
    print(i)
print("Blast off!")
```

或

```
for i in range(5, 0, -1):
    print(i)
print("Blast off!")
```

`range` 函数有三种形式：

- `range(b)` 将给出数字 0, 1, 2 等，直到 *但不包括* `b`。
- `range(a, b)` 将给出数字 `a`, `a`+1, `a`+2 等，直到 *但不包括* `b`。
- `range(a, b, c)` 将给出数字 `a`, `a`+`c`, `a`+`c`+`c`, `a`+`c`+`c`+`c` 等，直到（或直到） *但不包括* `b`。

**技术说明：** `range` 返回的值不是一个列表，而是一个迭代器（稍后会详细介绍）。如果你需要一个列表，请使用 `list(range(...))`。

#### 1.11.6 Import 语句

Python 有一些函数是“内置”的。例如，`abs(n)` 会给你数字 `n` 的绝对值。如果 `x` 是一个列表，`len(x)` 将返回列表的长度；或者如果 `x` 是一个字符串，`len(x)` 将返回字符串中的字符数。

还有更多函数可用，但不是内置的；你必须从一个模块中 **导入** 它们。（“模块”只是一个包含代码的文件。）例如，平方根和自然对数函数 `sqrt` 和 `log` 可以从 `math` 模块导入。

```
from math import sqrt, log
```

现在，如果你想求一个数字 `x` 的平方根，只需说 `sqrt(x)`。

对于小程序，很容易从一个模块中一次性导入所有内容。例如，

```
from math import *
```

这种形式的 import 语句不推荐用于较大的程序。除了不够明确之外，还有无意中从不同模块导入同名（但含义不同）的风险。

标准 Python 发行版附带大约 300 个模块；还有更多可用模块，包括商业和免费的；其中一些在附录中有所描述。在编写代码解决特定领域的问题之前，最好先检查一下已经有哪些可用的。

### 1.12 从用户获取输入

要向用户请求输入，你可以调用函数 `input(prompt)`。例如，

```
name = input("What is your name? ")
print("Hello,", name)
```

你可以省略提示，这种情况下用户将面对一个空白屏幕，想知道为什么程序没有做任何事情。

调用 `input` 的结果总是一个字符串。如果你期望一个整数或浮点数，你可以使用 `int` 或 `float` 函数将字符串转换为所需的类型。例如，

```
question = "How old are you, " + name + "? "
age = int(input(question))
```

如果用户输入的内容无法转换为整数，将会导致错误。我们将在后面讨论错误处理。

### 1.13 函数

要定义一个函数，使用关键字 `def`、函数名、括号括起来的参数列表和一个冒号。后面跟着缩进的函数体。当你有值要返回时，将值放在 `return` 语句中。例如：

```
def largest(a, b, c):
    """Return the largest of a, b, and c."""
    if a >= b and a >= c:
        return a
    elif b >= a and b >= c:
        return b
    return c
```

（我们本可以将最后的 return 语句放入 else 部分，但没有必要。如果 if-elif 语句没有执行任何操作，它就会被执行。）

函数的第二行是一个**文档字符串**，用于记录函数的目的。按照惯例，即使字符串只有一行，也使用三引号字符串。如果文档字符串超过一行，所有行的缩进量应相同，并且结束的三引号应单独占一行。

文档字符串应写成完整的句子。第一句（以句号结尾）被视为摘要行。以这种方式记录每个函数是可选的，但强烈推荐。对于任何有文档字符串的函数，`help(function_name)` 将打印该字符串。

每个函数都返回一个值。如果你没有指定返回值，函数将返回 `None`。（这是一个实际的合法值，类型为 `NoneType`，因此你可以将其赋值给一个变量，或询问一个变量是否等于它。）

函数可以在另一个函数内部定义，然后它就是包含它的函数的局部函数。

### 1.15 执行顺序

Python 程序中的所有内容并不都必须在函数内部；你可以（并且通常会）有一些“顶层”语句。非常短的程序可能完全由顶层语句组成，没有函数定义。

Python 程序从开头（第一行）到结尾（最后一行）进行评估。顶层语句将在出现时进行评估。评估函数定义（`def`）会导致函数被定义，但直到其他语句调用它时才会执行。

函数是动态定义的，也就是说，当 `def` 被评估时，并且必须在调用之前定义。与大多数语言一样，函数的*词法顺序*（它们在列表中出现的顺序）没有限制。

程序通常由一组函数组成，程序的最后一行是对“main”函数的单个顶层调用。通常这个“main”函数被命名为 `main`，但并非总是如此。

### 2.1 对象表示法

列表和字符串（你已经见过），以及集合和字典（即将介绍）是**对象**。谈论对象有专门的术语：

- *方法*是在对象上定义的函数。
- 你不是“调用”方法，而是*向对象发送消息*。这很笨拙，所以我喜欢说你*与对象交谈*。

与此一起，使用对象有特殊的表示法：

1. 命名你正在交谈的对象。
2. 放一个句点（`.`）。
3. 写出方法名称，以及任何参数（逗号分隔，放在括号内）。

例如，如果 `s` 是字符串 `"Hello World"`，那么 `s.lower()` 是字符串 `"hello world"`。你可以将其理解为：“嘿，`s`，给我一个你的小写版本。”

### 2.2 列表

Python **列表**是一个可索引的值序列，不一定都是相同类型。第一个索引是 0。你可以通过编写一个用逗号分隔的值序列并用方括号 `[]` 括起来来创建一个列表。

示例：

- `s = ["Mary", 23, "A+"]`
- `name = s[0]` # `name` 现在是 `"Mary"`
- `s[1] += 1` # `s` 现在是 `["Mary", 24, "A+"]`
- 空列表写为 `[]`。

你可以通过将列表“乘以”一个整数来创建任意大小的列表。整数必须在 `*` 运算符的右侧。

- `[0] * 4` # 结果是 `[0, 0, 0, 0]`
- `[None] * 100` # 结果是 100 个 `None`
- `[2, 4] * 3` # 结果是 `[2, 4, 2, 4, 2, 4]`

如果 *x* 是一个列表，你说 *y = x*，那么 *y* 就成为*同一列表的另一个名称*，而不是 *x* 的副本。也就是说，你对 *y* 所做的任何更改也会更改 *x*，反之亦然。

以下是列表上两个有用的*函数*：

- `len(**my_list**)` 返回列表中的元素数量。
- `sorted(**my_list**)` 返回列表的副本，元素按升序排列。

以下是列表上两个有用的*方法*：

- `**my_list**.append(*value*)` 将 *value* 添加到 `**my_list**` 的末尾并返回 `None`。
- `**my_list**.pop()` 移除 `**my_list**` 的最后一个元素并返回它。
- `**my_list**.sort()` 对列表进行排序并返回 `None`。

你可以获取列表的“切片”，这会给你一个包含原始列表部分元素的新列表。如果 `**my_list**` 是一个列表，*i* 和 *j* 是整数，那么

- `**my_list**[*i*:*j*]` 是从 `**my_list**[*i*]` 到但不包括 `my_list[*j*]` 的元素的副本。
- `**my_list**[*i*:]` 是从 `**my_list**[*i*]` 开始并持续到末尾的元素的副本。
- `**my_list**[:*j*]` 是从 `**my_list**[0]` 到但不包括 `**my_list**[*j*]` 的元素的副本。
- `**my_list**[:]` 是整个列表的副本。

负数可以用作索引，-1 表示列表的最后一个元素。可以包含步长，如 `**my_list** [*i*:*j*:*step*]`，*step* 也可以是负数。

你也可以在赋值运算符的左侧使用切片表示法。如果你将一个值列表赋值给一个切片，这些值将替换切片中的值。值列表的长度不必与它替换的切片长度相同。例如，如果 `x = [0, 1, 2, 3]`，你赋值 `x[1:3] = [11, 22, 33]`，结果将是 `x` 变为 `[0, 11, 22, 33, 3]`。

**专业提示：** 你可以将字符串视为字母列表，例如，`"abcdef"[2:5]` 是 `"cde"`。

由于任何值都可以放入列表中，你可以创建一个列表的列表，例如，`grades = [["Mary", "A+"], ["Donald", "C-"]]`。要获取单个元素，你必须分别索引外部和内部列表：`grades[1]` 是 `["Donald", "C-"]`，所以 `grades[1][0]` 是 `"Donald"`。

> **注意：** *不要*尝试通过使用 `[[None] * 3] * 2` 这样的构造来创建列表的列表。它看起来可以工作，但实际上外部列表中的每个值都是对同一个内部列表的引用。

你可以用这样的代码创建一个“二维”列表：

```python
x = [0] * 3
for i in range(0, 3):
    x[i] = [0] * 5
```

初始赋值创建一个零数组，然后循环将每个零替换为一个零数组。

### 2.3 元组

*元组*由零个或多个值组成，用逗号分隔并用括号括起来；例如，`("John", 23)`。当元组单独出现在赋值的左侧或右侧时，可以省略括号，例如，

```python
a, b, c = 1, 2, 3
```

**技术说明：** 零元组写为 `()`，而一元组通过在单个值后放置逗号来写，像这样：`(3,)`。这些形式通常可能不是很有用。

元组对于将少量值组合成一个单元非常有用。例如，你可能想编写一个函数，返回对象的x-y坐标，或者列表中的最大值和最小值。

内置函数`divmod`返回一个元组。7除以20得2余6（20 // 7 == 2 且 20 % 7 == 6）。如果你需要这两个值，可以使用`divmod(20, 7)`，它返回元组`(2, 6)`。

以下是一些你可以对元组执行的操作：

- x = (1, 2, 3) # x 现在持有元组 (1, 2, 3)
- x = 1, 2, 3 # 与上面相同
- a, b, c = x # a 现在是 1，b 是 2，c 是 3
- a, b, c = "xyz" # a 现在是 "x"，b 是 "y"，c 是 "z"
- y = x[1] # y 现在是 2（元组从零开始索引）
- z = x[1:] # 可以使用切片；z 现在是 (2, 3)
- a, b = b, a # a 和 b 的值互换

元组是**不可变的**：你不能更改其内容。例如，如果 x 是 (1, 2, 3)，那么 x[0] = 5 是非法的。

### 2.4 集合

集合是一组值的集合，这些值不一定都是同一类型。集合字面量写作花括号包围的逗号分隔列表。例如：

```
s = {10, "ten", "X"}
```

空集合（没有元素的那个）不能写作 `{}`，因为那是一个空字典（见下一节）。请改用 `set()`。

集合有两个重要属性：

- 集合没有重复元素。一个值要么*在*集合中，要么*不在*集合中。实际上，`in` 和 `not in` 是运算符，所以你可以询问 `10 in s`（结果为 True）或 `"TEN" not in s`（也为 True）。
- 集合中元素的顺序是不确定的，因此你不能对集合进行索引。

你可以使用 `for` 循环依次遍历集合的每个元素，只要你不关心元素的处理顺序。

```
for elem in s:
    print(elem)
```

如果元素的顺序很重要，请使用列表而不是集合。要在列表和集合之间转换，请使用 `list` 和 `set` 函数。

如果你熟悉数学中的集合，Python 提供了最常见的操作：并集、交集、差集和对称差集。这些可以写成运算符或方法调用。

- *set1* | *set2* 或 *set1*.union(*set2*) 返回在 *set1* 或 *set2* 中，或两者中都存在的元素集合。
- *set1* & *set2* 或 *set1*.intersection(*set2*) 返回同时在 *set1* 和 *set2* 中的元素集合。
- *set1* - *set2* 或 *set1*.difference(*set2*) 返回在 *set1* 中但不在 *set2* 中的元素集合。
- *set1* ^ *set2* 或 *set1*.symmetric_difference(*set2*) 返回恰好在一个集合中的元素集合。
- *set1*.issubset(*set2*) 如果 *set1* 的每个元素也在 *set2* 中，则返回 True。
- *set1*.issuperset(*set2*) 如果 *set2* 的每个元素也在 *set1* 中，则返回 True。

比较运算符（<, <=, ==, !=, >=, >）也可用于测试两个集合之间的子集/超集和相等/不相等关系。如果 *set1* 是 *set2* 的*真*子集，即 *set1* 是 *set2* 的子集但不等于它，则 *set1* < *set2* 为 True。

元素也可以添加到集合中或从集合中移除。

- *set*.add(*element*) 将 *element* 添加到 *set*。如果 *element* 已经在 *set* 中，则此操作无效。
- *set*.discard(*element*) 如果 *element* 存在，则从 *set* 中移除 *element*。如果 *element* 不在 *set* 中，则此操作无效。
- *set*.remove(*element*) 从 *set* 中移除 *element*，如果 *element* 不在 *set* 中，则引发 KeyError。
- *set*.pop() 从 *set* 中移除并返回一个*任意元素*，如果 *set* 为空，则引发 KeyError。你的代码不应依赖于返回某个特定元素。
- *set*.clear() 从 *set* 中移除所有元素。

### 2.5 字典

*字典*（类型为 dict）提供了一种快速查找内容的方式。字典字面量写作花括号 `{ }` 内的零个或多个键:值对。例如，你可以如下定义一个“电话簿”字典：

```
phones = {"Alice": 5551212, "Jill": 5556789, "Bob": 5559999}
```

键必须是唯一的；一个键只能关联一个值。值不必是唯一的；多个键可以具有相同的值。

一旦你有了一个字典，有三种方法可以在其中查找内容。特别注意括号 `( )` 和方括号 `[ ]` 的使用位置。

- `phones["Jill"]` 将返回 5556789。如果找不到键，例如你尝试 `phones["Xavier"]`，将导致 KeyError。
- `phones.get(key)` 的工作方式类似于 `phones[key]`，不同之处在于如果找不到键，则返回值 `None`。
- `phones.get(key, default_value)` 返回与键关联的值，如果没有该键，则返回 `default_value`。

你可以通过赋值向字典添加内容：

```
phones["Xavier"] = 5556666
```

这也适用于更改与键关联的值。例如，你可以用这种方式更新某人的电话号码。

有一个特殊命令 `del`，用于从字典中删除内容：

```
del phones["Xavier"]
```

如果键不在字典中，`del` 命令将引发 KeyError，因此在使用 `del` 前请检查。（你可以使用集合运算符 `in` 和 `not in` 来测试键是否在字典中。）

与集合一样，字典的元素以 Python 认为最佳的任何顺序存储。这可能因实现而异。

**并非所有内容都可以用作键。** 键必须是*不可变的*。这本身是一个很长的讨论，但简短的形式是：字符串、数字、元组和布尔值是很好的键。任何其组成部分可能更改的对象（如列表、集合和字典）*不能*用作键。

### 2.6 字符串方法

你可以对字符串执行的大多数操作是方法，而不是函数。也就是说，它们的形式是 *string.method(arguments)*。

字符串方法有很多。许多最有用的方法列在附录 A 中。这里，我们只讨论几个最有用和/或最令人困惑的字符串方法。

- *string1.split(string2)* 返回 *string1* 的子字符串列表，这些子字符串由 *string2* 分隔。如果省略 *string2* 参数，则使用空白作为分隔符。当你想要一个字符串列表时，这可以节省输入。例如，`'one two three'.split()` 返回列表 `['one', 'two', 'three']`。
- *string.join(list_of_strings)* 返回一个单一字符串，其中 *list_of_strings* 的元素由 *string* 分隔。例如，`'<='.join(['a', 'b', 'ab'])` 返回字符串 `'a<=b<=ab'`。这可能令人困惑，因为 *string* 最常见的是单个空格 `' '` 或空字符串 `''`。

格式化字符串，或 *f-string*，以 f 或 F 为前缀。在 f-string 中，花括号 `{}` 内的*任何表达式*都会被其值替换。例如，字符串 `'pi is {round(pi, 4)}.'` 的结果是 `'pi is 3.1416.'`。

要在 f-string 中放入花括号字符，请将其加倍。

### 2.7 对象的循环

可迭代对象，如列表、元组、集合和字典，可以使用 `for` 循环进行迭代。

#### 2.7.1 遍历列表

如果你想对列表或其他可迭代类型的每个元素执行某些操作，可以使用 `for` 循环。一个简单的 `for` 循环如下所示：

```
for e in my_list:
    print(e)
```

如果你不仅需要处理列表中的元素，还需要处理它们在列表中的位置，可以使用更复杂的 `for` 循环版本：

```
for i in range(0, len(my_list)):
    print(my_list[i], "is at", i)
```

`enumerate` 函数提供了一种更好的获取 (**index, value**) 元组的方法。上面的代码可以替换为

```
for index, value in enumerate(my_list):
    print(value, "is at", index)
```

请注意，返回的元组周围的括号是可选的。

简单通常更好，因此除非你确实需要索引，否则不要使用获取索引的循环。

#### 2.7.2 遍历集合

`for` 循环的简单形式适用于集合：

```
for e in my_set:
    print(e)
```

`enumerate` 方法也适用于集合。然而，值得重申的是，集合不能被下标访问，应始终被视为没有确定的顺序。

#### 2.7.3 遍历字典

简单的循环也适用于字典，但在这种情况下，赋值给循环变量的只是键，而不是整个 **键:值** 对：

```
for k in my_dict:
    print(k)  # 仅打印键
```

为了更明确，你可以使用字典的 `keys` 方法：

```
for k in my_dict.keys():
    print(k)  # 仅打印键
```

你也可以只使用 `values` 方法打印值：

```
for v in my_dict.values():
    print(v)  # 仅打印值
```

你可以通过只遍历键，然后为每个键查找值来打印键和值：

```
for k in my_dict:
    print(k, "->", my_dict[k])
```

字典有一个额外的方法，items。这个方法看起来返回一个包含（键，值）元组的列表：

```
for t in my_dict.items():
    print(t) # 打印（键，值）元组
```

实际上，items 返回的不是一个列表，而是字典项的**动态视图**。返回的每个项实际上是一个（键，值）元组，但如果你在遍历字典时修改了字典的内容，可能会得到一些意想不到且不受欢迎的结果。

元组可以在 for 循环中直接解包。元组周围的括号可以省略。

```
for k, v in my_dict.items():
    print(k, "is", v)
```

### 2.8 处理异常

错误会发生。程序可能尝试将一个数除以零，或者向 None 发送消息，或者读取一个不存在的文件。当这种情况发生时，Python 会**引发一个异常**。

每种异常类型都有一个名称。在 Python 中，“错误”和“异常”之间几乎没有区别。两者，如果不处理，都会导致程序终止。

如果你知道错误可能在哪里发生，你可以处理它。例如，你可能会使用 input 函数要求用户输入一个整数。无论用户输入什么，都会作为字符串返回，你可以使用 int 函数将其转换为整数——除非用户输入了非数字字符，否则你会得到一个 ValueError。

要处理这个问题，你可以使用 try-except 或 try-except-finally 语句。它的一般形式如下：

```
try:
    # 可能出错的代码
except SomeErrorType:
    # 出错时该怎么做
finally:
    # 之后该怎么做
```

finally 部分是可选的。如果存在，无论是否发生错误，它都会被执行。

这是一个例子：

```
number = None
while number is None:
    try:
        n = input("Enter an integer: ")
        number = int(n)
    except Exception:
        print("Try again!")
print("Your number is", number)
```

详细来说，上面的例子是这样工作的：

- 变量 number 被设置为 None。
- 进入 while 循环时，number 等于 None，因此循环体被执行。
- input 函数等待用户输入内容，然后将其（作为字符串）存储在 n 中。
- int 函数尝试将字符串 n 转换为数字。
- 如果 int 成功，
  - 结果被放入 number，except 部分被跳过，控制权返回到循环顶部，
  - 由于 number 不再等于 None，循环退出。
- 如果 int 失败，
  - int 引发一个异常，
  - 控制权转到 except 部分，跳过 try 部分中可能剩余的任何语句，
  - 打印出 "Try again!"，控制权返回到循环顶部，
  - 由于 number 仍然等于 None，循环体再次执行。
- 当用户输入一个有效的整数时，循环退出并打印出用户输入的数字。

以下是一些需要记住的重要点：

- 如果在 try 和 except 之间的任何地方发生错误，控制权会*立即*转到 except 部分。try 部分中任何剩余的代码将不会被执行。
- *finally* 部分中的代码将*始终*被执行。如果 try 或 except 部分中的代码执行了 return 语句，finally 中的代码将在函数返回之前执行。

如果错误发生在函数中，但错误不在 try-except 语句内，那么异常会被传递到调用位置。如果该位置在 try-except 语句内，异常将在那里被处理，否则该函数也会立即返回到其调用位置。这样，每个函数都将异常携带到其调用位置，一直沿着调用序列向上，直到其中一个捕获了异常。如果异常从未被捕获，程序将以错误终止。

Python 定义了*大量*的异常类型，以便你可以为每种异常类型做不同的处理。对于简单的程序，你可以只使用通用类型 Exception，但这不推荐用于较大的程序。

通常，处理错误的难点在于弄清楚当错误发生时该做什么。

### 2.9 类型转换

当作为函数调用时，类型*名称*充当构造函数。给定一个不同类型的参数，该函数将（在可能的情况下）返回指定类型的值。例如：

- int(5.7) # 返回 5
- int("5") # 返回 5
- float(5) # 返回 5.0
- float("5.7") # 返回 5.7
- bool("False") # 返回 True
  - 以下内容被视为假：False、None、0、0.0、空字符串、空列表、空集合、空元组和空字典。
- str([1, 2, 3]) # 结果是 '[1, 2, 3]'

其他可用作转换函数的数据类型包括 list、set、tuple 和 dict（字典）。

- 当你从 dict 转换为其他任何类型时，你只能得到键，而不是值。
- 只有当元素成对分组时，你才能将列表、集合或元组转换为 dict，例如，一个包含 2-元组（具有两个元素的元组）的列表。
- 集合没有内在的顺序，因此转换为集合或从集合转换不一定保留元素的顺序。

函数 type(x) 将返回一个可以与类型名称（而不是该名称的字符串表示）进行比较的值。例如，测试 type(5) == int 将返回 True，但 type(5) == "int" 将返回 False。

函数 isinstance(value, type) 测试 value 是否是命名类型。要测试值是否是多种简单类型之一，类型可以是这些类型的元组 (type1, type2, ...)，或类型联合 type1 | type2 | ...，在这种情况下，isinstance 测试 value 是否是元组中任何命名类型之一。但是，isinstance 不能测试泛型类型，如 list[int]。

### 2.10 作用域

名称的作用域是程序中该名称有意义且可以使用的部分。Python 的作用域规则不寻常；它们遵循“LEGB 规则”：Local（局部）、Enclosed（闭包）、Global（全局）、Built-in（内置）。要理解这个规则，记住以下两点会有所帮助：(1) 变量通过赋值来定义；(2) 函数/方法可以嵌套在其他函数/方法中。

- 变量可以通过 global var1, ..., varN 或 nonlocal var1, ..., varN 形式的语句声明为全局或非局部。声明必须在变量的任何使用之前。
- **局部：** 如果一个变量是参数，或者在该函数内被赋值且未明确声明为全局或非局部，则该变量是函数的局部变量。
- **闭包：** 函数中的变量可以引用在封闭函数中声明的变量，前提是 (1) 该变量未在此函数中被赋值，或者 (2) 它在 nonlocal 语句中被声明。
- **全局：** 在顶层（不在函数或方法中）声明的变量是全局变量，可以在整个程序中引用。函数中的变量可以引用全局变量，前提是 (1) 该变量未在此函数或封闭函数中被赋值，或者 (2) 它在 global 语句中被声明。
- **内置：** 程序中未以其他方式声明的变量可能是内置变量。一些例子是 list、print 和 divmod。

当变量是非局部（闭包）或全局时，即使该变量未在函数内被赋值，将其声明为非局部或全局也是良好的文档实践。

全局变量通常被认为是不可取的，应尽可能避免使用。使用全局变量的函数：

- 不再是自包含的，只能在上下文中理解和调试；
- 更难测试，因为测试必须包括正确设置全局变量的代码；并且
- 更难在另一个项目中重用，因为它们依赖于上下文。

可以通过向函数添加额外参数和向类添加实例变量来避免全局变量。

一些程序员使用以每个全局变量名称开头加下划线的约定。此约定不影响变量的实际作用域。

### 2.11 文件 I/O

要读取或写入文件，你必须做三件事：(1) 打开文件，(2) 使用文件，(3) 关闭文件。

- `with open(file_name, mode) as file: statements` 将打开文件，执行一次语句，然后关闭文件。这是处理文件最简单、最安全的方法。
  - 模式是以下之一：
    - 'r' 读取文件（如果省略 mode，这是默认值）。
    - 'w' 擦除并写入文件。
    - 'a' 追加到现有文件的末尾。
    - 'r+' 同时读取和写入。
    - 'rb'、'wb'、'ab'、'rb+' 用于二进制输入/输出。
  - 由于 statements 部分只执行一次，它通常由一个依次处理每一行的循环组成。
  - 缩进与往常一样：语句在 with 行下缩进。
- `file = open(file_name, mode)` 打开并返回（在变量 file 中）对命名文件的引用；mode 如 with open 所述。

- *file*.read() 将读取并返回整个文件作为一个字符串，包括任何换行符。
- *file*.readline() 读取并返回一行文本，包括终止换行符（如果有的话）。如果返回空字符串，则表示已到达文件末尾。
- *file*.readlines() 读取整个文件并将其作为字符串列表返回，包括终止换行符。
- 输入文件可以被迭代：`for line in file: *statements*` 将读取文件中的每一行，将其赋值给变量 *line*，并执行 *statements*。*statements* 按照惯例缩进。
- *file*.write(*string*) 将字符串写入文件，并返回写入的字符数。
- *file*.close() 关闭文件。**必须执行**。在完成文件操作后保持文件打开可能会导致问题。

例如，以下是一些用于计算文本文件行数的代码。

```
count = 0
with open(file_name, 'r') as f:
    while f.readline():
        count += 1
```

由于历史原因，用于表示“换行”（'\n'）的字符在 Windows 文件、旧版 Macintosh 文件和 Mac OS X/Linux 文件中并不相同。读取或写入文本文件（非二进制文件）会自动将特定于平台的行尾转换为当前平台的行尾。**将二进制文件当作文本文件读取或写入会损坏它们。** 另一方面，文本文件可以作为二进制文件读取和写入而不会造成损害。

### 2.12 PICKLING

序列化（有时称为编组）是将对象转换为线性字节流。这样做可以将对象保存到文件中，或将其传输到另一个进程。字节流可以被反序列化（解组）以重建原始对象。

序列化 Python 对象最常见的方法称为 pickling。Python 也可以使用 JSON 和 XML 进行序列化，但这些格式不支持那么多对象类型。

Python 值和大多数内置对象都可以被 pickle，包括模块顶层的用户定义类和函数。递归和相互连接的对象也可以被 pickle。生成器、lambda 函数、数据库连接和线程是少数不能被 pickle 的东西。

要 pickle 或 unpickle 对象，首先导入 pickle，然后使用以下方法：

- pickle.dump(object, file) — 将对象保存到文件，该文件必须以 'wb'（写入二进制）模式打开。
- variable = pickle.load(file) — 重建先前写入文件的对象，该文件必须以 'rb'（读取二进制）模式打开。
- str = pickle.dumps(object) — 将对象保存到变量 str 中作为字符串。
- object = pickle.loads(str) — 重建先前写入字符串 str 的对象。

Pickling 不安全。pickle 文件可能包含代码对象和数据来攻击你的系统。确保你 unpickle 的任何内容都来自可信来源，并且在传输过程中未被篡改。

## 类

类定义了一个新的*类型*，以及一个用于创建该类型值的**构造函数**。该类型的值是**对象**，每个这样的**对象**将包含类中描述的变量和方法。

这里有一个类比：*类*就像一个蛋糕食谱，而*对象*是你按照食谱制作的蛋糕。蛋糕类中的变量可能包括糖、面粉和黄油，而方法可能包括混合、烘烤和上菜。

### 3.1 类和继承

定义类的语法是

```
class ClassName(superclass):
    variable and method definitions
```

类被安排在一个**层次结构**中，object 类位于顶层。每个类（除了 object 本身）都有一个**超类**，并且它从该超类**继承**变量和方法。也就是说，超类中定义（或继承）的所有变量和方法对新类都是可用的。

例如，你可以定义一个 Person 类，包含变量 name 和 age 以及一个方法 greet。这种类型的每个对象都将拥有这些变量的各自副本，以及对 greet 方法的各自引用。然后，如果你创建一个 Customer 类作为 Person 的子类，那么 Customer 类型的每个对象都将拥有自己的 name 和 age 变量以及一个 greet 方法，外加你在 Customer 中声明的任何其他变量和方法（例如，一个购买列表）。

特殊的 object 类是所有类的“根”。如果你在定义类时省略了**超类**，它默认为以 object 作为超类。每个类都直接或间接地从 object 继承。

> **术语：** 作为名词，“实例”与“对象”含义相同。然而，我们通常在谈论特定对象时使用“实例”（john 是 Person 的一个实例），或者当我们用这个词作为形容词时（name 是类 Person 的一个实例变量）。

> **术语：** “字段”和“属性”是“实例变量”的其他名称。

示例（非常糟糕的）类定义：

```
class Person(object):
    name = 'Joe'
    age = 23

    def say_hi(self):
        print('Hello, Joe.')
```

- 要创建类的实例（对象），使用类名就像使用函数名一样，例如，`p = Person()`。以这种方式使用时，类名充当**构造函数**；它构造一个新对象。
- 我们可以使用**点表示法**访问对象 p 的字段：`p.name` 是 'Joe'，`p.age` 是 23。
- 我们可以修改 p 的字段，例如通过 `p.name = 'Jack'`。
- 我们可以使用点表示法与对象 p “对话”，或者更正式地说，“向对象 p 发送消息”：`p.say_hi()` 告诉 p 执行 `say_hi`，p 将通过打印 Hello, Joe 来响应。

**约定：** 类名总是以大写字母开头，变量和函数名总是以小写字母开头。

不幸的是，这个类的每个新创建的实例都将完全相同。我们希望创建具有不同 name 和 age 初始值的 Person 实例。为此，我们必须探索一个特殊变量：`self`。

### 3.2 构造函数和 self

如果你熟悉其他语言中的类和对象，那么 Python 中的类和对象非常相似——除了你可能对 `self` 这个词感到困惑，它似乎无处不在。

请记住，我们从类创建的每个对象都有自己对该类每个字段（实例变量）的副本。如果我们有两个名为 joe 和 jane 的 Person 对象，那么它们每个都将有一个 name 字段和一个 age 字段。

简而言之，类定义内部使用的名称 `self` 是我们正在与之对话的特定对象。类中的每个方法都必须将 `self` 作为其第一个参数。

这是一个更好的类定义：

```
class Person(object):
    def __init__(self, name, age):
        self.name = name
        self.age = age
    def say_hi(self):
        print('Hello', self.name)
```

当你构造一个对象时，你可能想要指定其实例变量的值。例如，每个 Person 对象都应该以其自己的 name 和 age 创建。这是通过在类定义中包含一个名为 `__init__` 的特殊方法来完成的。（那是 init，前后各有两个下划线。）`__init__` 是一个**初始化器**——当你构造一个新对象时，它*从构造函数自动调用*。

**技术说明：** Python 有许多 ***dunder 方法***，它们以双下划线开始和结束。Dunder 方法在内部使用，很少直接调用。

要创建一个对象，使用类名就像使用函数名一样，并提供除 `self` 之外的所有参数的值。在 `__init__` 方法内部，使用 `self.variable = value` 的形式来初始化实例变量。

在上面的例子中，我们会通过 `jill = Person('Jill', 30)` 来创建一个对象 `jill`。在初始化器 `__init__` 内部，通过将值赋给 `self.name` 来创建一个 name 实例变量，通过将值赋给 `self.age` 来创建一个 age 实例变量。

### 3.3 子类

在类定义之外，我们通过点号表示法“与”对象“对话”，例如 `jill.age` 或 `jill.say_hi()`。但在类定义*内部*，对象是在与“自身”对话，因此我们使用 `self.age` 和 `self.say_hi()`。

那么，`self` 是什么？它是每个方法的*显式*第一个参数，也是我们发送给对象的每条消息的*前缀*参数。在 `say_hi` 的定义中，`self` 必须作为第一个参数给出；当我们说 `jill.say_hi()` 时，`jill` 是作为 `self` 的值传递给该方法的参数（参见图 3.1）。

```python
class Thing(object):
    def do_something(self, x, y):
        ...

t.do_something(3, 5)
```

**图 3.1** `self` 作为前缀参数。

如果一个方法引用同一类中的实例变量，则必须使用 `self` 这个词。如果一个类的一个方法调用同一类的另一个方法，也必须使用 `self` 这个词。任何未用 `self` 名称标记的变量（或方法）都只是局部于它们所在的方法。

```python
def get_older(self):
    self.age += 1

def get_much_older(self, years):
    for i in range(0, years):
        self.get_older()
```

**技术说明：** `self` 只是一个参数名，因此理论上你可以用任何其他名称替换它。但这将严重违反惯例。

与某些其他面向对象语言不同，对象没有固定不变的实例变量集。你可以简单地通过赋值向对象添加实例变量；例如，你可以说 `jill.occupation = 'scientist'`。这只会影响 `jill`；其他 `Person` 对象保持不变。同样，你可以使用 `del` 操作从特定对象中删除实例变量。

**重要调试说明：** 在定义和使用类与对象时，最常见的错误是提供*过多的参数*或*参数不足*。这几乎总是由于在某处忘记使用 `self` 造成的。

**与 Java 的比较：** Java 的 `this` 等同于 Python 的 `self`，有两个区别：(1) `this` 是一个关键字，而 `self` 是一个变量名；(2) `this` 通常是隐式的，而 `self` 必须每次显式使用。

一个类继承其超类的所有实例变量和方法。（它是其超类的**子类**。）例如，你可以将 `Friend` 类定义为 `Person` 的子类。

```python
class Friend(Person):
    def smile(self):
        print('~\_(^~^)_/~')

meg = Person('Margaret', 25)
bob = Friend('Robert', 33)
```

当你将 `meg` 创建为 `Person` 的实例时，`meg` 将拥有 `say_hi`、`get_older` 和 `get_much_older` 方法以及 `__init__` 初始化器，但没有 `smile` 方法。当你将 `bob` 创建为 `Friend` 时，`bob` 将拥有所有这些，并且还会有一个 `smile` 方法。

你可以通过提供另一个同名方法来**覆盖**（替换）任何继承的方法，例如，

```python
def say_hi(self, extra):
    print('Hi!', extra)
```

如果你在子类中覆盖了一个方法，你仍然可以使用 `super()` 作为前缀来访问被覆盖的方法。例如，

```python
def say_hi(self, extra):
    super().say_hi()
    print(extra)
```

当你想要超类中的所有内容，但希望子类拥有额外信息时，通常会编写一个子类。例如，一个 `Friend` 对象是一个 `Person`，所以它应该有一个名字和年龄，但也许你还想让它有一个昵称。在这种情况下，你需要编写一个新的 `__init__` 方法，覆盖继承的那个。

在 `Friend` 的初始化器中，你*可以*复制并粘贴 `Person` 初始化器中完成的所有工作，但更好的风格是直接调用那个初始化器。在这种情况下，你*确实*直接调用 `__init__`。例如，`Friend` 初始化器可能看起来像这样：

```python
def __init__(self, name, age, nick):
    super().__init__(name, age)
    self.nickname = nick
```

如果你编写一个子类是因为你只想要超类中的部分内容，而不是全部，这表明你的类将受益于重新组织。

### 3.4 打印对象

`print` 方法接受任意数量的参数，通过调用每个参数的 `__str__` 方法将其转换为字符串，然后打印它们。（与 `__init__` 一样，名称前后各有两个下划线。）

所有类都有一个从 `object` 继承的 `__str__` 方法。覆盖这个方法是个好主意。如果你不这样做，你的对象将被打印成类似这样的内容：

```
<__main__.Friend object at 0x1064728d0>
```

对于 `Friend` 对象，你的方法可能看起来像这样：

```python
def __str__(self):
    return self.name + "'s age is " + str(self.age)
```

而 `print(meg)` 将产生：

```
Margaret's age is 19
```

在上面，Margaret 的年龄是一个整数，我们不能将整数与字符串相加，所以我们必须通过调用函数 `str(self.age)` 将她的年龄显式转换为字符串。一旦你在类中定义了 `__str__` 方法，你就可以对该类的对象调用 `str` 函数，例如通过说 `str(meg)`。

类似于 `str(x)`，`repr(x)` 函数返回一个表示对象 `x` 的字符串，由其 `__repr__(self)` 方法定义。区别在于 `str` 由 `print` 方法使用，应该提供人类可读的字符串，而 `repr` 方法在调试和开发期间使用，应该详细且明确。

`__str__` 和 `__repr__` 是继承的“双下划线”方法，你可以用自己的版本覆盖它们，然后通过 `str(x)` 和 `repr(x)` 间接调用。

### 3.5 比较对象

相等（`==`）和不等（`!=`）运算符对于 Python 的内置对象类型工作良好。然而，当你定义自己的类时，`==` 在比较两个不同的对象时将返回 `False`（但见下文）。

记住*相等*和*相同*之间的区别很重要。

- 如果 **a** 的每个部分都与 **b** 的相应部分相同，则两个对象 **a** 和 **b** 是**相等**的；如果你改变 **a**，**b** 可能会也可能不会改变。测试相等性需要测试所有部分。
- 如果 **a** 和 **b** 是**相同的**，那么它们只是同一对象的两个不同名称；通过对 **a** 做某事来改变该对象意味着 **b** 也会看到变化。测试相同性只涉及测试 **a** 和 **b** 是否都引用内存中的同一位置。
- 赋值 `a = b` 从不复制对象 **b**；它只是让 **a** 引用与 **b** 相同的对象。
- 当你将对象用作函数的参数时，函数获得的是对原始对象的引用，而不是副本。
- `a is b` 测试 **a** 和 **b** 是否引用同一对象；`a is not b` 测试它们是否是不同的（但可能相等的）对象。

对于你自己的类，你可以通过在类中定义一些特殊方法来定义相等性和排序。（所有这些名称都使用双下划线。）

- `__eq__(self, other)` 如果你认为对象 `self` 和 `other` 相等，则应返回 `True`，否则返回 `False`。相同的对象应被视为相等（你可以使用 `is` 运算符测试相同性）。当你的对象使用 `==` 进行比较时，将调用此方法。
- `__ne__(self, other)` 在对象使用 `!=` 进行比较时将被调用。
- `__lt__(self, other)` 在对象使用 `<` 进行比较时将被调用。
- `__le__(self, other)` 在对象使用 `<=` 进行比较时将被调用。
- `__ge__(self, other)` 在对象使用 `>=` 进行比较时将被调用。
- `__gt__(self, other)` 在对象使用 `>` 进行比较时将被调用。

**这些方法都是独立的。** 定义其中一些*不会*自动定义其他方法。例如，如果你定义了一个 `__eq__` 方法，`__ne__` 不会神奇地出现。如果你定义了这些方法中的一个，你应该定义全部六个。

我们通常认为比较运算符是比较大小或量级；但如果你定义自己的比较运算符，你可以将它们用于你喜欢的任何类型的排序。这已经在字符串中完成；这些运算符使用词典（字母）顺序，所有大写字母都小于所有小写字母。

### 3.6 绑定

从技术意义上讲，Python 中的每个值都是一个对象。然而，有些对象是**不可变的**：它们不能被更改。例如，数字 5 是不可变的；你不能将 5 改为 6。如果你有一个变量 `x`，其值为 5（我们说 `x` **绑定**到 5），你可以使用赋值语句将 `x` 的绑定更改为 6，但 5 仍然是 5。

当你执行一个赋值操作时，你将变量**绑定**到一个值。如果你随后将该变量赋值给一个新变量，绑定关系只是被复制了。

```
a = 5 # 将 a 绑定到 5
b = a # 将 a 的绑定复制到 b
a = 6 # 改变 a 的绑定
print(b) # b 仍然绑定到 5
```

数字、字符串、布尔值和元组是不可变的。然而，列表是**可变的**：你可以更改列表中的值，但它仍然是同一个列表。集合、字典和大多数其他对象也是可变的。

对于可变对象，绑定的工作方式与不可变对象相同。

```
a = [1, 2, 3] # 将 a 绑定到该列表
b = a # 将 a 的绑定复制到 b
a = [4, 5, 6] # 改变 a 的绑定
print(b) # b 仍然绑定到 [1, 2, 3]
```

更改可变对象中的值*不会改变*任何变量绑定。

```
a = [1, 2, 3] # 将 a 绑定到该列表
b = a # 将 a 的绑定复制到 b
a[0] = 99 # 更改列表，而不是绑定
print(b) # b 仍然绑定到同一个列表，
    # 但该列表现在是 [99, 2, 3]
```

当调用一个函数时，调用中参数的值被绑定到函数的参数；换句话说，这些值被赋值给参数。当函数返回时，它*不会*将参数值复制回参数列表，但传递给函数的任何可变对象都可能在函数内部被更改。

```
def foo(a, b):
    a[0] = 99
    b = [7, 8]

a = [1, 2, 3]
b = [1, 2, 3]
foo(a, b)
print(a, b) # [99, 2, 3] [1, 2, 3]
```

作为一般风格规则，函数应该返回一个值但不改变其参数，而方法可以改变其参数。例如，以下是内置的：

- **obj.sort()** 是一个方法，它对对象 **obj** 进行排序并返回 None，而
- **sorted(obj)** 是一个函数，它保持 **obj** 不变但返回一个排序后的副本。

### 3.7 浅拷贝和深拷贝

对象可以包含对其他对象的引用。如果我们复制该对象，我们通常会得到一个**浅拷贝**：对象中的引用被复制了，但被引用的对象本身没有被复制。

```
grades = [["Mary", "A+"], ["Don", "C-"]]
grades2 = grades[:]
grades2[0] = ["Bob", "B"]
grades2[1][1] = "F"
print(grades) # [['Mary', 'A+'], ['Don', 'F']]
print(grades2) # [['Bob', 'B'], ['Don', 'F']]
```

以下是发生的情况：

- grades 是一个包含两个引用的列表，一个指向列表 ["Mary", "A+"]，另一个指向列表 ["Don", "C-"]。
- 对 grades2 的赋值创建了一个包含这些引用副本的新列表。
- 对 grades2[0] 的赋值将 grades2 中的第一个引用更改为一个新列表 ["Bob", "B"]。
- grades2[1] 仍然是 grades[1] 中引用的副本，并且两者都指向同一个列表 ["Don", "C-"]，因此对 grades[1][1] 的赋值更改了该列表的内容。

Python 提供了一个 deepcopy 方法，可以创建一个完全独立的**深拷贝**对象，可以达到任意层级。这甚至适用于循环对象，即包含自身的对象。

```
import copy
grades3 = copy.deepcopy(grades)
grades3[1][1] = "D+"
# grades 未改变
```

## 进阶技巧

### 4.1 语句

本节介绍之前未涵盖的 Python 语句，或比之前涵盖的有更多选项的语句。

- import
  - import **module** — 使命名的 **module** 可用，之后可以通过 **module.name** 访问该模块中名为 **name** 的对象或方法。这可以防止从多个模块导入同名时出现问题。
  - from **module** import **names** — 导入给定的名称，这些名称不需要以 **module** 和点号为前缀。
  - from **module** import * — 从 **module** 导入所有名称。谨慎使用，因为它可能导入你甚至不知道的名称。
  - import **module** as **modname** — 使命名的 **module** 可用，但你将使用 **modname.name** 而不是 **module.name**。
  - from **module** import *name1* as *name2* — 导入 *name1* 但将其重命名为 *name2*。

- assert *expression1*, *expression2*
  - 如果 *expression1* 为真，则不执行任何操作，但如果 *expression1* 为假，则引发 AssertionError。可选的 *expression2* 用作 AssertionError 的消息。
  - assert 语句最好被视为“可执行文档”。它可以在函数开头使用，以表达对参数的要求，或者在中间或结尾使用，以声明刚刚计算的内容。它也可以用作内置测试用例。

- *var1*, . . . , *varN* = *expr1*, . . . , *exprN*
  - 同时赋值。左侧的变量被赋予右侧表达式的值。变量和表达式的数量必须完全相同。这只是元组打包和解包的一个特例。
  - a, b = b, a 是交换 a 和 b 值的一种特别简单的方法。

- break
  - 如果在 while 或 for 循环中执行 break，它会导致循环退出。
  - 如果在嵌套循环中执行 break，则只退出最内层的循环。
  - 如果循环因 break 而退出，则不会执行后面的 else 子句。（可能对 Python 独有，循环可以有 else 子句。）

- **continue**
  - 如果在 while 或 for 循环中执行 continue，则跳过循环中剩余的语句，控制返回到循环顶部（对于 while 循环是测试条件，对于 for 循环是下一个值）。
  - 在嵌套循环中，continue 适用于最内层的循环。

- **del variable**
  - 导致变量不再存在（变为未绑定）。

- **else:**
  - while 循环和 for 循环后面都可以跟一个 else 子句。当循环正常退出时，else 子句中的代码会被执行，但如果循环因 break 或 return 而终止，则不会执行。

- **exec(arg)**
  - 执行一个字符串、一个已打开的文件或一个代码对象 arg。本书不涵盖代码对象。
  - exec 实际上是一个函数，但它返回 None 作为其值，因此经常被当作语句使用。
  - exec **不安全**，因此如果 arg 可能包含恶意或有害代码，请不要使用它。

- **nonlocal variables**
  - 函数可以嵌套在其他函数内部。默认情况下，内部函数可以访问但不能更改外部函数的变量。如果内部函数声明变量为 nonlocal，则它可以访问并更改它们。
  - 规则与局部变量和全局变量的规则非常相似。

- pass
  - pass 语句不执行任何操作。当语法要求存在语句但没有特别需要做的事情时，它偶尔很有用。
  - 省略号（三个点，...）与 pass 相同。

- print(expr1, ... , exprN, sep=sepString, end=endString, file=outputFile)
  - 计算并打印表达式，表达式之间用 sepString 分隔，最后一个表达式之后用 endString，输出到 outputFile。关键字参数 sep、end 和 file 可以省略，分别使用默认值 ' '（单个空格）、'\n'（换行符）和 stdout。

- raise
  - raise Exception 引发命名的 Exception，该异常可能稍后由 try-except 语句处理。Python 提供了大量异常类型，或者你可以通过继承 Exception 来定义自己的异常。
  - raise Exception(expression) 引发命名的 Exception，并使用表达式的结果作为异常中的消息。
  - 在 except 子句中单独使用 raise 会重新引发相同的异常。

### 4.2 标识符

标识符以字母或下划线开头，可以包含字母、下划线和数字；区分大小写。可以使用 Unicode 字符。

按照惯例，类名以大写字母开头。其他名称则不是。

在类中，你通常希望有**私有**实例变量——即除了类内部之外无法访问或更改的字段。在 Python 中无法做到这一点。相反，一些程序员采用了一种惯例，即在他们希望是私有的变量名称前加上一个下划线，或者如果他们真的、真的希望它是私有的，则加上两个下划线。这是编程版本的在你家前门贴一张纸条，上面写着“请不要入室盗窃”。

变量名可以由单个下划线组成。当需要变量但实际不用于任何事情时，通常使用此名称。

```
for _ in range(0, 3):
    print("Hello!")
```

内置的“dunder”名称以两个下划线开头和结尾。你不应该以这种方式命名自己的变量和方法。

### 4.3 类型提示

Python 是一种**动态类型**语言。也就是说，变量名的类型是它当前持有的任何值的类型。

从 Python 3.5 开始，你可以输入**类型提示**来指定变量预期持有的值的类型。这些提示被视为注释；它们对运行中的程序绝对没有影响。一些 IDE 可以使用类型提示进行一些静态代码检查，这在未来可能会变得更加普遍。

以下示例演示了提示变量、参数和函数返回值类型的语法。

```
def indent(line: str, amount: int) -> str:
    indented_line:str = ' ' * amount + line
    return indented_line
```

要构建更复杂的类型提示，请导入 `typing` 模块。

```python
from typing import List, Set, Dict, Union, Optional
```

通过此导入，你可以使用诸如 `List[int]`、`Set[int]`、`Dict[str, str]`、`Union[int, float]` 和 `Optional[str]` 等类型提示，以及更复杂的类型提示如 `List[Set[int]]`。`Union` 类型提示指定该值可以是列出的任何类型，而 `Optional` 类型提示指定该值要么是给定类型，要么是 `None`。

类型提示可以被赋予一个名称，之后该名称就可以用作类型提示。这对于记录复杂数据结构非常有帮助。

```python
Number = Union[int, float]
```

### 4.4 数字

**十进制整数**由数字 0 组成，或者由不以 0 开头的数字序列组成。

**二进制整数**由二进制数字（0、1）组成，以 `0b` 或 `0B` 开头。

**八进制整数**由八进制数字（0 到 7）序列组成，以 `0o` 或 `0O` 开头。

**十六进制整数**由十六进制数字（0 到 9 和 a 到 f 或 A 到 F）序列组成，以 `0x` 或 `0X` 开头。

**浮点数**（"实数"）包含小数点、指数后缀，或两者兼有。指数由字母 `e` 或 `E`、一个可选的符号和一个或多个数字组成，例如 `1.7e-12`。

**虚数**由十进制整数或浮点数后跟 `j`（*不是* `i`）或 `J` 组成。

**复数**由整数或浮点数与虚数的和或差组成。

**要将字符串转换为数字**，请使用函数 `int(s)`、`float(s)` 和 `complex(s)`。如果字符串格式错误，将引发 `ValueError`；特别是，表示复数的字符串不能包含任何空格：`3+5j` 是可以的，但 `3 + 5j` 不行）。

**要将数字转换为字符串**，请使用 `str(x)` 获取十进制表示，或使用 `bin(i)`、`oct(i)` 或 `hex(i)` 分别获取整数 *i* 的二进制、八进制或十六进制字符串表示。

**要将数字转换为或从 Unicode 字符转换**，请分别使用函数 `chr(n)` 或 `ord(c)`。

### 4.5 字符串

**字符串**写为零个或多个字符的序列，用单引号（`'...'`）、双引号（`"..."`）、三单引号（`'''...'''`）或三双引号（`"""..."""`）括起来。它可以被视为字符列表。

**三引号字符串**可以跨越多行，并将换行符作为字符串的一部分包含在内，除非换行符前面紧跟着反斜杠（`\`）。

**原始字符串**是以 `r` 或 `R` 为前缀的字符串。在原始字符串中，反斜杠不会转义字符；所有字符都代表其自身。这在编写正则表达式时特别有用。

Python 中的**正则表达式**遵循 POSIX 标准，通常写为原始字符串。此处不作介绍。

函数 `eval(string)` 将 `string` 作为 Python 表达式进行求值并返回结果。如果使用此函数，请确保 `string` 不包含恶意代码。

### 4.6 F-字符串

格式化字符串有三种方式：使用 f-字符串、使用 `format` 方法以及使用旧式 `%` 格式化。通常推荐使用 f-字符串方法。

格式化字符串，或称 f-字符串，以 `f` 或 `F` 为前缀。在 f-字符串中，任何用花括号 `{}` 括起来的表达式都会被其值替换。

-   `print(f'Area is {5*7/2}.')`
    -   输出：`Area is 17.5`。

在 Python 3.8 及更高版本中，如果花括号中的表达式以等号结尾，则会同时打印表达式及其结果。

-   `print(f'Area is {5*7/2=}.')`
    -   输出：`Area is 5*7/2=17.5`。

如果你确实想在 f-字符串中打印花括号，用反斜杠转义它们是无效的。相反，应该使用双花括号（`'{{'` 或 `'}}'`）。

可以使用格式化代码。例如，格式 `:9.3f` 表示在总共 9 个空格中显示一个浮点数，小数点后保留 3 位数字。

-   `print(f'Area is {5*7/2:9.3f}.')`
    -   输出：`Area is 17.500`。

格式化代码的语法相当复杂，此处不作介绍。相反，我们只给出几个简单的例子。

```python
f'{"abc":6}' is 'abc   '
f'{"abc":>6}' is '   abc'
f'{123:6d}' is '   123'
f'{123:<6d}' is '123   '
f'{pi:6.2f}' is '  3.14'
f'{pi:<6.2f}' is '3.14  '
```

`format` 方法的语法是 **string**.format(**values**)。与 f-字符串一样，字符串包含花括号，但花括号中包含的是格式化代码，而不是表达式。例如：

```python
print('pi is {:8.4f}'.format(pi))
```

此代码将打印字符串 `'pi is    3.1416'`，单词 `is` 和 `3` 之间有三个空格。

除了格式化代码，传递给 `format` 方法的字符串中的花括号还可以包含简单的整数。每个整数被视为 **values** 列表的索引。因此，例如，`'{1} {0} {1}'.format(1, 2)` 返回字符串 `'2 1 2'`。

旧式 `%` 格式化在字符串内使用 `%` 百分号表示替换位置，并在字符串后作为运算符使用。例如，`'%s = %i' % ('x', 7)` 得到字符串 `'x = 7'`。这种格式化风格不再推荐，此处不再进一步讨论。

### 4.7 位运算

以下**位运算符**可应用于整数。

-   `~` 按位取反
-   `&` 按位与
-   `|` 按位或
-   `^` 按位异或
-   `<<` 左移（右操作数是左操作数要移动的位数）
-   `>>` 右移（右操作数是左操作数要移动的位数）

这些运算符作用于整数的二进制表示；即每个整数都表示为一个二进制数。整数 0 由全零位组成，-1 由全一位组成。

-   **按位取反**将每个 1 变为 0，每个 0 变为 1。在数值上，结果与改变数字的符号然后减去 1 相同。
-   **按位与**在两个操作数的对应位都为 1 时给出 1，否则为 0。
-   **按位或**在其中一个或两个操作数的对应位为 1 时给出 1，否则为 0。
-   **按位异或**在两个操作数的对应位不同时给出 1，否则为 0。
-   **左移**将所有位向左移动，并用零填充。
-   **右移**将所有位向右移动，一些位从左端消失，右端补零。

### 4.8 列表推导式

*列表推导式*是一种从值集合计算列表的方法。该集合可以是列表、元组、集合、字典的键、字符串或任何可以遍历的东西。列表推导式可以包含可选的 `if` 测试。

-   `[expression for variable in collection]` 是一个新列表，通过将集合中的每个值依次绑定到变量，然后计算表达式形成。
-   `[expression for variable in collection if condition]` 是一个新列表，通过将集合中的每个值依次绑定到变量，然后如果此绑定满足条件，则计算表达式形成。

列表推导式功能强大，值得了解。以下是一些简单用法：

-   将表达式应用于列表的每个元素：`[2 * x for x in [1, 2, 3]]` 返回 `[2, 4, 6]`。
-   从列表中移除不需要的元素：`[x for x in [1, 0, 2] if x > 0]` 返回 `[1, 2]`。

### 4.9 迭代器

可迭代对象是任何可以遍历的对象。可迭代对象包括列表、集合、元组、字符串、字典、范围和文件。迭代器是一个对象，它跟踪在遍历可迭代对象时的位置，并在需要时提供下一个值。

例如，你可以通过 `it = iter([2, 3, 5])` 获取一个列表迭代器。你可以通过重复调用 `next(it)` 来遍历此列表的元素；这将依次返回 2、3，然后是 5。再次调用 `next(it)` 将引发 `StopIteration` 异常。

这正是 `for` 循环的工作方式。它首先获取一个可迭代对象（如列表或范围）的迭代器。然后使用迭代器从对象中获取一个值，执行循环体，然后返回迭代器获取下一个值。当 `for` 循环遇到 `StopIteration` 异常时退出。

你可以创建自己的可迭代对象。为此：

-   可迭代类必须包含一个 `__iter__` 方法，用于创建并返回一个新的迭代器对象，该对象处于所有元素尚未被取出的状态。
-   迭代器类至少包含两个方法：
    -   一个 `__init__` 方法，它接受可迭代对象作为参数，将其保存在实例变量中，并执行任何其他必要的初始化操作；
    -   一个 `__next__` 方法，用于查找或计算下一个值。如果没有更多值可返回时调用它，应引发 `StopIteration` 异常。
-   这些类可以合并，使得可迭代对象本身就是它自己的迭代器。在这种情况下，该类包含一个 `__next__` 方法，而 `__iter__` 方法只需返回 `self`。

举个简单的例子，我们可以编写一个可迭代类 `MyList`，它持有一个列表，以及一个迭代器类 `Reverser`，它将按相反顺序遍历该列表。

```
class MyList():
    def __init__(self, ls):
        self.ls = ls

    def __iter__(self):
        return Reverser(self.ls)

class Reverser():
    def __init__(self, ls):
        self.ls = ls
        self.index = len(self.ls)

    def __next__(self):
        self.index = self.index - 1
        if self.index >= 0:
            return self.ls[self.index]
        raise StopIteration
```

迭代器的工作是使用自己的变量（而非可迭代对象的变量）来跟踪其在迭代过程中的位置。迭代器不应影响或修改可迭代对象。这一规则允许两个或多个迭代器同时在同一个对象上运行而互不干扰。

如果类被合并，使得可迭代对象就是它自己的迭代器，那么一次只能支持一个迭代器。

`for` 循环可以使用任何可迭代对象。在 `for` 循环中，`StopIteration` 异常不会导致错误；它只是终止 `for` 循环。例如：

```
ls = MyList([1, 2, 3, 4])
for e in ls:
    print(e)
```

或者，你可以直接使用 `iter` 和 `next` 方法，并自行处理 `StopIteration` 异常。这本质上就是前面 `for` 循环所做的事。

```
ls = MyList([1, 2, 3, 4])
it = iter(ls)
while True:
    try:
        print(next(it))
    except StopIteration:
        break
```

迭代器只能使用一次；当它引发 `StopIteration` 异常时，它就完成了，无法重置。与其重置迭代器，不如直接创建一个新的。

如果你有一个迭代器 `it`，它返回*有限*数量的值，并且你需要这些值的列表，只需说 `list(it)`。

### 4.10 生成器

**生成器**是一种迭代器。它按需一次生成一个值，但不一定与特定类型的对象绑定。

一种**生成器表达式**看起来就像列表推导式，只是外层的方括号被圆括号替代。生成器可以在任何可以使用迭代器的地方使用，例如在 `for` 循环中。示例：

```
word = 'generator'
gen = (c for c in word if c in 'aeiou')

for i in gen:
    print(i, end=' ')
```

这段代码将打印 `e e a o`。

> **注意：** 生成器是一种迭代器，因此在它返回所有值之后，它就“用完了”。如果上面的 `for` 循环被第二次调用，它将不会打印任何内容，因为变量 `gen` 将持有一个空的生成器。

你可以通过使用 `yield` 而不是 `return` 来编写充当生成器的函数。下面是一个生成 2 的幂的生成器示例：

```
def powers_of_two():
    n = 2
    for i in range(0, 5):
        yield n
        n *= 2
```

当这个函数被调用时，它返回一个*生成器*，而不是一个数字。你可以在 `for` 循环中使用这个生成器，方式与上面使用 `gen` 相同，你将得到值 2, 4, 8, 16 和 32。

```
gen = powers_of_two()
for n in gen:
    print(n)
```

由于 `gen` 是一个迭代器，你可以使用 `next(gen)` 来获取下一个值。在这种情况下，你必须自己处理异常。

```
gen = powers_of_two()
while True:
    try:
        print(next(gen))
    except StopIteration:
        break
```

以下是其工作原理：

-   调用 `gen = powers_of_two()` 返回一个生成器并将其放入 `gen` 中。
-   第一次调用 `next(gen)` 会执行生成器直到 `yield` 语句，并返回 `yield` 的值，就像普通的 `return` 语句一样。但除此之外，生成器还记住了它停止的位置。
-   后续调用 `next(gen)` 将导致生成器从它停止的地方恢复执行，即在 `yield` 语句之后立即执行。所有局部变量的值都将被保留。就生成器而言，就好像 `yield` 从未发生过一样。在这个例子中，`for` 循环会继续运行。
-   你可以在生成器中有多个 `yield` 语句。函数会记住它在哪个 `yield` 处停止，并从那里继续。
-   与迭代器一样，当没有更多值可返回时，生成器会引发 `StopIteration` 异常。当它到达函数末尾或遇到 `return` 语句时，这会自动发生。

### 4.11 参数与实参

在没有额外语法的情况下，**实参**（函数调用中的表达式）通过位置与**形参**（函数定义中的变量）匹配。第一个形参获取第一个表达式的值，依此类推。

函数定义中的**形参**可以是：

-   一个简单的变量名，通过位置匹配并绑定到一个实参。位置形参必须位于任何其他形参之前。
-   `*variable=value`，为缺失的实参提供默认值。
-   `*args`，接受多个实参作为一个元组。按照惯例，名称 `args` 通常用于此目的。
-   `**kwargs`，接受多个关键字实参作为一个字典。按照惯例，名称 `kwargs`（关键字实参）通常用于此目的。

`*args` 或 `**kwargs` 只能作为最后一个形参使用，因为它们中的任何一个都会收集所有剩余的实参。

函数调用中的**实参**可以是：

-   一个表达式。
-   `*name=value`，为具有该名称的形参提供一个值。这称为*命名实参*或*关键字实参*。
-   如果同时使用了位置实参和关键字实参，所有位置实参必须位于关键字实参之前。
-   `*iterable`，将可迭代对象的值作为单独的实参传递。
-   `**dictionary`，通过名称将值传递给多个形参。

从 Python 3.8 开始，实参可以强制为仅位置实参。在形参位置使用单个斜杠 `/` 意味着所有前面的实参必须是位置实参。在形参位置使用星号 `*` 意味着其后的所有实参必须是关键字实参。例如，给定函数：

```
def foo(a, b, /, c, d, *, e, f):
    print(a, b, c, d, e, f)
```

`a` 和 `b` 的实参必须仅通过位置传递；`e` 和 `f` 的实参必须是关键字实参。`c` 和 `d` 的实参可以是任意一种（但由于位置实参必须位于关键字实参之前，如果 `c` 是关键字实参，则 `d` 不能是位置实参）。

官方 Python 文档中描述的许多函数似乎有一个单独的星号作为形参，例如 `os.remove(path, *, dir_fd=None)`。这是一种文档约定，表示后续的实参只能作为命名实参给出。这*并不*意味着你应该在函数调用中放入一个星号。

### 4.12 函数式编程

*函数式编程*的定义各不相同，但通常包括以下内容：

-   函数是对象，可以像任何其他值一样被对待。
-   变量是*单次赋值*的；一旦赋予一个值，就永远不会改变。
-   所有函数都是*纯*的；函数返回的值仅取决于传递给它的实参，如果使用相同的实参再次调用，它将产生相同的值。这排除了任何全局变量或其他外部因素（如系统时钟）的使用。

函数式编程被广泛认为是一种象牙塔技术，不适合日常编程。这里不是挑战这一观点的地方，只是指出所有现代编程语言都在一定程度上支持函数式编程。

在本书中，我们只考虑上述第一点：函数是对象。我们将只考虑这种用法的一个例子。

以下函数将在一个非空值列表中找到最大值：

```
def biggest(values):
    big = values[0]
    for v in values:
        if v > big:
            big = v
    return big
```

由于 `>` 也可以用于比较字符串，此函数可用于在字符串列表中找到字典序最大的字符串。但对于几乎所有其他目的（找到最小的数字、最长的字符串等），你必须编写另一个几乎相同的函数。

与其编写越来越多的函数，我们可以用一个通用的测试来替换 `>`：

def most(values, more):
    best = values[0]
    for v in values:
        if more(v, best):
            best = v
    return best

def larger(a, b):
    return a > b

我们可以这样调用 most 函数：

```
most([1, 6, 1, 8, 0], larger)
```

或者这样：

```
def longer(a, b):
    return len(a) > len(b)

most(["a", "generic", "list"], longer)
```

这仍然会产生许多像 `larger` 和 `longer` 这样的小函数。Python 还提供了*字面函数*，有时也称为*匿名函数*，因为它们没有名字。字面函数旨在仅在其编写的位置使用。出于历史原因，字面函数由关键字 `lambda` 引入。例如，

```
lambda a, b: len(a) > len(b)
```

它可以这样使用：

```
print(most(["a", "generic", "list"],
    lambda a, b: len(a) > len(b)))
```

一个 lambda 表达式由关键字 `lambda`、任意数量的参数、一个冒号和一个单独的表达式组成。在这个例子中，表达式的结果是一个布尔值，但它可以是任何类型。

Python 有许多内置函数，它们接受一个函数作为参数。以下是一些最常用的：

- `map(function, iterable)` 返回一个迭代器，其 `next` 函数将获取可迭代对象的下一个值，将函数应用于该值，并返回结果。
- `filter(predicate, iterable)` 返回一个迭代器，其 `next` 函数将返回可迭代对象中满足谓词的下一个值。
- `functools.reduce(binaryFunction, iterable)` 将 `binaryFunction` 应用于可迭代对象的前两个元素。然后，它重复地将 `binaryFunction` 应用于当前结果和可迭代对象的下一个成员，返回一个单一值作为结果。（注意：`reduce` 必须从 `functools` 导入。）

**示例：** 如果 `my_list` 是 `[3, 1, 4, 1, 6]`，那么

- `map(lambda x: x * x, my_list)` 将返回值 9, 1, 16, 1, 36。
- `filter(lambda x: x > 1, my_list)` 将返回值 3, 4, 6。
- `reduce(lambda x, y: x + y, my_list)` 将返回 15。

# 第 5 章

## 测试

代码必须经过测试。任何重要的代码片段第一次就能正确运行的情况是不常见的。

一种过于常见的方法是在编写代码时就进行测试，得出一切正确的结论，然后不采取任何措施来保留测试。这无助于使代码在未来免受更微妙错误的影响，并且在需求变化时更难修改代码。

*测试框架*是这样的软件：

1. 允许以简单、标准化的形式编写测试；并且
2. 允许以最少的努力运行测试。

在接下来的章节中，我们将讨论两个框架：`doctest` 和 `unittest`。

### 5.1 理念

使用适当的测试框架有诸多优势：

- 代码更有可能是正确的。
- 代码通常花费更少的时间来生成，因为调试时间减少了。
- 为测试而编写的函数往往更小且功能单一。
- 为测试而编写的函数在计算和 I/O 之间提供了清晰的分离。
- *测试套件*（测试的集合）的存在使得在将来某个时候修改代码变得更容易和更安全。

一个好的测试框架使程序员能够快速运行测试，通常只需单击一次按钮，并且一目了然地看到是否有任何错误。任何需要更多步骤的测试都不会被频繁运行（如果有的话），从而抵消了拥有它们的价值。

要编写可测试的代码，

- 编写许多小函数，每个函数只做一件事，而不是几个大函数做很多事情。
- 边写边测试。许多权威人士建议在编写要测试的代码之前编写测试；这有助于澄清代码应该做什么，并且也帮助程序员从用户的角度看待接口。
  - 对于已存在或遗留的代码，编写测试可能很困难或不可能。
- 最小化全局变量的使用，如果可能的话完全避免它们。其值仅依赖于其参数的函数更容易进行隔离测试。
- 严格分离执行计算的函数和执行输入或输出的函数，并且只测试前者。
  - 执行 I/O 的函数涉及程序员，因此它们阻止了单击测试的使用。虽然有测试这些函数的技术，但这些是高级技术，此处不涉及。

### 5.2 DOCTEST

许多测试是临时进行的——程序员在 Python 提示符（`>>>`）处输入函数调用，并查看结果是否正确。`doctest` 提供了一种非常省力的方式来保留这些测试。

要使用 `doctest`，只需将提示符处完成的临时测试（包括 `>>>` 提示符）复制并粘贴到被测试函数的文档注释中。以下 `get_digits` 函数返回整数或字符串中所有数字的列表：

```
def get_digits(number):
    """Return a list of digits in an int or string."""
    string = str(number)
    return [x for x in string if x.isdigit()]
```

然后程序员可能会运行一些测试：

```
>>> get_digits("124c41")
['1', '2', '4', '4', '1']
>>> get_digits(1213141)
['1', '2', '1', '3', '1', '4', '1']
```

要使用 `doctest`，将 Python 提示符处完成的临时测试复制并粘贴到文档字符串中。

```
def get_digits(number):
    """Return a list of digits in an int or string."""
    >>> get_digits("124c41")
    ['1', '2', '4', '4', '1']
    >>> get_digits(1213141)
    ['1', '2', '1', '3', '1', '4', '1']
    """
    string = str(number)
    return [x for x in string if x.isdigit()]
```

要再次运行这些测试，请使用：

```
import doctest
doctest.testmod()
```

`testmod` 方法将定位注释中的所有临时测试并再次运行它们。它将打印有关所有失败测试的信息；如果所有测试都通过，`testmod` 不会打印任何内容（尽管它确实返回一个摘要结果）。

**注意：**

- 测试引发的异常可以像任何其他结果一样进行测试，方法是将打印的结果复制到文档字符串中。
- `>>>` 提示符可以从任何列开始，但输出必须从相同的列开始。
- 对于跨越多行的代码，`...` 续行提示符也应复制到文档字符串中。
- 输出中的制表符将被转换为空格，这会导致 `doctest` 失败。
- 输出中的空行被视为输出的结束。

测试可以放在一个单独的文件中，并使用 `doctest.testfile(path)` 方法执行。

虽然 `doctest` 是保留一些测试的最省力方式，但它不如单元测试灵活。

### 5.3 单元测试

**单元测试**是对单个类中方法的测试。**测试用例**测试单个方法对特定输入集的响应。要进行单元测试，

- 导入 `unittest`。
- 导入 **fileToBeTested** 或者 `from **fileToBeTested** import *`。
  - **提醒：** 如果你使用 `from file import *`，那么你不必在每个函数调用前加上它所导入的文件名。
- 编写一个类 **SomeName**(unittest.TestCase)。在该类中，
  - 如果需要，定义方法 `setUp(self)` 和 `tearDown(self)`。这两个都是可选的。
  - 提供一个或多个 `test**Something**(self)` 方法。你可以包含其他方法，但测试方法的名称必须以 `test` 开头。
- 在测试文件的末尾，放置 `unittest.main()`。

以下是 `unittest.main()` 的作用。对于每个名称以 `test` 开头的方法，`unittest.main` 方法会调用 `setUp()`（如果你提供了的话），然后调用测试方法，然后调用 `tearDown()`（如果你提供了的话）。因此，每个测试都夹在 `setUp` 和 `tearDown` 之间。

`setUp` 方法的目的是确保在调用测试方法之前，所有内容都处于已知的、原始的状态。通过这种方式，你可以确保运行一个测试的结果不会影响后续测试的结果。

`tearDown` 的目的是删除可能已创建的工件（例如文件）。它的使用频率远低于 `setUp`。

每个测试方法通常应该只测试一个函数，尽管它可以多次调用该函数。如果一个函数对不同类型的输入（例如，正数或负数）表现不同，那么编写多个测试方法是一个好主意。

这是一个测试方法的简单示例：

```
def test_add(self):
    self.assertEqual(4, add(2, 2))
    self.assertEqual(0, add(2, -2))
```

按照惯例，将预期结果（4 或 0）放在前面，函数调用（`add`）放在最后。

如果测试方法中的任何断言失败，测试就会失败，并且该方法中的其余断言将不会被测试。因此，测试方法不应变得太长。

如果要测试的方法在类 `C` 中，并且你使用了 `import C` 而不是 `from C import *`，那么你也必须使用类名，例如，`self.assertEqual(4, C.add(2, 2))`。

以下是最常用的断言方法：

- `self.assertEqual(expected, actual)`
- `self.assertAlmostEqual(expected, actual)` 用于浮点数。
- `self.assertTrue(boolean)` 和 `self.assertFalse(boolean)`。

更多断言方法请参见附录 F。

你可以测试一个函数在应该引发异常时是否确实引发了异常，但这种测试有特殊的形式。这是必要的，因为函数的参数在函数调用之前就会被求值。例如，如果你写

```
self.assertRaises(ZeroDivisionError, 5/0)
```

那么参数 `5/0` 会在 `assertRaises` 被调用*之前*就被求值并引发异常。

解决方案是将函数名与其参数分开传递：

```
self.assertRaises(exception, function_name, arguments)
```

这允许 `assertRaises` 函数在 try-except 块内调用被测试的函数，并进行适当的处理。

当不进行测试时，一个常见的惯用做法是将主函数的调用作为代码文件的最后一行，例如 `main()`。这会导致主方法在文件加载后立即运行。在进行单元测试时，这是不可取的。应将其替换为

```
if __name__ == '__main__':
    main()
```

并在测试文件的末尾添加以下代码：

```
unittest.main()
```

这样，如果从程序文件加载，程序将运行；如果从测试文件加载，测试将运行。

### 5.4 单元测试示例

在文件 `get_digits.py` 中：

```
def get_digits(number):
    """Return a list of digits in an int or string."""
    string = str(number)
    return [x for x in string if x.isdigit()]

def main():
    s = input("Enter something: ")
    digits = get_digits(s)
    print("Digits found:", digits)
    if digits != []:
        main()

# Call main() if and only if started from this file
if __name__ == '__main__':
    main()
```

在文件 `get_digits_test.py` 中：

```
import unittest
from get_digits import *

class test_get_digits(unittest.TestCase):

    def test_get_digits(self):
        s = get_digits("<0.12-34 56abc789x")
        self.assertEqual(list("0123456789"),
                         get_digits(s))
        self.assertEqual(list("1230"),
                         get_digits(1230))
unittest.main()
```

### 5.5 测试套件

单元测试可以组合成一个*测试套件*。如果文件 `test-foo.py` 包含类 `TestFoo`，文件 `testbar.py` 包含类 `TestBar`，测试套件可以这样编写：

```
import unittest
import testfoo, testbar

def suite():
    suite = unittest.TestSuite()
    suite.addTest(
        unittest.makeSuite(
            testfoo.TestFoo))
    suite.addTest(
        unittest.makeSuite(
            testbar.TestBar))
    return suite

if __name__ == '__main__':
    test_suite = suite()
    runner = unittest.TextTestRunner()
    runner.run (test_suite)
```

## 图形用户界面

### 6.1 对话框

有几种 GUI（图形用户界面，发音为“gooey”）系统可以与 Python 一起使用。本章讨论 Tkinter，它随标准 Python 发行版捆绑提供。

许多程序不需要完整的 GUI，只需要一两个对话框。Tkinter 提供了许多这样的对话框。要使用它们，请从 tkinter 导入 messagebox、simpledialog 和/或 filedialog。

- `messagebox.showinfo(title, message)`
- `messagebox.showwarning(title, message)`
- `messagebox.showerror(title, message)`
  - 以上所有本质上都是相同的；区别在于显示哪个图标。所有都提供标题、消息和一个“确定”按钮。
- *result* = `messagebox.askyesno(*title*, *question*)`
  - 这提供“否”和“是”按钮，分别返回 `False` 或 `True`。
- *result* = `simpledialog.askfloat(*title*, *message*)`
- *result* = `simpledialog.askinteger(*title*, *message*)`
- *result* = `simpledialog.askstring(*title*, *message*)`
  - 允许用户输入浮点数、整数或字符串。提供“取消”和“确定”按钮；如果对话框被取消，返回的值是 `None`。
- *input_file* = `filedialog.askopenfilename(initialdir=*path*, title=*title*)`
  - 请求读入一个文件。如果提供了 `initialdir` 参数，导航将从该点开始。
- *output_file* = `filedialog.asksaveasfilename(initialdir=*path*)`
  - 请求保存文件的位置。如果提供了 `initialdir` 参数，导航将从该点开始。

### 6.2 TKINTER

**GUI 程序的工作方式与没有 GUI 的程序不同。** 程序不是在主方法的控制下运行所有代码，而是创建一个 GUI，此后发生的一切都是用户与 GUI 交互的结果。例如，用户点击一个按钮，这会导致执行某些代码。

从以下开始

```
from tkinter import *
import tkinter.ttk
```

之后，你的代码应该创建一个“根”窗口，

```
top = Tk()
```

接下来，用控件填充窗口（见下文）。

最后，将执行权交给 GUI：

```
top.mainloop()
```

需要执行三个主要任务：

1. 创建一些控件（按钮、文本区域等）。
2. 在窗口中排列控件。
3. 将代码与某些控件关联。

你可以在调用 `mainloop` 之前执行一些初始化代码，但一旦调用 `mainloop`，GUI 就接管了控制。

一个小麻烦是，每当使用 Tkinter 构建 GUI 时，它必须显示一个“根窗口”。这是一个出现在你屏幕上的小窗口，没有功能。要消除它，你可以自己创建一个根窗口并立即隐藏它。这只需要做一次。

```
import tkinter
root = tkinter.Tk()
root.withdraw()
```

### 6.3 创建控件

运行 GUI 程序时打开的窗口是一个**容器**：它可以包含**控件**（按钮、文本区域等）。**框架**是一个容器，你可以将其放入窗口中，或放入另一个框架中。

Tkinter 中有 15 种类型的控件，每种都有许多选项，用 `option=value` 表示。这里只介绍最常见的类型和选项。在下文中，我们假设窗口名为 `top`。

- **`fr = Frame(parent, option, ...)`**
  - 这是一个用于放置控件的容器。`parent` 可以是顶层窗口（`top`）或另一个 `Frame`。一些有用的选项是 `bg=color`（背景颜色，作为颜色名称或十六进制数）和 `bd=n`（边框宽度，以像素为单位）。
- **`but = Button(parent, text=string, command=function_name)`**
  - 创建一个包含字符串的按钮，点击时将调用指定的函数。无法向函数提供参数。
- **`lab = Label(parent, text=string)`**
  - 创建一个可以显示但用户不能编辑的标签。
  - 要更改文本，使用 `lab.configure(text=new_text)`。
- **`ent = Entry(parent, width=n)`**
  - 创建一个足够显示大约 `n` 个字符的矩形，用户可以在其中输入单行文本。可以输入超过 `n` 个字符，但可能无法全部显示。
  - 要检索文本，调用 `ent.get()`。
- **`txt = Text(parent, width=num_characters, height=num_lines)`**
  - 创建一个宽度为 `num_characters`、高度为 `num_lines` 的矩形，用户可以在其中输入多行文本。
  - 可以输入任意数量的行，但只有指定数量的行会可见。
  - 要检索文本，调用 `txt.get(1.0, END)`。
  - 要删除所有文本，使用 `txt.delete(1.0, END)`。
  - 要插入文本，使用 `txt.insert(END, text)`。
- `var = IntVar()`
  - 将 `var` 定义为 `IntVar`（参见下面的 `Checkbutton`）。
- `chk = Checkbutton(parent, text=string, variable=var, command=function)`
  - `var` 必须已用 `IntVar()` 定义。
  - 创建一个带有给定文本的复选框。
  - 如果选中，`var.get()` 将返回 `1`，如果未选中则返回 `0`。

### 6.4 将控件放入 GUI

有三种方法可以将控件排列到主窗口和框架中：`pack`、`grid` 和 `place`。在任何给定的窗口或框架中，只能使用其中一种方法。

可以通过将多个框架放入窗口和/或其他框架中，并为窗口和每个框架使用不同的布局来创建复杂的布局。

`widget.pack(options)` 只是将控件添加到窗口或框架中。选项有：

- `side=side`，其中 `side` 是 `LEFT`、`RIGHT`、`TOP` 或 `BOTTOM` 之一，用于从该侧开始添加控件。
- `expand=True` 以扩展控件以填充可用空间。

### 6.5 TKINTER 示例

`fill` 指定如何扩展控件，其中 `how` 可以是 `X`（水平）、`Y`（垂直）、`BOTH` 或 `NONE`。

`widget.grid(options)` 将控件添加到网格（二维表格）中。选项如下：

- `row=n` 控件所在的行。默认为第一个可用行。
- `column=n` 控件所在的列；默认为 0。
- `rowspan=n` 控件应占据的行数。
- `columnspan=n` 控件应占据的列数。
- `ipadx=n`, `ipady=n` 控件的内边距（以像素为单位），水平和垂直方向。
- `sticky=d` 如果控件所在空间较大，将其放置在何处。`d` 是 `N`、`S`、`E`、`W`、`NE`、`SE`、`NW`、`SW` 之一。

`widget.place(options)` 精确指定每个控件的放置位置。选项如下：

- `x=pixels`, `y=pixels` 控件锚点相对于父容器的 x 和 y 位置。
- `anchor=d` x 和 y 指的是控件的哪个部分，其中 `d` 是 `N`、`S`、`E`、`W`、`NE`、`SE`、`NW`、`SW` 之一。默认为 `NW`（左上角）。
- `bordermode=OUTSIDE` 在定位控件时考虑父容器的边框，否则为 `INSIDE`（`INSIDE` 是默认值）。
- `height=pixels`, `width=pixels` 控件的高度和宽度，以像素为单位。
- `relx=float`, `rely=float` x 和 y 位置，作为父容器宽度和高度的分数（介于 0.0 和 1.0 之间）。
- `relwidth=float`, `relheight=float` 控件的大小，作为父容器宽度和高度的分数（介于 0.0 和 1.0 之间）。

这是一个掷单个骰子的程序。执行后，窗口应出现在屏幕的左上角。通过关闭窗口退出程序。

```python
from tkinter import *
import tkinter.ttk
from random import randint

def roll():
    n = str(randint(1, 6))
    result.configure(text=n)

top = Tk()

roll_button = Button(top, text='Roll', command=roll)
result = Label(top, width=12)

roll_button.grid(row=0)
result.grid(row=1)
```

- **`string.strip()`** 返回一个副本，其中所有前导和尾随空白字符都被移除。

## 附录 C：统计模块

以下是统计模块提供的一些函数。

-   statistics.mean(**data**) 返回数据的数学“平均值”，通过将所有值相加，然后除以值的数量来计算。
-   statistics.median(**data**) 返回数据的“中间”数，使得一半的值小于或等于它，另一半大于或等于它。如果值的数量是偶数，则结果是两个中间值的平均值。
-   statistics.mode(**data**) 返回数据中出现次数最多的数字（如果并列，则返回第一个这样的数字）。

例如，

```
from statistics import *
a = [1, 2, 3, 4, 5, 5, 78]
print(f"{mean(a)=}, {median(a)=}, {mode(a)=}")
```

输出：

```
mean(a)=14, median(a)=4, mode(a)=5
```

以下是更多函数：

-   statistics.stdev(data) 返回样本标准差。
-   statistics.variance(data) 返回样本方差。

使用与上面相同的 `a` 值，

```
print(f"{stdev(a):6.2f}, {variance(a):6.2f}")

输出：

28.26, 798.67
```

-   any(iterable) 如果可迭代对象中至少有一个元素评估为真值，则返回 True。
-   filter(test, iterable) 返回一个迭代器，用于可迭代对象中通过测试的项目。
-   len(iterable) 返回可迭代对象中的元素数量。
-   list(iterable) 返回可迭代对象中元素的列表，顺序相同。
-   map(function, iterable) 返回一个迭代器。迭代器返回的每个值都是将函数应用于可迭代对象相应值的结果。
-   max(iterable) 返回可迭代对象中的最大值。
-   min(iterable) 返回可迭代对象中的最小值。
-   set(iterable) 返回可迭代对象中值的集合。
-   sorted(iterable) 返回可迭代对象中元素的列表，按排序顺序排列。
-   sum(iterable) 返回可迭代对象中值的总和。
-   tuple(iterable) 返回可迭代对象中元素的元组，顺序相同。
-   zip(iterable1,...,iterableN) 返回一个 N 元组的迭代器，其中第一个元组包含每个可迭代对象的第一个值，第二个元组包含每个可迭代对象的第二个值，依此类推。当任何一个可迭代对象用尽值时，迭代停止。
-   element in iterable 如果 element 在可迭代对象中，则返回 True。
-   element not in iterable 如果 element 不在可迭代对象中，则返回 True。

通常，接受可迭代对象的函数也可以接受迭代器或生成器。如果这样做，迭代器或生成器将以其当前状态传入，该状态可能位于可迭代对象的开头，也可能不是。

上述一些函数必须检查可迭代对象的每个元素（例如 max）。其他函数，如 any，可能检查也可能不检查每个元素。应注意不要使用产生无限数量值的迭代器或生成器调用此类函数。

## 附录 E：操作系统命令

以下是导入 `os` 时可用的一些函数：

-   os.getcwd() 获取当前工作目录。
-   os.chdir(path) 将当前工作目录更改为 `path`。路径（以字符串形式给出）可以是绝对路径或相对路径。
-   os.listdir(path='.') 返回一个列表，包含目录 `'.'` 中条目的名称。
    -   请注意，`path`（非斜体）是一个关键字，而不是可以替换为其他名称的变量。在其他函数中，`path`（斜体）表示包含字符串的变量，或字符串本身。
-   os.mkdir(path, mode=0o777, dir_fd=None) 创建一个名为 `path` 的目录，数字模式为 `0o777`。
    -   模式是 Unix 风格的三位八进制代码。每个数字的三位分别指定所有者、组和其他用户的读、写和执行权限。
-   os.remove(path, dir_fd=None) 删除单个文件 `path`；不适用于目录。
-   os.rmdir(path, dir_fd=None) 移除（删除）空目录 `path`。
-   os.rename(src, dst) 将文件或目录 `src` 重命名为 `dst`。
-   os.walk(top, topdown=True, onerror=None, followlinks=False) 通过自上而下或自下而上遍历目录树来生成目录树中的文件名。对于以目录 `top` 为根的目录树中的每个目录（包括 `top` 本身），它会生成一个三元组 `(dirpath, dirnames, filenames)`。
