您是地理学家、地质学家还是计算机科学家？如果您回答“是”，那很了不起，但我是一名空间数据分析师。简而言之，我对探索数据和整合位置信息感兴趣。现在，位置数据和地理空间数据集的获取已相当普遍，无论专业头衔或研究领域如何，我们大多数人都对数据产生了好奇心。在分析中欣赏*位置*维度，为理解更广泛特征对特定观察或结果的影响引入了新的维度。我花费大量专业时间研究公共卫生数据和医疗保健领域的大型开源数据集。一旦您熟悉了地理编码和空间文件，您不仅可以跨多个领域整理洞察，还能识别出存在巨大差距的区域。

这也是公民科学家的时代。开源工具和大规模开放在线课程（MOOCs）的可及性不仅赋能了更广泛的专业人士，也赋能了对数据好奇的人们。也许您对某种鸟类有爱好或兴趣，并希望获取空间数据来了解它们的栖息地。它们在哪里筑巢？它们从哪里迁徙到哪里？哪些栖息地支持最多的物种，这种情况随时间如何变化？您或许能够创建您的观测点或其他感兴趣变量的地图。我们稍后在生成一个数据问题进行探索时，会进一步探讨这个想法。

与此同时，如果您对公民科学家项目感兴趣，不妨看看纽约公共图书馆（NYPL）的 Map Warper 项目。正如您可能想象的那样，NYPL 拥有大量历史地图。挑战在于通过搜索现代匹配的*地面控制点*（GCPs）来纠正过时测量技术中的错误，并相应地对图像进行扭曲。这被称为*地图校正*。

空间数据分析入门告诉我们，任何分析都需要一个明确的问题。一旦你提出了问题，你就能发现是否有数据可以帮助你回答或验证这个假设。我指的不是精心挑选的数据，而是所有对构建假设或产生洞察至关重要的数据。种族应该被视为其他事物的拙劣生物学代理，还是社会或政治建构？我们需要理解我们收集的变量，以便能够培养同理心、揭示政策差距或满足未实现的健康需求。如果面临缺失数据或无法获取的资源，我们可能需要重新表述问题。

当你熟悉人口普查数据时，你开始理解种族被赋予的繁重任务。现在能够检查关于地点的数据（例如住房、就业、交通和教育）阐明了空间数据的作用，并促使我将ArcGIS和QGIS等地理空间应用程序整合到关于贫困、种族不平等、健康结构决定因素以及不断涌现的各种新问题的讨论中。我认为，当你依赖电子表格或数据表格时，如果缺乏空间考量，你可能会错过关键的洞察。静态指标（如道路可能的位置或特定事件的坐标）或动态指标（如传染病的传播）在与位置智能整合时会变得更加强大。

空间分析独特解决的问题是检查地理边界内识别的特征之间的关系。例如，非地理数据描述了值的分布——我们可以依赖描述性统计。但如果我们好奇空间关系可能对这些值产生什么影响呢？

乍一看，我们可以欣赏来自美国社区调查的县级种族变量。我们可以看到分类变量的集群，但我们还知道其他信息吗？这些是每个地区的主要类别，但我们还想了解什么？这就是地理空间数据变得如此有价值的地方。

### 空间数据科学的概念框架

地球并非完美的球体。考虑到地球的化学性质，以及在太空中旋转产生的离心力会倾向于将中间部分向外推，导致其呈扁球体形状，这就说得通了。从技术上讲，地球的形状是一个椭球体，绕两极的周长比绕赤道的周长要短，就像这个星球从上到下被挤压过一样。当我们试图绘制地表地图时，我们会采用一个地理坐标系，并将其转换为投影坐标系——用于平面地图。*经纬网*（地图的纬度和经度线框架）的建立，由QGIS和其他GIS应用程序加速完成。坐标系试图以接近单一事实来源的有意义方式呈现位置数据。这些不同的投影被称为*圆锥投影*、*方位投影*和*圆柱投影*。每一种都会留下必须处理的变形。根据所选的坐标系，我们可能会在面积、距离、方向和大小上出现变形。你会很高兴知道，我们不需要独自应对这些妥协——软件处理了大部分复杂的数学计算。图1-5展示了一个地理坐标系。

如果你使用OpenStreetMap或谷歌地图，你会熟悉Web墨卡托坐标系。熟悉这些地图的变体将使你能够选择最佳投影，因为需要做出妥协。例如，在我从事人口健康领域的工作时，在投影中保持面积至关重要。如果我在地图上绘制百分比或原始数字，并且我希望尽可能公正，不让一个小地方看起来比其他地方大太多，那么其中就存在一种固有的偏见，影响我对该地图的解读。我将尽力审视投影的弱点和优势，并说：“我将选择一个保持面积的投影。”

保持面积的地图被称为*等面积投影*。后面的章节将更详细地讨论这一点。我将回顾QGIS和ArcMap，并展示它们如何转换投影以获得最佳制图效果。因为地球不是一个完美的球体，我们确实必须做出妥协并接受一点点变形。但如果我们确保与可视化最相关的度量标准在所选坐标系中得到体现，我们就已经成功了大半。自然，我们希望我们的值与现实世界中的实际值紧密匹配。一个有趣数据集中的“在哪里”问题的现实，往往是视觉传达的推动力。例如，空间现象通常是迭代的而非线性的。也许你对某个系统地理的范围或尺度感到好奇。你从哪里开始？整个世界无法放在一张纸或一个电脑屏幕上——至少无法以一种可供解读的可见方式呈现。

与地球仪精度（记住我们试图在平面上表示椭球体几何形状）的误差或偏差，通常由Goldberg-Gott误差分数来衡量。最近一次最小化误差的尝试是由J. Richard Gott及其同事研究的。作为参考，流行的墨卡托投影（图1-7）的误差分数为8.296，而温克尔三重投影为4.563。较低的分数反映了在平面地图渲染中更少的误差或妥协。新的J. Richard Gott地图（0.881）与地球仪（0.0）相比，是迄今为止误差分数最低的。

既然你已经了解了空间数据分析的广泛目标，那么请思考从地方社区一直到社会和全球层面所面临的挑战。例如，想想环境、医疗保健、生物学、地理学、经济学、历史、工程学、地方政府、城市规划和供应链管理。即使是像医疗保健可及性、环境法规、社区规划和交通这样看似地方性或区域性的问题，也跨越了物理和政治边界、生态区域、市政区和流域，并具有空间组成部分。由于地图是我们对数据可视化最早的欣赏之一，因此在审视我们的数据后，我们可能会对位置产生好奇，这是有道理的。

地理信息系统（GIS）持续分析数据，并在广泛的行业中提供实时洞察。尽管空间分析和非空间分析之间可能存在相似之处，但空间统计是专门为处理地理数据而开发的。两者都与地理要素相关，但空间统计专门研究地理编码的地理空间数据。例如，当想象机场数据时，有非空间统计变量，如区域或用途（军事、民用/公共）或准点到达和起飞，也有用于分析的空间组成部分（高程和地理坐标）。与传统的非空间统计方法不同，它们将空间（邻近性、面积、连通性和/或其他空间关系）直接纳入其数学计算中。此外，对于那些用Python编写的工具，源代码是可用的，以鼓励你学习、修改、扩展和/或与他人分享这些以及其他分析工具。这些复杂的问题是空间性的。这些问题发生在哪里？我们如何规划以在未来取得更好的结果？

*空间思维*的概念，涉及邻近性、重叠包含、邻接性，以及测量地理空间和要素与现象之间相互关系的方式。这就是我们通过空间素养入门可以学到的东西。*空间素养*始于内容知识以及对系统及其如何与人类影响的范围。阿斯彭全球变化研究所（AGCI）确定了地球的六大系统：

1. 大气圈
2. 冰冻圈
3. 水圈
4. 生物圈
5. 岩石圈
6. 人类圈

人类圈指的是人类在地球上的存在。地理空间数据使我们能够理解所有这些系统的相互关联性，并且我们拥有*大数据*——海量数据——可用于构建完善的数据问题。

你不必成为专家就能掌握应对更重大问题的重要技能。如果你从基础层面理解地理空间数据和技术的工作原理，你就已经踏上了通往更复杂概念的道路。你将学会构建数据问题，并准备好确定可操作的步骤，以推进开发你的创新应用或解决方案。

地理空间问题复杂多变，随空间和时间而变化，并日益影响我们的日常生活。只需看看当前的头条新闻，就能找到我们未来几年将面临的挑战实例：种族不平等、气候变化、健康的结构性决定因素、刑事司法、安全饮用水、可持续农业实践、海洋酸化、贫困、物种濒危与灭绝，以及经济困境——仅举几例。个人的位置如何影响其健康、福祉或经济机会？利用GIS，可以通过展示疾病传播率、到最近医院的距离、道路、水道、树木覆盖和城市步行便利性之间的模式来研究这类问题。

我们将深入探讨如何在Python中探索矢量数据，但首先我需要介绍一些在本书后续内容中会用到的概念。*矢量*数据由点、线和多边形表示。我们将使用Python脚本和QGIS集成将数据集加载到地图中，并检查矢量数据的结构。后续章节还将教授如何使用颜色和符号自定义地图，以提高清晰度和准确性。

Coverage、shapefile和地理数据库是ESRI产品的演进，代表了其文件格式的不同世代。我不会花太多时间讨论不同的格式，但你可能会看到数据文件被指定为shapefile（.shp）或地理数据库（.gdb），因此了解它们的优缺点是很有好处的。将文件格式与文字处理程序（如Microsoft Word）和简单文本文件进行比较。内容（页面上的文字）可能相同，但文本文件的复杂性和精细度肯定较低。不同的GIS文件格式也是如此。尽管内容相同，但功能元素却不同。如果你想分享一段文字，为什么选择文本文件？如果你希望文档能被所有人阅读，无论使用什么软件，或者你希望便于携带或使用更小的存储格式呢？GIS格式文件在功能上各不相同——shapefile没有拓扑或空间图层，而地理数据库可以选择包含此类图层。它们在简单性、冗余性、错误检测和存储大小方面也各不相同。在使用包含位置数据的数据集时，你可能没有选择余地，因此了解可用选项以及你能从地理角度说明什么非常重要。

### 理解空间关系

空间参考数据被定义为离散对象或连续（标量场）。客观视图关注特定位置可见的内容。地图上有特定的边界或区域，数据或对象要么存在，要么不存在。你不会期望在特定边界内的每个位置都存在一个建筑物，甚至是一个代表城市的多边形对象。

场视图代表缺乏特定边界但存在于整个地图视图中的连续数据。也许更直观一点，考虑地表高程、降水或地表温度。无论你的研究视图如何，你都可以在研究位置包含的每个位置记录这些测量值。

到目前为止，我只是在描述我们研究位置内点、多边形或线的分布。栅格，你可能还记得，将图像、地表温度和数字高程表示为一个值数组，该数组被划分为单元格网格。单元格或像素描述空间分辨率。这两个术语经常互换使用，描述代表所覆盖区域的单元格或像素的维度。空间数据模型将这些现实世界客观和/或场视图的抽象表示进行探索，以数学关系建模或预测关系。

照片和栅格图像之间的主要区别在于后者扩展的波长波段。这种超越简单的红、绿、蓝波长的增强数据，使得机器学习模型能够区分各种各样的物体。反射红外光的能力是物体特有的，并在多光谱图像中产生额外信息。太空中有数百颗地球观测卫星持续捕捉地球图像。世界各地的许多航天机构免费提供这些数据。这些数据集对科学家、研究人员、政府和企业具有巨大的价值。

### 空间素养

空间素养需要理解空间关系所必需的概念相互作用。你将在此基础上构建关于地理空间如何表示的知识，以便对空间概念进行推理和做出关键决策。流行的 Python 编程语言是实现超越简单地图投影、地理处理和地理可视化的更高级分析工具的强大工具。QGIS 拥有强大的编程接口，允许你扩展软件的核心功能以及编写脚本来自动化你的任务，并且 QGIS 支持 Python 脚本语言。即使你是编程和/或 QGIS 的新手，学习一点 Python 并熟悉 QGIS 编程接口，也能让你在工作中更加高效。

空间或关系数据库管理系统旨在存储和查询空间数据。这对我来说一直很困惑——PostGIS 和 PostgreSQL，到底有什么区别？以下是我的理解。如果你处理的是大型数据集，将数据存储在数据库中是合理的。关系数据库有模式，数据存储在列和行中。当你想要访问数据时，你查询关系数据库。结构化查询语言 (SQL) 执行查询，作为关系数据库的原生语言，允许无缝集成。

PostgreSQL 数据库管理系统 (DBMS) 是一个数据库，PostGIS 作为扩展器与其打包在一起，为 PostgreSQL 添加空间功能。例如，空间数据库通常会有一列来标识由空间参考标识符 (SRID) 定义的坐标系。SQL Server Spatial 和 ESRI ArcSDE 是其他空间数据库的例子。

对于本书，我主要使用我在培训或项目中首次使用的数据资源，并更新到最新报告的数据。我想确保无论你的操作系统如何，你都可以访问不同的数据集。QGIS 有各种各样的工具，这将使操作变得简单直接。我也在专业上使用 ArcGIS，但由于 ArcGIS Pro 只能在 Microsoft Windows 上运行，这会太受限。如果你是高级用户，并希望用本地数据或更能反映你个人或专业兴趣的数据替换数据，这里有一些建议：

1.  你需要一个健康混合的城市和农村地点，具有不同类型的道路、边界、水体特征和地形。
2.  当有选择时，我更喜欢最近的数据，但对于教学目的目的，但我将参考具有最广泛属性的来源。

3.  不要忘记将数据保存为GeoPackage文件格式，以便能够继续探索。
4.  如果你正在创建区域的栅格文件，请定位最大图层的范围坐标。

在探索属性表时，描述不同类型的属性可能会有所帮助。我们将在第5章一起构建地图时详细讨论这一点。我们希望制图的度量是否取决于单位的大小？作为参考，我选择了`pop_max`作为属性，这是一个*空间广延属性*的例子。人口计数与一个区域相关联，但如果我们将人口划分为更小的区域，计数不一定成比例，因为人口很少均匀分布。因为我们是在汇总一个特定区域，所以人口计数只有在代表整个区域时才是准确的。这些属性适用于长度、面积或体积等。如果你改变长度，或增加或减少面积或体积，你就是在改变单位的大小，因此度量值会改变。

像人口密度这样的*空间密集属性*不依赖于区域的大小。如果假设单位是均匀的，那么这些值可以描述其任何部分，而空间*广延*变量，如人口计数，则是空间依赖的。

### 映射不平等

在我们开始将 Python 与 QGIS 及其他平台集成之前，我想以一个强大的可视化来结束本章，该可视化展示了在考虑数据问题时，地点和位置的价值。

“新政时期的美国映射红线”网站展示了房主贷款公司（HOLC）的数据。该网站是这样介绍这个项目的：

在 1935 年至 1940 年间，联邦政府房主贷款公司的代理人创建了数千份区域描述，其中为现在称为佐治亚州萨凡纳市卡弗高地社区所写的描述尤为突出。HOLC 工作人员使用每个城市当地房地产专业人士（贷款人、开发商和房地产评估师）组织的数据和评估，为住宅社区分配了反映其“抵押贷款安全性”的等级，然后这些等级会在彩色编码的地图上可视化。获得最高等级“A”（在地图上显示为绿色）的社区，在银行和其他抵押贷款机构决定谁应该获得贷款以及城市中哪些地区是安全投资时，被视为风险最小。获得最低等级“D”（显示为红色）的社区则被视为“危险”。

该交互式网站允许选择感兴趣的具体城市。我选择了纽约市，以查看我经常光顾的历史区域。多年来，我一直在纽约市的社区中漫步，目睹了经济投资涌入那些曾经危险到不敢涉足的社区。1930 年代的红线政策将靠近公园的社区标记为“仍然理想”和“最佳”，只要与哈莱姆区和主要的黑人社区保持距离即可。

“地点”的力量一直延续到现代，抵押贷款安全红线不再是“正式”做法，但许多这些地区已经锁定了土地利用系统。

### 图 2-9. 1930 年代纽约市红线地图中的 HOLC 等级

*红线政策*锁定了土地利用系统，使得最佳等级（绿色或蓝色）之外的某些区域可能遭受更多洪水（更多沥青）、更少公园、有限的树线、更靠近废物处理设施以及更靠近州际高速公路（污染），并且可能经历过更高的地表温度，增加了不利气候影响的额外风险。

城市热岛是温度较高的人造或自然地点。拥有更多绿化、更少不透水表面或更少工业或建筑的社区显示为黄色，较温暖的区域则用橙色到红色表示。自红线政策将某些社区标记为危险以来，纽约市令人难以置信的演变，使其目前整体较低的城市热岛效应，似乎是这些土地利用趋势的例外。如果你探索城市热岛集群（用箭头标记）并重新审视图 2-9，这些可能是需要深入调查的区域。
