## 1.0 简介

当您购买树莓派时，您本质上购买的是一块组装好的印刷电路板，或者对于树莓派 400 而言，是安装在键盘外壳中的电路板。要获得一个功能完整的系统，您至少需要一个合适的电源、操作系统和鼠标。

本章中的食谱涉及设置您的树莓派并使其准备就绪。

由于树莓派使用标准的 USB 键盘和鼠标，大部分设置都非常简单，因此您将只专注于那些特定于树莓派的任务。

## 1.1 选择树莓派型号

### 问题

树莓派有许多型号，您不确定使用哪一种。

### 解决方案

选择使用哪种树莓派型号在很大程度上取决于您计划用它做什么。表 1-1 列出了一些用途和我的型号建议。

表 1-1. 选择树莓派型号

| 用途 | 建议型号 | 备注 |
|---|---|---|
| 替代台式电脑 | 树莓派 400 或树莓派 4 型号 B (4 GB) | 如果您进行网页浏览，您将需要 4 GB 内存。Pi 400 提供了内置在键盘外壳中的便利性。 |
| 电子实验 | 树莓派 2 或 3 型号 B | 相对较新的硬件将最大限度地减少软件问题。无需更高性能。 |
| 计算机视觉 | 树莓派 4 型号 B (4 GB) | 需要最高性能。 |
| 家庭自动化 | 树莓派 2 或 3 型号 B | 低功耗且性能绰绰有余。 |
| 媒体中心 | 树莓派 3 或 4 | 为了视频性能。 |
| 电子显示板 | 任何型号 | 带有 WiFi 的型号，有利于远程访问。 |
| 嵌入式无线电子项目 | 树莓派 Zero W | 低成本且支持 WiFi，便于远程访问。 |
| 嵌入式电子项目 | Pico | Pico 成本非常低，除了名称外，与大多数树莓派几乎没有共同之处。更多信息请参见第...章。 |

如果您想要一款通用的树莓派，我推荐树莓派 4 型号 B。它的内存是原始树莓派的四倍，并且拥有四核处理器，它将比 Pi Zero 更好地处理大多数任务，但它不会像树莓派 4 那样发热或消耗那么多电力。树莓派 3 型号 B+ 还有一个巨大的优势，即内置了 WiFi 和蓝牙，因此无需额外的 USB WiFi 适配器或蓝牙硬件。

#### 树莓派 4 型号 B

在撰写本文时，树莓派 4 型号 B（图 1-1）是最新款的*标准*树莓派。

![](img/54ed1f8e884bc184601bf6f652461a95_6_0.png)

图 1-1. 树莓派 4 型号 B

这款新型号还首次允许用户选择内存大小（1 GB、2 GB、4 GB 或 8 GB），价格根据您的内存选择而定。

最重要的变化之一是，为早期版本供电的 micro-USB 插座已被 USB-C 连接器取代。此外，早期版本的单个全尺寸 HDMI 连接器已被一对 micro-HDMI 连接器取代——因此您需要一根特殊的 HDMI 线或适配器。是的，您可以同时连接两个显示器。

在内部，树莓派比其前代产品快得多（特别是如果您选择 4 GB 或 8 GB 内存版本）。事实上，一些基准测试表明，它比任何以前的树莓派快三到四倍。这是以板上主芯片比早期版本运行温度高得多为代价的——事实上，热到足以伤人。

另一方面，如果您将树莓派嵌入到一个单一用途的项目中，使用紧凑的 Pi Zero W 并节省几美元可能是一个不错的选择。

#### 树莓派 400

树莓派是一款相当不错的台式电脑替代品，而没有哪款树莓派比树莓派 400（图 1-2）更适合此用途。

![](img/54ed1f8e884bc184601bf6f652461a95_7_0.png)

图 1-2. 树莓派 400

树莓派 400 本质上与树莓派 4 的硬件相同，但内置在键盘外壳中，让人想起 1990 年代的家用电脑。HDMI 和 USB 端口都位于 Pi 400 的背面。GPIO 引脚位于计算机背面，但不像普通树莓派那样容易访问，因此如果您购买树莓派是为了学习电子学，普通 Pi 可能是更好的选择。但是，如果您只是想要一台台式电脑替代品，那么 Pi 400 确实是一个非常好的选择。

### 讨论

图 1-3 显示了 Pi Zero W、树莓派 3 B 和树莓派 4。

![](img/54ed1f8e884bc184601bf6f652461a95_8_0.png)

如图 1-3 所示，Pi Zero W 的大小大约是 Pi 3 B 或 Pi 4 B 的一半，它有一个用于通信的 micro-USB 插座，另一个用于供电。Pi Zero W 还通过使用 mini-HDMI 插座和 micro-USB OTG 插座来节省空间。如果您想将键盘、显示器和鼠标连接到 Pi Zero，您需要为 USB 和 HDMI 端口都准备适配器，然后才能连接标准外围设备。树莓派 A+ 比 Pi Zero 大，并且具有全尺寸的 USB 和 HDMI 端口。

表 1-2 总结了迄今为止所有树莓派型号之间的差异，最新发布的型号位于顶部。

表 1-2. 树莓派型号

| 型号 | 内存 | 处理器（核心数 * 时钟频率） | USB 插座 | 以太网端口 | 备注 |
|---|---|---|---|---|---|
| 400 | 4 GB | 4 * 1.8 GHz | 4 (2 x USB3) | 是 | 内置于键盘中 |

## 1.3 封装树莓派

### 问题

你需要为树莓派准备一个外壳。

### 解决方案

除了树莓派 400，树莓派本身不附带外壳，除非你购买的是套件。这使得它有些脆弱，因为电路板底部有裸露的连接点，如果树莓派放置在金属物体上，很容易造成短路。

为树莓派购买一个外壳是个好主意。如果你打算使用树莓派的通用输入/输出（GPIO）引脚（用于连接外部电子设备的引脚），图 1-6 所示的 PiBow Coupé 是一个美观实用的设计，适用于树莓派 4 及更早版本。

### 讨论

有各种各样的外壳样式可供选择，包括：

- 简单的两件式卡扣塑料盒
- 可 VESA 安装的盒子（用于安装在显示器或电视背面）
- 兼容乐高的盒子
- 3D 打印的盒子设计
- 激光切割的亚克力设计
- 充当大型散热器用于冷却的设计
- 内置风扇的设计
- 用于实验室和车间的 DIN 导轨设计

![](img/54ed1f8e884bc184601bf6f652461a95_14_0.png)

图 1-6. 装在 PiBow Coupé 中的树莓派 2

购买哪种外壳很大程度上取决于个人品味。然而，以下是你需要考虑的一些事项：

- 你是否需要访问 GPIO 连接器？如果你计划将外部电子设备连接到树莓派，这一点很重要。
- 外壳通风是否良好？如果你计划超频树莓派（配方 1.12）或高强度运行播放视频或游戏，这一点很重要，因为这些都会产生更多热量。
- 最后，确保你购买的外壳适合你的树莓派型号。

如果你有 3D 打印机，你也可以打印自己的外壳。在 [Thingiverse](https://www.thingiverse.com/) 或 [MyMiniFactory](https://www.myminifactory.com/) 上搜索 Raspberry Pi，你会发现很多设计。

## 1.4 选择电源

### 问题

你需要为树莓派选择一个电源。

### 解决方案

适用于树莓派的电源的基本电气规格是提供稳定的 5V 直流电（DC）。
电源必须能够提供的电流大小取决于树莓派的型号及其连接的外围设备。值得购买一个能轻松应对树莓派的电源，你应该考虑任何型号的树莓派至少需要 1A。
如果你从购买树莓派的同一家店购买电源，卖家应该能告诉你它是否适用于树莓派。
树莓派 4 应该使用 3A 电源。这部分是因为它比早期型号具有更强的处理能力，需要更多电力，但也因为它的两个 USB3 端口能够向高功率 USB 外围设备（如外部 USB 驱动器）提供高达 1.2A 的电流。
如果你要在树莓派 4 之前的型号上使用 WiFi 或需要大量电力的 USB 外围设备，你应该购买能够提供 1.5A 甚至 2A 的电源。同时要小心那些可能无法提供准确或可靠 5V 电压的超低成本电源。

### 讨论

树莓派 4 是第一款改用更现代的 USB-C 连接器的树莓派。与早期主板上使用的 micro-USB 连接器不同，这种连接器是可逆的（图 1-8）。

![](img/54ed1f8e884bc184601bf6f652461a95_18_0.png)

在图 1-8 中，你可以看到树莓派 4 的 USB-C 电源连接器位于树莓派 3 的 micro-USB 连接器下方。顺便说一句，你还可以看到取代单个全尺寸 HDMI 连接器的两个 micro-HDMI 视频端口。

无论你的树莓派使用的是 USB-C 连接器还是 micro-USB 连接器，电源和连接器实际上与许多智能手机充电器中使用的相同。如果它们以 micro-USB 插头结尾，它们几乎肯定是 5V（但请检查）。那么，唯一的问题是它们是否能提供足够的电流。

如果不能，可能会发生一些不好的事情：

- 它们可能会变热，存在火灾风险。
- 它们可能会直接失效。
- 在高负载时（例如，当树莓派使用 WiFi 或播放视频时），电压可能会下降，树莓派可能会自行重置。

如果你使用的是树莓派 3 或更早的型号，请寻找标称能提供 1A 或更多电流的电源。如果它指定的是瓦特数（W）而不是安培数（A），将瓦特数除以 5 即可得到安培数。因此，一个 5V 10W 的电源可以提供 2A（2,000mA）。

使用最大电流为 2A 的电源并不会比使用 700mA 的电源消耗更多电能。树莓派只取用它所需的电流。

如果你要购买新的树莓派，我认为作为通用计算机的最佳选择是树莓派 4 或 400。如果你不需要 WiFi 或想要更小的设备，也可以考虑 3 B、2 B 或 Zero W。

## 1.5 选择操作系统

### 问题

树莓派有多种不同的操作系统，你不确定该使用哪一个。

### 解答

这个问题的答案取决于你打算用树莓派做什么。

如果只是作为通用计算机使用，或用于电子项目，你应该使用 **Raspberry Pi OS**，这是树莓派的标准官方发行版。

如果你计划将树莓派用作媒体中心，有一些专门为该目的设计的发行版（Linux 版本）（参见配方 4.1）。

在本书中，我们几乎只使用 **Raspberry Pi OS**，尽管大多数配方也适用于任何基于 Debian 的 Linux 发行版。

### 讨论

如果你有兴趣尝试一些不同的发行版，可以购买一些 microSD 卡（它们并不贵），并将各种发行版复制到上面。如果你这样做，最好将不想丢失的文件保存在插入树莓派的 USB 闪存盘上。

请注意，如果你使用后续的某个配方来写入自己的 SD 卡，你需要一台带有 SD 卡插槽（以及 SD 到 microSD 适配器）的计算机，或者你可以购买一个便宜的 USB SD 卡读卡器。

树莓派官方发行版列表。

## 1.6 使用 Pi Imager 安装操作系统

### 问题

你想将树莓派的操作系统直接写入 microSD 卡，以便你的树莓派可以直接使用。

### 解答

在使用树莓派之前，你必须通过将磁盘映像写入 microSD 卡来准备一张装有 **Raspberry Pi OS** 操作系统的 microSD 卡。

将磁盘映像写入 microSD 卡的过程如下：

1.  使用 Mac、Windows 或 Linux 计算机（不是你的树莓派），下载 [Raspberry Pi Imager](https://www.raspberrypi.com/software/)。
2.  将 microSD 卡插入你的计算机。最好断开任何其他可移动介质的连接，以免意外覆盖错误的设备。
3.  启动 Raspberry Pi Imager（图 1-10）。
4.  将操作系统选择为 **Raspberry Pi OS (32 bit)**，并选择 SD 卡。
5.  点击 **Write**，等待映像文件复制到可移动介质上。

准备好 SD 卡或其他可移动介质后，你可以将其连接到树莓派。当树莓派通电时，它将启动到你安装的任何操作系统发行版。

### 讨论

硬件供应商有时会提供自己的磁盘映像，其中内置了对其硬件的支持。最好避免使用此类映像，因为这样做意味着你将无法获得使用标准 **Raspberry Pi OS** 发行版和所有预装软件的所有好处。这也意味着，如果你在某个软件上遇到问题，找到支持会困难得多，因为你使用的是非标准发行版。

### MICROSD 卡

并非所有的 microSD 卡都生而平等，使用更好的卡，你的树莓派性能会更好。因此，请寻找标明为“class 10”的卡。**Raspberry Pi OS** 包含一个实用程序（[链接待补充]）来测试你的 microSD 卡。你可以在“附件”下的“Raspberry Pi 诊断”中找到它。

在容量方面，你应该寻找至少 16 GB 的卡，实际上，考虑到价格差异很小，32 GB 的卡是更好的选择，因为它将为你提供充足的扩展空间。

### 另请参阅

树莓派安装 **Raspberry Pi OS** 的指南可在此处找到：[https://www.raspberrypi.com/software/](https://www.raspberrypi.com/software/)

## 1.8 从真正的硬盘或 USB 闪存盘启动

### 问题

你的 microSD 卡太小，和/或你担心整个操作系统都在 SD 卡上运行。

### 解答

在树莓派 4 和 400 出现之前，这是可能的，但并不容易，涉及一些复杂的命令，如果操作不当，有可能“变砖”你的树莓派。如果你有树莓派 4 或 400，那么现在这变得非常简单，可以从 Raspberry Pi Imager 开始。

该过程类似于使用 Raspberry Pi Imager 设置 microSD 卡。要遵循此配方，你需要一台 Windows、Mac 或 Linux 计算机；一个 USB SSD（固态硬盘）以及一张 microSD 卡。尽管此配方的全部意义在于用 USB 驱动器替换 microSD 卡作为启动设备，但你仍然需要两张 microSD 卡（一张是你现有的用于树莓派的 microSD 启动设备，另一张是空白的 microSD 卡）。

1.  将空白的 microSD 卡插入读卡器。
2.  启动 Raspberry Pi Imager，然后从“操作系统”下拉菜单中，选择 *Misc utility Images*，然后选择 *Bootloader*，再选择 *USB Boot*（图 1-14）。请注意，如果这些选项在你的 Raspberry Pi Imager 中不存在，那么你可能需要下载 [最新版本](https://www.raspberrypi.com/software/)。
3.  从“存储”下拉菜单中选择你的 microSD 卡，然后点击 **Write**。
4.  将新写入的 microSD 卡放入你的树莓派并启动它。这张 microSD 卡映像的唯一目的是重新配置你的树莓派以进行 USB 启动。一旦完成其工作，屏幕将变为绿色。
5.  关闭树莓派电源并更换 microSD 卡，这样你现在使用的是原来的 microSD 系统卡。
6.  启动你的树莓派，然后从菜单中选择 *Accessories*，再选择 *SD Card Copier*（图 1-15）。选择你的系统 microSD 卡作为源，外部 USB 驱动器作为目标，然后点击 **Start**。
7.  复制完成后，你可以再次关闭树莓派并取出 microSD 卡。下次打开树莓派时，它应该会从 USB 驱动器启动。

## 1.9 连接 DVI 或 VGA 显示器

### 问题

你的显示器没有 HDMI 接口，但你想将它与树莓派一起使用。

### 解决方案

很多人都遇到过这个问题。幸运的是，可以为带有 DVI 或 VGA 输入但没有 HDMI 接口的显示器购买适配器。

DVI 适配器是最简单且最便宜的。如果你搜索“HDMI 公头转 DVI 母头转换器”，价格可以低于 5 美元。

### 讨论

使用 VGA 适配器更复杂，因为它们需要一些电子元件将信号从数字转换为模拟，因此要小心那些不包含这些元件的线缆。官方转换器名为 Pi-View，在任何销售树莓派的地方都可以买到。Pi-View 的优势在于经过测试，已证实可与树莓派配合使用。你可以在网上找到更便宜的替代品，但这些通常无法工作。

然而，实际上，鉴于新显示器都带有 HDMI 接口，你最好将钱花在购买新显示器上，而不是适配器上。

### 另请参阅

## 1.10 使用复合视频显示器/电视

### 问题

你的低分辨率复合显示器上的文字难以辨认。

### 解决方案

你需要为小屏幕调整树莓派的分辨率。树莓派有两种类型的视频输出：(1) HDMI，和 (2) 来自音频插孔的复合视频，后者需要一根特殊的线缆。其中，HDMI 是质量好得多的选择。如果你打算将复合视频作为主屏幕，你可能需要重新考虑。如果你使用的是复合视频屏幕——比如，因为你需要一个非常小的屏幕——你需要进行一些调整以使视频输出适应屏幕。你需要对文件 `/boot/config.txt` 进行一些更改。你可以通过将 SD 卡重新插入 SD 卡读卡器，在笔记本电脑或 PC 上编辑此文件，或者你也可以在树莓派上直接编辑它（无需将其取出）。在树莓派本身上编辑文件通常使用 nano 编辑器。这有点棘手，我建议你在尝试编辑第一个文件之前彻底阅读[配方 3.7](http://elinux.org/)。如果你愿意继续使用 nano 编辑文件，请在终端会话中输入以下命令：

```
$ sudo nano /boot/config.txt
```

请注意，要保存并退出 nano，请按 Ctrl-X，然后按 Y（确认），然后按 Enter。

如果文字太小无法阅读，最好将 SD 卡从树莓派中取出并插入你的计算机。然后该文件将位于 SD 卡的顶层目录中，你可以使用 PC 上的文本编辑器（如 Notepad++）进行修改。

你需要知道屏幕的分辨率。对于许多小屏幕，这将是 320 x 240 像素。在文件中找到以下两行：

```
#framebuffer_width=1280
#framebuffer_height=720
```

删除每行开头的 # 并将两个数字更改为屏幕的宽度和高度。删除 # 会启用该行。在以下示例中，这些行已被修改为 320 乘 240：

```
framebuffer_width=320
framebuffer_height=240
```

保存文件并重新启动树莓派。你应该会发现一切都变得更容易阅读了。你可能还会发现屏幕周围有一个又大又粗的边框。要调整这一点，请参阅[配方 1.11](Recipe 1.11)。

### 讨论

有许多低成本的闭路电视显示器，当你制作像复古游戏机（[配方 4.4](Recipe 4.4)）之类的东西时，它们可以成为树莓派的好伴侣。然而，这些显示器通常分辨率非常低。

### 另请参阅

有关使用复合显示器的另一个教程，请参阅[这个 Adafruit 教程](this Adafruit tutorial)。

另外，当你使用 HDMI 视频输出时，请参阅配方 1.9 和 1.11 来调整你的画面。

## 1.12 最大化性能

### 问题

你的树莓派似乎非常慢，所以你想超频它以使其运行得更快。

### 解决方案

        return 1

    factorial(200)
after_time = time.clock()

print(after_time - before_time)
```

## 2.0 简介

树莓派设计用于连接互联网。其在互联网上通信的能力是其关键特性之一，并开启了各种可能的用途，包括家庭自动化、网络服务、网络监控等。

连接可以通过以太网线进行（至少对于大多数型号而言），较新的型号内置了 WiFi。

## 2.1 连接到有线网络

### 问题

你想通过有线网络连接将树莓派连接到互联网。

### 解决方案

首先，如果你拥有树莓派（1、2 或 3）的 A 型或 A+ 型，或 Pi Zero，则没有用于以太网的 RJ45 接口。在这种情况下，访问互联网的最佳选择是使用无线 USB 适配器（参见方法 2.5）。

如果你拥有树莓派（1、2、3、4 或 400）的 B 型或 B+ 型，那么你很幸运；只需将以太网跳线插入其 RJ45 插座，然后将另一端连接到家庭路由器背面的空闲插座（图 2-1）。

### 讨论

如果 LED 闪烁，但你无法在树莓派上使用浏览器连接到互联网，请检查你的网络管理控制台是否启用了 DHCP。查找类似图 2-2 所示的选项。

## 2.2 查找你的 IP 地址

### 问题

你想知道树莓派的 IP 地址，以便与之通信，无论是将其作为 Web 服务器连接、交换文件，还是通过 SSH（方法 2.7）或 VNC（方法 2.8）远程控制它。

### 解决方案

IPv4 地址（用于本地地址）是一个四部分数字，唯一标识网络中计算机的网络接口。每个部分之间用点分隔。

要查找树莓派的 IP 地址，你需要在终端窗口中发出此命令：

```
$ hostname -I
192.168.1.16 fd84:be52:5bf4:ca00:618:fd51:1c .....
```

响应的第一部分是树莓派在家庭网络上的本地 IP 地址。

### 讨论

树莓派可以拥有多个 IP 地址（即每个网络连接一个）。因此，如果你同时通过有线和无线连接到你的 Pi，它将有两个 IP 地址。不过，通常你只会通过其中一种方式连接，而不是同时使用两种。要查看所有网络连接，请使用 `ifconfig` 命令：

## 2.3 设置静态IP地址

### 问题

你希望为你的树莓派设置一个不会改变的IP地址。

### 解决方案

有两种方法可以实现。一种方法使用你的网络控制器（你的家庭集线器）。第二种方法使用树莓派本身。

使用网络的方法通常是首选，但第二种方法，即由树莓派自行确定其IP地址，如果你打算在不同网络之间移动树莓派并希望保持IP地址不变，那么这种方法值得一试。

你家中的所有电脑、电视、手机和其他支持互联网的设备通常都通过集线器连接到互联网，该集线器将你的电话线、4G或光纤电缆连接到你的房子。所有这些设备，无论是通过WiFi还是通过直接电缆连接到集线器，都被称为你的局域网的一部分。

默认情况下，当你将新设备连接到你的局域网时（例如树莓派），无论是通过以太网电缆插入还是使用WiFi，局域网控制器（你的集线器）将使用称为DHCP的系统为新设备分配IP地址。该地址将从IP地址池中分配，范围可能从192.168.1.2到192.168.1.199（或可能是10.0.0.2到10.0.0.199）。换句话说，对于连接到局域网的每个设备，只有四部分IP地址的最后一部分会改变。

当DHCP为设备分配IP地址时，它会设置一个租用时间，即该设备将保证保留该IP地址多长时间，而不会面临被分配给其他设备的风险。一般来说，此租用时间的默认值相当短；在我的集线器上，它是一周。这意味着我的树莓派的IP地址可能会在一周后毫无预警地改变，如果树莓派在没有键盘、鼠标和显示器的项目中使用，可能很难找到其IP地址以允许我连接。这就是为什么我可能想为你的树莓派设置一个*静态IP地址*。

### 使用网络设置IP地址

确保你的树莓派IP地址不变的一种方法是直接进入你的集线器控制界面，并将DHCP租用时间更改为更高的值。要访问此界面，你需要使用一台计算机（可以是你的树莓派，但不一定是），并转到一个特定地址，该地址通常写在集线器上，描述为*路由器*地址或管理控制台地址。对于我的路由器，这是http://192.168.1.1。还需要输入用户名和密码。这些与WiFi接入点名称和密码不同。它们通常也写在集线器的某个地方，并且通常默认值分别为admin和password。

连接后，你需要在管理控制台的各个页面中查找任何提及的DHCP设置，这些设置应类似于图2-3。

![](img/54ed1f8e884bc184601bf6f652461a95_60_0.png)

将“租用持续时间”（或你的集线器上具体标记的任何内容）更改为允许的最大值。

像这样延长租用时间的一个缺点是它适用于你局域网上的所有设备。因此，如果你有很多设备，你可能会用完IP地址，因为DHCP在租用期到期之前无法重新分配旧的IP地址。

更好的方法是使用称为DHCP保留的东西。这指示DHCP将特定IP地址永久分配给特定设备。在图2-4中，你可以看到我已将IP地址192.168.1.3分配给设备raspberrypi-Ethernet（一个通过以太网电缆连接到集线器的树莓派）。

![](img/54ed1f8e884bc184601bf6f652461a95_61_0.png)

从现在起，每当该树莓派连接到局域网时，它将被分配IP地址192.168.1.3，并且DHCP不会将该IP地址分配给任何其他设备。

### 使用树莓派设置IP地址

为树莓派设置静态IP地址的第二种方法使用树莓派本身而不是局域网。实际上，树莓派正在向集线器请求它想要使用的IP地址。

让你的树莓派决定自己的IP地址与网络上通常的工作方式略有相反，即局域网决定IP地址。当然，危险在于局域网的DHCP控制器可能已将该IP地址分配给了其他计算机。如果你有多个树莓派，都试图使用相同的IP地址，也会出现类似的问题。IP地址不能共享；在这种情况下，其中一个树莓派将无法连接到网络。

如果你想使用本节中描述的方法，你需要确保你的IP地址对于特定的树莓派是唯一的，并且你为树莓派选择的静态IP地址在DHCP范围之外。例如，查看图2-3，你可以看到DHCP分配的IP地址范围是从192.168.1.2到192.168.1.199。因此，如果我们选择静态IP地址192.168.1.210，我们知道它不会被重新分配。请注意，IP地址的每个数字部分都在0到255之间，因此，例如，192.168.1.300不是有效的IP地址。

你的树莓派对于每个网络连接将有不同的IP地址。在本例中，我们将使IP地址静态化，并且对于WiFi和以太网电缆都相同。

要编辑此文件，请在终端中键入以下命令：

```
$ sudo nano /etc/dhcpcd.conf
```

首先，决定要使用的IP地址。你需要选择一个既未被网络上任何其他机器使用，又在DHCP使用的IP地址范围之外的地址。在本例中，我将使用192.168.1.210。然后我将修改文件内容，在文件末尾添加以下行：

```
interface eth0
static ip_address=192.168.1.210/24
static routers=192.168.1.1
static domain_name_servers=192.168.1.1

interface wlan0
static ip_address=192.168.1.210/24
static routers=192.168.1.1
static domain_name_servers=192.168.1.1
```

请注意，正如我之前提到的，我已为两个网络接口（以太网和WiFi）设置了相同的IP地址。如果两个连接都可用，第一个连接的将阻止另一个使用该IP地址。实际上，这意味着以太网连接将获胜，因为它比WiFi连接快得多。

用于`routers`和`domain_name_servers`设置的IP地址是你用于连接到集线器管理控制台的IP地址。许多集线器使用上述IP地址范围。`routers`和`domain_name_servers`条目旁边的地址与你将在集线器上找到的管理控制台地址相同。但是，如果你的集线器使用不太常见的地址格式10.0.0.1，你需要将我们前面示例中的所有192.168.1实例更改为10.0.0。

进行更改后，按Ctrl-X然后按Y保存文件。有关使用nano编辑器的指南，请参见配方3.7。你需要通过运行`sudo reboot`重新启动树莓派，更改才能生效。

### 讨论

随着不同版本的树莓派操作系统，网络发生了很大变化。这些说明适用于最新（截至撰写本文时）版本的树莓派操作系统。如果你没有最新版本的树莓派操作系统，你应该获取它，因为树莓派操作系统总是在发展和改进。你可以在配方3.40中了解如何执行此操作。要了解你拥有的树莓派操作系统版本，请参见配方3.39。

### 另请参阅

- [维基百科](https://en.wikipedia.org/wiki/IP_address)包含你想了解的关于IP地址的所有信息。

## 2.4 设置树莓派的网络名称

### 问题

你想更改树莓派的网络名称，使其不再只是“raspberrypi”。

### 解决方案

有几种方法可以实现。无论使用哪种方法，请确保你选择的网络名称不包含空格，并且只包含字母、数字和连字符（-）。

在所有三种方法中，你还需要重启树莓派才能使更改生效。

### 使用树莓派配置工具设置网络名称

除非你的树莓派是无头运行（未连接显示器和键盘），否则设置树莓派网络名称最简单的方法是使用树莓派配置工具。要打开此工具，请转到树莓派菜单，选择“首选项”，然后单击“树莓派配置”。然后单击“系统”选项卡（图 2-5）。

![](img/54ed1f8e884bc184601bf6f652461a95_65_0.png)

图 2-5. 使用树莓派配置工具更改主机名

将“主机名”字段中的名称更改为你偏好的名称，然后单击“确定”。然后系统会提示你重启以使更改生效（配方 1.14）。

### 使用命令行设置网络名称（简单方法）

你也可以使用 `raspi-config` 工具从命令行更改树莓派的网络名称。在终端会话中运行以下命令：

```
$ sudo raspi-config
```

这将打开 `raspi-config` 工具。使用上/下方向键选择“网络选项”，然后按回车键。这将打开一个表单，你可以在其中输入新的网络名称（图 2-6）。请注意，此界面仅使用命令行，因此你可以从 SSH 会话中使用它（配方 2.7）。

![](img/54ed1f8e884bc184601bf6f652461a95_66_0.png)

### 使用命令行设置网络名称（困难方法）

如果你真的想用困难的方法，你也可以直接编辑控制树莓派网络名称的文件。你需要更改两个文件。

首先，编辑文件 /etc/hostname。你可以通过打开终端窗口并输入以下命令来完成：

```
$ sudo nano /etc/hostname
```

将 `raspberrypi` 替换为你选择的名称。

其次，使用以下命令在编辑器中打开文件 /etc/hosts：

```
$ sudo nano /etc/hosts
```

该文件将如下所示：

```
127.0.0.1       localhost
::1             localhost ip6-localhost ip6-loopback
fe00::0         ip6-localnet
ff00::0         ip6-mcastprefix
ff02::1         ip6-allnodes
ff02::2         ip6-allrouters

127.0.1.1       raspberrypi
```

将文件末尾的名称（`raspberrypi`）更改为你偏好的新名称。

### 讨论

更改树莓派的名称可能非常有用，特别是如果你有多个树莓派连接到你的网络。

## 2.6 使用控制台线连接

### 问题

没有可用的网络连接，但你仍然想从另一台计算机远程控制树莓派。

### 解决方案

使用 *控制台线*（一种需要单独购买的特殊线缆——参见 [待添加链接]）连接到树莓派。

要使用此方法，你需要启用串行接口。这意味着，至少在设置树莓派期间，你需要连接键盘、显示器和鼠标。

要启用串行接口，请转到树莓派菜单，选择“首选项”，然后单击“树莓派配置”。选择“接口”选项卡，然后单击“串行端口”的“启用”按钮，如图 2-9 所示。

![](img/54ed1f8e884bc184601bf6f652461a95_71_0.png)

图 2-9. 使用树莓派配置工具启用串行接口

与大多数树莓派配置一样，你也可以通过在终端会话中运行以下命令来使用命令行 `raspi-config` 工具：

```
$ sudo raspi-config
```

选择“接口选项”，然后选择“串行”，如 [图 2-10](#figure-2-10) 所示。

![](img/54ed1f8e884bc184601bf6f652461a95_72_0.png)

图 2-10. 使用 `raspi-config` 工具启用串行接口

控制台线非常适合将用于 *无头* 模式的树莓派——即不连接键盘、鼠标或显示器。

[图 2-11](#) 中所示的控制台线可从 [Adafruit](#) 购买。

![](img/54ed1f8e884bc184601bf6f652461a95_73_0.png)

图 2-11. 控制台线

按照以下说明并参考图 2-11 连接线缆。

1. 将红色（5V）线连接到 5V 引脚，该引脚位于 GPIO 接口边缘左侧一个引脚处。
2. 将黑色（GND）线连接到树莓派上左侧下一个引脚的 GND。
3. 将白色线（Rx）连接到树莓派 GPIO 14（TXD），位于黑色线左侧。
4. 将绿色线（Tx）连接到 GPIO 15（RXD），位于树莓派上白色线左侧。

如果你使用不同的线缆，线缆颜色可能不同，因此请务必检查线缆的文档，否则有损坏树莓派的风险。

请注意，USB 线也在红线上提供 5V，其功率足以单独为树莓派供电，但不足以支持大量设备。

如果你是 Windows 或 macOS 用户，你需要为 USB 线安装驱动程序，可在此处获取 [Windows 驱动程序](https://example.com) 和 [macOS 驱动程序](https://example.com)。Linux 用户通常不需要为这些线缆安装任何驱动程序。

要从 macOS 连接到树莓派，你需要运行终端并输入以下命令：

```
$ sudo cu -l /dev/cu.usbserial -s 115200
```

连接后，按回车键，树莓派登录提示应该会出现（图 2-12）。默认用户名和密码分别是 *pi* 和 *raspberry*。

![](img/54ed1f8e884bc184601bf6f652461a95_74_0.png)

如果你尝试从 Windows 计算机连接到树莓派，你需要下载名为 [Putty](https://www.putty.org/) 的终端软件。

当你运行 Putty 时，将“连接类型”更改为串口，并将速度设置为 115200。你还需要将“串行线路”设置为电缆正在使用的 COM 端口。这可能是 COM7，但如果不行，请使用 Windows 设备管理器进行检查。
当你点击“打开”并按回车键时，终端会话应启动并显示登录提示。

### 讨论

如果你轻装出行，控制台电缆可以成为使用树莓派的一种非常便捷的方式，因为它同时提供了电源和远程控制树莓派的方法。
控制台引线在 USB 端有一个芯片，提供 USB 转串口接口。这有时（取决于你的操作系统）需要在你的 PC 上安装驱动程序。只要拥有适用于你 PC 的必要驱动程序，你应该可以使用任何 USB 转串口转换器。
如果你小心地将四个插座按正确的顺序粘合（或用胶带固定）在一起，使它们作为一个整体套在 GPIO 排针上，那么将引线的插座插入正确的位置会更容易。
使用像 Raspberry Leaf 这样的 GPIO 模板（参见[待添加链接]）可以更容易地找到 GPIO 排针上的正确位置。树莓派的引脚排列显示在[待添加链接]中。

### 另请参阅

你可以在这个 [Adafruit 教程](https://www.adafruit.com/)中找到更多关于使用串口控制台的信息。Adafruit 也出售控制台电缆。这里使用的引线是 Adafruit 提供的（产品代码 954）。

## 2.7 使用 SSH 远程控制树莓派

### 问题

你想通过安全外壳（SSH）从另一台计算机连接到远程的树莓派。

### 解决方案

在使用 SSH 连接到你的树莓派之前，你必须先开启 SSH。在较新版本的树莓派操作系统上，你可以使用树莓派配置工具（图 2-13），你可以在主菜单的“首选项”下找到它。只需选中 SSH 的复选框，然后点击“确定”。然后系统会提示你重启。

如果你更喜欢使用命令行，请使用 `raspi-config` 应用程序。你可以随时在终端中输入以下命令来启动它：

```
$ sudo raspi-config
```

点击“接口”选项卡，向下滚动到 SSH 选项，然后点击“启用”按钮。

![](img/54ed1f8e884bc184601bf6f652461a95_77_0.png)

图 2-13. 使用树莓派配置工具开启 SSH

如果你使用的是 macOS 或者在你想连接树莓派的计算机上安装了 Linux，你只需打开一个终端窗口并输入以下命令即可连接：

```
$ ssh 192.168.1.16 -l pi
```

这里，IP 地址（192.168.1.16）是你树莓派的 IP 地址（参见配方 2.2）。然后系统会提示你输入密码，并登录到树莓派（图 2-14）。

![](img/54ed1f8e884bc184601bf6f652461a95_78_0.png)

要从 Windows 计算机连接，你需要使用 Putty（[配方 2.6](Recipe 2.6)）来启动一个 SSH 会话。

### 讨论

SSH 是连接远程计算机的一种非常常见的方式；任何你可以在树莓派本身上执行的命令，都可以通过 SSH 使用。正如其名所示，它也是安全的，因为通信是加密的。

也许唯一的缺点是它是一个命令行环境，而不是图形环境。如果你需要远程访问完整的树莓派桌面环境，你需要使用 VNC（[配方 2.8](Recipe 2.8)）或 RDP（[待添加链接]）。

### 另请参阅

另请参阅这个 [Adafruit 教程](Adafruit tutorial)。

你也可以使用 PiBakery 来启用 SSH。

## 2.8 使用 VNC 远程控制树莓派

### 问题

你想通过虚拟网络计算（VNC）从 PC（Windows 或 Linux）或 macOS 访问树莓派的完整树莓派操作系统图形桌面。

### 解决方案

使用树莓派操作系统上预装的 VNC 软件。然而，为此，你首先需要配置你的树莓派以开启它。你可以使用树莓派配置工具来完成此操作，该工具可以在树莓派桌面主菜单的“首选项”部分找到。点击“接口”选项卡，向下滚动到 VNC 选项，选择“启用”按钮，然后点击“确定”（图 2-15）。

![](img/54ed1f8e884bc184601bf6f652461a95_80_0.png)

图 2-15. 开启 VNC 接口

如果你不介意虚拟屏幕分辨率非常低，你可以跳过本配方的这部分内容，直接远程连接到你的树莓派，但很可能你会想要增加分辨率。为此，请使用以下命令编辑 *config.txt* 文件：

```
$ sudo nano /boot/config.txt
```

找到如下所示的行：

```
# framebuffer_width=1900
# framebuffer_height=1024
```

删除每行开头的 # 字符以启用新的屏幕宽度和高度，如下所示：

```
framebuffer_width=1900
framebuffer_height=1024
```

你需要重启才能使更改生效。
要从远程计算机连接到树莓派，你需要安装 VNC 客户端软件。RealVNC VNC Viewer 是一个流行的选择，适用于 Windows、Linux 和 macOS。
当你在 macOS 或 PC 上运行客户端程序时，系统会要求你输入要连接的 VNC 服务器的 IP 地址（即你树莓派的 IP 地址）。
然后系统会提示你输入密码（图 2-16）。

![](img/54ed1f8e884bc184601bf6f652461a95_82_0.png)

“口令”和“签名”是安全设备，旨在在有人入侵你的树莓派时向你发出警报。如果在你再次认证时其中任何一个发生变化，你的树莓派可能已被入侵。

### 讨论

虽然你可以使用 SSH 命令行完成大多数操作，但有时能够访问树莓派的图形环境会很有用。
只要启用了 VNC 选项，树莓派的 VNC 服务器会在你重启时自动启动。

### 另请参阅

另请参阅这个 [Adafruit 教程](https://www.adafruit.com/)。

## 2.9 在 Mac 网络上共享文件

### 问题

你希望你的树莓派出现在 macOS Finder 的计算机列表中，以便你可以连接到它并使用 Finder 浏览文件系统。

### 解决方案

macOS 操作系统内置了通过网络在 Finder 中查看文件的支持（图 2-17）。但是，你必须在树莓派上进行一些配置更改，macOS 才能查看树莓派上的网络文件。

![](img/54ed1f8e884bc184601bf6f652461a95_83_0.png)

你需要知道树莓派的 IP 地址（配方 2.2）。

在树莓派上，使用以下命令安装 `netatalk`：

```
$ sudo apt-get update
$ sudo apt-get install netatalk
```

接下来，回到你的 Mac 上，在 Finder 菜单中，选择“前往”，点击“连接服务器”，然后输入 `afp://192.168.1.210` 作为服务器地址（当然，使用你树莓派的 IP 地址，而不是这里显示的地址）。然后，点击“连接”。然后系统会提示你登录（图 2-18）。

![](img/54ed1f8e884bc184601bf6f652461a95_84_0.png)

使用名称 **pi** 和你的密码登录，密码将是 **raspberry**，除非你已更改过。然后 Finder 应该会显示你树莓派上主目录的内容。

还需要在树莓派上进行一些配置更改，这将自动将你的树莓派挂载到 Mac 的文件系统上：

```
$ sudo apt-get update
$ sudo apt-get install avahi-daemon
$ sudo update-rc.d avahi-daemon defaults
```

接下来，输入此命令：

```
$ sudo nano /etc/avahi/services/afpd.service
```

将以下代码粘贴到文件中。这需要输入很多内容，所以如果你正在阅读本书的纸质版，你可以从[这个网页](this web page)复制并粘贴。只需向下滚动直到找到本章和本配方：

```
<?xml version="1.0" standalone='no'?><!--*-nxml-*->
<!DOCTYPE service-group SYSTEM "avahi-service.dtd">
<service-group>
  <name replace-wildcards="yes">%h</name>
  <service>
    <type>_afpovertcp._tcp</type>
    <port>548</port>
  </service>
</service-group>
```

要设置守护进程运行，以便在树莓派每次启动时自动执行此操作，请输入以下命令：

```
$ sudo /etc/init.d/avahi-daemon restart
```

切换回你的 Mac，你现在应该能在 Finder 中看到你的树莓派了。

### 讨论

能够在 Mac 和树莓派之间轻松移动文件非常有用。这意味着你可以在树莓派上使用文件，而无需连接单独的键盘、鼠标和显示器。
你也可以像在 Mac 上一样打开树莓派上的文件。这样做的好处是你可以使用 TextMate 或你最喜欢的 macOS 文本编辑器来编辑它们。
如果你使用的是 Windows 或 Linux，你也可以通过配置树莓派作为网络附加存储（NAS）来共享文件；参见[配方 2.10](Recipe 2.10)。

### 另请参阅

此处的说明改编自[本教程](this tutorial)，该教程将Matt Richardson和Shawn Wallace的著作*Getting Started with Raspberry Pi*（O'Reilly）列为原始来源。

## 2.10 使用树莓派作为网络附加存储

### 问题

你希望将树莓派用作网络附加存储（NAS），以便从网络上的计算机访问连接到树莓派的大容量USB驱动器。

### 解决方案

解决此问题的方法是安装和配置Samba。为此，请执行以下命令：

```
$ sudo apt-get update
$ sudo apt-get install samba
$ sudo apt-get install samba-common-bin
```

接下来，将USB硬盘连接到树莓派。它将自动挂载到你的`/media/pi`文件夹中。要检查它是否在那里，请运行此命令：

```
$ cd /media/pi
$ ls
```

驱动器应以其格式化时赋予的名称列出。每当树莓派重启时，它都会自动挂载自身。

接下来，你需要配置Samba，以便在网络上共享驱动器。为此，你首先需要添加一个Samba用户（pi）。输入以下命令并输入密码：

```
$ sudo smbpasswd -a pi
New SMB password:
Retype new SMB password:
Added user pi.
```

现在，你需要对文件`/etc/samba/smb.conf`进行一些更改，因此请输入此命令：

```
$ sudo nano /etc/samba/smb.conf
```

你要找的第一行在文件顶部附近：

```
workgroup = WORKGROUP
```

只有当你计划从Windows机器连接时才需要更改此项。这应该是你的Windows工作组的名称。对于Windows XP，默认值是MSHOME；对于较新版本的Windows，它是HOME（但请检查你的Windows网络）。请注意，通常在Mac和Windows PC（以及Linux机器）的混合网络中连接到NAS工作正常。

下一个更改在文件下方的“身份验证”部分。找到以下行：

```
# security = user
```

删除行首的`#`以激活该行并开启安全性。最后，滚动到文件末尾并添加以下行：

```
[USB]
path = /media/pi/NAS
comment = NAS Drive
valid users = pi
writeable = yes
browseable = yes
create mask = 0777
public = yes
```

保存文件，然后通过输入以下内容重启Samba：

```
$ sudo /etc/init.d/samba restart
```

如果一切正常，你的USB驱动器现在应该在网络上共享了。

### 讨论

要在macOS上连接到驱动器，请选择“前往”，然后从Finder菜单中单击“连接服务器”。接下来，在“服务器地址”字段中，输入**smb://raspberrypi/USB**。将打开一个登录对话框，你需要在其中将用户名更改为**pi**（图2-19）。

![](img/54ed1f8e884bc184601bf6f652461a95_88_0.png)

如果你从Windows机器连接到NAS，具体过程将因Windows版本而异。但是，基本原理是，在某个时候你需要输入网络地址，该地址应为`\raspberrypi\USB`（图2-20）。

![](img/54ed1f8e884bc184601bf6f652461a95_89_0.png)

然后，系统会提示你输入用户名和密码，然后才能使用NAS磁盘（图2-21）。你可能只需要在第一次执行此操作。添加网络位置后，你应该能够在文件资源管理器中直接导航到它。

![](img/54ed1f8e884bc184601bf6f652461a95_90_0.png)

如果你是Linux用户，以下命令应该可以为你挂载NAS驱动器：

```
$ sudo mkdir /pishare
$ sudo smbmount -o username=pi,password=raspberry //192.168.1.16/USB /pishare
```

## 3.4 使用终端浏览文件系统

### 问题

你需要了解如何使用终端更改目录并在文件系统中移动。

### 解决方案

用于浏览文件系统的主要命令是 `cd`（更改目录）。在 `cd` 之后，你需要指定要切换到的目录。这可以是当前目录内某个目录的*相对*路径，也可以是文件系统中其他位置的*绝对*路径。
要查看当前目录是什么，请使用命令 `pwd`（打印工作目录）。

### 讨论

尝试几个例子。打开一个终端会话，你应该会看到如下提示符：

```
pi@raspberrypi: ~ $
```

你在每个命令后看到的提示符（`pi@raspberrypi: ~ $`）是你的用户名（`pi`）和计算机名（`raspberrypi`）的提醒。`~` 字符是你的主目录（`/home/pi`）的简写。因此，在任何时候你都可以按如下方式将当前目录切换到主目录：

```
$ cd ~
```

> **提示**

在本书中，我使用 `$` 作为每行开头需要你输入命令的地方。这被称为提示符。命令行的响应不带任何前缀；它就像在树莓派屏幕上显示的那样出现。

你可以通过使用 `pwd` 命令来确认命令确实将目录设置为了主目录：

```
$ pwd
/home/pi
```

如果你想在目录结构中向上移动一级，可以在 `cd` 命令后使用特殊值 `..`（两个点），如下所示：

```
$ cd ..
$ pwd
/home
```

正如你现在可能已经推断出的，特定文件或目录的路径是由用 / 分隔的单词组成的。因此，整个文件系统的根目录是 /，要访问 / 内的主目录，你会引用 /home/。然后，要找到其中的 *pi* 目录，你会使用 /home/pi/。你可以省略路径末尾的 /。

路径可以是绝对路径（以 / 开头并指定从根目录开始的完整路径），也可以是相对于当前工作目录的路径，这种情况下它们不能以 / 开头。

你对主目录中的文件拥有完全的读写权限，但当你进入存放系统文件和应用程序的位置时，你对某些文件的访问将被限制为只读。你可以覆盖此限制（配方3.12），但需要谨慎操作。

通过输入命令 `cd /` 和 `ls` 来查看目录结构的*根*，如图3-6所示。

```

boot  etc   lib   media       opt  root  sbin sys  usr
pi@raspberrypi:/ $

图3-6. 列出目录内容

## 3.5 复制文件或文件夹

### 问题

你需要复制一个文件或目录。

### 解决方案

使用 `cp` 命令来复制文件和目录。

### 讨论

`cp`（复制）命令用于创建文件或目录的副本。其基本语法是 `cp 源 目标`。

例如，要在当前目录中创建名为 `myfile.txt` 的文件的副本，你可以使用：

```
$ echo "hello" > myfile.txt
$ ls
myfile.txt
$ cp myfile.txt myfile2.txt
$ ls
myfile.txt    myfile2.txt
```

虽然在此示例中两个文件路径都位于当前工作目录内，但文件路径可以指向文件系统中任何你有写入权限的位置。以下示例将原始文件复制到 `/tmp` 目录，这是一个用于存放临时文件的位置（不要在该文件夹中存放任何重要文件）：

```
$ cp myfile.txt /tmp
```

请注意，在这种情况下，没有指定新文件的名称，只指定了它要存放的目录。这将在 `/tmp` 中创建一个名为 *myfile.txt* 的副本，与原始文件同名。

有时，你可能不想只复制一个文件，而是想复制一个包含许多文件甚至其他子目录的整个目录。要复制一个目录及其所有内容，你需要使用 `-r` 选项（表示递归）：

```
$ cp -r mydirectory mydirectory2
```

无论何时复制文件或文件夹，命令的结果都会告诉你是否没有权限。如果是这种情况，你需要更改目标文件夹的权限（参见配方 3.14），或者使用超级用户权限复制文件（参见配方 3.12）。

### 讨论

## 3.12 使用超级用户权限执行任务

### 问题

某些命令无法运行，因为你*权限不足*。

### 解决方案

你需要使用超级用户权限来执行命令。`sudo`（substitute user do）命令允许你以超级用户权限执行操作。只需在命令前加上 `sudo` 即可。

### 讨论

大多数你想在命令行上执行的任务通常不需要超级用户权限。最常见的例外情况是安装新软件和编辑配置文件。`apt-get` 命令是向 Raspberry Pi OS 安装新软件的主要方式。你将在 [配方 3.17](Recipe 3.17) 中正式遇到它。

另一个需要超级用户权限的例子是 `reboot` 命令。如果你尝试以普通用户身份运行它，你会收到一些错误消息：

```
$ reboot
Failed to set wall message, ignoring: Interactive authentication required.
Failed to reboot system via logind: Interactive authentication required.
Failed to open /dev/initctl: Permission denied
Failed to talk to init daemon.
```

如果你在命令前加上 `sudo`，命令就能正常工作：

```
$ sudo reboot
```

如果你有一大堆命令需要以超级用户身份运行，又不想在每个命令前都加上 `sudo`，可以使用以下命令：

```
$ sudo sh
#
```

注意提示符如何从 `$` 变为 `#`。所有后续命令都将以超级用户身份运行。当你想恢复为普通用户时，输入命令：

```
# exit
$
```

### 另请参阅

要了解更多关于文件权限的信息，请参阅 [配方 3.13](Recipe 3.13)。
要使用 `apt-get` 安装软件，请参阅 [配方 3.17](Recipe 3.17)。

## 3.13 理解文件权限

### 问题

你已经看到文件名被列出时伴随的奇怪字符。你想知道它们的含义。

### 解决方案

要查看与文件和目录相关的权限和所有权信息，请使用带有 `-l` 选项的 `ls` 命令。

### 讨论

运行命令 `ls -l`（选项字母是小写 *L*），你会看到类似这样的结果：

```
$ ls -l
total 16
-rw-r--r-- 1 pi pi     5 Apr 23 15:23 file1.txt
-rw-r--r-- 1 pi pi     5 Apr 23 15:23 file2.txt
-rw-r--r-- 1 pi pi     5 Apr 23 15:23 file3.txt
drwxr-xr-x 2 pi pi  4096 Apr 23 15:23 mydir
```

`ls` 命令响应的第一行告诉你目录中有 16 个文件。

图 3-11 显示了列表信息的不同部分。第一个块包含权限。在第二个块中，数字 1（标记为“文件”）表示涉及多少个文件。此字段仅在列表条目是目录时才有意义；如果是文件，则通常只是 1。接下来的两个条目（都是 `pi`）是文件的所有者和组。大小条目（第五个块）表示文件的大小（以字节为单位）。修改日期会在每次编辑或更改文件时更改，最后一个条目是文件或目录的实际名称。

![](img/54ed1f8e884bc184601bf6f652461a95_119_0.png)

权限块分为四个部分（类型、所有者、组和其他）。第一部分是文件的类型。如果是目录，它将是字符 `d`；如果是文件，条目将只是 `-`。

下一部分包含三个字符，指定文件的所有者权限。每个字符都是一个开或关的标志。如果所有者有读权限，第一个字符位置将有一个 `r`。如果所有者有写权限，第二个位置将有一个 `w`。第三个位置（在此示例中为 `-`）如果文件对所有者可执行（程序或脚本），则将有一个 `x`。

第三部分具有相同的三个标志，但适用于组中的任何用户。用户可以被组织成组。因此，在此情况下，文件有一个用户 `pi` 和一个组所有权 `pi`。如果组 `pi` 中有任何其他用户，他们将拥有此处指定的权限。

最后一部分指定既不是 `pi` 也不在 `pi` 组中的用户的权限。因为大多数人只会以用户 `pi` 的身份使用 Raspberry Pi，所以最相关的权限在第一部分。

### 另请参阅

要更改文件权限，请参阅 [配方 3.14](Recipe 3.14)。

## 3.14 更改文件权限

### 问题

你需要更改文件的权限。

### 解决方案

你可以使用 `chmod` 命令来修改文件权限。

### 讨论

你可能想要更改文件权限的常见原因包括需要编辑标记为只读的文件，以及赋予文件执行权限以便它可以作为程序或脚本运行。
`chmod` 命令允许你添加或删除文件的权限。有两种语法可以做到这一点；一种需要使用八进制（基数 8），另一种是基于文本的。你将使用更容易理解的文本方法。
`chmod` 的第一个参数是要进行的更改，第二个是要应用更改的文件或文件夹。此更改参数采用权限范围（+、-、= 分别表示添加、删除和设置）后跟权限类型的形式。

例如，以下代码将为文件 *file2.txt* 的所有者（用户）添加执行（x）权限：

```
$ chmod u+x file2.txt
```

如果我们现在列出目录，我们可以看到 x 权限已被添加：

```
$ ls -l
total 16
-rw-r--r-- 1 pi pi     5 Apr 23 15:23 file1.txt
-rwxr--r-- 1 pi pi     5 Apr 24 08:08 file2.txt
-rw-r--r-- 1 pi pi     5 Apr 23 15:23 file3.txt
drwxr-xr-x 2 pi pi  4096 Apr 23 15:23 mydir
```

如果我们想为组或其他用户添加执行权限，我们将分别使用 `g` 和 `o`。字母 `a` 为所有人添加权限。

### 另请参阅

有关文件权限的背景信息，请参阅 [配方 3.13](Recipe 3.13)。
有关更改文件所有权，请参阅 [配方 3.15](Recipe 3.15)。

## 3.15 更改文件所有权

### 问题

你需要更改文件的所有权。

### 解决方案

你可以使用 `chown`（更改所有者）命令来修改文件或目录的所有权。

### 讨论

正如我们在配方 3.13 中发现的，任何文件或目录都有一个所有者和一个组与之关联。因为大多数 Raspberry Pi 用户只有一个用户 pi，所以我们实际上不需要担心组。

偶尔，你会发现系统上安装的文件具有与 pi 不同的用户。如果是这种情况，你可以使用 chown 命令更改文件的所有权。

要更改文件的所有者，请使用 chown，后跟新的所有者和组（用冒号分隔），然后是文件名。

你可能会发现你需要超级用户权限来更改所有权，在这种情况下，你应该在命令前加上 sudo（配方 3.12）：

```
$ sudo chown root:root file2.txt
$ ls -l
total 16
-rw-r--r-- 1 pi    pi       5 Apr 23 15:23 file1.txt
-rwxr--r-- 1 root  root     5 Apr 24 08:08 file2.txt
-rw-r--r-- 1 pi    pi       5 Apr 23 15:23 file3.txt
drwxr-xr-x 2 pi    pi    4096 Apr 23 15:23 mydir
```

使用名称巧妙的 `scrot`（SCReenshOT）屏幕截图软件。

### 讨论

## 3.17 使用 apt-get 安装软件

### 问题

你想使用命令行安装软件。

### 解决方案

从终端会话安装软件最常用的工具是 `apt-get`（高级打包工具）。
该命令的基本格式（必须以超级用户身份运行）如下：

```
$ sudo apt-get install <软件名称>
```

例如，要安装 AbiWord 文字处理软件，你需要输入以下命令：

```
$ sudo apt-get install abiword
```

### 讨论

`apt-get` 包管理器使用一个可用软件列表。此列表包含在你使用的 Raspberry Pi 操作系统发行版中，但可能已过时。因此，在使用 `apt-get` 安装新软件之前，最好始终运行以下命令来更新列表：

```
$ sudo apt-get update
```

该列表和用于安装的软件包都在互联网上，因此除非你的 Raspberry Pi 有互联网连接，否则这些操作都无法进行。

> **提示**

如果你在更新时遇到类似 `E: Problem with MergeList /var/lib/dpkg/status` 的错误，请尝试运行以下命令，这将删除有问题的文件并用一个新的空文件替换它：

```
$ sudo rm /var/lib/dpkg/status
$ sudo touch /var/lib/dpkg/status
```

安装过程通常需要一些时间，因为文件必须下载并安装。某些安装还会在你的桌面或开始菜单的程序组中添加快捷方式。

你可以使用命令 `apt-get search` 后跟搜索字符串（如 `abiword`）来搜索要安装的软件。这将显示你可以安装的匹配软件包列表。

### 另请参阅

## 3.18 删除使用 apt-get 安装的软件

### 问题

使用 `apt-get` 安装了大量程序后，你现在发现想要删除其中一些。

### 解决方案

`apt-get` 工具有一个选项（`remove`）可以删除软件包，但它只会删除那些通过 `apt-get install` 安装的软件包。

例如，如果你想删除 AbiWord，你会使用以下命令：

```
$ sudo apt-get remove abiword
```

### 讨论

像这样删除软件包并不总是会删除所有内容，因为软件包通常还有作为依赖项安装的先决条件软件包。要删除这些，你可以使用 `autoremove` 选项，如下所示：

```
$ sudo apt-get autoremove abiword
$ sudo apt-get clean
```

`apt-get clean` 选项将对未使用的软件包安装文件进行进一步的清理。

### 另请参阅

参见配方 3.17 了解如何使用 `apt-get` 安装软件包。

## 3.23 在启动时自动运行程序或脚本

### 问题

你希望一个程序或脚本在树莓派启动时自动运行。

### 解决方案

修改你的 *rc.local* 文件以运行你想要的程序。
使用以下命令编辑文件 `/etc/rc.local`：

```
$ sudo nano /etc/rc.local
```

在以 # 开头的第一段注释行之后添加以下行：

```
$ /usr/bin/python /home/pi/my_program.py &
```

在命令行末尾包含 `&` 非常重要，这样它才能在后台运行；否则你的树莓派将无法启动。

### 讨论

这种自动运行程序的方式需要非常小心地编辑 *rc.local*，否则你可能会导致树莓派无法启动。

### 另请参阅

## 3.24 作为服务自动运行程序或脚本

### 问题

你希望安排一个脚本或程序在每次树莓派重启时自动启动。

### 解决方案

大多数树莓派发行版所基于的 Debian Linux 使用一种基于依赖的机制来自动化在启动时运行命令。这有点棘手，涉及为你想要运行的脚本或程序创建一个配置文件，该文件将位于一个名为 *init.d* 的文件夹中。

### 讨论

以下示例向你展示如何运行主目录中的一个 Python 脚本。该脚本可以做任何事情，但在本例中，该脚本运行一个简单的 Python Web 服务器，这在[待补充链接]中有进一步描述。
以下是涉及的步骤：

- 1. 创建一个 *init* 脚本。
- 2. 使 *init* 脚本可执行。
- 3. 将新的 *init* 脚本告知系统。

首先，创建 *init* 脚本。你需要在文件夹 `/etc/init.d/` 中创建它。脚本可以任意命名，但在本例中，我们将其命名为 `my_server`。

使用以下命令通过 nano 创建新文件：

```
$ sudo nano /etc/init.d/my_server
```

将以下代码粘贴到编辑器窗口中并保存文件。这需要输入很多内容，因此如果你正在阅读本书的纸质版，你可以从[此网页](this web page)复制并粘贴代码；只需向下滚动直到找到本章和本技巧：

```
### BEGIN INIT INFO
# Provides: my_server
# Required-Start: $remote_fs $syslog $network
# Required-Stop: $remote_fs $syslog $network
# Default-Start: 2 3 4 5
# Default-Stop: 0 1 6
# Short-Description: Simple Web Server
# Description: Simple Web Server
### END INIT INFO

#! /bin/sh
# /etc/init.d/my_server

export HOME
case "$1" in
  start)
    echo "Starting My Server"
    sudo /usr/bin/python /home/pi/myserver.py 2>&1 &
    ;;
  stop)
    echo "Stopping My Server"
    PID=`ps auxww | grep myserver.py | head -1 | awk '{print $2}'`
    kill -9 $PID
    ;;
  *)
    echo "Usage: /etc/init.d/my_server {start|stop}"
    exit 1
    ;;
esac
exit 0
```

要自动化运行一个脚本，这确实需要做很多工作，但其中大部分是每个服务都相同的样板代码。要运行不同的脚本，只需通读脚本，更改描述和你想要运行的 Python 文件的名称即可。
下一步是使此文件对所有者可执行，你可以使用以下命令完成：

```
$ sudo chmod +x /etc/init.d/my_server
```

现在程序已设置为服务，你可以使用以下命令在将其设置为启动序列的一部分以自动启动之前，测试一切是否正常：

```
$ /etc/init.d/my_server start
Starting My Server
Bottle v0.11.4 server starting up (using WSGIRefServer())...
Listening on http://192.168.1.16:80/
Hit Ctrl-C to quit.
```

最后，如果运行正常，使用以下命令使系统知道你已定义的新服务：

```
$ sudo update-rc.d my_server defaults
```

有关更改文件和文件夹权限的更多信息，请参阅技巧 3.13。

## 3.25 在固定时间间隔自动运行程序或脚本

### 问题

你希望每天运行一次脚本或按固定间隔运行。

### 解决方案

使用 Linux 的 `crontab` 命令。
为此，树莓派需要知道时间和日期，因此需要网络连接或实时时钟；参见[待补充链接]。

### 讨论

`crontab` 命令允许你安排事件按固定间隔发生。这可以是每天或每小时，你甚至可以定义复杂的模式，以便在一周的不同日子发生不同的事情。这对于你可能希望在半夜运行的备份任务非常有用。
你可以使用以下命令编辑计划事件：

```
$ crontab -e
```

如果你想要运行的脚本或程序需要由超级用户运行，请在所有 `crontab` 命令前加上 `sudo`（技巧 3.12）。
注释行（以 # 开头）指示了 `crontab` 行的格式。数字依次是分钟、小时、月份中的日期、月份和星期几，后面是你想要运行的命令。

## 3.26 查找文件

### 问题

你想找到一个你知道存在于系统某处的文件。

### 解决方案

使用 Linux 的 `find` 命令。

### 讨论

`find` 命令从命令中指定的目录开始，搜索你指定的文件，如果找到该文件，则显示其位置。

例如：

```
$ find /home/pi -name gemgem.py

```

你可以从目录树的不同位置开始搜索，甚至可以从整个文件系统的根目录（/）开始。搜索整个文件系统会花费更长时间，并且会产生错误消息。你可以通过在行末添加 `2>/dev/null` 来重定向这些错误消息。

要在整个文件系统中搜索文件，请使用以下命令：

```
$ find / -name gemgem.py 2>/dev/null
/home/pi/python_games/gemgem.py
```

请注意，`2>/dev/null` 会重定向那些会干扰最终找到文件时显示的输出。你可以在 [配方 3.31](Recipe 3.31) 中了解更多关于重定向的信息。

你也可以在 `find` 命令中使用通配符，如下所示：

$ find /home/pi -name match*

/home/pi/python_games/match3.wav
/home/pi/python_games/match0.wav
/home/pi/python_games/match5.wav
```

## 3.27 使用命令行历史记录

### 问题

你希望能够重复执行命令行上的命令，而无需再次输入它们。

### 解决方案

使用向上箭头和向下箭头键从命令历史记录中选择以前的命令，并使用 `history` 命令与 `grep` 结合来查找更早的命令。

### 讨论

你可以通过按向上箭头键来访问你之前运行的命令。再次按下它将带你到再之前的命令，依此类推。如果你错过了想要的命令，向下箭头键可以带你反向查找。

如果你想取消而不运行选定的命令，请使用 Ctrl-C。

随着时间的推移，你的命令历史记录会变得太大，以至于无法使用箭头键找到很久以前使用的命令。要找到很久以前的命令，你可以使用 `history` 命令：

```
$ history
    1  sudo nano /etc/init.d/my_server
    2  sudo chmod +x /etc/init.d/my_server
    3  /etc/init.d/my_server start
    4  cp /media/4954-5EF7/sales_log/server.py myserver.py
    5  /etc/init.d/my_server start
    6  sudo apt-get update
    7  sudo apt-get install bottle
    8  sudo apt-get install python-bottle
```

这会列出你所有的命令历史记录，很可能条目太多，让你难以找到想要的命令。为了解决这个问题，你可以将 `history` 命令*管道*（参见配方 3.33）到 `grep` 命令，后者将只显示与搜索字符串匹配的结果。因此，例如，要找到你发出的所有 `apt-get`（配方 3.17）命令，你可以使用以下命令：

```
$ history | grep apt-get
    6  sudo apt-get update
    7  sudo apt-get install bottle
    8  sudo apt-get install python-bottle
   55  history | grep apt-get
```

每个历史记录项旁边都有一个编号，所以如果你找到了你要找的行，你可以使用 `!` 后跟历史记录编号来运行它，如下所示：

```
$ !6
sudo apt-get update
.....
```

### 另请参阅

要查找文件而不是命令，请参阅配方 3.26。

## 3.28 监控处理器活动

### 问题

树莓派有时运行得有点慢，所以你想看看是什么在占用处理器。

### 解决方案

使用任务管理器实用程序，你可以在主菜单的附件程序组中找到它（图 3-15）。

任务管理器允许你一目了然地查看 CPU 和内存的使用情况。你还可以右键单击一个进程，并从出现的弹出菜单中选择终止它的选项。

窗口顶部的条形图显示了总的 CPU 使用率和内存使用率。进程列在下方，你可以看到每个进程占用的 CPU 份额。

### 讨论

如果你更喜欢从命令行执行此类操作，请使用 Linux 的 `top` 命令来显示关于处理器和内存使用情况以及哪些进程使用最多资源的非常相似的数据（图 3-16）。然后你可以使用 `kill` 命令来终止一个进程。你需要以超级用户身份执行此操作。

在这种情况下，你可以看到 `top` 进程是一个使用了 97% CPU 的 Python 程序。第一列显示其进程 ID（2447）。要终止此进程，请输入以下命令：

```
$ kill 2447
```

很有可能以这种方式终止一些重要的操作系统进程，但如果你这样做了，关闭你的树莓派电源并重新打开它将使一切恢复正常。

有时你可能有一个正在运行的进程，在使用 `top` 时并不立即可见。如果是这种情况，你可以使用 `ps` 命令搜索所有正在运行的进程，并将结果管道（配方 3.33）到 `grep` 命令（配方 3.27），后者将搜索结果并突出显示感兴趣的项目。

例如，要找到我们占用 CPU 的 Python 进程的进程 ID，我们可以运行以下命令：

```
$ ps -ef | grep "python"
pi         2447  2397 99 07:01 pts/0    00:00:02 python speed.py
pi         2456  2397  0 07:01 pts/0    00:00:00 grep --color=auto python
```

在这种情况下，Python 程序 `speed.py` 的进程 ID 是 2447。列表中的第二个条目是 `ps` 命令本身的进程。

> ### KILLALL 命令
>
> `kill` 命令的一个变体是 `killall` 命令。请谨慎使用，因为它会终止所有与其参数匹配的进程。因此，例如，以下命令将终止树莓派上运行的所有 Python 程序：
>
> ```
> $ sudo killall python
> ```

### 另请参阅

另请参阅 `top`、`ps`、`grep`、`kill` 和 `killall` 的手册页。你可以通过使用 `man` 命令后跟你想了解信息的命令名称来查看这些手册页，如下所示：

## 3.33 使用管道

### 问题

你想将一个 Linux 命令的输出作为另一个命令的输入。

### 解决方案

使用 `pipe` 命令（即键盘上的*竖线*符号 `|`）将一个命令的输出管道传输到另一个命令。

例如：

```
$ ls -l *.py | grep Jun
-rw-r--r-- 1 pi pi 226 Jun  7 06:49 speed.py
```

此示例将查找所有扩展名为 `py` 且目录列表中包含 `Jun` 的文件，表明它们最后修改于六月。

### 讨论

乍一看，这很像使用 `>` 进行输出重定向（配方 3.31）。区别在于，如果目标是另一个程序，`>` 将不起作用。它只能用于重定向到文件。
你可以将任意多个程序链接在一起，如图所示，尽管这不是你经常做的事情：

```
$ command1 | command2 | command3
```

### 另请参阅

另请参阅配方 3.28，了解使用 `grep` 查找进程的示例，以及配方 3.27，了解使用 `pipe` 和 `grep` 搜索命令历史记录。

## 3.34 隐藏终端输出

### 问题

你想运行一个命令，但不希望输出填满屏幕。

### 解决方案

使用 `>` 将输出重定向到 `/dev/null`。

```

`dev` 目录包含操作系统设备，包括串行端口等。在此目录中，定义了一个特殊设备（空设备），它会简单地丢弃发送给它的所有内容。

### 讨论

此示例说明了语法，但除此之外没什么用。更常见的用途是当你运行一个程序，而开发者在其代码中留下了很多你并不真正想看到的跟踪消息时。以下示例隐藏了 `find` 命令的多余输出（参见配方 3.36）：

```
$ find / -name gemgem.py 2>/dev/null
/home/pi/python_games/gemgem.py
```

## 3.35 在后台运行程序

### 问题

你想在运行程序的同时处理其他任务。

### 解决方案

使用 `&` 命令在后台运行程序或命令。
例如：

```
$ python speed.py &
[1] 2528
$ ls
```

命令行不会等待程序运行完成，而是显示进程 ID（第二个数字），并立即允许你继续运行任何其他命令。然后你可以使用此进程 ID 来终止后台进程（配方 3.28）。

要将后台进程带回前台，请使用 `fg` 命令：

```
$ fg
python speed.py
```

这会报告正在运行的命令或程序，然后等待其完成。

### 讨论

后台进程的输出仍会出现在终端中。
将进程置于后台的另一种方法是打开多个终端窗口。

## 3.37 设置日期和时间

### 问题

你想手动设置树莓派上的日期和时间，因为它没有互联网连接。

Fri 19 Jul 10:59:08 BST 2019

## 3.39 查看运行的操作系统版本

### 问题

你想确切知道你运行的是哪个版本的树莓派操作系统。

### 解决方案

在终端或安全外壳（SSH）会话中输入以下命令：

```
$ cat /etc/os-release
PRETTY_NAME="Raspbian GNU/Linux 9 (stretch)"
NAME="Raspbian GNU/Linux"
VERSION_ID="9"
VERSION="9 (stretch)"
ID=raspbian
ID_LIKE=debian
HOME_URL="http://www.raspbian.org/"
SUPPORT_URL="http://www.raspbian.org/RaspbianForums"
BUG_REPORT_URL="http://www.raspbian.org/RaspbianBugs"
pi@raspberrypi:~ $
```

### 讨论

从上一个示例的结果中可以看出，第一行告诉了我们所有需要知道的信息。在这种情况下，我的树莓派运行的是 Raspbian 版本 9，其昵称为 *stretch*。

如果你在使用某个特定软件时遇到问题，了解你运行的树莓派操作系统版本会很有用。通常，技术支持人员问你的第一个问题是*你运行的是哪个版本的树莓派操作系统？*

你可能需要知道树莓派上 Linux 内核的版本。你可以使用以下命令找到它：

```
$ uname -a
Linux raspberrypi 4.14.71-v7+ #1145 SMP Fri Sep 21 15:38:35 BST 2018 armv7l GNU/Linux
```

这里你可以看到 Raspbian stretch 使用的是 v4.14 内核。

要查看 SD 卡或其他启动盘上剩余的空间，请参阅 [配方 3.38](Recipe 3.38)。

## 3.40 更新树莓派操作系统

### 问题

你想将你的树莓派更新到最新版本的树莓派操作系统。

### 解决方案

打开终端（[链接待补充]）中的命令行，并输入以下命令以将系统更新至最新版本：

```
$ sudo apt-get update
$ sudo apt-get dist-upgrade
```

这将花费一些时间，特别是当有大量内容需要升级时。最重要的是，如果你的系统中有任何重要文件，我建议在升级前将它们复制到USB闪存驱动器（配方3.2）上。

### 讨论

这些命令中的第一个实际上并不更新RaspberryPi OS；它只是更新`apt-get`包管理器，使其了解构成你的操作系统及相关软件的最新版本包。

`dist-upgrade`命令会升级操作系统本身。在此过程中，你将收到关于所需磁盘空间的警告，因此你应该使用配方3.38来检查是否有足够的空间，然后再按Y继续升级。

保持你的发行版更新很重要，原因有几个。首先，操作系统更改的主要原因之一是修复错误。因此，软件安装过程中出现的问题通常在更新后会消失。其次，如果你将你的树莓派暴露在互联网上，新版本的RaspberryPi OS通常会修补安全漏洞。

### 另请参阅

要从全新的RaspberryPi OS安装重新开始，请参阅[链接待补充]。

本章中的一些配方涉及将树莓派转换为一次性使用的设备，而其他配方则使用树莓派上的特定软件。

## 4.1 制作媒体中心

### 问题

你想将你的树莓派转换为一个超级媒体中心。

### 解决方案

要将你的树莓派用作媒体中心，你应该选择性能更优的树莓派4 B，因为视频播放非常消耗处理器资源。

你可以在使用[树莓派磁盘映像工具](https://www.raspberrypi.com/software/)将镜像写入microSD卡的同时，将你的树莓派设置为媒体中心。不要选择Raspberry Pi OS作为要安装的发行版，而是从操作系统按钮的媒体播放器操作系统部分选择LibreELEC（图4-1）。

![](img/54ed1f8e884bc184601bf6f652461a95_160_0.png)

LibreELEC是一个优化你的树莓派作为媒体中心的发行版。它包括Kodi媒体中心软件，该软件基于XBMC开源项目，该项目最初是为了将Xbox游戏机转换为媒体中心而开发的。此后，该代码已被移植到许多平台，包括树莓派（图4-2）。

![](img/54ed1f8e884bc184601bf6f652461a95_161_0.png)

图4-2. 树莓派作为媒体中心

树莓派完全能够播放全高清视频以及流媒体音乐、MP3文件和网络电台。

### 讨论

Kodi是一款功能强大的软件，具有许多功能。也许检查其是否工作的最简单方法是将一些音乐和/或视频文件放入USB闪存驱动器或外部USB硬盘，并将其连接到树莓派。你应该能够从Kodi播放它们。

由于树莓派可能放在你的电视附近，你可能会发现你的电视有一个USB端口，可以提供足够的电流来运行树莓派。如果是这种情况，你将不需要单独的电源。

无线键盘和鼠标是个好主意，因为如果你成对购买它们，它们将使用单个USB端口的接收器，这避免了电线到处拖拽的需要。你也可以购买带有内置触摸板的迷你键盘，这在这种情况下很有用。

有线网络连接通常比WiFi连接性能更高、更好，但将树莓派放在以太网插座附近并不总是方便。如果是这种情况，你可以设置XBMC使用WiFi。设置Kodi非常直观，你可以在http://kodi.wiki找到使用该软件的完整说明。

### 另请参阅

你可以为树莓派添加[红外（IR）遥控器](infrared (IR) remote)来控制Kodi。

## 4.4 制作网络摄像头服务器

### 问题

你想将树莓派设置为网络摄像头服务器。

### 解决方案

你可以使用USB网络摄像头或树莓派摄像头模块作为摄像头，你可以通过网络上的其他地方使用浏览器访问它。

下载一个名为*motion*的软件包。这将允许你设置一个连接了USB网络摄像头的树莓派，这样你就可以连接到一个网页并查看网络摄像头。

要安装软件，请在终端窗口中输入以下命令：

```
$ sudo apt-get update
$ sudo apt-get install motion
```

你需要进行一些配置更改。首先，使用以下命令编辑文件`/etc/motion/motion.conf`：

```
$ sudo nano /etc/motion/motion.conf
```

这是一个相当大的配置文件。在文件顶部附近，你会找到这一行：

```
daemon off
```

按如下方式更改该行：

```
daemon on
```

另一个更改在文件更下方。你需要更改这一行：

```
webcam_localhost = on
```

使其读取如下：

还有另一个文件需要修改。输入以下命令：

```
$ sudo nano /etc/default/motion
```

将：

```
start_motion_daemon=no
```

改为：

```
start_motion_daemon=yes
```

然后将：

```
stream_localhost on
```

改为：

```
stream_localhost off
```

要启动网络服务，请执行以下命令：

```
$ sudo service motion start
```

如果你使用的是树莓派摄像头模块，还有一个额外的步骤。你需要使用以下命令编辑 `/etc/modules` 文件：

```
$ sudo nano /etc/modules
```

将以下行添加到该文件末尾，然后像往常一样保存并退出 nano：

```
bcm2835-v4l2
```

重启你的树莓派，现在你应该能够打开网络浏览器查看网络摄像头了。为此，你需要知道树莓派的 IP 地址（[配方 2.2](Recipe 2.2)）。你也可以通过访问 *http://localhost:8081* 来检查网络摄像头是否在树莓派的浏览器上正常工作。

从同一网络上的另一台计算机，打开浏览器并访问 URL *http://192.168.0.210:8081/*。你需要更改 URL 以匹配你的树莓派的 IP 地址，但保留 URL 末尾的 `:8081` 端口号。

如果一切顺利，你应该会看到类似 [图 4-7](Figure 4-7) 的内容。

![](img/54ed1f8e884bc184601bf6f652461a95_170_0.png)

图 4-7. 树莓派网络摄像头

### 讨论

motion 软件功能非常强大。还有许多其他设置可以调整，以改变你的网络摄像头的工作方式。

默认情况下，网络摄像头只能从你的网络内部查看。如果你想让网络摄像头对整个互联网可见，你需要在你的家庭集线器上设置 *端口转发*。这需要你登录到集线器的管理控制台，找到端口转发选项，并为树莓派的 IP 地址启用端口 8081。

然后你就可以使用你的 ISP 分配的外部 IP 地址来查看网络摄像头。这通常显示在你的管理控制台的首页上。但要注意：除非你向 ISP 支付费用获取静态 IP 地址，否则每次重启家庭集线器调制解调器时，这个 IP 地址都可能会改变。

像 No-IP 这样的服务也可以用来提供动态域名服务（DNS），这样你就可以注册一个域名，该域名会自动跟随你的 IP 地址的变化。

### 另请参阅

- 你可以在 https://oreil.ly/_z4GH 找到与树莓派兼容的网络摄像头列表。

## 4.6 将你的树莓派变成无线电发射器

> **注意**
> 请务必查看 [本配方的配套视频](https://www.youtube.com/watch?v=1vV8pE4l0p0)。

### 问题

你想将你的树莓派转换成一个低功率 FM 发射器，向普通的 FM 收音机接收器发送无线电信号（[图 4-11](https://www.oreilly.com/library/view/make-great-stuff/9781449399191/graphics/0411.png)）。

### 解决方案

在树莓派的早期，伦敦帝国理工学院的一些聪明人编写了一些 C 代码，让你可以做到这一点。下载包甚至播放 *星球大战* 主题曲作为示例。如果你有 [原版树莓派 1](https://www.raspberrypi.org/products/raspberry-pi-1-model-b/)，这个项目仍然有效。此后代码没有针对更新版本的树莓派进行更新。
幸运的是，该项目在一个名为 *rpitx* 的更先进的项目中得以延续。

你只需要一根短导线连接到通用输入/输出（GPIO）引脚 4。一根 10 厘米的母对公排线就能很好地完成这项工作。事实上，即使没有天线，它也应该能在收音机紧挨着你的树莓派时工作——这就是传输的强度。

![](img/54ed1f8e884bc184601bf6f652461a95_177_0.png)

第一步是使用以下命令安装 rpitx 软件。请注意，此安装将更改树莓派配置的一些内容，包括 GPU（图形处理器）的工作频率。因此，如果这是你的主要树莓派，请确保首先备份任何重要数据。你需要的代码如下：

```
$ sudo apt-get update
$ git clone https://github.com/F5OEO/rpitx
$ cd rpitx
$ ./install.sh
```

现在你需要做其他事情大约 15 分钟，等待软件安装。你可能会看到看起来像错误消息和警告的内容，但这些都是正常的。在安装结束时，安装脚本会询问：

```
In order to run properly, rpitx need to modify /boot/config.txt.
Are you
    sure (y/n)
```

按 Y，脚本将通过以下消息确认它所做的更改：

```
Set GPU to 250Mhz in order to be stable
```

如果你需要撤销此更改，请编辑 /boot/config.txt，删除最后一行 `gpu_freq=250`，然后重启。

接下来，找一个 FM 收音机接收器并将其调谐到 103.0 MHz。如果这个频率已被其他传输占用，请选择另一个频率并记下来。

现在运行以下命令（如果你不得不更改频率，请将频率参数从 103.0 更改）：

```
sudo ./pifmrds -freq "103.0" -audio src/pifmrds/stereo_44100.wav
```

如果一切正常，你应该能听到开发者谈论左右声道的声音。

### 讨论

你需要知道，这个项目在你的国家可能不合法。其功率输出高于用于 MP3 播放器的 FM 发射器。

如果你把树莓派放在你的车里，这将是通过车辆音频系统输出声音的好方法。

### 另请参阅

要了解更多关于 rpitx 项目的信息，请参阅 [https://github.com/F5EOEO/rpitx](https://github.com/F5EOEO/rpitx)。

## 4.10 播放网络电台

### 问题

你想在 Raspberry Pi 上播放网络电台。

### 解决方案

VLC 媒体播放器应该已随 Raspberry Pi OS 预装。你可以在菜单的“声音和视频”下找到它。

如果由于任何原因它不在那里，你可以使用“首选软件”工具（配方 4.2）安装它。

如果你更喜欢从命令行安装 VLC 媒体播放器，可以通过运行以下命令来完成：

```
sudo apt-get install vlc

## 5.1 在 Python 2 和 Python 3 之间做选择

### 问题

你需要使用 Python，但不确定应该使用哪个版本。

### 解决方案

两个都用。使用 Python 3，直到你遇到一个最好通过回退到版本 2 来解决的问题。

### 讨论

尽管 Python 的最新版本 Python 3 已经存在多年，但你会发现很多人仍然在使用 Python 2。事实上，在树莓派操作系统发行版中，两个版本都提供了，版本 2 只被称为 Python，而版本 3 被称为 Python 3。Python 3 通过使用命令 `python3` 来运行。除非另有说明，本书中的示例都是为 Python 3 编写的。大多数示例无需修改即可在 Python 2 和 Python 3 上运行。

Python 社区不愿放弃旧版本，这在很大程度上是因为 Python 3 引入了一些破坏与版本 2 兼容性的更改。因此，为 Python 2 开发的大量第三方库中的一些将无法在 Python 3 下工作。

我的策略是尽可能使用 Python 3 编写，当由于兼容性问题需要时，再回退到 Python 2。

## 5.6 从终端运行 Python 程序

### 问题

从 Mu 内部运行程序（配方 5.4）很好，但有时你想从终端窗口运行 Python 程序。

### 解决方案

在终端中使用 `python` 或 `python3` 命令，后跟包含你要运行的程序的文件名。

### 讨论

要从命令行运行 Python 3 程序，请使用如下命令：

```
$ python3 myprogram.py
```

如果你想使用 Python 2 运行程序，请将命令 `python3` 改为 `python`。在这两种情况下，你要运行的 Python 程序都应该在一个扩展名为 `.py` 的文件中。
你可以以普通用户身份运行大多数 Python 程序；但是，有些你需要以超级用户身份运行。如果你的程序属于这种情况，请在命令前加上 `sudo`：

```
$ sudo python3 myprogram.py
```

在前面的例子中，你需要在命令中包含 `python3` 来运行程序，但你也可以在 Python 程序的开头添加一行，以便 Linux 知道它是一个 Python 程序。这个特殊的行被称为 *shebang*（是“hash”和“interrobang”两个符号名称的缩写），下面的单行示例程序将其作为第一行：

```
#!/usr/bin/python3
print("I'm a program, I can run all by myself")
```

在你可以直接从命令行运行此程序之前，你必须使用以下命令（参见配方 3.14）赋予文件写权限；这里假设文件名为 *test.py*：

```
$ chmod +x test.py
```

参数 +x 表示向文件添加执行权限。
现在你可以使用单个命令运行 Python 程序 *test.py*：

```
$ ./test.py
I'm a program, I can run all by myself
$
```

行首的 ./ 是命令行找到文件所必需的。

## 5.7 为值命名（变量）

### 问题

你想给一个值命名。

### 解决方案

你使用 = 将值赋给一个名称。

### 讨论

在 Python 中，你不必声明变量的类型；你可以直接使用赋值运算符 (=) 为其赋值，如以下示例所示：

```
a = 123
b = 12.34
c = "Hello"
d = 'Hello'
e = True
```

你可以使用单引号或双引号定义字符串常量。Python 中的逻辑常量是 `True` 和 `False`，它们区分大小写。

按照惯例，变量名以小写字母开头，如果变量名由多个单词组成，则用下划线字符连接。给你的变量起描述性名称总是一个好主意，这样当你在休息后回到程序时，就能弄清楚它是如何工作的。

一些有效的变量名示例包括 `x`、`total` 和 `number_of_chars`。

>>>
```

## 5.20 条件执行命令

### 问题

你想仅在某个条件为真时运行一些 Python 命令。

### 解决方案

使用 Python 的 `if` 命令。
以下示例仅在 `x` 的值大于 100 时打印消息 `x is big`：

```
>>> x = 101
>>> if x > 100:
...     print("x is big")
...
x is big
```

### 讨论

在 `if` 关键字之后，有一个*条件*。这个条件通常（但并非总是）比较两个值，并给出一个 `True` 或 `False` 的答案。如果为 `True`，则后续缩进的行都将被执行。
通常，当条件为 `True` 时想做一件事，而当答案为 `False` 时想做另一件事。在这种情况下，`else` 命令与 `if` 一起使用，如本例所示：

```
x = 101
if x > 100:
    print("x is big")
else:
    print("x is small")

print("This will always print")
```

你还可以将一系列 `elif`（else if）条件链接在一起。如果其中任何一个条件成功，则执行该代码块，并且不会尝试其后的任何其他条件。
例如：

```
x = 90
if x > 100:
    print("x is big")
elif x < 10:
    print("x is small")
else:
    print("x is medium")
```

此示例将打印 x is medium。

## 5.26 在 Python 中定义函数

### 问题

### 解决方案

创建一个函数，将代码行组合在一起，允许从多个地方调用它。
以下示例说明了如何在 Python 中创建然后调用函数：

```
def count_to_10():
    for i in range(1, 11):
        print(i)

count_to_10()
```

此示例使用 `def` 命令定义了一个函数，该函数在每次调用时都会打印数字 1 到 10：

```
count_to_10()
```

### 讨论

命名函数的约定与配方 5.7 中变量的约定相同；也就是说，它们应该以小写字母开头，如果名称由多个单词组成，则单词之间应使用下划线分隔。
示例函数有点不灵活，因为它只能计数到 10。如果我们想让它更灵活——例如，让它可以计数到任何数字——我们可以将最大数字作为函数的*参数*包含在内，如本例所示：

```
def count_to_n(n):
    for i in range(1, n + 1):
        print(i)
```

除了使用`[ ]`表示法从列表中读取值外，你还可以使用它来更改特定位置的值，如下所示：

```python
>>> a = [34, 'Fred', 12, False, 72.3]
>>> a[1] = 777
>>> a
[34, 777, 12, False, 72.3]
```

如果你尝试使用过大的索引来更改（或读取）列表元素，你将得到一个`Index out of range`错误：

```python
>>> a[50] = 777
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
IndexError: list assignment index out of range
>>>
```

## 6.3 查找列表长度

### 问题

你需要知道列表中有多少个元素。

### 解决方案

使用Python函数`len`。
例如：

```python
>>> a = [34, 'Fred', 12, False, 72.3]
>>> len(a)
5
```

函数`len`从Python版本1就存在，这与更面向对象、基于类的Python版本2和3相当不同。对于后者，能够这样写会更有意义：

>>> a = [34, 'Fred', 12, False, 72.3]

```

但你不能——Python就是这样。

### 讨论

## 6.4 向列表添加元素

### 问题

你需要向列表添加一个项目。

### 解决方案

使用Python函数`append`、`insert`或`extend`。
要向列表末尾添加单个项目，请使用`append`，如下所示：

```python
>>> a = [34, 'Fred', 12, False, 72.3]
>>> a.append("new")
>>> a
[34, 'Fred', 12, False, 72.3, 'new']
```

### 讨论

有时你不想将新元素添加到列表末尾，而是想将它们插入到列表中的某个位置。为此，请使用`insert`命令。第一个参数是应插入项目的索引，第二个参数是要插入的项目：

```python
>>> a.insert(2, "new2")
>>> a
[34, 'Fred', 'new2', 12, False, 72.3]
```

请注意，新插入元素之后的所有元素都向上移动了一个位置。
`append`和`insert`都只向列表添加一个元素。`extend`函数将一个列表的所有元素添加到另一个列表的末尾：

```python
>>> a = [34, 'Fred', 12, False, 72.3]
>>> b = [74, 75]
>>> a.extend(b)
>>> a
[34, 'Fred', 12, False, 72.3, 74, 75]
```

## 6.5 从列表中移除元素

### 问题

你需要从列表中移除一个项目。

### 解决方案

使用Python函数`pop`。
不带参数的`pop`命令移除列表的最后一个元素：

```python
>>> a = [34, 'Fred', 12, False, 72.3]
>>> a.pop()
72.3
>>> a
[34, 'Fred', 12, False]
```

### 讨论

请注意，`pop`会返回从列表中移除的值。
要移除最后一个元素以外的位置的项目，请使用带参数的`pop`，参数为要移除项目的位置：

```python
>>> a = [34, 'Fred', 12, False, 72.3]
>>> a.pop(0)
34
```

如果你使用的索引位置超出了列表的末尾，你将得到一个`Index out of range`错误。

## 6.6 通过解析字符串创建列表

你需要将一个由某个字符分隔的单词字符串转换为一个字符串数组，其中数组中的每个字符串都是其中一个单词。

### 解决方案

使用Python字符串函数`split`。

>>> "abc def ghi".split()

```

如果你为`split`提供一个参数，它将使用该参数作为分隔符来分割字符串。
例如：

```python
>>> "abc--de--ghi".split('--')
['abc', 'de', 'ghi']
```

### 讨论

例如，当你从文件导入数据时，此命令非常有用。`split`命令可以选择性地接受一个参数，该参数是分割字符串时用作分隔符的字符串。因此，如果你使用逗号作为分隔符，你可以这样分割字符串：

```python
>>> "abc,def,ghi".split(',')
['abc', 'def', 'ghi']

>>> a = [34, 'Fred', 12, False, 72.3]

## 6.8 枚举列表

### 问题

你需要依次对列表中的每个项目运行一些代码行，但你也需要知道每个项目的索引位置。

### 解决方案

参数`n`包含在括号内，然后在`range`命令中使用，但在1被加到它之前不使用。

使用参数作为要计数到的数字意味着，如果你通常计数到10但有时计数到不同的数字，你将总是必须指定该数字。然而，你可以为参数指定一个默认值，从而两全其美，如本例所示：

```python
def count_to_n(n=10):
    for i in range(1, n + 1):
        print(i)

count_to_n()
```

现在，除非在调用函数时指定了不同的数字，否则它将计数到10。

如果你的函数需要多个参数，也许是在两个数字之间计数，参数用逗号分隔：

```python
def count(from_num=1, to_num=10):
    for i in range(from_num, to_num + 1):
        print(i)

count()
count(5)
count(5, 10)
```

所有这些示例都是不返回任何值的函数；它们只是执行某些操作。如果你需要一个函数返回值，你需要使用`return`命令。

以下函数接受一个字符串作为参数，并在字符串末尾添加单词*please*：

```python
def make_polite(sentence):
    return sentence + " please"

print(make_polite("Pass the cheese"))
```

当函数返回值时，你可以将结果赋给一个变量，或者，如本例所示，你可以打印结果。

## 6.9 对列表进行排序

### 问题

你需要对列表的元素进行排序。

### 解决方案

使用 Python 语言的 `sort` 命令：

```
>>> a = ["it", "was", "the", "best", "of", "times"]
>>> a.sort()
>>> a
['best', 'it', 'of', 'the', 'times', 'was']
```

`sort` 命令使用标准的 Python 比较运算符。这意味着对于字符串，它们将按字母顺序升序排列。

### 讨论

当你对列表进行排序时，你实际上是在修改它，而不是返回原始列表的排序副本。这意味着如果你还需要原始列表，你需要在排序之前使用标准库中的 `copy` 命令来制作原始列表的副本：

```
>>> from copy import copy
>>> a = ["it", "was", "the", "best", "of", "times"]
>>> b = copy(a)
>>> b.sort()
>>> a
['it', 'was', 'the', 'best', 'of', 'times']
>>> b
['best', 'it', 'of', 'the', 'times', 'was']
>>>
```

`copy` 模块是复制对象所必需的。你可以在[待补充链接]中了解更多关于模块的信息。

## 6.10 切割列表

### 问题

你需要使用原始列表元素的一个范围来制作一个子列表。

### 解决方案

使用 Python 语言的 [ : ] 构造。以下示例返回一个列表，其中包含从索引位置 1 到索引位置 2 的原始列表元素（: 之后的数字是排他的——即，包含到但不包括元素 2：

```
>>> l = ["a", "b", "c", "d"]
>>> l[1:3]
['b', 'c']
```

请注意，字符位置从 0 开始（而不是从 1 开始），因此位置 1 表示字符串中的第二个字符，3 表示第四个；然而，字符范围在高端是*排他*的，因此在此示例中不包含字母 *d*。

### 讨论

[ : ] 表示法实际上非常强大。你可以省略任一参数，此时将根据情况假定为列表的开头或结尾。

例如：

```
>>> l = ["a", "b", "c", "d"]
>>> l[:3]
['a', 'b', 'c']
>>> l[3:]
['d']
>>>
```

你也可以使用负索引从列表末尾开始计数。以下示例返回列表中的最后两个元素：

```
>>> l[-2:]
['c', 'd']
```

顺便说一下，在前面的示例中，`l[:-2]` 返回 `['a', 'b']`。

### 另请参阅

参见配方 5.17，其中对字符串使用了相同的语法。

## 6.11 列表推导式

### 问题

你想要一种更简洁的方法，通过过滤或转换原始列表来从另一个列表构建一个列表。

### 解决方案

使用列表推导式。

列表推导式并不能做任何你用常规循环做不到的事情，但它们确实简化了涉及列表的操作，并且如果使用得当，可以使你的代码更具可读性。

列表推导式将接受一个现有列表，并从原始列表的元素创建一个新列表，要么过滤列表，使只有符合条件的元素包含在新列表中，要么可以创建一个包含与原始列表相同数量元素的新列表，但每个元素都以某种方式被操作。

提醒一下，如果不使用列表推导式，以下代码将获取一个列表并将其过滤到一个新列表中，该新列表仅包含以字母 'a' 开头的元素。

```
new_list = []
people = ['agnes', 'andrew','jane','peter']
for person in people:
    if person[0] == 'a':
        new_list.append(person)
print(new_list)
```

当你运行这个程序（ch_07_filter.py）时，你会看到 `new_list` 只包含以 'a' 开头的名称。

```
$ python3 ch_07_filter.py
['agnes', 'andrew']
```

我们可以通过使用列表推导式（ch_07_filter_comp.py）来大大缩短这个示例。

```
people = ['agnes', 'andrew','jane','peter']
new_list = [person for person in people if person[0] == 'a']
print(new_list)
```

这将产生与 ch_07_filter.py 完全相同的输出，但代码减少了两行。列表推导式包含在方括号中，推导式中的第一个单词是 `person`。这表明 `person`（尚未指定）将被添加到列表副本中。然后我们有 `for person in people`，这是遍历列表的常规方式，列表名为 `people`，列表中的每个元素名为 `person`。推导式的最后一部分是条件 `if person[0] == 'a'`。这是拒绝所有不以 'a' 开头的列表元素的部分。

推导式的条件是可选的，你可能只想使用推导式来修改列表的每个元素。ch_07_change_comp.py 的示例将名称大写，返回一个新列表，其中每个名称的首字母大写。

```
people = ['agnes', 'andrew','jane','peter']
new_list = [person.capitalize() for person in people]
print(new_list)
```

```
$ python3 ch_07_change_comp.py
['Agnes', 'Andrew', 'Jane', 'Peter']
```

### 讨论

列表推导式是操作列表的一种非常强大且有用的技术。起初，语法可能看起来有点奇怪，因为它被封闭在方括号中。但这是保持代码简洁而不增加理解其工作原理难度的好方法。

### 另请参阅

关于不使用列表推导式遍历列表，请参见[配方 6.7](Recipe 6.7)。

## 6.12 创建字典

### 问题

你需要创建一个查找表，将值与键关联起来。

### 解决方案

使用 Python 字典。
当你需要按顺序访问项目列表，或者总是知道要使用的元素的索引时，列表非常有用。字典是存储数据集合的另一种方式，但它们的组织方式非常不同，如[图 6-1](Figure 6-1)所示。

![](img/54ed1f8e884bc184601bf6f652461a95_249_0.png)

字典以一种方式存储键/值对，使你可以使用键非常高效地检索该值，而无需搜索整个字典。

要创建字典，你使用 { } 表示法：

```
>>> phone_numbers = {'Simon':'01234 567899', 'Jane':'01234 666666'}
```

### 讨论

在这个示例中，字典的键是字符串，但它们不需要是；它们可以是数字，实际上可以是任何数据类型，尽管字符串最常用。

值也可以是任何数据类型，包括其他字典或列表。以下示例创建一个字典（a），然后将其用作第二个字典（b）中的值：

```
>>> a = {'key1':'value1', 'key2':2}
>>> a
{'key2': 2, 'key1': 'value1'}
>>> b = {'b_key1':a}
>>> b
{'b_key1': {'key2': 2, 'key1': 'value1'}}
```

当你显示字典的内容时，请注意字典中项目的顺序可能与创建字典并用一些内容初始化时指定的顺序不匹配：

```
>>> phone_numbers = {'Simon':'01234 567899', 'Jane':'01234 666666'}
>>> phone_numbers
{'Jane': '01234 666666', 'Simon': '01234 567899'}
```

与列表不同，字典没有保持项目顺序的概念。由于它们内部的表示方式，字典内容的顺序——出于所有实际目的——将是随机的。
顺序看起来随机的原因是底层数据结构是一个*哈希表*。哈希表使用 `hashing` 函数来决定存储每个值的位置；`hashing` 函数为任何对象计算一个数字等效值。
你可以在 [Wikipedia](https://en.wikipedia.org/wiki/Hash_table) 上了解更多关于哈希表的信息。

### 另请参阅

字典与[待补充链接]中描述的 JSON 数据结构语言有很多共同之处。

## 6.13 访问字典

### 问题

你需要查找并更改字典中的条目。

### 解决方案

提供 Python 的 [ ] 表示法。在方括号内使用你需要访问的条目的键，如下所示：

```
>>> phone_numbers = {'Simon':'01234 567899', 'Jane':'01234 666666'}
>>> phone_numbers['Simon']
'01234 567899'
>>> phone_numbers['Jane']
'01234 666666'
```

### 讨论

查找过程是单向的，从键到值。
如果你使用一个不在字典中的键，你将得到一个 `KeyError`。例如：

```
>>> phone_numbers = {'Simon':'01234 567899', 'Jane':'01234 666666'}
>>> phone_numbers['Phil']
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
KeyError: 'Phil'
>>>
```

除了使用 [ ] 表示法从字典中读取值，你还可以用它来添加新值或覆盖现有值。
以下示例向字典中添加了一个新条目，其键为 `Pete` **，值为** `01234 777555`：

```
>>> phone_numbers = {'Simon':'01234 567899', 'Jane':'01234 666666'}
>>> phone_numbers['Pete'] = '01234 777555'
>>> phone_numbers['Pete']
'01234 777555'
```

如果该键在字典中未被使用，则会自动添加一个新条目。
如果该键已经存在，那么之前的任何值都将被新值覆盖。
这与尝试从字典中读取值形成对比，未知的键会导致错误。

有关处理错误的信息，请参阅 [待补充链接]。

## 6.15 遍历字典

### 问题

你需要依次对字典中的每个项目执行某些操作。

### 解决方案

使用 `for` 命令遍历字典的键：

```
>>> phone_numbers = {'Simon':'01234 567899', 'Jane':'01234 666666'}
>>> for name in phone_numbers:
...     print(name)
...
Jane
Simon
```

注意键的打印顺序与它们创建时的顺序不同。这是字典的一个特性。条目的顺序不会被记住。

### 讨论

还有其他几种技术可以用来遍历字典。如果你需要同时访问键和值，以下形式可能很有用：

```
>>> phone_numbers = {'Simon':'01234 567899', 'Jane':'01234 666666'}
>>> for name, num in phone_numbers.items():
...     print(name + " " + num)
...
Jane 01234 666666
Simon 01234 567899
```
