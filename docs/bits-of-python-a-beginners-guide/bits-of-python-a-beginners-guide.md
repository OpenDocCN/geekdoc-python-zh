# 第 01 章：入门

## 背景

在我们准备编码之前，让我们先了解一些基础知识。我们人类有多种交流语言，如英语、西班牙语、法语等等，而要与计算机交流，我们也需要一种语言。计算机无法理解英语或任何其他语言，但它能够将这些语言翻译成它能理解的语言：二进制。

二进制由 1 和 0 组成。每个 1 和 0 被称为一个**比特**。像 101 这样的二进制数可以转换为数字 5。你可能已经发现，用这种方式交流对我们来说相当困难，因此**编程语言**应运而生。编程语言允许我们以更易理解的格式输入内容，计算机则可以使用**编译器**将我们输入的内容转换为二进制，以便计算机能够理解。如今有许多编程语言可供选择，每种语言都有其独特的优势。1991 年，Python 诞生了。现在，我们已经准备好开始编码了。

# 第三章：注释

Python中的**注释**是解释和理解代码的非常有用的工具。注释只是一条语句，可用于解释代码段并说明其用途。

注释可以这样写：

```python
# This is a comment
```

当你以井号（#）开始新行时，Python就知道你在写注释。注释**对你的代码没有任何影响**，它们只是供你（程序员）阅读的语句，以帮助你或他人理解代码。

程序会忽略所有注释，所以尽情注释吧！

# 第四章：基础

现在是时候学习Python中的数据类型了。这可能不是最令人兴奋的部分，但它至关重要。

### 字符串

你上一个项目中的“Hello World!”被称为**字符串**。字符串本质上是所有文本输入，必须始终用‘单引号’或“双引号”括起来。字符串可以是单词，如“*Hello*”，甚至是数字，如“5”。任何用引号括起来的东西都会变成字符串。

### 整数

像10、15、-17和39这样的整数被称为**int**，即整数的缩写。你可以对它们执行数学运算，如加法、减法和乘法。我们不能对字符串执行数学运算。在前面的例子中，我们通过用双引号将数字5括起来，使其变成了字符串，这意味着我们也不能对它执行数学运算。

### 浮点数

像10.1、23.5和-9.54这样的数字被称为**浮点数**。它们代表小数，与整数类似，可以对它们执行数学运算。

### 布尔值

最后一个主要的数据类型是**布尔值**，它表示***True***或***False***语句。

### 变量

**变量**是任何编程语言的重要组成部分，我们使用它们将数据存储在计算机中以备后用。例如，我们可以说*price* = 10，或者*name* = “John”。*price*和*name*都是变量。变量可以存储任何东西，字符串、整数、布尔值或浮点数。

当我们说*price* = 10和*name* = "John"时，这被称为变量的**初始化**，即我们为其赋值。

将变量赋值想象成将值放入一个盒子中：

![](img/68bd89f6eb7eca3e8b8bb158e8d40be2_26_0.png)

变量的一个有趣特性是，它们可以用来打印，而无需输入所有内容。以之前的“Hello World!”程序为例。我们不必说*print("Hello World!")*，而可以这样说：

```python
Text = "Hello World!"

print(Text)
```

这与之前的结果完全相同。

Python是**区分大小写的**（大小写很重要！），因此在输入时请确保所有内容完全相同。如果我们在上一个例子中打印变量*Text*时改变了大小写，我们会得到这个错误：

```
NameError: name 'text' is not defined. Did you mean: 'Text'?
```

另一件需要注意的事情是，如果我们有类似这样的代码：

```
Value = 10

Value = 20

print(Value)
```

输出是什么？Python的工作方式是从上到下执行，因此存储在value中的数据将是**最后更新的版本**，所以我们的结果将是20。

使用与之前相同的例子，将盒子想象成一次只能容纳一个物品的东西：

![](img/68bd89f6eb7eca3e8b8bb158e8d40be2_28_0.png)

使用变量的另一个酷炫功能是，如果你的变量是一个数字，我们可以通过这样说来递增它：

```
Value = 10

Value = Value + 1

print(Value)
```

从数学的角度来看，这可能令人困惑，一个数字怎么能等于那个数字加一呢？嗯，在编程中，这所做的只是将变量的值**重新赋值**为比它大一的值，所以*Value = Value + 1*将变量*Value*从10重新赋值为11：

![](img/68bd89f6eb7eca3e8b8bb158e8d40be2_29_0.png)

另一种写法是使用快捷方式+=：

```
Value = 10

Value += 1

print(Value)
```

这会打印出相同的结果：

![](img/68bd89f6eb7eca3e8b8bb158e8d40be2_30_0.png)

## 字符串索引

索引是编程中最重要的工具之一，我们很快会进一步探讨，但现在，我们可以使用索引来引用字符串的特定部分。

**索引**指的是字符串中的每个字符，每个字符都有自己的索引。索引从 0 开始，一直到字符串的末尾（例如，字符串 “Hello” 的索引 0 是字符 “H”，索引 4 是 “o”。字符串的最后一个索引总是比字符串长度小 1，因为索引从 0 开始）。

我们可以通过以下方式编写代码：

```python
String = "How are you?"
print(String[0])
```

这将打印索引 0 的字符，即 “H”。

要打印最后一个索引，我们可以这样写：

```python
String = "How are you?"
print(String[11])
```

*记住空格也算！*

另一种方法是使用负数来实现：

```python
String = "How are you?"
print(String[-1])
```

这将给你最后一个索引，即 “?”。当我们不知道字符串长度时，这非常有用。

类似地，如果我们说：

```python
String = "How are you?"
print(String[-12])
```

这将给你 “H”。

## 字符串切片

字符串切片的设置如下：

我们输入字符串名称，后跟方括号，就像平常一样，但然后我们在方括号中遵循以下语法：

```python
String[start : stop : step]
```

本质上，这是通过指定我们从哪个索引开始和结束来打印字符串的特定部分。如果我们使用 String[0:]，我们将得到整个字符串：

![](img/68bd89f6eb7eca3e8b8bb158e8d40be2_44_0.png)

（将字段留空将默认为字符串的末尾）

我们可以根据索引进行分割，所以 *String[2:-2]* 将给我们从第 3 个字符（索引 2）到倒数第 3 个字符的所有内容：

![](img/68bd89f6eb7eca3e8b8bb158e8d40be2_45_0.png)

至于最后一个参数，步长，这是我们跳过多少个字母。将其留空意味着我们打印起始和结束范围内的每个字母，但如果我们将其更改为 3，那么这将使我们从起始索引开始，打印该索引处的字符，跳过 3 个索引，打印该索引处的字符，依此类推，直到我们达到结束索引。

如果步长使我们超过了结束索引，那么打印的最后一个索引将是步长超过结束索引之前的那个索引。

假设我们使用与之前相同的示例，但将步长设置为 3：

![](img/68bd89f6eb7eca3e8b8bb158e8d40be2_46_0.png)

如你所见，步长打印了每第三个字符，但当它到达字符串末尾前 2 个字符时，它无法再打印任何内容，因此它打印了字符串结束前的最后一个索引。

## 连接

我们也可以将变量添加到字符串中，但其格式有点不同：

```python
string = "I'm good"

print("Hello, how are you " + string)
```

这的结果将是：

```
Hello, how are you I'm good
```

如你所见，要将一个字符串添加到另一个字符串，我们需要添加一个加号，然后是变量的名称。这种字符串的组合称为**连接**。但是请注意，你只能将字符串与字符串连接，所以如果你有类似这样的代码：

```python
num = 7

print("I like donuts so much, I have " + num)
```

这可能不会立即在代码中产生错误，但当你运行代码时，你会得到：

```
print("I like donuts so much, I have " + num)
TypeError: can only concatenate str (not "int") to str
```

这个错误只是意味着你不能将字符串与任何其他数据类型连接，它必须是字符串与字符串连接。

我们不需要将字符串赋值给变量才能连接它，我们也可以简单地在两个字符串之间放置一个加号并打印，这将完美运行。

# 第八章：数据类型转换

这是一个测试你到目前为止Python知识的谜题：

```
Number = input("Pick a number: ")

Half = Number / 2

print(Half)
```

如果你说输入总是一个字符串，而字符串不能用于数学运算，那么你是正确的！

这就引出了一个问题，我们如何从用户那里获取输入并对它们进行数学运算？

这就是**数据类型转换**的用武之地。字符串、整数、布尔值和浮点数都是不同的**数据类型**，你可以通过说出类型的名称，然后将要转换的内容括在括号中来在它们之间进行转换。

*字符串 <=> 整数/浮点数*

例如，我们只需使用**int()**就可以解决之前的问题：

```
Number = input("Pick a number: ")

Half = int(Number) / 2

print(Half)
```

这将使*Number*成为一个真正的整数，从而允许它被2除。

但是，请注意，*Number*必须是一个实际的数字，转换才不会产生错误。你不能将像“ten”或“cat”这样的字符串转换为整数，因为它们没有整数版本，但*int("12")*将返回数字12。

只要字符串具有浮点表示，浮点转换也是相同的。浮点转换的命令是**float()**，所以*float("12")*将返回12.0。

同样，反向操作也可以，我们可以将整数或浮点数转换为字符串，没有任何限制（任何东西都可以是字符串）。需要注意的是，当将某些内容转换为字符串时，命令是**str()**，而不是**string()**

### *整数 <=> 浮点数*

将整数转换为浮点数时，它只是在末尾添加一个额外的.0，所以*float(20)*是20.0。

同样，反向操作也可以，*int(20.0)*将得到20。

### *布尔值 <=> 任何类型*

对于布尔值，我们也可以转换它们，只是工作方式有点不同。*True*布尔值对应于1，而*False*布尔值对应于0。

这意味着如果我们执行*int(True)*，我们得到1，如果我们执行*int(False)*，我们得到0。它们的浮点表示只是在每个末尾添加一个.0。

布尔转换使用**bool()**命令，所以执行**bool(1)**或**bool(0)**分别得到True和False。字符串转换与普通转换相同，*str(True)*得到字符串“True”，*str(False)*得到“False”。

# 第 12 章：错误

在阅读本书的过程中，你肯定遇到过一些错误，其中大多数可能不太容易理解。

在这里，我将讨论一些最常见的错误以及你应该如何解决它们。

## 语法错误

你最可能遇到的第一个也是最常见的错误是**语法错误**，这意味着你的代码的某些部分不符合 Python 的规则。

错误的好处在于它们会准确地告诉你哪里出错了，这样你就可以修复它。这里有一个例子：

```
txt = "Hello" print(txt)
```

如果我们有这个程序，我们的输出将是：

```
txt = "Hello" print(txt)
SyntaxError: invalid syntax
```

正如你可能注意到的，print 语句与变量在同一行，这不是正确的 Python 代码。因此，我们的 IDE 告诉我们犯了一个错误，并在我们出错的地方加了下划线，以便我们可以修复它。

## 名称错误

![](img/68bd89f6eb7eca3e8b8bb158e8d40be2_79_0.png)

这里的错误是我们输入了 *text* 而不是 *txt*，所以这给 Python 发出了红色警报：

![](img/68bd89f6eb7eca3e8b8bb158e8d40be2_79_1.png)

**名称错误**发生在某个东西的名称不匹配时，无论是变量名还是 Python 命令的名称。IDE 不仅会告诉我们哪里出错了，还会告诉我们我们可能想表达什么！当然，这因 IDE 而异，但错误信息应该大致相同。

## 缩进错误

*阅读第 16 章后再回到这个错误*

正如你所看到的，Python 是一种基于缩进的语言，如果这是你的第一门语言，这可能看起来并不奇怪，但许多其他语言使用 {花括号} 来表示代码包含在某个东西中。

为了简单和可读性，Python 使用缩进，然而，这有时会令人困惑。

如果你有这样的代码：

```
x = 10

if x < 20:
    print("x is less than 20")

else:
    print("x is not greater than 20")
```

它将产生这个错误：

```
print("x is not greater than 20")
^
IndentationError: expected an indented block after 'else' statement
```

这个**缩进错误**发生是因为 print 语句没有缩进，以表明它是 else 语句执行的结果。

这种错误的一个更棘手的形式是这样的：

```
x = 10

if x < 20:
    print("x is less than 20")
    print("What's wrong?")
```

这将导致相同的错误，但解释有点不同：

```
print("What's wrong?")
^
IndentationError: unindent does not match any outer indentation level
```

你可能想知道哪里出错了，但如果你仔细观察，你会发现第二个 print 语句的缩进不足以与前一个语句对齐，从而导致错误。

## 类型错误

当你尝试对无法执行该操作的数据类型执行操作时，就会发生这种情况。有很多情况可能发生这种情况，但下面列出了一些：

```
str = "Python has too many rules."

str[13] = 'r'

print(str)
```

这段代码的结果是：

```
str[13] = 'r'
TypeError: 'str' object does not support item assignment
```

这可能一开始看起来令人困惑，但一旦你意识到字符串是**不可变数据类型**，它就变得清晰了。这意味着一旦它们被创建，就不能被更改。因此，用另一个字符替换一个字符会导致**类型错误**。如果你想这样做，你只需要：

```
python
str = "Python has too many rules."
print(str[:13] + "r" + str[14:])
```

本质上，这做的是打印所有字符直到你要添加的字符，添加该字符，然后使用字符串切片打印字符串的其余部分。

这种错误的另一个例子是：

```
python
dogcount = 12
print("I want " + dogcount + "dogs")
```

这将导致：

```

print("I want " + dogcount + "dogs")
TypeError: can only concatenate str (not "int") to str
```

正如你所看到的，这是因为字符串只能与字符串连接，所以我们不能将整数与字符串组合。一个简单的解决方法是将变量 *dogcount* 转换为字符串，然后连接它们。

## 索引错误

***阅读第 14 章和第 17 章后再回到这个错误***

当你尝试引用字符串、列表等中超出范围的内容时，就会发生此错误。这里有一个例子：

```
mylist = [12, 14, 1, 2, 16, 72]
print(mylist[10])
```

我们得到：

```
print(mylist[10])
IndexError: list index out of range
```

我们尝试引用列表范围之外的内容。

此列表的最大索引为 5，因此索引 10 不存在，从而导致此错误。此错误也可能发生在字符串以及任何其他你可以引用其中内容的数据类型上。

我想分享的一个例子是循环：

```
mylist = [12, 14, 1, 2, 16, 72]

for i in range(0, 6):
    print(mylist[i])
```

在这里，我们可以看到我们最终会打印 *mylist[6]*，它不存在，因此我们将收到**索引错误**，即索引超出范围。

## 值错误

## ValueError

当尝试将一种数据类型转换为另一种，但结果无效时，就会发生 **ValueError**：

```python
str = "this isn't working"
print(int(str))
```

我们无法将该字符串转换为整数，因为它不包含数字，因此我们会收到以下错误：

# 第16章：IF、ELSE和ELSE IF语句

### If和Else

像if或else这样的语句被称为**条件语句**，因为其中涉及一个条件：

如果约翰喜欢冰淇淋，就给他冰淇淋。否则，就给他糖果。

在Python中，工作原理相同，唯一不同的是格式。

在Python中，if和else语句通常像这样格式化：

```
x = 7

if x > 10:
    print("x is greater than 10")

else:
    print("x is not greater than 10")
```

注意，我们没有为else子句放置语句，因为它与if语句相关联。

else意味着与if语句相反的情况，所以在我们的例子中，这意味着else代表x不大于10的情况。

你可能注意到的另一件事是缩进。缩进仅仅意味着代码位于另一个语句内部，并且只有在该语句为*True*时才会执行。缩进告诉程序，这段代码是某个其他部分的一部分并包含在其中，比如条件语句，但也包括循环或函数等，我们稍后会讲到。

因此，*if*语句可以被评估为*True*或*False*。

如果它是*True*，我们将能够执行*if*语句内部的代码，然后跳过*else*语句；但如果它不是*True*，那么我们将跳过*if*语句，转到*if*语句之后的下一行。

### ==?

一个需要记住的运算符是**比较运算符，==**。这可能看起来很奇怪，为什么我们需要一个双等号符号？

嗯，它有一个非常有用的用途，你将在编码过程中大量使用它。普通的=是一个**赋值运算符**，我们可以给某物赋值。现在，让我们创建一个if语句，如果为真，我们将打印“wow”：

```
x = 10

if x / 2 == 5:
    print("wow")
```

如你所见，等号下面有红色下划线，因为我们本应在那里使用比较运算符。

通过说 $x / 2 = 5$，我们是在将值 $x / 2$ 赋给5，而不是检查它是否等于5。因此，我们需要使用比较运算符==。一个经验法则是，只在给变量赋值时使用普通的=，而在其他所有地方都使用==。

### Else If

Else if语句（写作*elif*）用于当我们有多个条件需要检查，但我们只想在前一个条件失败时才进入下一个条件的情况。

例如，如果我们有一个变量*age*，那么我们可以这样写：

```
age = int(input ("What is your age?:"))

if age < 18:
    print ("Minor")

elif age >= 18 and age < 40:
    print( "Adult")

elif age >= 40 and age < 60:
    print("Middle Aged")

else:
    print("Elderly")
```

如你所见，我们只在前一个语句失败时才进入下一个语句。例如，如果*age*是16，我们将通过第一个条件，直接打印"Minor"，而不会执行其余代码，因为我们没有失败。一旦通过了其中一个条件，其余的条件将被忽略，我们继续执行程序。

思考*if*、*elif*和*else*命令的一个好方法是使用树状图：

使用*elif*语句而不是多个*if*语句的好处在于，即使前一个条件满足，其他*if*语句仍然会被执行。

一种思考方式是，*elif*语句都检查是否满足其中一个条件并相应地做出响应，而多个*if*语句都是独立的，无论是否满足都会被检查。

### And、Or和Not

现在让我们稍微复杂一点。

我相信你们都知道**and**、**or**和**not**这些词的意思，在编程的上下文中，它们非常相似。

*And*意味着两个条件都必须为*True*，整个语句才为*True*。

*Or*意味着其中一个条件必须为*True*，语句才为*True*。

*Not*会反转条件的结果（*True* => *False*，*False* => *True*）。

让我们看一些例子来消除任何疑虑。使用之前的例子：

```
x = 7

if x > 10 and x > 5:
    print("wow!")
else:
    print("uh oh")
```

这里，我们看到我们满足了*if*语句的第二个条件，但不满足第一个条件，所以整个语句为*False*。然后我们将进入*else*语句。

现在轮到*Or*了：

```
x = 7

if x > 10 or x > 5:
    print("wow!")

else:
    print("uh oh")
```

这里我们的结果不同，$x$可能不满足第一个条件，但这是一个*or*运算符，意味着只要其中一个为*True*，整个语句就为*True*。7大于5，所以我们将打印"wow!"。

最后是*Not*：

```
x = 7

if not(x > 10) :
    print("wow!")

else:
    print("uh oh")
```

我们看到7不大于10，所以第一个条件不满足，但我们前面有一个*not*，这意味着*False*变成了*True*。

由于if语句被评估为*True*，我们执行其内部的代码并打印“wow!”。

循环！循环通过自动重复执行任务指定次数，让我们的生活更轻松。

### While 循环

一种循环类型被称为 **While 循环**。*while* 循环通过 *True* 和 *False* 语句工作。只要给定的条件在下一次运行时仍为 *True*，它就会持续执行命令。每次循环执行称为一次 **迭代**。

例如，如果我们有类似这样的代码：

```
x = 1

while x<10:

    x = x + 1

print ("Done")
```

如果我们逐步分析这个程序，会看到我们首先将 *x* 初始化为 1，然后有一个 while 循环，它会一直运行直到 *x* 大于或等于 10。

在 while 循环内部，我们打印分配给 *x* 的值，并将 *x* 递增 1。一旦 *x* 为 10，循环结束，我们打印“Done”。

注意“Done”不在循环内部，因为它没有缩进。这是因为我们只想在循环结束后打印一次“Done”，所以如果把它放在循环内部，“Done”会在每次迭代时都被打印。

因此，这个程序的结果是：

![](img/68bd89f6eb7eca3e8b8bb158e8d40be2_122_0.png)

最后需要意识到的是，我们需要 $x = x + 1$ 这个条件，因为没有它，$x$ 将始终为 1，而 1 始终小于 10。

### 无限 While 循环

这看起来可能没用，为什么要有永远无法结束的循环呢？

然而，这在几乎所有使用循环的场景中都非常有用，因为更多时候，我们永远不知道可能需要循环多少次。

例如，如果你正在设计一个游戏，想要有一个结束屏幕显示“*再玩一次？*”，如果用户选择“是”，循环再次运行，如果选择“否”，游戏结束。我们不知道应该循环多少次，因为用户可能在玩 1 轮或 50 轮后就结束了。

因此，我介绍 **while True** 循环。

*while True* 循环会永远继续，没有任何条件能让它停止。它只能通过 **break** 语句停止。

通过在循环内部放置 break 语句，它会立即结束循环，语句之后的所有代码都不会执行。还有一个 **continue** 语句，它会强制程序回到循环顶部，程序不会执行 continue 语句之后的任何内容。

为了说明这些语句在哪里有用，让我们使用之前的场景。如果我们想制作一个游戏，让用户决定是否再玩一次，如果用户说“否”，break 语句可以派上用场；如果用户说“是”，continue 语句可以派上用场，所以我们可以这样做：

```
while True:
    print("Wow Python Wow")
    answer = input("Play Again? (Y/N)")
    if answer == "N":
        break
    elif answer == "Y":
        continue
```

创建无限循环的另一种方法是使用布尔值。

我们可以将一个变量初始化为布尔值 *True*，只要该变量的求值结果为 *True*，我们就继续循环：

```
condition = True
while condition == True:
    print("Wow Python Wow")
    answer = input("Play Again? (Y/N)")
    if answer == "N":
        condition = False
    elif answer == "Y":
        continue
```

这与 *while True* 循环具有相同的功能，只是在这里，我们只有在变量 *condition* 为 *True* 时才继续循环，并且不需要 break 语句。

因此，如果用户拒绝再玩，我们只需将 condition 的值更改为 *False*，当我们再次循环时，程序结束，因为变量 *condition* 不是 *True*。

如果用户决定继续玩，*condition* 保持不变，我们再次循环。

### For 循环

另一种循环类型是 **for 循环**，它类似于 while 循环。

*for* 循环仍然循环，但它与 *while* 循环的不同之处在于它遍历对象。例如，*for* 循环可以遍历字符串并打印字符串中的每个元素，或者可以遍历数字列表。

对于这两种情况，工作方式是：选择对象中的第一个元素，执行 *for* 循环中的代码，然后循环回到顶部，选择下一个元素，依此类推。这个过程持续进行，直到没有更多元素可供选择。

*for* 循环的一般格式是有一个临时变量，该变量被分配给对象中的每个元素，以便随后在 *for* 循环中使用。

它的唯一目的是遍历数据集。

如果之前的解释有点混乱，这里有一个例子：

```
MyList = [2,4,7,12,13,6,5,16,7,16,2]

divcount = 0

for i in MyList:
    if i % 3 != 0:
        divcount = divcount + 1

print("There are " + str(divcount) + " numbers in this list that are not divisible by three")
```

这里的目标是找出列表中不能被三整除的数字的数量，为此，我们首先获取列表，然后创建一个变量 *divcount* 来跟踪不能被三整除的数字的数量。然后我们创建一个 *for* 循环来遍历列表并检查列表中的每个元素是否能被三整除，如果不能，我们将 *divcount* 变量增加一，并在程序结束时打印最终计数。

在继续之前，让我们澄清一下关于前面程序的一些混淆点。我们在 for 循环中使用了变量 *i* 作为临时变量来存储列表中的每个元素。每次循环迭代时，*i* 都会被重新分配给列表中的下一个元素。

*i* 是制作循环时常用的变量，但任何其他你选择的变量名也可以使用。

最重要的是要记住，它在程序中的唯一目的是作为一个变量，在遍历列表时被重新分配给列表中的项目。

### 嵌套循环

嵌套 *for* 循环起初是一个难以理解的概念，但一旦你掌握了它们，它们在各种情况下都非常有用。

它们有用性的一个例子是，当我们有一个二维列表，我们不仅想遍历该列表，还想遍历列表中包含的列表元素。

在这里，我们会这样做：

```
my2dlist = [[2,1,13,1023], [6,1,27,1], [10,12], [1,16,1,54,39], [18,1]]

for i in my2dlist:
    for j in i:
        if j == 1:
            print("1 is here!")
```

如你所见，外层循环遍历每个元素并将其分配给 *i*。内层循环然后遍历列表元素 *i* 以查找数字 1 是否存在。

运行后，我们得到结果：

```
1 is here!
1 is here!
1 is here!
1 is here!
1 is here!
1 is here!
```

我们可以得出结论，这个二维列表中总共有 6 个 1。

嵌套循环也可以用于普通列表，而不仅仅是二维列表。

如果我们想将一个元素与列表中的每个其他元素进行比较，我们会有一个外层循环遍历元素，然后有一个内层循环遍历所有其他元素以便进行比较。

我们还可以有嵌套的 *while* 循环和 *for* 循环在 *while* 循环内，反之亦然，但你必须确保 while 循环被正确处理，并且有一个 break 语句可以访问，否则循环将永远不会结束。

---

value = 14

if value % 2 == 0:
    if value % 6 == 0:
        print("multiple of 2 and 6!")

我们检查值是否为偶数的条件，只有当它是偶数时，我们才检查它是否是 6 的倍数的条件。这是一个嵌套的 *if* 语句。

我们还可以在 else 语句中有 if 语句，逻辑与之前相同。假设我们在程序中添加另一个子句：

```
value = 20

if value % 2 == 0:
    if value % 6 == 0:
        print("multiple of 2 and 6!")
else:
    if value % 5 == 0:
        print("multiple of 5 and odd!")
    else:
        print(":(")
```

在这里，我们说的是，如果我们发现我们的值不满足第一个条件，意味着它不是偶数而是奇数，我们才会检查它是否能被 5 整除。如果它也不能被 5 整除，那么我们只打印一个悲伤的表情 :(

# 第18章：函数

函数

现在你已经理解了如何编写代码，接下来是组织代码的部分。不过别跑开，这不像打扫房间那样枯燥，组织代码实际上可以相当有趣，而且从长远来看还能节省你的时间！

函数似乎是一个新概念，但我们从本书开始就一直在使用它们。

函数本质上是一块可以通过一条命令“调用”或执行的代码，这意味着我们只需编写一次，之后每当需要再次使用该代码块时，就可以调用它。这里有一个例子：

```
def myfunc():
    a = 12
    b = 20
    return a+b

myfunc()
```

如你所见，我们使用 *def* 关键字创建了函数，然后写了函数名，后跟括号和冒号。

接着我们在函数内部编写了一些代码来对两个数字求和，并在最后写了 **return** a + b。

**Return** 是一个新关键字，出现在函数的末尾。

return 关键字的目的是**存储**函数输出的最终值，并让程序返回到调用函数时中断的地方。return 语句不会打印任何内容。

下面是一个图示，向你展示 return 和 call 语句是如何工作的：

![](img/68bd89f6eb7eca3e8b8bb158e8d40be2_134_0.png)

为了确保你理解这个图示，这里有一个解释。

这个程序不是从函数开始的，Python 实际上会忽略所有这些代码，因为函数还没有被调用。

程序从调用语句 *myfunc()* 开始。然后它回到函数，执行函数，接着 return 语句将程序带回 print 语句处中断的地方。

## 参数

我们一直在使用函数，如果你回顾一下，你会注意到它们通常在括号里有一些东西，无论是文本还是数字。这些被称为**参数**。

参数就像函数的输入，我们输入参数，函数使用这些作为输入来执行其内部的代码。

例如，如果我们回顾之前的程序：

```
def myfunc(a, b):
    return a+b

print(myfunc(12, 20))
```

现在，我们在括号内放置了两个参数 a 和 b，表示函数需要传入这两个参数才能执行。

然后我们可以调用函数并输入我们喜欢的任何数字。这些输入被称为**实参**。实参是传递给函数的输入，参数是函数接收的变量占位符。

然而，它们不仅限于数字。它们可以是列表、字符串、布尔值以及你能想到的任何其他最适合你函数的数据类型。任何东西都可以传递给函数以满足你的编程需求。

参数的好处在于，如果我们想在不使用参数的情况下添加不同的数字集，我们就需要每次都更改函数内部的代码，这在这里可能看起来不是什么麻烦，但在需要大量输入计算的程序中，它会变得相当繁琐。

通过使用参数，我们所要做的就是在调用语句中更改一次实参的值，然后这些新值就会被传入，这样我们就不需要对实际函数做任何更改。

## 输入作为实参

我们之前已经使用过用户的输入作为程序中的值，对于函数来说，这也没有什么不同。

如果我们想在函数中使用输入，我们所要做的就是将这些输入设置为函数的实参，如下所示：

```
def myfunc(a, b):
    return a+b

value1 = int(input("select a number: "))
value2 = int(input("select a number: "))
print(myfunc(value1, value2))
```

如你所见，我们只是创建了变量来接收输入，将输入从字符串转换为整数，然后使用这些变量作为实参。

你可能还注意到的另一件事是，变量的名称与函数中参数的名称不同。这看起来可能是个错误，但这是完全没问题的。

输入实参的变量名不必与函数接收的参数名相同。

我们必须在函数内部使用正确的参数名（函数接收的 *a* 和 *b*），但在函数外部，我们可以传入任何变量名作为输入。

函数接收分配给我们创建的变量的值，并将其重新分配给它定义的参数名。

在我们的例子中，*value1* 中包含的值被分配给 *a*，*value2* 中包含的值被分配给 *b*。

你可能还注意到函数的设置与我们正常的代码不同，比如我们在函数之后的调用语句中给函数输入，而不是像往常一样在代码开头。这是因为，如你所知，Python 是自上而下运行的，所以我们不能在函数之前放置调用语句，因为 Python 还没有看到该函数，所以它会给你一个 NameError。

这就是为什么调用语句必须在函数定义之后。

至于输入变量的初始化，它们可以放在函数之前，然后在函数定义之后的调用语句中使用，但是，最好将不在函数中的代码部分放在一起，以避免混淆。

# 第19章：TRY-EXCEPT

我相信我们都能说，之前的错误信息并不完全是我们所说的愉快的阅读体验，有时，错误并没有确切地告诉你程序中出了什么问题，就退出了，没有给你自己处理错误的机会。

## Try, Except

为了解决这个问题，我们有 **try-except** 语句。try-except 语句用于错误处理，如下所示：

```
try:
    txt = "hello"
    print(text)
except:
    print("An error happened, oops")
```

这会打印：

```
An error happened, oops
```

try except 的格式是通过一个 **try** 语句来放置代码，然后是一个 **except** 语句，如果发生错误，程序不会立即退出，程序员有机会通过在发生错误时有一个完全独立的代码段，或者简单地告诉用户他们哪里出错了来处理错误。

## Try, Except...两次？

我们也可以对特定错误使用 *except* 语句：

```
try:
    txt = "hello"
    print(txt)
except NameError:
    print("A variable is not defined")
```

在这里，我们指定了 *except* 语句可以检查的错误类型，然后如果发生该错误，就打印我们的自定义错误消息：

```
A variable is not defined
```

但如果我们得到的错误与预期的不同呢？在这种情况下，我们实际上可以有 2 个 *except* 语句：

```
try:
    txt = "hello"
    print(text)
except SyntaxError:
    print("You did not follow the rules of Python")
except:
    print("Something else went wrong")
```

这里的另一个 *except* 语句是为了以防引发的错误不是 SyntaxError。

在这种情况下，错误实际上是 NameError，所以程序会引发 NameError，检查 except 语句是否提到了 NameError，如果没有，它就会进入第二个 *except* 语句：

```
Something else went wrong
```

## Else

如果一切正常，我们可以放置一个 *else* 语句：

```
try:
    txt = "hello"
    print(txt)
except NameError:
    print("A variable is not defined")
else:
    print("All good!")
```

由于这个程序没有错误，没有引发错误，变量 *txt* 和 else 语句被打印：

![](img/68bd89f6eb7eca3e8b8bb158e8d40be2_145_0.png)

## Finally

一旦 try-except 语句完成，我们可以放置 **finally** 代码块，无论发生什么都会执行。它可以通知我们 try-except 已经完成：

```
try:
    txt = "hello"
    print(txt)
except NameError:
    print("A variable is not defined")
else:
    print("All good!")
finally:
    print("The try-except is done!")
```

这将打印变量 *txt*，因为程序没有错误。它会跳过第一个 *except* 语句，执行 *else* 语句，并执行 *finally* 语句：

```
hello
All good!
The try-except is done!
```

最后需要注意的是，try-except 语句不会停止程序，它们只是让程序在 except 语句中继续执行。

Try-Except 块可能看起来没有必要，我们完全可以直接阅读错误信息，但它们是防止程序崩溃的好方法，让我们能够自己处理错误。

## Raise

**raise** 命令实际上可以让我们主动引发一个错误。我们为什么会想这样做呢？嗯，当用户输入了不允许的内容时，我们可以引发一个错误并告诉用户他们不能输入该内容：

```
txt = 17

if type(txt) != str:
    raise TypeError("Only strings are allowed")
```

我们这里所做的就是指出用户输入了一个整数，而他们只能输入字符串，因此我们引发了一个带有自定义消息的 TypeError，解释了他们做错了什么：

```
TypeError: Only strings are allowed
```

我们也可以根据 *raise* 命令想要实现的目标，简单地使用 *raise Exception* 或任何其他错误类型来代替 TypeError。

请注意，*raise* 语句会立即结束程序，它不像 try-except 语句那样让程序继续运行。
