## 什么是MicroPython和用于微控制器的Python

所以你们中的一些人可能会想，MicroPython是什么？这个超级小的蛇是什么？或者可能和Monty Python有关？IEEE Spectrum在2018年发布了最常用编程语言的排名。

### 什么是MicroPython？

- **精简**且**高效**的**Python**重写版本
- 包含完整的解析器、编译器、虚拟机、运行时系统和垃圾回收器
- **字节码**或**原生机器码**
- 支持**内联汇编**
- **在芯片上编译**
- **REPL**（读取、求值、打印 - 循环）

![](img/610f3057ec812c0d14bb9f163e39f201_5_0.png)

所以MicroPython是Python的完全重新实现，它被设计得精简高效，以便在微控制器上运行，因为你不能像在台式电脑上那样直接使用Python。它需要从头开始重写以适应微控制器。它有一个虚拟机和一个运行时系统，带有垃圾回收以及你需要让它真正高效运行的一切。你可以使用字节码或原生机器码。MicroPython还支持内联汇编。当你有一个项目想使用Python，因为你希望它容易上手，但你需要在某些微小的部分让它更高效时，它也支持内联汇编。编译发生在芯片上。你写的程序不是在你的台式电脑上编译，而是在你使用的硬件上编译。我这里有一个很棒的摄像头，我会给你们展示一些演示。

### 什么是 MicroPython 以及为什么你应该关注它

大家好，欢迎回来。在这个项目中，我将谈论 MicroPython，谈谈它的存在意义，它与 Python 的关系，以及它的一些最重要的特性。

大约在 2014 年，Damien George 发布了一种用于微控制器的新编程语言，名为 MicroPython。这次发布是一个雄心勃勃的 Kickstarter 项目的成功完成，该项目始于 2013 年。当时，微控制器编程主要由 C 语言主导。如果你熟悉 C 语言，那么你知道它是什么样子的。在微控制器上使用 C 语言并不难学，然而，随着程序变得越来越大，事情确实会变得更复杂。随着微控制器变得越来越强大，越来越多的人开始对它们感兴趣并对其进行编程。其中许多人是第一次编程，这包括各个年龄段的人。因此，Damien 想创造一种能在微控制器上运行的语言，它比 C 语言更容易学习和使用。他不想重新发明轮子，所以他选择了 Python 作为原型。他当时的挑战是创造一种能够模拟 Python，但也能在微控制器的裸金属上运行的语言，而不是在操作系统之上。这就是 MicroPython 的由来。以下是 MicroPython 网站上对该语言的描述。其中的强调部分是我加的。MicroPython 是 Python 3 编程语言的一个精简高效的实现，包含 Python 标准库的一个小子集，并针对在微控制器和受限环境中运行进行了优化。

现在，因为 MicroPython 包含 "Python" 这个词，很容易混淆，认为 MicroPython 只是 Python 的一个更小版本。这种混淆就像我经常看到的 Java 和 JavaScript 之间的混淆一样，尽管 Python 和 MicroPython 名字相似，但它们是完全不同的语言，有着不同的目标和实现。我将在后面的项目中更详细地讨论 Python 和 MicroPython 之间的区别，但现在我只想确保你不会因为名字的相似性而感到困惑。

MicroPython 从 Python 借鉴的是其语言架构、注重代码可读性的编程哲学，以及庞大的、已经知道如何使用 Python 的程序员群体。Python 开发者可以很快成为 MicroPython 开发者，并为微控制器编写程序。根据 TIOBE 编程语言流行度指数，Python 是世界上最流行的编程语言，占有 30% 的份额。该指数是基于 Google 上对编程语言、教程和资源的搜索量计算的。作为对比，有趣的是，开发板上使用的 C/C++ 在该指数中排名大约第五。这种流行度转化为一个充满你所需的所有文档、库和社区支持的 Python 生态系统。MicroPython 和 Python 一样容易学习，并且遵循 Python 在优秀开发工具和文档方面的传统。在本课程中，你会看到我不断浏览 Python 代码、文档以及我们将要使用的许多优秀库。在工具方面，你有很多选择。特别是在本课程中，我们将使用 Thonny，但你也可以选择 uPyCraft 和新的 Mu 等工具。我真正喜欢 Thonny 的地方在于，它本身就是一个功能齐全的 Python IDE，拥有出色的调试工具，同时也完全支持 ESP32 上的 MicroPython，以及其他目标板，如 Raspberry Pi Pico 和 BBC micro:bit。MicroPython 语言的另一个巨大优势是，一旦你学会了它，你就可以将你的技能应用于多种硬件目标。在我录制这个项目时，MicroPython 支持最初的主板版本一和 Pyboard，以及第三方开发板，如 STM32 NUCLEO 和 Discovery 开发板、Pyboard、Raspberry Pi Pico、WiPy、ESP8266 和 ESP32、TinyPICO 以及我之前提到的 BBC micro:bit，而这还只是部分列表。

### MicroPython 特性

1. 旨在实现 Python 3.4 (CPython) 的语言和语法标准。
2. MicroPython 标准库实现了 CPython 标准库的一个子集（以 "u" 前缀标记）。
3. MicroPython 中提供 REPL 交互式提示符（读取-求值-打印循环）。
4. 易于安装和使用第三方代码包。许多包可在 PyPI 上找到。
5. 支持设备上的文件系统。
6. 简单的命令行工具，用于与 MicroPython 设备交互：pyboard.py

现在让我们快速浏览一下 MicroPython 最重要的特性。首先，对于任何这门语言的新手来说最重要的是，MicroPython 旨在实现 Python 3.4（以及部分 3.5）的语言和语法标准。这简单意味着任何已经会用 Python 3 编程的人都可以立即开始用 MicroPython 编程。Python 3 的保留关键字、运算符、函数以及那令人头疼的空白符规则都在 MicroPython 中得到了忠实的实现。其次，由于 MicroPython 面向嵌入式计算机和微控制器，不可能实现包含所有模块和方法的 Python 标准库。物联网设备上根本没有足够的存储空间来容纳它。因此，MicroPython 实现了 CPython 标准库的一个精选子集，而且即使是这个子集，也是以效率为重点来实现的。Python 库的 MicroPython 版本名称带有 `u` 或小写 `u` 字母前缀。这样你就可以区分它们。每当你看到一个带有 `u` 字母前缀的 MicroPython 库时，要知道它是为 MicroPython 设备实现的原始 CPython 库的更高效或优化版本。MicroPython 有一个交互式解释模式，也称为 REPL。REPL 代表 Read-Eval-Print Loop（读取-求值-打印循环）。可以把它想象成 Python 的命令行。你可以使用这个命令行来发出 Python 指令甚至代码块。REPL 会立即求值 Python 代码，MicroPython REPL 功能齐全，具有补全能力、可以使用 Ctrl-C 中断正在运行的程序以调用软重置等等。还有一个粘贴模式，你也可以使用下划线变量来存储上一次的输出。

### MicroPython 概述

MicroPython 的一个关键特性是能够使用 `uasyncio` 库在测试代码中广泛演示异步计算。其次，除了 MicroPython 标准库之外，还有无数由用户贡献并发布在 GitHub 和 PyPI（Python 包索引，类似于 CPython）等在线仓库上的库。MicroPython 有一个简单的机制来包含外部代码程序。在本课程中，我将向你展示如何查找和使用这些外部库，它们使得将屏幕和传感器等硬件组件集成到你的 MicroPython 项目中变得轻而易举。

第三，MicroPython 能够访问目标设备上的一个小文件系统。这个文件系统使得存储你的 MicroPython 程序、支持库文件以及任意文件（例如用于存储传感器数据或网络和 IoT 服务凭证的文本文件，甚至是位图图像文件）成为可能。你希望在屏幕上显示它们，例如，我将在本课程后面展示一个类似的例子。在本课程中，我准备了几个示例，演示如何在 ESP32 上使用文件系统。

最后，第六点，MicroPython 有一个简单的命令行 Python 工具，允许你在目标设备上运行脚本或访问文件系统。这个工具叫做 `mpremote`。在本课程中，我们不会使用这个工具，因为 Thonny IDE 内置了对多种目标设备（包括 ESP32）的 MicroPython 支持。然而，我想提到 `mpremote`，因为它对你来说是有用的，值得了解。

让我们现在来谈谈在各种硬件目标上运行 MicroPython。正如你可能已经知道的，MicroPython 可以在许多不同的微控制器上运行。硬件的多样性意味着并非所有 MicroPython 代码都能在不修改的情况下跨设备工作。通常，在跨不同目标共享 MicroPython 代码时，有两点需要记住：

1.  大多数使用 MicroPython 标准库函数和语言核心的代码无需任何修改即可工作。语言语法、保留关键字、控制结构以及来自这些标准库的函数（例如用于数学的 `math` 模块、用于基本操作系统服务的 `os` 模块以及用于时间和日期相关功能的 `time` 模块）将在所有 MicroPython 硬件目标上工作。这一点无需担心。
2.  另一方面，任何在特定微控制器上独特实现的功能都需要在 MicroPython 中进行独特的实现。例如，数字引脚功能在 ESP32 等设备上的实现方式与在树莓派上的实现方式不同。有些树莓派引脚能做的事情，ESP32 可能做不到。网络接口、SPI 接口、模数转换器等功能在不同开发板上的实现方式也类似。这些差异反映在每个开发板的 MicroPython 实现中。因此，除了标准库之外，MicroPython 还有专门为每个受支持的开发板实现的库。你应该花点时间研究你的目标设备的特定库，以便了解有哪些功能可用，哪些不可用。因此，你可以充分利用你所选择的设备的特定能力。

还有一点，当我谈到这个时，并非所有设备功能都可以通过 MicroPython 访问。例如，在 ESP32 的 MicroPython 固件中，没有对蓝牙的支持。没有蓝牙模块，即使你知道 ESP32 具备蓝牙功能。但是有对 Wi-Fi 的支持。这只是你无法使用 MicroPython 使用的一个功能示例。

这两种语言的另一个差异体现在 Python 函数中使用 `self` 关键字时的处理方式。Python 不会将其计为额外的参数，但 MicroPython 会。因此，如果你向一个包含 `self` 关键字的函数传递了错误数量的参数，那么你在 Python 和 MicroPython 中得到的错误信息将会不同。这里有一个例子。你可以看到这个例子。我在 Python 和 MicroPython 中调用了同一个函数 `Calculator`，并且只传递了一个参数。我本应传递两个参数，但我犯了个错误。这里只传递了一个参数。你可以看到它们编码了具有相同参数的完全相同的函数。但返回的指示错误的信息却不同。所以，你看，Python 告诉我缺少一个必需的参数，而 MicroPython 告诉我总共应该有三个参数，而我只给了两个。实际上，我只给了一个。所以你可以看到，Python 中返回的这个信息可能会有点令人困惑，这可能会误导你并导致你调试延迟。只要你意识到 `self` 关键字的这种情况，我认为你就能克服这类问题。
