# Python的特点

Python是一种解释型语言

## 编译器

从其操作功能来看，编译器会一次性将整个源文件转换为可读的机器语言。随后，机器语言被执行。

## 解释器

另一方面，解释器采用逐行方式将高级语言转换为机器语言，然后执行。Python解释器会获取文件的开头，将第一行转换为机器语言，然后执行它。

区分高级语言和编译语言至关重要。例如，像C和C++这样的编译语言使用编译器来翻译和解释（从高级语言到机器语言）。另一方面，像Python这样的解释语言则使用解释器来进行这种翻译和后续执行的过程。解释语言和编译语言之间的重要区别在于，广泛使用的编译语言在运行和执行性能上通常优于使用解释语言编写的程序。然而，Python并不受此缺点的影响。

总之，Python是一种解释语言，因为程序直接从源文件执行。每当运行Python程序时，都需要源文件。Python所做的就是将开发者编写的源文件转换为中间代码，然后再将其转换为可执行的机器语言。因此，Python是一种解释语言，因为它由解释器实时处理。源文件在执行前无需编译。编译意味着代码必须在运行前转换为机器语言。

## Python是动态类型的

Python的另一个特点是动态类型，这意味着数据类型在执行时即时检查，而静态类型则是在运行前检查数据类型。

## Python是强类型的

强类型语言的主要特点是它不能自动将一种形式（类型）的数据转换为另一种。另一方面，像PHP和JavaScript这样的松散类型语言则可以自由且自动地将数据从一种类型转换为另一种。

在这方面，在将12加到字符串之前，JavaScript语言会尝试将数字12转换为字符串“12”，然后将其附加到字符串的末尾。

Python语言会引发错误，因为它没有将数字12转换为字符串的能力。

总之，Python是一种强类型语言，即变量的类型必须是已知的。这意味着值的类型不会以意外的方式改变。例如，仅包含数字的字符串不会自动变成数字，这在Perl中可能发生。在Python中，每种数据类型转换都需要特定的转换。

## 庞大的库集合

Python为用户提供了广泛的库，这使得添加新功能和能力变得容易，而无需重新发明新的方法。

## Python 变量

理解 Python 变量、类及其工作方式，对于初学者和希望扩展编程技能的程序员都至关重要。

## 什么是 Python 中的变量？

在编写复杂代码时，随着程序的执行，你的程序将需要数据以进行必要的更改。因此，变量是在程序开发过程中赋值后创建的、用于存储代码值的区域。与其他相关编程语言不同，Python 没有声明变量的命令，因为变量在设置后可以修改。此外，Python 中的变量不像大多数其他计算机语言编程那样有限制。

因此，Python 中的变量被定义为用于保存数据值的内存资源。这样，Python 变量充当存储单元，为计算机提供处理所需的数据。在 Python 编程中，每个值都有其数据库，每个数据被分类为数字、元组、字典和列表等。作为程序员，你了解变量的工作原理以及它们在使用 Python 创建高效程序时的作用。因此，本教程将使学习者了解如何声明、重新声明和连接局部变量和全局变量，以及如何删除变量。

## 变量与常量

变量和常量是 Python 编程中使用的组件，但执行不同的功能。变量和常量都使用在程序创建期间用于创建要执行的代码的值。变量充当内存中数据的基本存储位置，而常量是值保持不变的变量。相比之下，变量存储数据的储备，而常量是一种变量文件，其值以大写字母书写并用下划线分隔。

## 变量与字面量

变量也是字面量的一部分，字面量是变量或常量消耗的数据，在 Python 编程中使用多种字面量。一些常见的字面量类型包括数字、字符串、布尔值、特殊字面量以及元组、字典、列表和集合等字面量集合。变量和字面量之间的区别在于，两者都影响未处理的数据，但变量存储数据，而字面量将数据提供给常量和变量。

## 变量与数组

    def __init__(self, value):
        self.value = value
    def __str__(self):
        return repr(self.value)

    raise UserDefinedException(" input is null ")
except UserDefinedException as userdefinedexception:
    print 'userdefinedexception : ', userdefinedexception.value
```

在上面的程序中，第一个（`try`、`except`、块）处理了除零异常。
第二个（`try`、`except`）块引发了一个用户自定义异常。

## 什么是 Python 中的序列？

程序执行的序列不是一条连接南北的道路。它可以从北向南运行到终点。程序执行的序列也可能像繁忙街区的道路一样复杂，有 9 个转弯和 18 个转弯，容易让人头晕。

要编写一个好的程序，控制程序执行的过程至关重要。因此，有必要使用程序的过程控制结构。没有它们，就不可能使用程序来完成任何复杂的工作。

编程语言已经发展了多年。结构化编程逐渐成为程序开发的主流思想。其主要思想是按照从上到下的顺序执行整个程序。Python 语言主要按照源代码的顺序从上到下执行，但有时执行顺序会根据需要而改变。

此时，可以通过流程控制指令告诉计算机优先执行程序的哪个序列。程序的过程控制就像为交通规划一个向各个方向延伸的交通方向。

众所周知，大多数用于过程控制的程序代码都是从上到下逐行顺序执行的，除了具有高重复性的操作外，不适合顺序执行。任何 Python 程序，无论其结构多么复杂，都可以使用 3 种基本控制流程来表达或描述：顺序结构、选择结构和循环结构。

顺序结构程序的第一行语句是入口点，从上到下执行到程序的最后一行语句。选择结构允许程序根据检查条件是否成立来选择要执行的程序块。如果条件为 True，则执行某些程序语句。如果条件为 False，则执行其他程序语句。

通俗地说，如果你遇到场景 A，就执行操作 A；如果是场景 B，就执行操作 B。就像我们开车到十字路口看到信号灯一样，红灯停，绿灯行。此外，目的地也有不同的方向，你可以根据不同的情况选择路线。换句话说，选择结构表示程序将根据所需条件确定程序的“方向”。

循环结构的循环流程控制功能是在程序块中重复执行程序语句，直到满足特定的结束条件。Python 包括一个 `for` 循环和一个 `while` 循环。

## 循环重复结构

这主要指的是循环管理结构。一个精确的程序语句会根据设定的条件被重复执行，并且只有在条件判断不成立时，循环才会终止。简而言之，重复结构用于设计需要反复执行的程序块，也就是让程序代码符合结构化设计的精神。

例如，如果你希望计算机计算 1+2+3+4+...+10 的值，你不会希望在程序代码中从一累加到十，这原本是繁琐且重复的，你只需利用循环管理结构就能轻松达到目标。Python 包含 `while` 循环和 `for` 循环，其具体用法如下所述。

## While 循环

如果要执行的循环次数是确定的，那么使用 `for` 循环语句是最佳选择。然而，`while` 循环更适合用于循环次数不确定的情况。`while` 循环语句与 `for` 循环语句类似，都属于前测循环。前测循环的运行模式是在循环程序块开始时检查循环条件判断表达式。

当判断表达式的结果为真时，程序会执行循环块内的程序语句。我们通常将循环块内的程序语句称为“循环体”。`while` 循环也使用条件表达式来评估其真假，以控制循环流程。一旦条件表达式为真，循环内的程序语句就会被执行。一旦条件表达式为假，程序流程将跳出循环。

`while` 循环语句的格式如下：

```
While 条件表达式:
    # 如果条件表达式成立，则执行此程序块中的 while 循环语句的流程图。
```

`while` 循环应包含控制变量的初始值以及递增或递减的表达式。编写循环程序时，应检查是否存在退出循环的条件。如果条件不存在，循环体将无限执行，导致“无限循环”，也称为“死循环”。

循环结构通常需要 3 个条件：

- (1) 循环变量的初始值。
- (2) 循环条件表达式。
- (3) 改变循环变量的值（递增或递减）。

例如，以下程序：

```
first = 1
While first < 10:  # 循环条件表达式
    print(first)
    first += 1  # 调整循环变量的递增或递减值。
```

当 `first` 小于十时，`while` 循环内的程序语句将被执行，然后 `first` 会加一，直到 `first` 等于十。如果条件表达式的结果为假，程序将跳出循环。

## For 循环

`for` 循环，也称为计数循环，是编程中常用的一种循环类型。它会重复执行固定次数的循环。如果在设计程序时认为所需的循环执行次数是固定的，那么 `for` 循环语句是最佳选择。Python 语言中的 `for` 循环可用于遍历任何序列的元素或表项。序列可以是元组、列表或字符串，它们按顺序执行。

语法如下：

```
For 元件变量 in 序列:
    # 执行的指令
Else:
    # else 的程序块可以补充也可以不补充，也就是说，使用 for 循环时，else 语句可以补充也可以不补充。
```

上述 Python 语法所描绘的含义是，`for` 循环按照当前序列中元素（项或表项）的顺序，遍历序列（如字符串或列表）中的所有部分。

例如，以下变量值都将被用作遍历序列的部分：

```
first = "abcdefghijklmnopqrstuvwxyz"
second = ['january', 'march', 'may', 'july', 'august', 'october', 'december']
result = [a, e, 3, 4, 5, j, 7, 8, 9, 10]
```

此外，如果你希望计算循环执行的次数，你需要在 `for` 循环管理语句中设置循环的初始值、结束条件以及每次循环执行时循环变量的递增或递减值。`for` 循环每轮执行时，如果没有特别指定递增或递减值，它将自动累加一，直到满足条件。

例如，以下语句是一个元组（11 ~ 15），并使用 `for` 循环打印出元组中的数字元素：

双重嵌套`for`循环的构造格式：

例如，可以使用双重嵌套`for`循环轻松完成一个表格。让我们通过以下示例程序，看看如何使用双重嵌套`for`循环制作九九乘法表。

进入x循环时，x=1。当y循环从1执行到9后，会返回x循环继续执行。y循环中的`print`语句不会换行。外层x循环中的`print()`语句在y循环完成并离开y循环之前不会换行。执行完成后，就得到了九九乘法表的第一行。当所有x循环完成时，乘法表就完成了。

## Python 类继承

继承是面向对象编程中的一个特性，它允许一个类从父类（也称为基类）继承方法和属性。这是编程中一个非常便捷的特性，因为它允许程序员为一个类创建一组功能，然后将它们传递给子类。因此，程序将能够在子类中创建新的甚至覆盖现有的功能，而不影响父类的功能。

继承创建的子类将具有父类的特化。Python 中有四种继承类型：单继承、多级继承、层次继承和多重继承。

## 单继承

没有继承特性的编程类可以准确地称为基于对象的编程。程序运行时，应创建新的抽象数据类型，每个类型都有自己的操作。然而，面向对象编程与基于对象编程的区别在于继承。单继承是指一个类或子类从一个父类继承方法和属性。

## 多重继承

在多重继承中，子类或子类从多个类继承方法和属性。例如，一个类 C 可以同时继承类 A 和类 B 的特性，就像孩子同时继承父母的特征一样。在某些情况下，子类可以继承两个以上父类的特性和功能。

子类可以继承方法和属性的父类数量没有限制。请注意，虽然多重继承以减少程序冗余而闻名，但它也可能给程序带来更高程度的复杂性和模糊性，因此在实现之前必须在程序设计中充分考虑。

## 多级继承

我们已经确定 Python 中的一个类可以继承多个父类的特性。当一个类继承其他类的方法和函数，而这些类又从其他类继承这些方法和函数时，这种方法称为多级继承。与 C++ 和其他面向对象编程语言一样，Python 允许在任意深度实现多级继承。

## 层次继承

当多个类源自一个父类或基类时，就会发生层次继承。子类或子类继承的特性包含在父类中。层次继承与多级继承的区别在于建立类之间关系的顺序。在多级继承中，顺序可能是随意的，父类可以从先前的子类继承特性。

## 为什么继承在 Python 编程中有用？

继承是面向对象编程的一个非常便捷的特性，因为它允许程序员轻松遵守软件开发最重要的原则之一——不要重复自己（DRY）。简单来说，在程序中实现类继承是用更少的代码行和更少的重复来获得更多功能的最有效方式。

继承还将迫使您更加关注编程的规划阶段，以确保您编写的程序代码清晰、简洁且有效。

继承的另一个用途是为程序的不同部分添加功能。

在计算中，多态性和抽象是高级编程特性，它们扩展了继承的应用和实用性。

多态性意味着如果一个类 Y 从类 X 继承，它不一定需要继承该类的所有内容。它可以以不同的方式实现一些继承的方法和属性。Python 作为隐式多态语言，可以重载运算符以根据单个上下文赋予它们所需的行为。多态性的概念是，根据情况的需要，相同的过程可以以多种方式执行。这在 Python 中可以通过两种方式完成：方法重载和方法重写。

方法重载是用不同的参数两次定义相同的函数。例如，我们可以为 Vehicle 类提供两个不同的初始化函数。目前，它只假设有 4 个门。如果我们想具体说明汽车有多少个门，我们可以在当前初始化函数下面创建一个新的初始化函数，带有另一个 doors 参数，如下所示（较新的在底部）：

def __init__(self, makeAndModel, prodYear, airConditioning):
    self.makeAndModel = makeAndModel
    self.prodYear = prodYear
    self.airConditioning = airConditioning
    self.doors = 4

def __init__(self, makeAndModel, prodYear, airConditioning, doors):
    self.makeAndModel = makeAndModel
    self.prodYear = prodYear
    self.airConditioning = airConditioning
    self.doors = doors
```

现在，当创建 Vehicle 类的实例时，可以选择是否定义门的数量。如果不定义，则假定门的数量为 4。方法重写是子类用自己的代码覆盖父类的函数。例如，创建另一个扩展 Vehicle 的类 Moped。将门设置为 0，因为这很荒谬，并将空调设置为 false。唯一相关的参数是型号/品牌和生产年份。它应该看起来像这样：
