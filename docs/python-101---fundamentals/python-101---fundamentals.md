# 介绍

本书是一个名为 *Python 101* 系列的第一部分。在这个系列中，我们将专注于以一种高效且易于学习的方式来学习Python编程语言。目标是聪明地、实际地、快速地学习它，而无需阅读数千页的内容。我们将保持内容的简单和精准。在这第一本书中，我们将向您介绍这门语言并学习基础知识。

这本书是为完全初学者编写的，理解它不需要任何*编程、信息技术或数学*技能。最后，您将能够编写各种简单有趣的程序，并获得继续学习第二卷的必要基础。

## 什么是Python？

实际上，关于Python有一个很大的误解。大多数程序员说Python是一种解释型语言，而另一些则说它是编译型语言。但如果您想找到一个恰当的答案，那么两者都是。Python既是一种解释型语言，也是一种编译型语言。

因为尽管我们没有直接编译源代码，但在后台有一个过程会将源代码编译成字节码，然后Python解释器将该字节码转换成机器语言。所有这些过程都由Python虚拟机（我们将在本书的第一章中下载和安装它）完成。这就是为什么我们可以称Python既是编译型也是解释型语言。

正如我们之前了解到的，Python是一种解释型和编译型语言。但Python也是一种面向对象、具有动态语义的高级编程语言。

## 为什么选择Python？

你现在可能在问自己的一个问题是——‘为什么要学习Python？为什么不学习Java、C、C++或其他语言？为什么？’

首先，一个优秀的程序员精通所有编程语言。所以，学习Python并不意味着你不能学习其他编程语言。但其次，Python可能是开始编程之旅的最佳语言。

Python的语法非常简单，甚至孩子也能轻松掌握。在其他语言中，有很多事情需要你手动处理。而Python让我们的生活变得更轻松。因为庞大的库能够借助已有的工具让你有能力去创造新的东西。意思是，你不需要从零开始制作一切，你可以在你的程序中使用别人的程序来创造新的东西，显然也能节省你宝贵的时间和辛勤的工作。

除此之外，Python的人气正在飞速增长。根据TIOBE指数，Python是人气最高且呈上升趋势的语言（2022年8月）。

## 数据类型、变量与注释

## 3.1 – 什么是数据类型？

数据类型是数据的分类或归类。它表示该值的种类，告诉我们可以在特定数据上执行什么操作。在 Python 编程中，数据类型实际上是类，而变量是该特定类的实例或对象。

## 3.2 – 数据类型

Python 编程中有不同类型的数据类型。在这里我们将全面学习它们。

### 3.2.1 – 字符串

*字符串*不过是一系列字符或一段文本。你应该记得我们在第一个程序中打印了一个字符串。在 Python 编程中，我们在引号（`""` 或 `''`）内编写的任何文本或字符都被称为*字符串*。如果文本没有被引号包围，那么 Python 解释器永远无法将该文本识别为*字符串*。字符串的关键字是 **`str`**。

### 3.2.2 – 整数

整数也被称为数值数据类型。*整数*就是一个普通的整数，我们可以用它进行基本计算。整数的关键字是 **`int`**。

- 示例：1, 54, 69, 23, 0, 32, 1947 等。

### 3.2.3 – 浮点数

浮点数与整数相同。它们也是数值数据类型。但在*整数*中我们不能使用小数位，而*浮点数*允许我们使用小数位。因为它是浮点数。*浮点数*的关键字是 **`float`**。

示例：32.42, 5.01, 0.122 等。

### 3.2.4 – 布尔值

布尔值基本上是一种二进制数据类型，只包含两个值，即*真*或*假*。当我们将进入条件或循环时，我们会经常使用它。布尔数据类型的关键字是 **`bool`**。

### 3.2.5 – 序列

我们将在后面的章节中学习序列。但由于*序列*也是数据类型，我们至少应该在这里提及它们。

| 数据类型 | 关键字 | 描述 |
| --- | --- | --- |
| 列表 | `list` | 值的集合 |
| 元组 | `tuple` | 不可变列表 |
| 字典 | `dict` | 键值对列表 |

## 3.3 – 什么是变量

变量不过是一个包含某些值的容器。每个变量都有特定的数据类型。但 Python 编程的好处是我们不需要在使用前声明变量或其数据类型。变量在我们给它赋值的那一刻就被创建。因为 Python 不是“静态类型”的。

## 3.4 – 创建变量

在 Python 中创建变量非常简单。你只需选择一个合适的名称，然后借助*赋值运算符* (`=`) 为其赋值。例如：

```python
myString = "Hello World!"
myInteger = 35
myFloat = 65.03
```

这里我们定义了三个变量。第一个是*字符串*类型，第二个是*整数*类型，最后一个是*浮点数*类型。基本上，你可以为变量选择任何你想要的名称，但存在一些限制。不允许使用任何保留关键字，如 `str`、`int`、`dict`、`if`、`for`、`while` 等。此外，变量名不能以整数或除下划线 (`_`) 以外的特殊字符开头。

## 3.5 – 使用变量

myString = "Hello World!"

这里你可以看到我们没有使用引号，因此圆括号内的文本被当作变量名处理。Python 解释器打印的是变量的值，而不是圆括号内实际写的文本。上述代码的输出如下所示。

## 3.6 – 类型转换

有时我们会得到某种数据类型的值，无法直接正常处理。为此，我们将这些值转换为我们期望的数据类型，这个过程称为*类型转换*。

例如，我们可能得到一个字符串作为输入，但字符串的值是数字。在这种情况下，“11” 不等于 11。即使字符串表示一个数字，我们也无法用字符串进行计算。因此，我们需要进行类型转换。

语法：

```
<所需数据类型>(表达式)
```

示例：

```python
value = "11"
number = int(value)
```

类型转换通过使用特定的数据类型函数来完成。在这个例子中，我们正在将一个*字符串*转换为*整数*。因此，我们需要使用 **`int()`** 函数。你也可以使用 **`str()`** 函数反向执行此过程。在 Python 中，这是一个非常重要的内容，我们在编码过程中会经常用到。

## 3.7 – 注释

Python 中的注释不过是代码中的一行或多行文本，解释器在执行脚本时会忽略它们。它帮助我们记录关于代码块的简短说明，并增强代码的可读性。Python 编程中有三种类型的注释。

**单行注释：** 如果我们在脚本中任何一行的开头放置一个井号 (`#`)，解释器会将该行视为单行注释。

```python
# 这是一条单行注释
print("Python 101")
```

我们也可以在任何语句后添加单行注释。例如，

```python
# 这是一条单行注释
print("Python 101")   # 我们也可以在这里添加注释
```

**多行注释：** Python 没有提供编写多行注释的单独方式。但是，有其他方法可以解决这个问题。我们可以在注释的每一行开头使用井号 (`#`) 来实现多行注释。

```python
# 这是一条
# 多行
# 注释
print("Python 101")
```**文档字符串注释：** Python文档字符串是出现在函数之后的带三引号的字符串字面量。它用于关联写在Python函数内部的文档，直接添加在函数下方以描述其功能。

```
def sum(a, b):
    """此函数接受两个参数并返回它们的和"""
    result = a + b
    return result
```

我们将在后面的 *函数* 章节中使用文档字符串注释。同时，无需过多思考上述函数的语法，我们将在该章节中学习。

## 4.2 – 赋值运算符

## 第六章

### 条件语句

决策在编程中与在现实生活中同样重要。Python 中的条件语句会根据条件求值为 *True* 还是 *False* 来执行不同的计算。这些条件在 Python 中由 if-else 语句处理。

## 6.1 – If-Elif-Else 条件

基本上，一个条件需要为 *True*，这样我们的脚本才会继续执行其代码块中编写的代码。这里三个重要的关键字是 **if**、**elif** 和 **else**。

**语法：**

```
if Condition:
    <Statement>

elif Condition:
    <Statement>

else:
    <Statement >
```

**示例：**

```
number = input("Enter a Number: ")
number = int(number)

if number < 20:
    print("Number is less than 20.")

elif number > 30:
    print("Number is greater than 30.")

else:
    print("The Number is in 20 - 30")
```

在这里你可以看到，我们从用户那里获取 'number' 输入，并将该数字转换为整数。然后我们的第一个 *if 语句* 检查给定的数字是否小于 20。请记住，条件 (number < 20) 总是返回 True 或 False。因此，如果条件返回 True，那么只有 **if 代码块** 会被执行。解释器永远不会检查其余的条件。

```
Output:
Enter a Number: 1
Number is less than 20.
```

现在假设给定的数字是 39。那么会发生什么？
我们的第一个条件将为 False，因为 39 不小于 20。所以，解释器会忽略 **if 代码块**，并继续检查下一个条件。当一个条件返回 True 时，解释器将只执行那个特定的代码块。在这种情况下，下一个 **elif 代码块** 将被执行。因为 39 大于 30。如果没有条件返回 True，解释器将只执行 **else 代码块**。

```
Output:
Enter a Number: 39
Number is greater than 30.
```

请记住，你可以使用无限个 *elif* 条件，但不能在 **if-else** 梯子中使用超过一个的 **if** 或 **else**。整个 *if-else* 条件集也被称为 *if-else* 梯子。
当然，你不需要 **elif** 或 **else** 代码块。你只需编写一个 **if** 语句，如果条件不满足，它就会跳过代码并继续执行脚本的其余部分。

# 第 8 章 – 序列

序列是 Python 最基本的数据结构。它包含多个元素，并通过特定的编号进行索引。在本章中，我们将讨论不同类型的序列及其函数。

## 8.1 – 列表

我们之前对 *列表* 有所了解。这里我们将详细学习。顾名思义，它就是项目的列表。

`Numbers = [2, 5, 6, 7, 1, 6]`

在 Python 中，我们使用 *方括号* 来定义一个 **列表**。然后将所有项放在这些方括号之间，并用逗号分隔。最有趣的是，Python 列表对数据类型没有限制。你可以使用任何数据类型，也可以混合使用。例如：

```python
names = ["Sam", "Alex", "Virat", "jack"]
ages = [22, 19, 25, 24]
mixed = ["India", 9, True, 98.3]
```

### 8.1.1 – 访问值

为了访问列表的值，你需要先了解索引。基本上，索引是列表中每个元素的位置。在 Python 中，索引从零开始。这意味着第一个元素的索引是 0，第二个元素的索引是 1，依此类推。现在，让我们尝试使用索引访问值。

```python
print(names[0])
print(ages[1])
print(mixed[2])
```

这里我们访问的是 names 的第一个项目（Sam）、ages 的第二个项目（19）和 mixed 的第三个项目（True）。要访问值，我们需要在变量名后面的方括号中放入索引。这是 Python 中访问值的基本语法。

```
Output:
Sam
19
True
```

在上面的例子中，我们只访问了每个列表的一个值。但如果我们想访问特定范围的多个值或整个列表呢？让我们看看下面的例子。

```
Example - 1:
print(names[0:3])
print(names[:3])
print(names[1:])
Output:
['Sam', 'Alex', 'Virat']
['Sam', 'Alex', 'Virat']
['Alex', 'Virat', 'jack']
```

这里你可以看到，我们传递了一个索引范围来访问值。在第一个 print 语句中，我们使用 [0:3] 来访问从第零个到第三个项目。也许有个问题会困扰你：我们传递了结束索引 3，为什么没有得到第四个项目？因为索引 3 对应的是第四个元素。

原因在于，我们在冒号后传递的（这里是 3）不是结束索引。它是第 n 个元素（这里，第三个元素是 “Virat”）。

**语法：变量名[起始索引 : 第 n 个元素]**

还有一件事要记住，如果我们不传递起始索引，它默认为 0（第二个语句）。同样，你可以不传递长度来访问列表的其余部分。

希望你理解了索引的概念。再看一些例子来更好地理解。

```python
# 返回整个列表

>>> [22, 19, 25, 24]

# 返回从第3个项目到列表末尾

print(names[1:3])

## 8.3 – 字典

Python 字典是 `键` 和 `值` 的集合。此序列中的每个条目都有一个键和一个对应的值。在其他编程语言中，这种结构称为 `哈希映射`。

```python
dict = {
    "name" : "Sam",
    "age" : 22,
    "dept" : "Programmer"
}
```

我们使用 `花括号` 定义字典，键值对用 `逗号` 分隔。键和值本身用 `冒号` 分隔。左边是 `键`，右边是对应的 `值`。

由于键类似于列表或元组的索引，它应该是唯一的。但值不一定是唯一的。我们可以让多个键对应相同的值，但当我们引用某个特定键时，该键名必须是唯一的。同时，键不能被更改。

有趣的一点是，我们可以将列表或元组作为值存储。

```python
dict = {
    "name" : ["Sam", "Alex", "Virat", "jack"],
    "age" : [22, 19, 25, 24],
    "dept" : ("Programmer", "Writer", "Photographer", "Editor")
}

## 函数

函数不过是一段语句块。有时我们需要在不同地方为不同值使用一些通用代码。而这会降低代码的可读性和美观性。这就是为什么我们使用*函数*。

*函数*允许我们只编写一次代码，然后多次使用。函数可以看作是组织好的代码块，我们在脚本的不同地方重用它们。它们使我们的代码更具模块化，并提高了可重用性。

## 9.1 – 定义函数

我们使用关键字 **def**，后跟**函数名**和**圆括号**来定义一个*函数*。冒号之后的代码需要缩进。

```python
# 定义函数
def greet():
    print("Welcome to Python 101")
```

这是一个非常基础的函数示例。这里我们定义了一个名为 **greet** 的函数，然后在函数内部我们只是打印了一段文本。但如果你现在运行这段代码，什么也不会发生。因为我们必须调用函数才能执行函数内部的代码块。

```python

```

我们可以在脚本中的任何地方调用函数。而且，我们可以在同一个脚本中多次调用函数。对于这个函数，每次调用它都会打印 *Welcome to Python 101*。

### 9.1.1 – 参数

在上面的例子中，我们看到函数 greet 只是打印相同的值。但我们可以通过使用参数使其动态化。这意味着我们可以通过传递不同的参数来获得不同的值。

```python
# 定义带参数的函数
def sum(num1, num2):
    print(num1 + num2)

# 调用函数
sum(33, 16)
# 输出：49
```

如你所见，我们在圆括号之间有两个参数 num1 和 num2。函数 sum() 打印这两个数字的和。当我们用两个参数 33 和 16 调用函数时，它给我们输出 49。现在你可以通过使用不同的参数来获得不同的输出。但要记住一件事，参数的数量必须与参数的数量相同。否则，它会抛出一个错误。

### 9.1.2 – 返回值

到目前为止，我们看到函数只是执行语句。但它也可以返回一个特定的值。在 sum() 函数的情况下，它也可以返回求和值，而不是自己打印出来。这个值可以被保存或处理。为了返回值，我们使用关键字 return。

```python
# 定义带返回值的函数
def sum(num1, num2):
    return(num1 + num2)
```

这里我们返回 num1 和 num2 的和，而不是打印它。

```python
# 调用函数 - 1
result = sum(33, 16)
print(result)

# 调用函数 - 2
print(sum(33, 16))
# 输出：
# 49
# 49
```

由于函数返回求和值，我们可以将其存储到变量中或直接使用。在任何情况下，输出都是相同的。

## 9.3 – Args 和 KWArgs

到目前为止，我们看到参数和实参的数量应该相同。但如果我们不知道用户将传递的实际实参数量怎么办？在这种情况下，Args 和 KWArgs 允许我们通过只定义一个参数来传递多个实参。

让我们用一个现实生活中的例子来理解。

假设我们正在收集员工的信息。所以显然姓名是必填的，但有些人可以跳过一些信息。所以，我们可以这样做，

```python
# 定义带 Args 的函数
def employee(name, *info):
    print(name, info)

# 用多个实参调用函数

# 输出：
# Sam (22, 'Kolkata', 'Programmer')
# Rio (20, 'Designer')
```

在这里你可以看到，我们定义了一个带有两个参数 name 和 info 的函数。但我们在 info 前面使用了一个星号 (*) 符号。因为 info 是一个可变参数。它可以以元组的形式存储多个值。

有趣的是，元素的数量没有限制。实际上，如果你仔细看输出，你会看到我们的第一个参数 name 是一个普通的字符串，但其余所有参数都显示为一个元组。

如果我们想访问 Args 的值，我们在打印时也需要使用星号 (*) 符号。比如，

```python
def employee(name, *info):
    print(name, *info)

employee("Sam", 22, "Kolkata", "Programmer")
# 输出：Sam 22 Kolkata Programmer
```

但这里所有的信息都不清楚。因为当我们为不同的员工调用函数时，它返回不同的结果。所以，很难决定哪个是年龄、地址或职位。在这种情况下，KWArgs（关键字可变参数或实参）帮助我们克服这个问题。

```python
def employee(name, **info):
    print(name, info)

employee("Sam", age = 22, address = "Kolkata", desig = "Programmer")
# 输出：Sam {'age': 22, 'address': 'Kolkata', 'desig': 'Programmer'}
```

我们用双星号 (**) 符号定义 *kwargs*。如果你注意到上面的例子，它返回一个键值对的字典。所以，现在我们可以应用字典函数或处理数据。

## 异常处理

程序充满了错误和异常。我相信你肯定遇到过一些，如果你没有，那你可能还不够努力！错误或异常是编程中每天都会遇到的事情。让我们看看下面这段代码。

```

```

如果我们尝试运行这段代码，它会抛出一个 **ZeroDivisionError**。这是因为除以零是未定义的，而我们的脚本不知道如何处理这种情况。因此，它会崩溃。让我们再看一个例子。

```
name = "Sam"

输出：
Traceback (most recent call last):
  File "c:\...\test.py", line 2, in <module>
    name = int(name)
ValueError: invalid literal for int() with base 10: 'Sam'
```

或者，当我们尝试将一个 *字符串* 转换为 *整数* 时，它会抛出一个 **ValueError**，脚本同样会崩溃。

## 10.1 – Try-Except

通过使用 try 和 except 块，我们可以轻松地处理这些类型的错误。try-except 的结构与 if-else 条件语句相同。但没有条件。我们将那些可能存在冲突、网络问题或其他类似问题的代码写在 try 块中。然后在 except 块中处理异常。

```
try:
    name = "Sam"
    name = int(name)
except ValueError:
    print("Code block for Value-Error")
```

语句必须至少有一个 except 块。但它可以有多个 except 块。

### 10.1.1 – Else 语句

我们还可以使用 else 语句。当我们的代码没有错误时，它将会被执行。

```
try:
    name = "Sam"
    # name = int(name)
except ValueError:
    print("Code block for Value-Error")
else:
    print("All ok")
```

首先，它会执行 try 块中的代码。由于没有错误，它会执行 else 块。

### 10.1.2 – Finally 语句

```
try:
    name = "Sam"
    # name = int(name)
except ValueError:
    print("Code block for Value-Error")
finally:
    print("It will execute always")
```

*Finally* 语句与 *else* 语句类似。它们之间基本且唯一的区别是：*else* 只在代码没有错误时执行，而 finally 无论代码是否有错误都会执行。

## 12.1 – 打开与关闭文件

### 12.1.1 – 访问模式

在Python中打开文件时，我们使用特定的访问模式。访问模式定义了打开文件的目的，例如读取或写入。Python中有多种访问模式。下表将对其进行更好的总结。

| 访问模式 | 描述 |
| :--- | :--- |

希望你对访问模式有了概念。现在该关闭文件了。当我们处理完文件后，需要关闭它，否则可能会引发一些不必要的错误。为此，我们只需使用`*close()*`函数即可。

```python
file = open('myfile.txt', 'r')

# 所有必要的代码

```

## 12.2 – With 语句

或者，我们可以使用`with`语句更有效地打开和关闭流。`with`语句会打开一个流，执行缩进的代码，然后关闭流。

```python
with open('myfile.txt', 'r') as file:
    # 所有必要的代码
```

如果你像我一样懒，`with`语句可以帮到你。因为我们不需要手动关闭文件流，它会自动完成。此外，使用`with`语句可以缩短代码并使其更具可读性。

## 12.3 – 从文件读取

一旦我们以读取模式打开了文件流，就可以开始从流中读取数据。为此，我们只需使用`*read()*`方法。

```python
# 以普通方式读取
file = open('myfile.txt', 'r')
print(file.read())
file.close()

# 使用 with 语句读取
with open('myfile.txt', 'r') as file:
    print(file.read())
```

这里我们以读取模式打开文件，并使用`read`方法打印文件的所有数据。

如果我们不需要全部数据，只需要文件的前50个或100个字符怎么办？我们可以通过向`read`方法传递一个参数来实现。

```python
with open('myfile.txt', 'r') as file:
    print(file.read(50))
```

这段代码将打印文件的前50个字符。

## 12.3 – 写入文件

向文件写入内容与读取文件非常相似。只需要改变两样东西：访问模式和方法。要写入文件，我们需要使用`*write()*`方法，而不是`*read()*`方法。

这里我们使用`with`语句来进行操作，你也可以两种方式都用。

```python
with open('myfile.txt', 'w') as file:
    file.write("Welcome to Python-101")
```

这里你可以看到，我们向`write`方法传递了一个字符串，该字符串将被写入我们的文件（*myfile.txt*）。你也可以向`write`方法传递一个变量，而不是直接传递字符串。

到目前为止一切似乎都很好。但有一个大问题：如果你每次运行代码时文件名都相同，它会覆盖该文件。因此，在写入之前你必须决定你想要什么。如果你每次都想覆盖，那就继续。但如果你不想丢失过去的数据，那么你应该使用追加访问模式，即‘**a**’。

```python
with open('myfile.txt', 'a') as file:
    file.write("Welcome to Python-101")
```

现在，即使数据相同，它也会将作为参数传递的数据追加到文件中。如果文件名相同，每次迭代的数据都会被追加到文件中。
