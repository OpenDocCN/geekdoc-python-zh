## 选择语言

初学者常遇到的一个问题是，在尝试弄清楚应该先学习哪种编程语言时感到无所适从。市面上有成千上万种选择，但没有一种“最佳”语言。请记住这一点：归根结底，语言本身并不那么重要。真正重要的是掌握知识、理解编程范式和设计模式。

当然，每种编程语言，甚至基础的脚本语言，都有其特性，这可能会让其他语言更容易理解。许多程序员从未正式上过课程，而是在整个职业生涯中自学每种语言。这通常通过复用已知概念，并借助文档和书籍来掌握其语言结构来实现。因此，与其纠结于先学哪种语言，不如选择你想要尝试的开发类型，然后直接从对你来说最容易上手的那种开始。

你可以为多种平台开发多种有趣的编程应用：Web开发、桌面开发、移动开发和命令行。

## 桌面脚本

在Windows或Mac桌面上开始编程最简单的方法，仍然是使用一种脚本或宏语言，例如AutoHotkey（适用于Windows）或Automator（适用于Mac）。当然，一些资深程序员可能会认为AutoHotkey或AppleScript不是“真正的”编程，因为这些工具本质上只是执行高级脚本。然而，对于那些只想动手实践、自动化桌面任务的新手程序员来说，使用这些免费工具是迈向“真正”编程的第一步。请记住，一个应用程序何时包含脚本，何时被视为编写计算机应用程序，其界限通常是模糊的。当你的代码被编译时，它就被视为“真正的”编程。大多数应用程序的最终用户通常对此一无所知，只要程序设计良好，并能以动态高效的方式满足其预定需求，他们就不必在意。

## Web开发

如果你不想局限于特定的编程语言和特定操作系统的架构与理念，那么可以考虑为整个网络环境开发你的应用程序，并将其作为Web应用程序发布给更广泛的受众。

HTML和CSS：要创建一个网站，你需要掌握超文本标记语言（HTML），它是构成网页的标记语言；以及层叠样式表（CSS），它是控制标记语言外观的样式信息。

HTML和CSS是构建语言，主要包含页面结构和样式信息。然而，在开始构建Web应用程序之前，你应该熟悉手写代码，因为构建基本网页是开发一个令人兴奋的Web应用程序的关键。在掌握了使用HTML和CSS构建静态页面之后，学习JavaScript就是在应用程序中构建动态网页的下一步。书签脚本、Greasemonkey用户脚本、Chrome Web Apps和Ajax都是使用JavaScript构建的。

服务器端脚本：一旦你能够熟练地在应用程序中构建动态网页，你可能就需要在后端添加一些服务器端逻辑。为此，你需要学习一种服务器端脚本语言。例如，要创建一个网页联系表单，将用户输入的信息通过电子邮件发送到某个地方，就需要一个服务器端脚本。像Python、Perl或Ruby这样的脚本语言可以让你的网站连接到数据库，因此，如果你想创建一个用户可以注册并保存数据的页面，这可能是正确的解决方法。

Web框架：为了避免为每个新的Web开发项目重复解决常见问题，许多程序员开发了开发框架，这些框架省去了反复编写相同代码来构建动态网页的繁琐工作。许多脚本语言都提供了特定于Web的框架，以简化常见的Web应用程序任务。Web开发框架包括：Ruby on Rails框架（用于Ruby开发者）、CakePHP（用于PHP程序员）、Django（用于Python工程师）和jQuery（用于JavaScript开发者）。

Web API：API（应用程序编程接口）是一种编程方式，允许软件的不同部分相互通信。例如，如果你想在网站上添加一个很棒的地图，你可以使用Google地图，而不是构建自己的自定义地图。Google Maps API使得通过JavaScript在网页中轻松集成地图成为可能。几乎每个现代互联网公司都使用API，允许你将信息和工具集成到你的应用程序中。这些包括：Twitter、Facebook、Google Docs、Google Maps等。通过API将其他Web服务集成到你的Web应用程序中，是增强富Web开发的宝贵资源。每个主要的互联网公司API都应该提供详尽的文档和多个快速入门指南。

## 命令行脚本

如果你想编写一个程序，接收文本或文件信息并输出有用的结果，那么命令行是合适的选择。虽然命令行脚本不像Web应用程序或桌面应用程序那样直观有趣，但对于编写自动化流程的短脚本来说，它是最简单直接的选择。

一些在基于Linux的服务器上运行的脚本语言同样适用于命令行，例如Perl、Python和PHP。学习其中一种语言将使你熟悉这两种环境。如果熟悉Unix是你的编程目标之一，那么你需要掌握shell脚本和Bash。Bash是*nix环境的命令行脚本语言，它可以完成从帮助你创建数据和文件的自动备份，到构建具有用户交互功能的完整应用程序等所有任务。

## 扩展功能

现代Web应用程序和程序可以通过插入额外的代码模块来扩展，以添加更多功能。随着越来越多的现代开发者审视现有程序和框架，并希望添加特定功能以使其更完善，附加开发正变得越来越流行。

即使只掌握HTML、JavaScript和CSS的基础知识，你仍然可以在任何应用程序中做很多事情。书签脚本、Greasemonkey用户脚本和现代用户样式都是用与构建传统网页相同的代码编写的，因此它们非常值得学习，即使你只是想用一小段代码来修改一个现有的网站。

## 工作区开发

进一步创建的软件具有更多功能，例如 Firefox 和 Chrome 的特性，能帮助你获取更多能力。要实现 Firefox 和 Chrome 的特性，你需要熟悉 JavaScript、XML 和 JSON——这些标记语言在功能上与 HTML 类似，但具有更严格的规范规则。

许多开源网络程序还提供了开发框架，例如 WordPress 和 Drupal。这两者都是用 PHP 编写的，使得这门语言成为开发的必备技能。

学习网络开发最初可能是一个艰难的过渡，但能帮助你从一个领域获取必要技能并应用到另一个领域，比如工作区软件开发。工作区开发编程将涉及操作系统（OS）、提供的软件开发工具包（SDK）的使用，以及跨平台开发的规划。运用网络开发的额外技能，也可以重新应用于通过网络分发你的工作区软件，以连接更广泛的用户群体。

## 计算机编程秘诀：轻松入门

PC软件是由一系列PC需要执行的指令创建的。这些指令可能伴随着执行这些指令所需的不同重要数据。因此，最常见的编程方式包括定义指令和数据。为了在PC上存储数据，你必须熟悉计算机编程基础和三个相关组件：

1.  必须执行的指令。
2.  执行指令的顺序。
3.  执行指令所需的数据。

计算机编程的主要部分基本上包括纸面步骤。在这个阶段，你不需要使用电脑工作，但如果你使用文字处理器，它将使你能够将作品写入磁盘文件而不是纸上。你需要这样做，以便在开始训练计算机执行之前，能够清晰、更明确地输入你希望计算机执行的内容。

请记住，计算机将完全按照给定的指令执行。它无法理解你想要什么；它只能做你让它做的事情。因此，在你的程序处理计算机之前，你需要完成一些步骤，包括：

-   定义问题

在你教计算机做什么之前，你必须首先熟悉它。你需要明确地向计算机说明它需要完成什么或创建什么，作为它将执行的所有活动的结果。

-   识别关联

如果你确实知道计算机将提供什么最终产品，你必须检查你拥有的数据，并确定你需要的数据。你还需要表示逻辑系统、条件和你需要应用的其他策略，以便将原始数据引导至你想要实现的最终产品。

-   映射关联

计算机编程中的这个步骤必须按适当的顺序展开。请记住，执行活动的顺序与实际活动本身同样重要。在将顺序安排成适当的顺序时，你可以考虑各种选择。

当你完成定义问题以及规划和设计关联后，接下来你需要做的首先是编写程序代码到一种或多种计算机语言的任务。但是，在继续之前，你必须首先确定你想要使用的编程语言，就像特定的计算机级别一样。

通常，编程语言更适合不同类型的任务。因此，首先选择最适合所需挑战的语言至关重要。当然，学习计算机编程基础是有抱负的程序员需要武装自己的第一步，特别是如果他希望在该领域寻求职业发展。

### 简单的学习曲线

已经掌握了使用 CSS 和 HTML 的基础知识？放心。你不会觉得学习这项新技术很困难。这种语言主要处理视图层。通常，Ember 和 Angular 被认为是那种不太容易学习的“距离表达语言”。因此，为了获得更简单、更快的学习体验，这种语言在普通设计师中非常受欢迎。

### 清晰的抽象

这是 React JS 的一个突出特点。你不需要遵循或掌握任何特定的架构（如 MVVM 或 MVC）来使用这种语言构建新的应用程序。只需理解组件的生命周期、属性和状态，你就可以完全掌握 React JS 语言来完成你的工作。按照你想要的方式构建应用程序的布局。

### 可重用的组件

组件就像乐高积木，React JS 是一种基于组件的语言。每个组件都按照其自身的内部逻辑工作。这对设计师有什么好处？嗯，这种语言允许重用这些组件。显然，这也带来了代码的重用。现在，当设计师可以重用代码库时，不仅处理代码变得非常容易，扩展它也是如此。

### 数据绑定

这种语言使用一种独特的架构 FLUX，通过一个称为调度器的单一控制点来管理流向组件的数据流。借助这个单向数据绑定元素，调试所有独立的顶级 ReactJS 应用程序变得更加简单。

在这个有限的范围内提及所有令人兴奋的组件确实不太可能。然而，除了所有上述令人信服的理由外，ReactJS 简单、高性能、采用本地方法、拥有出色的工具来测试应用程序等等。

因此，如果你试图构建任何大型或单页应用程序，可以选择这个开发平台。
