## 变量和数据类型

变量是源代码中的一个名称，它与内存中的一个区域相关联，你可以用该区域存储数据，然后在整个代码中调用这些数据。数据可以是多种类型之一，包括：

| 类型 | 描述 |
|---|---|
| 整数 | 存储整数 |
| 浮点数 | 存储小数 |
| 布尔值 | 可以是 True 或 False |
| 字符串 | 存储字符集合。"Hello World" 就是一个字符串 |

> "变量与内存中的一个区域相关联，你可以用该区域存储数据"

![](img/3303e52de17b2e01ada468e51372cd68_10_0.png)

此时，值得解释的是，Python 文件中任何跟在 # 字符后面的文本都会被解释器忽略。这样你就可以在代码中编写注释。

除了这些主要数据类型外，还有序列类型（从技术上讲，字符串是一种序列类型，但由于其非常常用，我们将其归类为主要数据类型）：

| 类型 | 描述 |
|---|---|
| 列表 | 包含按特定顺序排列的数据集合 |
| 元组 | 包含按特定顺序排列的不可变数据集合 |

元组可用于存储坐标之类的数据，其中包含作为单个变量存储的 x 和 y 值，而列表通常用于存储较大的集合。元组中存储的数据是不可变的，因为你无法更改元组中单个元素的值。但是，你可以在列表中这样做。

了解 Python 的字典类型也很有用。字典是一种映射数据类型。它以键值对的形式存储数据。这意味着你可以使用值对应的键来访问存储在字典中的值，这与使用列表的方式不同。在列表中，你会使用元素的索引（一个表示元素在列表中位置的数字）来访问列表的元素。

### 缩进详解

如前所述，缩进级别决定了代码块属于哪个语句。在 Python 中缩进是强制性的，而在其他语言中，则使用花括号来组织代码块。因此，使用一致的缩进风格至关重要。在 Python 中，通常使用四个空格表示一个缩进级别。你可以使用制表符，但制表符定义不明确，尤其是在多个编辑器中打开文件时。

## 控制结构

在编程中，控制结构是任何可以改变代码执行路径的语句。例如，一个控制结构如果决定在数字小于 5 时结束程序，它可能看起来像这样：

```
#!/usr/bin/env python2
import sys # 用于 sys.exit 函数
int_condition = 5
if int_condition < 6:
    sys.exit("int_condition must be >= 6")
else:
    print("int_condition was >= 6 - continuing")
```

代码执行的路径将取决于整数 int_condition 的值。只有当条件为真时，'if' 块中的代码才会执行。import 语句用于加载 Python 系统库；后者提供了 exit 函数，允许你退出程序并打印错误消息。注意缩进（在这种情况下每个缩进级别为四个空格）用于指示代码块属于哪个语句。'If' 语句可能是最常用的控制结构。其他控制结构包括 'for' 循环和 'while' 循环。

> "代码执行的路径将取决于整数 int_condition 的值"

## 关于 Python 列表的更多信息

Python 列表类似于其他语言中的数组。Python 中的列表（或元组）可以包含多种类型的数据，这在其他语言的数组中通常不是这种情况。因此，我们建议你只在列表中存储相同类型的数据。由于列表中数据的处理方式，这几乎总是应该如此。

## 函数和变量作用域

函数在编程中用于将过程分解为更小的部分。这通常使代码更易于阅读。如果设计得当，函数也可以是可重用的。函数可以接收传递给它们的变量。Python 中的变量总是按值传递，这意味着传递给函数的是变量的一个副本，该副本仅在函数的作用域内有效。在函数内部对原始变量所做的任何更改都将被丢弃。然而，函数也可以返回值，所以这不是问题。函数使用关键字 `def` 定义，后跟函数名称。任何可以传递的变量都放在函数名称后面的括号中。多个变量用逗号分隔。在这些括号中赋予变量的名称是它们在函数作用域中将具有的名称，无论传递给函数的变量叫什么名字。

作用域是需要掌握的重要内容，否则它可能会让你养成一些坏习惯。让我们编写一个快速程序来演示这一点。它将有一个名为 `cont` 的布尔变量，该变量将决定是否在 `if` 语句中将数字赋值给一个变量。然而，除了在 `if` 语句的作用域内，该变量在其他任何地方都没有定义。我们将通过尝试打印该变量来结束。

```
#!/usr/bin/env python2
cont = False

    var = 1234
print(var)
```

在上面的代码段中，Python 会在打印之前将整数转换为字符串。然而，显式地将内容转换为字符串总是一个好主意——尤其是在将字符串连接在一起时。如果你尝试对字符串和整数使用 `+` 运算符，将会出错，因为不清楚需要发生什么。`+` 运算符通常会将两个整数相加。话虽如此，我们之前演示的 Python 字符串格式化器是更简洁的方法。你能看出问题所在吗？`var` 仅在 `if` 语句的作用域内定义。这意味着当我们尝试访问 `var` 时，会得到一个非常严重的错误。

```
[liam@liam-laptop Python]$ ./scope.py
Traceback (most recent call last):
  File "/scope.py", line 8, in <module>
    print var
NameError: name 'var' is not defined
```

如果 `cont` 设置为 `True`，那么变量将被创建，我们可以正常访问它。然而，这是一种不好的做法。正确的方法是在 `if` 语句的作用域之外初始化变量。

```
#!/usr/bin/env python2

cont = False

var = 0
if cont:
    var = 1234

if var != 0:
    print(var)
```

```
class Player(Widget):
    def on_touch_down(self, touch):
        self.direction = (
            'right' if touch.x >
            self.parent.center_x else
            'left')

    def on_touch_up(self, touch):
        self.direction = 'none'

    def on_key_down(self, keypress, scancode, *args):
        if scancode == 275:
            self.direction = 'right'
        elif scancode == 276:
            self.direction = 'left'
        else:
            self.direction = 'none'

    def on_key_up(self, *args):
        self.direction = 'none'

    def update(self, dt):
        dir_dict = {'right': 1,
        'left': -1, 'none': 0}
        self.position += (0.5
        * dt * dir_dict[self.direction])
```

## 用Python替换你的shell

Python是一门很棒的编程语言，但你知道它甚至可以替换你的主要shell吗？

我们每天都使用shell。对我们大多数人来说，shell是进入我们Linux系统的门户。多年来，甚至直到今天，Bash一直是Linux的默认shell。但它有点过时了。

别生气：我们仍然相信Bash是目前最好的shell，与一些其他UNIX shell如Korn Shell (KSH)、C Shell (CSH)甚至TCSH相比。

本教程不是关于Bash无能，而是关于如何为shell注入全新的生命，以便方便地完成旧任务，并完成以前根本不可能完成的新任务。那么，事不宜迟，让我们开始吧。

虽然Python编程语言可能需要你编写更长的命令来完成一项任务（由于Python模块的组织方式），但这并不是特别需要担心的事情。你可以轻松地为你打算替换的Bash命令编写等效的别名。大多数时候，做一件事会有不止一种方法，但你需要决定哪种方法最适合你。

Python提供了直接执行系统命令的支持（通过os或subprocess模块），但只要可能，我们将在这里专注于Python原生实现，因为这允许我们开发可移植的代码。

## 解析 JSON 数据

JSON（也称为 JavaScript 对象表示法）是一种轻量级的现代数据交换格式。JSON 是 ECMA-262 下的一个开放标准。它是一种文本格式，完全独立于语言。JSON 也被用作现代应用程序（如 Mozilla Firefox 和 Google Chrome）的配置文件格式。JSON 在现代 Web 服务（如 Facebook、Twitter、Amazon EC2 等）中也非常流行。在本节中，我们将使用 Python 模块 'simplejson' 来访问 Yahoo Search（使用 Yahoo Web Services API），它输出 JSON 数据。

在本节中，我们将使用 `simplejson.load` 函数，该函数允许我们将 JSON 对象反序列化为 Python 对象。

要使用本节，你应该具备以下条件：

1. Python 模块：simplejson。
   注意：你可以使用命令 'easy_install <module name>' 安装 Python 模块。此命令假设你拥有可用的互联网连接。
2. Yahoo App ID：
   Yahoo App ID 可以从 https://developerapps.yahoo.com/dashboard/createKey.html 创建。Yahoo App ID 将在下一页生成。详情请参见下面的屏幕截图。

simplejson 非常易于使用。在下面的示例中，我们将使用将 JSON 数据结构直接映射到 Python 数据类型的能力。这使我们无需开发任何 XML 解析代码即可直接访问 JSON 数据。

JSON 与 Python 数据映射

| JSON | Python |
|---|---|
| object | dict |
| array | list |
| string | unicode |
| number (int) | int, long |
| number (real) | float |
| TRUE | TRUE |

```
@code: LUDSearch.py
import simplejson, urllib
APP_ID = 'xxxxxxxx' # Change this to your APP ID
SEARCH_BASE = 'http://search.yahooapis.com/WebSearchService/V1/webSearch'

class YahooSearchError(Exception):
    pass

def search(query, results=20, start=1, **kwargs):
    kwargs.update({
        'appid': APP_ID,
        'query': query,
        'results': results,
        'start': start,
        'output': 'json'
    })
    url = SEARCH_BASE + '?' + urllib.urlencode(kwargs)
    result = simplejson.load(urllib.urlopen(url))
    if 'Error' in result:
        # An error occurred; raise an exception
        raise YahooSearchError, result['Error']
    return result['ResultSet']
```

**grub.py**：GRUB 配置文件 (grub.conf) 解析器（可在 FileSilo 获取）。它实现了两个主要函数 `readBootDB()` 和 `writeBootFile()`，负责读取和写入 GRUB 配置文件。

```python
import sys
from snack import *

from grub import (readBootDB, writeBootFile)

def main(entry_value='1', kernels=[]):
    try:
        (default_value, entry_value,
         kernels)=readBootDB()
    except:
        print >> sys.stderr, ("Error reading /boot/grub/grub.conf")
        sys.exit(10)

    screen=SnackScreen()

    while True:
        g=GridForm(screen, ("Boot configuration"),1,5)
        if len(kernels)>0 :
            li=Listbox(height=len(kernels), width=20,
                       returnExit=1)
            for i, x in enumerate(kernels):
                li.append(x,i)
            g.add(li, 0, 0)
            li.setCurrent(default_value)

        bb = ButtonBar(screen, (("Ok", "ok"),
                               ("Cancel", "cancel")))

        e=Entry(3, str(entry_value))
        l=Label(("Timeout (in seconds):"))
        gg=Grid(2,1)
        gg.setField(l,0,0)
        gg.setField(e,1,0)

        g.add(Label(''),0,1)
        g.add(gg,0,2)
        g.add(Label(''),0,3)
        g.add(bb,0,4,growx=1)
        result = g.runOnce()
        if bb.buttonPressed(result) == 'cancel':
            screen.finish()
            sys.exit(0)
        else:
            entry_value = e.value()
            try :
                c = int(entry_value)
                break
            except ValueError:
                continue

    writeBootFile(c, li.current())
    screen.finish()

if __name__ == '__main__':
    main()
```

## 我们自己的部件：井字棋

由于Kivy没有井字棋部件，我们必须自己创建一个！创建一个新的部件类来包含这种行为是很自然的：

```python
from kivy.uix.gridlayout import GridLayout
class TicTacToeGrid(GridLayout):
    pass
```

现在这显然还没有做任何事情，除了它继承了Kivy GridLayout部件的所有行为——也就是说，我们需要告诉它有多少列，然后它会自动排列任何子部件，以尽可能多的行数整齐地适配。井字棋需要三列和九个子部件。

这里我们引入了Kivy语言（kv），一种用于描述Kivy部件树规则的特殊领域特定语言。它非常简单，但消除了用Python代码操作图形用户界面所需的大量样板代码——作为一个不太严谨的类比，你可以把它想象成Python的JavaScript对应的HTML/CSS。Python赋予我们做任何事情的动态能力，但如果我们只想声明图形用户界面的基本结构，所有这些能力反而会碍事。请注意，你永远不需要kv语言，你总是可以只用Python做同样的事情，但本教程的其余部分可能会展示为什么Kivy程序员通常喜欢使用kv。

Kivy自带了使用kv语言所需的所有工具；最简单的方法是将其写在一个基于我们的App类命名的文件中。也就是说，我们应该将以下内容放在一个名为“tictactoe.kv”的文件中：

```
<TicTacToeGrid>:
    cols: 3
```

这是kv语言的基本语法；对于每种部件类型，我们可以编写一条规则来定义其行为，包括设置其属性和添加子部件。这个例子演示了前者，通过声明每个实例化的“TicTacToeGrid”都应将其“cols”属性设置为3，为“TicTacToeGrid”部件创建了一条规则。

这继承自Kivy的“Button”部件，它与鼠标或触摸输入交互，在交互切换其状态时分发事件。我们可以接入这些事件，在用户按下按钮时调用我们自己的函数，并可以设置按钮的“text”属性来显示“X”或“O”。我们还为我们的部件创建了一个新的Kivy属性“coords”——我们稍后会展示这为什么有用。这几乎与在“GridEntry.__init__”中通过编写“self.coords = [0, 0]”来创建一个普通的Python属性相同。

与之前一样，此语法定义了“GridEntry”部件应如何构建的规则，这次设置了控制按钮标签中文本大小的“font_size”属性。额外的魔法在于，kv语言自动检测到我们引用了Button自身的高度，并将创建一个绑定来更新这种关系——当“GridEntry”部件的高度改变时，其“font_size”也会改变，以使文本完美适配。我们本可以直接从Python创建这些绑定（这是稍后使用的“bind”方法的另一种用法），但这很少像引用我们想要绑定的属性那样方便。

```python
import random

def tux_spawn(right):
    global tux_pos, tux_vel
    tux_pos = [0,0]
    tux_vel = [0,0]
    tux_pos[0] = w/2
    tux_pos[1] = h/2
    if right:
        tux_vel[0] = random.randrange(2, 4)

    global score1, score2
    tux_spawn(random.choice([True, False]))

    if paddle1_pos > (h - (pad_h/2)):
        paddle1_pos = (h - (pad_h/2))

    else:
        paddle1_pos += paddle1_vel
    if paddle2_pos > (h - (pad_h/2)):

    elif paddle2_pos < (pad_h/2):
        paddle2_pos = (pad_h/2)
    else:
        paddle2_pos += paddle2_vel
    canvas.draw_line([w / 2, 0],[w / 2, h], 4, "Green")
    canvas.draw_line([(pad_w/2), paddle1_pos + (pad_h/2)], [(pad_w/2), paddle1_pos - (pad_h/2)], pad_w, "Green")

    tux_pos[1] += tux_vel[1]
    if tux_pos[1] <= tux_r or tux_pos[1] >= h - tux_r:
        tux_vel[1] = -tux_vel[1]*1.1
```

07 在 Pivader 的类中，除了为对象创建所需的属性外，你还会注意到除了 Block 类之外，所有类都有一个 update() 方法（方法是类中的一个函数）。update() 方法在主游戏的每次循环中被调用，它只是要求我们创建的类的实例进行移动。对于 Ammo 类中的子弹，我们要求它在屏幕上向下移动。如果它移出任一端，我们就销毁它。

11 首先，游戏检查 end_game 属性是否为 false——如果为 true，则跳过 main_loop() 中的整个循环，我们直接进入 pygame.quit()，退出游戏。只有当玩家关闭游戏窗口或在 start_screen 上按下 Esc 键时，此标志才会被设置为 true。假设 end_game 和 start_screen 都为 false，主循环就可以正式开始了，首先是 control() 方法，它检查玩家的位置是否需要改变。接下来我们尝试制造一个敌方导弹，并使用 random 模块来限制可以创建的导弹数量。然后我们使用一个简单的 for 循环调用屏幕上每个角色的 update() 方法。这确保了每个人在我们检查 calc_collisions() 中的碰撞之前都已经更新并移动了。

## 弹药

08 然而，类最有趣的地方在于，你可以使用一个类来创建许多不同的东西。例如，你可以有一个宠物类。从这个类中，你可以创建一只猫（会喵喵叫）和一只狗（会汪汪叫）。它们在许多方面不同，但它们都是毛茸茸的，有四条腿，所以可以从同一个父类创建。我们正是用我们的 Ammo 类做了这件事，用它来创建玩家子弹和外星导弹。它们颜色不同，射击方向相反，但本质上是同一种东西。

## 主循环关键逻辑 – 第二部分

12 一旦计算了碰撞，我们需要看看游戏是否仍然应该继续。我们通过 is_dead() 和 defenses_breached() 来做这件事——如果这些方法中的任何一个返回 true，我们就知道需要返回到开始屏幕。另一方面，我们还需要检查是否已经杀死了所有的外星人，这在 win_round() 中进行。假设我们没有死，但外星人死了，我们知道可以调用 next_round() 方法，该方法会创建一批新的外星人并增加它们在屏幕上移动的速度。最后，我们刷新屏幕，以便所有移动、射击或被杀死的东西都可以被更新或从屏幕上移除。记住，主循环每秒发生 20 次——所以我们在循环结束时没有要求屏幕更新这一事实并无影响。

```python
#!/usr/bin/env python

# An advanced Hello World program that will demonstrate the basics of
# programming in Python via a series of examples. Created by Liam Fraser
# for Linux User and Developer :: 22/04/2012.

# Import the sys library for the sys.exit function
import sys
# Import everything from the Decimal library
from decimal import *

# Get the user's first name and output a welcome message
firstName = raw_input("Please enter your first name: ")
print("Welcome " + firstName + "\n")

# Ask the user for a number and square it, double it and halve it
number = raw_input("Please enter a number: ")
number = Decimal(number)
numberHalved = number / 2
numberDoubled = number * 2
numberSquared = number * number

# Print out the values we just worked out, converting each float value
# to a string
print("The result of halving that number: " + str(numberHalved))
print("The result of doubling that number: " + str(numberDoubled))
print("The result of squaring that number: " + str(numberSquared))
print("")

# The stopping condition for a while loop
yesOrNo = False

# A while loop that will run until a user enters either "yes" or "no"
while yesOrNo == False:
    result = raw_input("Do you want to continue? (yes/no) ")

    if result == "yes" or result == "no":
        yesOrNo = True
    else:
        print("Error, please type yes or no" + "\n")

# Deal with the result
if result == "yes":
    print("\nContinuing")
else:
    print("\nExiting")
    sys.exit()

# Create the count which will be a stopping condition for a while loop
count = 1

# Use a while loop to add 5 to the number and output the value each time
print("Incrementing the number by 5\n")
while count <= 5:
    number += 1
    print("number + " + str(count) + " = " + str(number))

    # Increment the count:
    count += 1

# Finish off by printing that we are exiting
print("\nExiting")
```

### 使用 While 循环和 If 语句进行输入验证

12 为了演示 while 循环和 if 语句，我们将向用户输出一个需要是或否回答的问题。我们将询问他们是否想继续——为此，我们需要小写的 'yes' 或小写的 'no'。while 循环是一个运行直到满足条件的循环。在这种情况下，我们将创建一个名为 yesOrNo 的变量，while 循环将在 yesOrNo 为 false 时运行。yesOrNo 变量将是一个布尔类型，可以是 True 或 False。该变量将被初始化为 False，否则 while 循环将不会运行。

while 循环的格式是 'while [条件]:' —— 任何属于 while 循环的代码都需要在冒号下面的行中缩进。任何未缩进的代码都不属于 while 循环。if 语句也是如此。条件使用比较运算符 '==' 进行检查。单个 '=' 是赋值运算符，而双等号是比较运算符。另一个常见的比较运算符是 '!=' —— 意思是 '不等于'。

我们创建一个名为 'result' 的变量，它保存问题的结果：你想继续吗？然后我们用 if 语句检查这个结果是否有效。注意 'or' 运算符，它允许测试两个条件。如果用户输入了正确的值，那么我们将 yesOrNo 设置为 True，这将在下一次运行时停止 while 循环。否则，我们将输出一条错误消息，while 循环将再次运行。用户可以随时在终端使用 Ctrl+C 命令退出程序。

```python
# 使用 while 循环将数字加 5 并每次输出该值

    print("number + " + str(count) + " = " + str(number))
```

## 嗯，甜甜圈

**05** 你将在 MCPiPy 集合中找到的预制脚本之一是甜甜圈：

```
python
from mc import *

def draw_donut(mcx,mcy,mcz,R,r,mcblock):
    for x in range(-R-r,R+r):
        for y in range(-R-r,R+r):
            xy_dist = sqrt(x**2 + y**2)
            if (xy_dist > 0):
                ringx = x / xy_dist * R # nearest point on major ring
                ringy = y / xy_dist * R
                ring_dist_sq = (x-ringx)**2 + (y-ringy)**2

    for z in range(-R-r,R+r):
        if (ring_dist_sq + z**2 <= r**2):
            mc.setBlock(mcx+x, mcy+z, mcz+y, mcblock)

mc = Minecraft()

playerPos = mc.player.getPos()

draw_donut(playerPos.x, playerPos.y + 9, playerPos.z, 18, 9, GLASS)
mc.postToChat("Glass donut done")
draw_donut(playerPos.x, playerPos.y + 9, playerPos.z, 18, 6, WATER_STATIONARY)
mc.postToChat("Water donut done")
```

## 处理多个任务

你的树莓派项目可能需要同时处理多件事情。学习如何处理多个任务

在本教程中，我们将探讨如何使用 Python 中的多任务功能来管理多个任务。在标准库中，有三个主要模块可用。它们是 threading、multiprocessing 和 concurrent。每个都有其自身的优点和缺点。由于这些都是标准库的一部分，因此你不需要安装任何额外的东西。

首先，我们将看看 threading 模块。你可以使用此模块的两种方式。第一种是使用它来创建新的线程对象，这些对象可以被告知在你的程序中运行某个目标函数。以下是一个简单的例子：

```
import threading
def my_func():
    print("Hello World")
my_thread = threading.Thread(target=my_func)
my_thread.start()
```

假设你的任务可以划分为单独的函数，你可以为每个函数创建一个线程。需要注意的一点是，这些新线程在你调用 start 方法之前不会开始执行函数代码。在那时，目标函数将开始在后台异步运行。你可以使用如下代码来检查给定线程是否完成：

```
if my_thread.is_alive():
    print('This thread is still running')
```

在你的程序主体中的某个时刻，你将希望使用在这些线程中运行的函数的结果。当这种情况发生时，你可以使用线程对象的 join() 方法。这会暂停你的程序核心并强制它等待直到线程退出。线程在运行函数退出时默认退出。

但是，如何编写使用线程的好代码呢？首先要考虑的是你将使用全局可用的数据，还是使用仅应在当前线程内可见的数据。如果你确实需要仅本地数据，你可以创建一个可以存储这些值的本地对象。以下代码存储一个包含作者姓名的字符串：

```
mydata = threading.local()
mydata.myname = 'Joey Bernard'
```

这将是在线程运行的函数内使用的代码。如果你需要使用全局数据，你需要考虑不同的线程可能如何尝试使用此全局数据。如果每个人都从给定变量读取，你不会遇到任何问题。问题出现在你有多个线程可能尝试写入给定变量时。在这种情况下，你最终会遇到一种称为竞态条件的情况，其中一个线程可能会覆盖另一个线程的数据。在这些情况下，你需要使用锁对象来管理对这些全局变量的访问。一个基本示例如下所示：

```
mylock = threading.Lock()
counter = 0
def func1():
    mylock.acquire()
    counter = counter + 1
    mylock.release()
```

如你所见，你在程序主体中创建锁对象。然后，在函数代码中，你尝试获取锁。如果锁是空闲的，你获得访问权并将其锁定。如果锁对象已被另一个线程锁定，那么对 acquire 的调用将阻塞并等待直到锁被释放。这就是为什么你需要非常小心，始终为每个 acquire 语句都有一个 release 语句。否则，你将有一个几乎不可能在事后找到的错误。这也为你的程序引入了一个瓶颈，因此你要确保 acquire 和 lock 之间的任何代码都是完成必要工作所需的最低限度。这是 Python 中可用的最简单的锁定机制形式。如果你的需求更大，你可以查看其他选项，看看它们是否可能提供更好的访问控制。除了控制对全局数据的访问外，你可能还需要在线程之间直接通信。这可以通过事件对象来处理，该对象可用于将标志设置为 true 或 false，并使其对其他线程可见。例如，下面的代码展示了如何设置和使用这样的标志：

```
event1 = threading.Event()
def func1():
    ....
    event1.set()
    ....
def func2():
    ....
    if event1.set():
        print('I got a flag from func1')
    ....
```

有时，你唯一需要的通信是知道所有线程何时完成了它们工作的某个阶段。假设你有多个线程加载数据文件，你需要等待每个人都完成才能进入下一阶段。在这种情况下，你可以使用屏障对象来实现。下面，你可以看到如何为两个线程（上面）添加屏障：

```
barrier1 = threading.Barrier(2)
def func1():
    ....
    barrier1.wait()
    ....
def func2():
    ....
    barrier1.wait()
    ....
```

在上面的代码中，你需要在创建屏障对象时设置有多少线程将参与其中。然后，当线程使用它并调用 wait 方法时，它们将阻塞直到所有线程都调用 wait 方法。threading 模块是一种轻量、快速且简单的方法，可以为你的代码添加划分处理的能力，但它确实存在一个主要问题。在 Python 核心引擎内部，有一个结构

### 使用蓝牙

01 蓝牙是一种无线标准，用于在设备之间通过一到十米的短距离交换数据。当前版本Bluetooth v5于2016年6月发布，范围增加到200多米。它于2017年发布，很可能成为许多物联网设备和应用的主流。蓝牙使用标准IEEE 802.11，与Wi-Fi相同。它们有相似之处，例如建立连接、传输和接收文件以及流式传输音频和媒体内容。如果你使用的是Pi 2或更低版本，你仍然可以通过使用蓝牙USB加密狗来创建和使用这些程序。
