## 第 1 章：入门

编程技能需求旺盛，且短期内没有放缓的迹象。无论是网络开发、机器学习还是物联网，编程无处不在。过去，人们在大学学习编程。现在，编程已经在学校里教给孩子们。很快，大多数领域都会期望具备一定的编程知识。与这些变化相伴的是，学习编程正变得免费、易得且简单。整个生态系统都在推动这些变化。

### 为什么选择 Python？

Python 由 Guido van Rossum 于 1991 年开发，他以自己最喜欢的英国喜剧《蒙提·派森》为该项目命名。时至今日，官方文档中仍有许多蒙提·派森的“彩蛋”。

自那时起，Python 因其直观的语法以及创建复杂程序和处理大量数据的能力，一直被用于向人们介绍编程。作为初学者，你将学习如何快速编写一个基础程序，以及如何轻松地将你的项目扩展为更具商业性的应用。

Python 易于阅读。它具有类似人类的英语结构，使其易于理解和记忆。你可以添加到代码中的库和预制函数的数量也在增加，从而节省你的时间。如果你认真学习并付诸实践，你将能够编写出十年后依然易于阅读的程序。作为一名程序员，你将了解程序维护的重要性，以及与其他语言相比，Python 在管理上的便捷性。让我们通过一个列表简要探讨学习 Python 的原因。列表易于理解和记忆。

- 1. 用户友好：编程语言允许人与机器之间进行交流。Python 是一种高级编程语言，这意味着它比机器语言更接近人类语言。机器本身并不理解 Python。有一个解释器将 Python 代码翻译成机器语言。
- 2. 功能强大：尽管易于学习，Python 的功能却极其强大。它与 C++ 等低级编程语言一样有用。谷歌、微软、IBM、施乐和 NASA 等公司都在使用 Python。你甚至可以使用 Python 来开发游戏。
- 3. 面向对象编程：面向对象编程是最有效的编程范式之一。它允许程序员将问题、动作和数据视为代码中表示的真实世界对象。然而，它并不总是最佳选择，这使得 Python 很有趣，因为它是一种多范式语言。这意味着你可以将其用于面向对象编程、函数式编程等。它很灵活。你可以在使用其他范式技术的同时受益于面向对象编程。
- 4. 计算机友好：Python 运行不需要大量的处理能力或内存。例如，开发者使用 Python 创建由 5 美元计算机操作的小型机器人。它也可以在任何操作系统上运行。这意味着你可以同时在不同的操作系统上开发你的应用程序，并且它仍然能正常工作。这也意味着你的程序将能够在多个平台上运行。
- 5. 语言适应性：你可以将 Python 与其他语言结合使用，以利用不同的特性和优势。例如，你可以将其与 C++ 结合，以利用其在后端提供的系统优化和速度。
- 6. 免费：Python 是免费且开源的。其许可证允许你以适合你目的的方式修改源代码。目前，这对你来说可能不重要，但以后可能会。
- 7. 社区：网上有一个庞大的 Python 社区。这是因为 Python 是开源的，应用广泛，且易于学习。所以，你会发现许多致力于学习或教授这门语言的人。如果你遇到困难，你会很容易找到帮助。你还会发现许多与志同道合者合作的机会。

为此，请访问 [www.python.org](http://www.python.org) 并转到“下载”部分。到达那里后，选择适合你系统的正确安装程序。确保你下载的是最新版本

版本也一样。

当安装程序下载完成后，运行它并按照步骤操作。不要更改任何设置。

我们建议你安装 Python 并习惯在你的系统上使用它。在线平台适合练习，但不适合构建项目。因此，在你的系统上工作将帮助你更好地准备。我们也建议你自己编写代码。如果你复制粘贴代码，请尝试修改它。这样做将帮助你巩固所学知识。

### 使用文本编辑器

你几乎可以在任何文本编辑器中编写 Python 代码。有很多选择。有像 Notepad、Notepad++、gedit 等程序。有些文本编辑器比其他的更好，因为它们提供了对程序员有用的功能。例如，Notepad++ 具有语法高亮功能，这意味着你将能够看到代码中的错误并修复它们。其他编辑器如 Notepad 则比较基础，这意味着在你尝试运行代码之前，你无法发现错误。尝试选择一个你觉得自然的文本编辑器。

避免使用文字处理器。

### 使用 IDE

IDE，即集成开发环境，是一个用于编写代码的程序，它附带了许多对编程重要的功能。许多 IDE 具有图形界面、自动完成功能、历史记录功能，并且集成了其他调试工具，可以运行命令、部署等等。使用 IDE 的优势在于便利性。IDE 有许多支持功能、快捷键、提醒以及关于如何修复错误的建议。

最受欢迎的 IDE 之一是 IDLE。IDLE 随 Python 一起提供。IDLE 可以在两种模式下运行：交互模式和脚本模式。当你希望 Python 立即响应你的代码时，交互模式是最佳选择。

市面上有许多 IDE。做一点研究，选择你最喜欢的一个。

### 你的第一个程序

现在你已经准备好了，是时候编写你的第一个程序了。在这个例子中，我们将使用 IDLE，但如果你愿意，也可以使用文本编辑器或在线 Python 控制台。我们建议你使用 IDLE，以便熟悉 IDE。

以交互模式运行 IDLE。它很容易找到。只需在你的系统上搜索它并点击即可。你看到的窗口称为 Python shell。会有一个闪烁的光标，表示你可以输入。输入以下内容：

```
print ("Hello World!")
```

下一行应该显示：Hello World!

就是这样！你已经编写了你的第一个 Python 程序，它显示了消息 "Hello World!"。正如你所看到的，代码是不言自明的。它读起来就像一个英语命令。这就是我们所说的 Python 易于阅读和理解。

我们使用 "print()" 函数来显示括号内用引号括起来的文本。如果你没有引号，你会得到一个错误。代码将无法运行。Python 也是区分大小写的，所以如果你将函数大写，它将无法工作。

在 IDLE 的交互模式下，你可以立即获得程序的输出，但缺点是你无法保存项目并在以后继续工作。为此，你需要使用脚本模式。要执行此操作，请单击 "File" 并选择 "New File"。它将带你到一个新窗口。再次编写代码：print ("Hello World!")。

你会注意到，当你按 Enter 键时，什么也不会发生。闪烁的光标跳到下一行。这是一个重要的功能，因为它意味着你可以编写更多指令。要运行代码，你需要单击 "File"，然后单击 "Save As"。给文件起一个你想要的名字。你会看到它有默认的 ".py" 扩展名。扩展名很重要。如果缺少 ".py"，解释器将不会读取/运行代码。单击 "Run" 来运行脚本。你会看到 IDLE 将在交互窗口中显示结果。

最终，你希望能够像运行其他程序一样运行你的程序。为此，你需要一个可执行文件，双击它即可运行代码。如果你现在这样做，一个窗口会打开然后关闭。看起来好像什么都没发生，但这是不准确的。程序已经运行并自行终止，速度太快以至于你没有注意到。你需要给它指令，使其保持打开足够长的时间以便你看到。在我们深入讨论之前，让我们先讨论一下代码注释。

### 代码注释与你的程序

打开我们刚刚开始的脚本并输入：

```
# Hello World!
# This is a demonstration of the "print" function.
```

当你运行程序时，你会看到它仍然显示相同的消息。你写的行没有出现。Python 忽略了它们。这些行被称为注释；它们总是以 # 开头，它们的存在是为了通过解释代码来提高代码的可读性。现在看起来可能没有必要，因为你理解你正在编写的代码。当你与他人合作，或者你的应用程序变得复杂，包含大量变量和函数时，情况就不同了。你的代码需要让你的合作者以及几个月后的你自己能够理解。你的合作者可以研究你的代码做了什么，但这不会是一次愉快的体验。这样做效率太低了。

如果你担心更多的注释会影响应用程序的性能，请不要担心。不会的。Python 总是忽略注释。

你可以通过留空行来使你的注释更具可读性。这不会对你的代码产生任何影响。在代码块和部分之间留空行是可以的；它们会被忽略。回到你的脚本并添加以下行：

```
input ("\n\n Hit the Enter key to exit!")
```

运行它，它将显示一个控制台，其中包含消息 "Hello World!"，然后显示 "Hit the Enter Key to exit!"。程序将等待你按回车键。这是保持程序打开的一种方式。

## 第 2 章：软件设计周期

在我们开始之前，值得花点时间了解一下软件设计周期。大多数编程书籍不讨论这一点，但理解程序是如何设计和制作的非常重要。随着你成为一名程序员，这将是一项不可或缺的技能。你还应该知道，我们即将讨论的很多内容在面试中经常出现。

我们将学习如何识别问题并使用编程来解决它。这个过程的第一阶段与设计应用程序有关。你将看到如何将解决方案分解为小的、可操作的步骤来解决问题。你必须这样做，才能看到我们在解决问题时认为理所当然的步骤背后的逻辑。我们将要讨论的所有问题都将帮助你学习这个过程。它也有助于记忆。在本章的第二部分，我们将运用所学知识编写代码。在那之后，你可以根据你的能力测试和完善程序。

我们还将学习伪代码，也称为设计语言。伪代码不向机器发出指令，而是作为指南。它是那些很少被讨论但重要的编程技能之一，能让你作为开发者更有条理。

### 解决问题

编程使用逻辑来解决问题。在你学习本书和其他资源的过程中，你会看到这一点。将有大量的练习和实践。编程就是解决问题，无论你是否在为工作构建应用程序。当你创建另一个程序的克隆版本以便改进它时，这一点仍然成立。现在，当你阅读这篇文章时，你可能心中有一些问题。也许这就是你拿起这本书的原因。也许你有一个游戏或应用程序的想法，想要与世界分享。要达到这些目标，你必须从问一些问题开始。为什么？因为应用程序可能很快变得非常复杂，你需要知道你在做什么，这样才不会迷失方向。不经过这个过程就编写应用程序，就像 J.R.R 托尔金在不问自己关于他所写世界的文化、历史、人民和特征的问题的情况下写《指环王》一样不切实际。如果你坐下来制定一个计划，你将更快地开发你的应用程序，因为你面临的问题更少。

可能性在于，无论你有什么想法，很可能已经有人构建过它了。关键不在于想出全新的东西。虽然你可以这样做，但关键在于问自己是否有什么可以改进的地方。这也让你的事情变得简单，因为你不必从头开始构建任何东西。你可以找到已经编写并共享的代码，然后对其进行优化。你可能认为这是作弊，但事实并非如此。我们今天使用的大多数工具都是这样构建的。人们在彼此的技术基础上进行构建，以创造出更美好的东西。

### 问题识别

在编码之前，你需要一个想法。为了快速找到一个想法，想想你经常做的事情。选择一些重复性的、你认为可以通过代码自动化的任务。编程总是非常适合处理平凡的任务。这就是你的第一个问题。下一步是找到一种方法，告诉计算机为你执行这个任务。计算机需要遵循什么指令——用Python编写——来解决这个问题？你还需要获得结果，以表明计算机正在做你希望它做的事情。现在让我们来了解一下基本的开发周期。需要记住的重要一点是，你不必对此过于严格。你可以根据情况调整方法。这些原则不一定是蓝图，所以你可以创造性地运用它们。

如果不考虑结构，开始可能会很困难，边做边添加解决方案和构建可能会导致不愉快的体验。以下是你应该采取的步骤：

1.  **软件的目的：** 写下问题以及你的程序应该做什么来解决这个问题。由于你是编程新手，从简单的东西开始。一个向用户打印消息的程序就足够了。一旦你知道你想做什么，你就必须清楚地思考如何解决这个问题。这是头脑风暴环节。你所有的想法不必完美，但你只需要认为它们可行，这样如果它们不行，你就会知道该做什么。
2.  **用户类型：** 接下来，你必须问自己你的目标用户是谁。即使你是在做个人项目，你也应该养成这样做的习惯。问问自己谁会觉得这个程序有用。是你的朋友还是家人？普通大众？你预期他们将如何与程序互动？他们有技术知识吗？他们会期待什么？一旦你问自己这些问题，你就会开始对你的程序有一个更清晰的图景。你不仅会知道它应该做什么，还会知道它应该如何做，以及在做的过程中它应该是什么样子。
3.  **计算机系统：** 然后，你需要预判你的受众的操作系统。他们使用的是Windows、Apple还是Linux电脑？有时你需要考虑他们更可能使用的设备。是Apple平板电脑还是Android手机？你不必太具体。你需要有一个想法，以便在编码时为其做一些准备。

所有这些问题都将指导你的应用程序设计。当你在项目开始时回答它们时，你会让前方的道路变得简单得多。如果由于某些原因你没有所有这些问题的答案，也没关系。其中一些答案会在你开发过程中出现。

### 寻找解决方案

你现在对你想要构建什么、它在哪里工作、它是为谁服务的以及它必须如何工作有了一个基本的概念。看起来你可能不需要再继续思考了，但仍然有工作要做。许多开发者花大量时间设计他们的程序，而不是实际编码。现在你需要问自己，你预期的目标用户的需求是什么。然后理解他们的需求。如果你是为特定的人构建的，这可能涉及你和客户之间的协议。一旦你理解了他们想要什么，你就需要思考一个解决方案。解决方案可能并不明显，它可能是脆弱的，也可能会变得复杂。尽管如此，如果你写下了你的过程，你将更容易调整你的策略并避免其他问题。你也会为自己节省大量时间。构建它只是为了发现你必须重新开始并改变它，因为没有人想要它，这是没有意义的。以下是一些此时需要考虑的事情：

### 开始程序

### 结束程序

所有伪代码都被注释掉，这样当你运行它时，Python会忽略它。如果你不注释掉它，你会得到错误。上面的例子向我们展示了程序的主要设计。它是对程序实现其目标所需的所有代码的描述。当你完成其中的每一项时，程序就会执行。

## 第三章：变量与数据类型

变量就像用来存储数据值的容器，这样计算机才能处理它们。没有变量，处理过程就无法进行，因为没有东西可供处理。数据值有不同的类型：列表、整数、字符串、浮点数、字典等等。在编程中，许多函数的功能都依赖于变量的使用。变量使我们能够编写操作，而无需知道函数将处理的具体数据。事实上，计算机会将大多数类型的数据转换为变量。

在本章中，我们将学习变量和数据类型。我们将看到变量在任何程序开发中所扮演的基础角色。我们将通过实际例子来学习，其中我们将操作字符串并执行数学运算。

### Python 标识符

在编码时，你需要能够通过标签来识别数据类型。标识符是未被注释掉或未放在引号内的标签/单词。标识符为事物贴上标签，使代码更具可读性，前提是这些名称与你的项目相关。不要对多个变量或函数使用相同的标识符；否则会发生冲突。因此，你的标识符必须对每个变量都是唯一的。

标识符允许你使用任何你想要的单词，除了 Python 保留的那些。Python 使用的那些是语言不可分割的一部分，被称为关键字。其中一些关键字是：False、lambda、class、import、global、while、yield、continue。随着学习的深入，你会逐渐理解这些。因为你不会总是记住这些关键字，所以当你使用其中一个时，Python 会警告你。

现在，让我们学习如何命名变量。变量应始终以下划线或字母开头。这不是一条规则；你可以打破它，你的程序仍然会运行。这是其他程序员出于良好原因建立的一种约定。只要你的标识符不是以数字开头，你可以在变量中使用数字。此外，Python 区分大小写，这意味着以下名称变量不会被视为同一个变量："newVariable"、"NewVariable"、"NEWVARIABLE" 和 "_newVariable"。Python 会将每一个都视为一个唯一的变量，尽管从语义上讲，它们是相同的。这意味着你需要注意大小写。经验丰富的开发者建议你在整个项目中坚持使用一种风格。所以，如果你的变量以大写字母开头，你应该在整个项目中遵循相同的规则。这可以防止你和其他程序员之间产生不必要的混淆。

### 变量简介

定义、声明或引入一个变量分两个阶段完成。你会发现这在大多数编程语言中都是如此。第一阶段是初始化，即创建一个带有适当标识符的容器。第二阶段是赋值，即将一个值放入容器中。这两个阶段在一次操作中完成。下面是一个例子。

```
variable = value
```

你通过等号执行初始化和定义赋值。
一段执行某些操作的代码块称为语句。赋值就是语句。表达式是计算以获取值/结果的代码。下面是一组赋值，最后我们有一个获取总价的表达式。

```
number = 0
product_weight = 2.1
price_per_kilogram = 10
filename = 'file.txt'
trace = False
sentence = "this is an example"
total_price = product_weight * price_per_kilogram
```

在编程中，你需要有条理。你应该将每条语句写在一行中。在上面的例子中，我们有一组类似于购物清单的赋值。在编写这样的程序时，你需要可以用来执行各种任务的变量。目前，理解变量有不同的类型，这些类型构成了一个程序。它们就像食谱中的配料。在我们开始探索类型之前，让我们讨论一下 Python 中的动态类型。

### Python 与动态类型

在 Python 中，变量的数据类型是自动推断的，这意味着你在编写代码时不必指定它。这被称为动态类型。大多数编程语言要求你指定你的变量是数字、列表还是字典。动态类型使我们不必那么冗长。它让我们能更专注于功能。
这样做的缺点是，当你犯某些错误时，比如将不应该相加的变量相加，Python 不会警告你。Python 会假设这就是你想要做的。初学者面临的一个棘手任务是跟踪他们的变量：它们在哪里被赋值，它们的值是什么，以及它们如何变化。做到这一点的一个技巧是在代码块的顶部声明你的变量。这样，它们都在同一个地方，你可以轻松地检查它们，从而轻松地跟踪你的代码。另一个技巧是创建一个设计文档。你也可以创建一个表格，在其中声明你所有的变量。了解并跟踪你的变量将使你能够轻松地调试和维护你的代码。

动态类型允许 Python 标记使用错误数据类型的变量。因此，如果你同时使用两者执行操作，你的操作将是无效的，这意味着你会收到一个 "TypeError"，如下所示：

```
x = 2
y = 'text'
trace = False
x + y Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: unsupported operand type(s) for +: 'int' and 'str'
y - trace Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: unsupported operand type(s) for -: 'str' and 'bool'
```

你不需要理解代码就能欣赏刚刚发生的事情。变量 x 是一个数字，变量 y 是文本。你不能用单词和数字进行计算。这就是你得到错误的原因。你可以将单词组合在一起形成句子，甚至可以将句子组合在一起。你也可以对数字进行数学运算。你不能做的是将两者混合。这个 TypeError 被称为回溯。它告诉你需要回去修复程序逻辑中的某些东西。这是 Python 会警告你的许多其他 TypeErrors 的一个例子。你需要确保你想要执行的操作不违反现实世界中关于它们应如何工作的规则。

在前面的例子中，我们有两种类型的变量：一个整数（int）和一个字符串（str）。字符串是文本或单词，整数是整数。这些是 Python 的一些数据类型。Python 提供了组合变量类型以形成更复杂数据类型的方法；我们将在后面探讨这一点。现在，让我们通过实践来构建我们的编程基础。

### 基本文本操作

字符串是任何编程语言中流行的数据类型。我们在制作 "Hello World" 程序时就用到了一个。在那个程序中，我们使用了一个 print 语句。那么如何打印多个语句呢？如果你使用我们在第一个程序中使用的方法，你只需在下一行添加另一个 print 语句。还有其他方法可以做到这一点，比如连接。连接是将字符串组合成一个。你可以通过几种方式执行连接。下面，它通过用逗号分隔变量来完成：

```
characterClass = "warrior"
characterGender = "male"
print (characterGender, characterClass)
```

输出将是 "male warrior."
你也可以使用字符串本身而不是保存它们的变量：

```
print ("male" "warrior")
```

输出将是 "malewarrior." 使用此方法时，你需要添加空格，以便输出中的单词之间有空格。你可以在第一个单词的末尾或第二个单词的开头添加一个额外的空格。以下是操作方法：

```
print ("male " "warrior")
或者像这样：
print ("male" " warrior")
```

输出将是 "male warrior." 然而，以这种方式连接字符串可能会导致问题。下面是一个例子：

```
print (characterGender "warrior")
```

结果将如下所示：

```
File "<stdin>", line 1
print (characterGender "warrior")
^ SyntaxError: invalid syntax
```

当你尝试以这种方式将字符串连接到变量时，你会得到如上所示的错误。你只能对两个独立的字符串执行此操作。
你可以使用加号运算符来组合字符串。在 Python 和大多数编程语言中，它被称为连接运算符，因为它组合字符串。它的样子如下：

x = "male"
y = "warrior"
x + y

输出结果将是 "malewarrior"。

Python 也使用加号进行数学运算。因此，以这种方式拼接字符串需要 Python 先评估变量，这意味着它需要做更多工作。所以不要不必要地使用这种方法，因为频繁使用会导致程序对系统要求更高。这是设计规划的一个例子。当你需要拼接多个字符串时，尽量寻找其他方法，只在找不到更好选择时才使用它们。

### 引号

在 Python 中，引号很重要。在之前的示例中你可能已经注意到了。引号将字符和单词标记为文本。字符被定义为单个字母、标点符号、数字或空格。文本实际上是一串字符。这就是为什么这种数据类型被称为“字符串”。Python 使用引号将文本与命令和关键字分隔开来。到目前为止，我们只使用了双引号，但 Python 提供了其他方式。

例如，你可以使用单引号或双引号编写 "Hello World" 程序。方法如下：

```
print("Hello World")
print('Hello World')
```

这两种写法会产生相同的结果，因为 Python 不会区分它们。所以你可以选择你喜欢的任何一种。那么，如何测试包含多行的文本块呢？Python 允许你使用块引号，保留所有的换行和空格。方法如下：

```
x = ''' this is a long line of text
    that requires multiple lines'''
y = """ this is another long line of strings
    but this time, we're using double quotes
    because they're cooler """
```

这两种方法会产生相同的结果，无论使用单引号还是双引号。Python 会以相同的方式读取它们，所以这完全取决于你的偏好。

那么，如何在字符串内部使用引号呢？为此，你必须使用不同的引号。例如，字符串用双引号，被引用的内容用单引号。像这样：

```
x = "She told me to 'quote' like this"
```

在上面的例子中，我们有一个引号层级，双引号构成字符串，单引号标记被引用的文本。在单引号内，我们有一个子字符串；在双引号内，我们有一个主字符串。这被称为嵌套引号。

### 数字

数字的赋值方式与字符串或任何其他数据类型相同。这是一个例子：

```
x = 5

```

Python 能够识别它处理的是数字，因为没有引号或字母。在这个例子中，是一个整数。在我们继续之前，你需要对计算机的工作原理有一个基本的了解。

计算机以二进制形式存储数据，即一串 1 和 0。信息是通过触发大量可以开或关（1 或 0）的开关来处理的。关于这个主题还有很多需要学习，但这些信息对于我们的目的来说已经足够了。

让我们看看一些数字类型：

- 1. 整数：在编程和数学中，整数被称为整数。整数没有小数点，但可以是负数或正数。整数在运算中经常使用。

- 3. 布尔值：它们是最基本的类型，代表了计算机的工作方式。布尔值可以是 True(1) 或 False(0)。它们经常与逻辑运算符如 "and"、"or" 和 "not" 一起使用。我们稍后会详细探讨这一点。变量使用 True 和 False 关键字赋值布尔值。它们总是大写的；否则将无法工作。

### 基本运算

你已经看到了我们如何将值赋给变量。现在我们将执行一些基本运算，如加法 (+)、减法 (-)、乘法 (*) 和除法 (/)。当你执行这样的运算时，你就创建了一个表达式。表达式是计算机必须处理以获得结果的代码。表达式可以赋值给变量。下面是一个例子：

### “While”循环

"while"循环首先检查一个条件是否为真。只要条件为真，就会执行一条语句。因此，它检查条件是否为真并执行，然后再次检查并执行，如此循环。以下是一个示例：

```
x = 1

    print(x)
    x += 1
```

这个循环检查x是否小于十，然后执行代码。它检查x是否仍然小于十，并继续执行，直到x不再小于十。在执行过程中，x的值每次增加一。循环中代码的每次执行都称为一次迭代，因为它被重复多次。

在创建"while"循环时，你需要注意两件事。循环检查的变量需要在循环运行前初始化，并且该变量需要在每次迭代中更新/更改，否则你将得到一个无限循环。无限循环可能是危险的，Python不会告诉你何时创建了一个无限循环。如果你发现程序陷入无限循环，你需要终止它，但这在某些情况下可能不起作用。无限循环不会损坏你的计算机，但它们会占用并耗尽系统资源，导致系统变慢或重启。在某些罕见的情况下，无限循环是有帮助或需要的。但作为初学者，最好避免它们。

我们的输出显示得到了改善。但如果我们使用这些测量值去购买布料，最终会有点短缺。因此，我们需要知道预期的值，以确认程序运行良好。如果不运行测试，就无法判断我们的计算是否准确和有用。虽然我们可能觉得目前的输出有些冗长，但将我们的数字与预期值进行比较可以帮助我们找出错误所在。花些时间检查程序，运行更多测试，并找出哪些地方不工作。需要进行一些微调。我们该如何实现呢？将为正值。为此，我们需要一个包含哨兵值的循环。哨兵值用于在检测到特殊值时向程序发出中断循环的信号。程序需要检查用户输入的值是否为脚本可接受的值。例如，只要存在正值，循环就会持续迭代，当检测到负值时停止。下面是一个示例：

```
offset = 0

    x = int(input("Enter a positive value: "))
    total += x
    offset += 1

print(mean)
```

另一个问题是，我们不知道用户输入值的类型，因此应该自动将其转换为我们可以处理的格式：整数。如果用户输入字符串，程序将因值错误而终止。

循环的另一个重要方面是如何退出它们。`break` 和 `continue` 方法帮助我们无问题地跳出循环。例如，当满足某个条件时，你可能希望退出循环而不检查其他条件。在这种情况下，你使用 `break`。如果你想跳过特定的迭代，可以使用 `continue`，循环将进入下一次迭代。

在某些情况下，你希望程序验证一个条件而不采取任何指定的操作。为此，你使用 `pass` 关键字。`pass` 关键字形成一个空语句，意味着程序将跳过循环并移动到下一个代码块。下面是一个示例：

```
while True:
    my_input = int(new_input("$? : >> "))
    if my_input > 0:
        pass
    else:
        print("This input is negative")
```

现在让我们看一个使用 `break` 和 `continue` 的例子：

Python 允许你嵌套多个循环和条件语句。有趣的是，你可以有无限层级的循环结构，但不建议这样做。当你嵌套太多循环时，一旦发生错误，你可能会很快感到困惑。从众多选项中知道程序应该遵循哪个条件是困难的。所以如果可以的话，尽量保持简单。缩进也会使事情难以阅读。每个循环和条件都需要缩进，从而创建大量复杂的块。对于任何程序员来说，无论技能水平如何，它看起来都会很杂乱。一般规则是，如果你需要超过两层循环，请检查你的程序是否有其他解决方案。根本不需要这种复杂性。简单的解决方案正等着被发现。找到它。

### “For” 循环

循环也可以用 `for` 语句创建。它与 `while` 循环具有相同的结构，并且在大多数情况下，你可以互换使用它们。但 `for` 循环最适合用于遍历序列，如列表、字典和字符串。在第一次迭代中，检索序列中的第一个元素，然后是第二个，依此类推。
要理解 `for` 循环，你需要了解一些关于序列的基础知识。我们将在下一章中更多地讨论序列，但我们已经处理过一个：字符串：字符串只是字符的序列。还有列表，它们是被操作的元素序列。还有元组，它们类似于列表但不能编辑。下面是一个 `for` 循环的示例：

```
buildings = ["store", "house", "hospital", "school"]
for x in buildings:
    print(x)
```

列表中的每个元素都将被打印出来。你也可以遍历字符串，像这样：

```
for x in "store":
    print(x)
```

“store” 中的每个字符都将被单独打印出来。
我们可以像中断 `while` 循环一样中断 `for` 循环。下面我们在列表中找到 “hospital” 时停止循环。

```
buildings = ["store", "house", "hospital", "school"]

    print(x)
    if x == "hospital":
        break
```

如你所见，`break` 发生在 “hospital” 被打印之后。下面是另一种情况：

```
buildings = ["store", "house", "hospital", "school"]

        break
    print(x)
```

在这个例子中，循环在 “hospital” 被打印之前中断。接下来是 `continue` 关键字，它将跳过指定的迭代，在本例中是 “hospital”：

```

如你所见，除了 “hospital” 之外，每个元素都被打印出来了。

### 序列

正如我们所说，字符串、列表和元组是序列的例子。有多种方法可以访问序列内部的数据。首先，我们应该分析数据结构。可以修改的数据结构称为可变的。它们具有可变性。字符串和元组不能被修改，所以它们是不可变的。但我们可以使用元组和字符串来形成新的。列表是可变的，因为我们可以添加和删除它们包含的元素。

那么，我们如何访问序列中的元素呢？我们可以使用项目的索引。索引表示元素的位置。索引写在方括号内的整数。下面是一个示例：

```

由于索引从零开始，“hospital” 位于第二个位置。从零开始的索引是编程中的一个常见特征。这就是为什么 2 访问列表中的第三个元素。你也可以使用负整数来访问项目：

```

"school"
```

负索引从列表或字符串的末尾开始计数。
如何同时访问多个元素？你可以使用切片来完成。你可以通过输入两个索引来切片序列的一部分。第一个索引表示切片应从哪里开始；第二个选择切片应在哪里结束。所以像 [1:4] 这样的切片意味着切片将从第二个位置开始，到列表中的第五个位置结束。你也可以反向访问它们。你也可以忽略输入第一个元素。Python 将假设它应该从第一个位置的元素开始。如果你留空第二个值，也会发生同样的事情。

```
[:3]
[3:]
```

切片方法创建了一个新的序列，这意味着原始数据不会被更改。你将能够操作和处理原始列表中的元素。因此，如果你编辑父序列和切片序列中的一个元素，它们都会改变。编辑与切片不同，因为它会改变序列。

### 字典

字典的工作方式与黄页相同。只要你有名字，你就能收集到关于这个人的所有其他联系信息。在字典中，名字是键，信息被称为值。键是不可变的，它们只能是字符串、元组或数字。值可以是任何支持的数据类型。虽然键不可变，但字典本身是可变的，这意味着你可以编辑它们。字典也被称为映射，因为键被映射到特定的对象。

如果你来自另一种编程语言，你会知道像字典这样的数据类型被称为哈希或关联数组。它们之间存在差异，因为决定键的值是哈希值。这是什么意思？哈希键可以存储在更大的键中，因为它们是从键定义的小值。这可能令人困惑，但你应该理解的是，Python 通过哈希键而不是字母顺序来排列字典数据。所以，如果你改变一个值，由此产生的哈希值会被修改，程序将找不到该值。这就是为什么键不能被更改。

## 第6章：函数

### 全局变量

我们使用的函数彼此封闭且独立。这就是我们喜欢它们的原因。它们允许我们移动它们，并将它们与我们喜欢的任何东西连接起来。要访问它们内部的数据，我们需要使用参数。为此，它们应该返回一个值。这就是大多数情况下的做法。但还有另一种方法。

我们可以使用全局变量在应用程序的不同部分之间共享数据。要理解什么是全局变量，我们需要理解作用域。

### 作用域

正如我们所看到的。将程序划分为可以相互独立访问的部分是可取的。这就是我们所说的作用域。我们创建的每个函数都会创建一个作用域。这就是为什么一个函数不能直接访问另一个函数的变量。这是一个简单的表示：

```python
def function_1():
    variable_1
### 第一个函数作用域的结束

def function_2():
    variable2
### 第二个函数作用域的结束

### 这代表整个应用程序的全局作用域
variable()
```

在上面的示例中，我们有三个作用域。默认情况下，程序有一个全局作用域。在我们的示例中，我们有两个由两个函数定义的作用域。但这一切意味着什么？

当你没有在内部做任何事情时，这两个函数都在全局作用域内。在此空间中定义的变量称为全局变量。在函数块内定义的任何其他变量属于该函数，称为局部变量。在我们的示例中，"variable_1"在第一个函数内部，使其成为局部变量。因此，我们无法从另一个作用域（如第二个函数或全局作用域）访问它。我们甚至无法修改它。

作用域就像一辆装有有色玻璃的汽车。如果你在车内，你可以看到车内和车外的一切。但如果你在车外，你就看不到车内的任何东西。函数就是这样工作的。这些变量在它们的函数（汽车）内部，从全局作用域（车外）是看不见的。在我们有许多属于不同函数的同名变量的情况下，它们将包含不同的数据，并且彼此不可见，不会引起命名冲突。因此，如果我们在第一个函数中有"variable2"，在第二个函数中也有"variable2"，它们将是不同的，持有不同的数据。

### 处理全局变量

我们将编写一个简短的程序来说明函数如何读取和编辑全局变量，然后我们将分析代码。下面是代码：

```python
### 使用全局变量
def read_global():
    print("在read_global()内部，值是：", value)

def shadow_global():
    value = -10
    print("在shadow_global()内部，值是：", value)

def change_global():
    global value
    value = -10
    print("在change_global()内部，值是：", value)
```

### 读取全局变量

我们已经了解了局部变量和全局变量，并讨论了作用域。现在，是时候让你稍微困惑一下了。无论你处于哪个作用域，都可以读取全局变量。你可以在全局作用域内读取函数内部的变量，反之亦然。这就像我们之前提到的汽车有色玻璃。当你在车内时，你可以看到车内的一切，但也能看到外面的景象。因此，函数可以读取全局变量。

我们之前编写的程序说明了可读性。我们定义了一个名为 `read_global` 的函数，它打印全局变量的值。它运行时没有任何错误，但有一个问题。你可以读取这些变量，但不能直接对它们进行任何更改。因此，如果你尝试像下面这样操作，你会得到一个错误：

```
value += 1
```

### 遮蔽全局变量

查看上面的例子，你可能想知道“shadow”代表什么。遮蔽（Shadowing）是指给一个局部变量赋予与全局变量相似的名称。但你仍然不能编辑全局变量。你所做的任何更改都发生在局部变量上。局部变量是全局变量的影子，因此称为“遮蔽”。以下是我们如何为 `shadow_global` 函数赋值：

```
value = -10
```

全局值没有改变。但在函数内部创建了一个相同的局部版本，现在该版本的值为 -10。你可以看到实际效果，因为我们已经告诉程序打印全局版本。

但请记住，像这样的遮蔽并不总是个好主意。为什么？因为它们令人困惑。你可能在查看程序时试图弄清楚哪个是哪个时就经历过这种情况。

### 更改全局变量

你可以使用全局 `global` 关键字来访问全局变量。在我们的例子中，我们用它来将值更改为 -10。当执行该值时，将打印新值。

那么，我们什么时候应该使用全局变量呢？事实是，你可以随时使用它们。但这明智吗？仅仅因为你能做某事，并不意味着你应该做。你需要权衡利弊，并问自己这是否适合你的项目。全局变量通常是有问题的，因为它们可能令人困惑。只在绝对必要时使用它们，当你找不到更好的替代方案来实现你想要的目标时。

或者，你可以将全局变量转换为常量，这样就不难跟踪了。假设在一个税收计算器中，你将有许多函数，但它们都使用 .45 作为值。你应该将该值放在一个全局常量中，这意味着我们不再需要在每个函数内部调用该值。这看起来可能没什么，但在处理复杂项目时会产生很大差异。拥有清晰的变量而不是可能任意的数字，更有意义且更容易处理。

## 第7章：面向对象编程简介

Python被认为是一种面向对象编程语言。这是什么意思？OOP，简称，是一种新的编程范式，已成为软件和游戏开发的标准。我们今天使用的大多数程序都使用OOP。OOP被定义为对象。在本章中，我们将学习构成OOP的概念。我们将探讨：

- 1. 定义对象和创建类。
- 2. 编写方法和声明对象的属性。
- 3. 多态性。
- 4. 创建模块。

### 类

类是OOP的基础。本质上，类是构建你自己的数据类型的模板。为了更容易理解这一点，不要将数据类型视为数字或文本。想想像汽车这样的东西，它可以是一个具有品牌型号、颜色等属性的数据类型。

类允许你创建可以在项目中使用的对象。它们是编程中最基本的概念之一。它通常是大多数编程课程的最后一步。数据中的内容代表属性，语句/命令是方法。

理解起来可能有点困难。但OOP使冗长、困难的程序变得更容易，因为它减少了你需要做的工作。它允许你重用几乎所有的代码。对象可以从其他类继承属性和方法，从而更容易修改和创建新对象。因此，我们可以有一个通用对象，并构建一个从它衍生出来的整个家族。最终，我们会得到类似家谱的东西。

假设你有植物、动物和矿物作为基础类。它们将具有传递给属于它们的任何对象的属性。类允许你创建子类。例如，动物父类有鸟类和鱼类作为子类。鸟类和鱼类将从动物类继承一些属性，这使其与其他类区分开来。除此之外，Bird 和 Fish 也会有各自的属性，使它们与众不同。你可以创建另一个子类，比如 Chicken 或 Raven 子类，它继承自 Bird 类。假设我们需要创建一个名为 Charlie 的 Raven 类实例。Charlie 将继承 Raven 的所有属性，比如 "caw()" 方法。因此，当我们编写 "charlie.caw()" 时，会得到字符串 "Caw! Caw!" 作为输出。

对象、类和方法类似于我们思考世界的方式。这就是为什么开发者青睐面向对象编程。它允许他们进行直接的思考。

### Python 命名空间

在深入研究类之前，我们需要先学习如何定义类。这就是为什么我们必须先了解命名空间，以便更深入地理解面向对象编程。

命名空间将名称映射到对象。默认情况下，命名空间是 Python 字典。命名空间的一个例子是包含 "abs" 等函数的默认名称集合。命名空间包含模块名称或局部函数调用。类的属性构成其自己的命名空间。属于多个命名空间的相同名称不会以任何方式相互影响。让我们通过模块来更好地理解这一点。两个定义了同名相同函数的模块可以共存，但它们不会让 Python 感到困惑。Python 将它们视为独特且独立的。

一旦你运行解释器，Python 就会创建命名空间，但有些命名空间在不同时间处于活动状态，包括最后时刻。例如，Python 的内置命名空间永远不会被删除。解释器读取一个模块时，会附加一个全局命名空间，但当解释器关闭时，它就会消失。

虽然特定的命名空间是预定义的，但我们不能从任何地方访问它们。这是因为作用域。在这个上下文中，作用域指的是我们无法直接访问的应用程序部分。我们上次讨论作用域时，有两个作用域：全局和局部。在这个上下文中，我们有三个作用域。第一个是具有局部命名空间的函数的作用域。第二个是创建全局命名空间的模块的作用域。第三个是包含所有预构建命名空间的通用作用域。

### 定义类

让我们看看类的语法。下面是一个例子：

```
class Raven:
    <myStatement1>
    <myStatement2>
    <myStatement3>
```

定义类与定义函数有些相似。类定义在我们看到任何结果之前执行。类内部的语句通常包含函数定义。

定义类时，我们使用 "class" 关键字，后跟类的名称。类名以大写字母开头。因此，类的命名与变量和函数的命名不同。开发者遵循这个标准，但这并不是 Python 本身的运作方式。如果你忽略使用变量命名约定，不会出错。但这仍然是个坏主意，因为公司和其他开发者讨厌这样，原因我们在此不深入探讨。

现在，我们使用创建的类来构建一个对象：

```
myObject = Raven()
print(myObject.x)
```

我们创建了一个名为 "myObject" 的对象，并打印了属于它的值 "x"。

### 定义方法

对象可以包含方法，也称为函数。下面是一个例子：

```
class Man:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    def myfunction(self):
        print("Hi, I am " + self.name)

x = Man("Thomas", 25)
x.myfunction()
```

注意，"self" 参数指的是类的当前实例。"self" 参数用于访问属于它的任何变量。要知道 "self" 不是一个关键字，而是属于类的函数的关键第一个参数。

你可能还注意到我们在这个例子中使用了名为 "__init__" 的函数。这个函数被称为构造函数。Python 还有其他构造函数，它们的特点是行首和行尾有下划线。

基本的类和对象在实际应用中用处不大，但像 "init" 这样的函数可能赋予它们意义。该函数在类初始化时执行。它用于为对象分配多个值。我们也可以用它在对象创建期间分配其他重要操作。

### Python 继承

继承是一个重要的编程概念。当类继承父类的功能时，就会发生继承。子类称为子类，父类称为父类。让我们看看语法：

```
class MainClass:
    #add functions and statements

class ChildClass(MainClass):
    #add functions and statements
```

继承允许你在子类中添加或修改功能。
Python 中有几种继承类型：

- 1. 单继承：这是我们到目前为止所做的。当一个子类从一个父类继承时，就会发生这种情况。
- 2. 多重继承：多重继承是指你创建一个子类，它可以继承多个父类的功能。语法变化不大；父类用逗号分隔。
- 3. 多级继承：当一个子类下面有另一个类继承其类功能时，就是多级继承。它也被称为孙类，因为子类成为第二个类的父类，而父类成为第二个子类的祖父类。
- 4. 层次继承：当一个父类创建多个子类时，我们就有层次继承。
- 5. 混合继承：当继承类型组合时，我们说我们有混合继承。

### 多态

多态意味着具有多种形式。在编程中，它指的是语言根据类或数据类型处理对象的能力。对象可以重新定义继承的方法，并为它将处理的数据类型调整代码。
多态是一个有些高级的主题。所以如果你还不太明白，没关系，但让我们通过一个类比来解释。想象你给朋友现金让他为你买午餐。如果你告诉一个 11 岁的孩子为你做同样的事，他会骑自行车给你买汉堡和薯条。但你的朋友可能会开车给你买素食三明治和葡萄。这就是多态：你有一个目标，不同的对象可以用不同的方式完成。当我们谈论同样的事情但用编程术语时，请记住这一点。

多态之所以重要有两个原因。它允许对象根据其参数执行方法的不同实现。其次，为一种数据类型编写的代码可以用于另一种相关数据类型。理解这一点对于掌握 Python 很重要。

多态的另一个类似于现实世界的特征是，它赋予语言穿越类层次结构的能力。这意味着它的性能是它习惯的，因为它过去学过。

这里有一个类比给你：就像读这本书。你知道如何读它，因为你在年轻时学会了如何读书。用面向对象编程的术语来说，你是一个可以与书交互、翻页并将字母转换成有意义的单词的对象。但如果你能读这本书，这意味着你也能读其他东西，比如网页。它们都包含单词，但没有翻页。但你也可以用它做其他事情，比如阅读路标和短信。所有这些之所以可能，是因为你学会了读书。读书是基础对象。

在编程中，这就是多态的含义。为特定类型编写的代码可以对相关数据类型执行操作。

### 创建模块

现在，你已经有一些使用 Python 模块和导入的经验。在 Python 中，你可以编写自己的模块。

编写模块是重用和重新利用代码的另一个例子。假设你正在开发一个纸牌游戏，在此期间，你创建了像 cards 和 deck 这样的类。在创建另一个纸牌游戏时，你可能需要这些类，即使游戏不同——像 cards 和 decks 这样的类。

模块还提供程序管理。当程序增长时，很难跟踪所有部分。模块允许你将它们分解成易于管理的小块。假设你正在处理一个商业项目，这意味着你有数千行代码。你可以使用模块将它们分成可管理的包。这意味着某些程序可以在某些部分上工作，而其他程序在其他部分上工作。它们不会因为谁在做什么而感到困惑，因为程序被很好地划分了。

你还可以与朋友和其他程序员共享模块。你可以上传它们。

### 编写模块

我们将创建一个用于简单游戏的模块。编写模块的方式与编写任何其他程序相同，但需要进行规划。你需要考虑所需的函数和类，然后将它们放入一个可以导入到其他项目中的文件中。

让我们创建一个名为 "game" 的模块。我们将创建一个类和两个函数，用于执行游戏中的常见任务。以下是代码。

```python
### game
### Example of writing a custom module
class Player(object):
    """ A basic player. """
    def __init__(self, name, score = 0):
        self.name = name
        self.score = score
    def __str__(self):
        rep = self.name + ":\t" + str(self.score)
        return rep
def ask_yes_or_no(question):
    """Ask a yes or no question."""
    response = None
    while response not in ("y", "n"):
        response = input(question).lower()
    return response
def ask_number(question, low, high):
    """Ask for a number within a range."""
    response = None
    while response not in range(low, high):
        response = int(input(question))
    return response
if __name__ == "__main__":
    print("This module was executed directly without importing.")
    input("\n\nPress the enter key to quit.")
```

该模块将被命名为 "games"，因为模块以其文件名命名。代码易于阅读，因为我们已经在示例中使用过其中一些函数。有一个 "player" 类定义了一个对象。该对象在构造函数中设置了两个属性。我们还有一个 "if" 语句，用于验证程序是否直接执行。如果检查为真，则应用程序直接运行；如果为假，则它将是一个被导入的模块。

## 第 8 章：异常

无论我们多么熟练或准备多么充分，项目永远不会像我们希望的那样顺利进行。作为一名程序员，你需要为此做好心理准备。

异常是当你的代码出现问题时出现的错误。你将经常需要修复这些错误。一些开发者的全部工作就是修复这些错误。这就是为什么我们将在本章中重点介绍如何修复错误。

### 当事情出错时

较小的简单程序永远不会显示奇怪的行为，因为代码很简单。以我们开始时的 "Hello World" 应用程序为例。它非常简单，你知道如果你运行一次，它将始终运行。但是，你编写的代码行数越多，技术越先进，遇到异常和错误的机会就越多。查看你的代码可能具有欺骗性。它可能并不总是按预期工作。

这里有一个例子。如果你的应用程序配置文件设置错误怎么办？或者如果它丢失了怎么办？拥有数千行代码的大型程序在某个时候会包含或更新不再有用的数据。但在某些情况下，代码和数据都很好，但你的应用程序依赖于与 Web 服务器的连接来收集信息，而该连接已断开。在某些情况下，错误是设计的产物，可以忽略，因为它不影响你的程序需要做的事情。

Python 只有在无法执行正常指令流时才会通知你。因此，你会得到一个异常。现在，你可能想进入你的代码并添加额外的行来处理异常。但这是一个糟糕的策略。Python 有一个易于遵循的异常处理方法，其异常处理工具是最有用的。

### 基本异常

为了演示，我们将从基本错误开始，例如除以零。这是你不会犯的错误类型，但它会触发异常，我们有兴趣学习如何处理这些异常。输入以下代码：

```python
1 / 0
```

```
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
ZeroDivisionError: int division or modulo by zero
```

在前面的章节中，我们遇到了一个错误，它给了我们一个类似的报告。让我们看看这个操作。除法总是需要被除数和除数。Python 会进行多次验证以确保满足这些标准。它检查两个参数是否都是数字，以及除数是否不为零。只有在执行这些检查后，才能执行操作。如果操作不符合这些标准，则会生成一个描述错误的异常。在编程中，我们说该操作引发了一个异常。

“引发”这个词是描述性的，因为它指的是层次结构。所以第一个函数调用第二个包含除以零的函数。第二个函数处理异常，因此它向第一个函数引发异常。最终，它无法被完全处理，因此它向你引发异常。异常很容易理解。它产生的报告称为回溯。

在某些编程语言中，异常不是引发的；它们是抛出或捕获的。因此，它被抛给第二个函数，该函数未能捕获它并将其带给第一个函数。但它仍然指的是同一件事。

处理异常的最简单方法之一是在脚本中使用 "try" 语句。"Try" 查找错误，而 except 块处理它们。以下是此示例：

```python
print(x)
```

"X" 未定义，这意味着它将导致错误。没有 try 语句，程序将崩溃。现在让我们处理异常：

```python
try:
    print(x)
except:
    print("An exception occurred")
```

也可以定义多个异常语句来捕获你想要的尽可能多的错误。这是一个例子：

```python
try:
    print(x)
except NameError:
    print("There is no definition for variable x")
except:
    print("Something else went wrong")
```

如果你得到一个 NameError 异常，程序将打印第一条消息，其他错误将打印第二条消息。

你还可以使用 "else" 关键字来指定如果没有错误时要执行的代码。这是一个例子：

```python
try:
    print("Hello World")
except:
    print("An error occurred")
else:
    print("Everything is fine")
```

### 异常类

Python 有几个异常类，用于告诉程序员代码出了什么问题。它们是主要异常类 "BaseException" 的子类，这意味着每个异常都源自此类。

我们将引发几个异常以了解它们继承了哪些特性。我们将通过使用数组处理异常来做到这一点，这样我们就可以存储它们并检查它们。以下是代码：

```python
store = []
### Create a number of different exceptions and handle them
try: {}["foo"]
except KeyError as e: store.append(e)
try: 1 / 0
except ZeroDivisionError as e: store.append(e)
try: "".bar()
except AttributeError as e: store.append(e)
### Loop through the errors and print the class ladder
for exception_object in store:
    ec = exception_object.__class__
    print(ec.__name__)
    indent = " +-"
    while ec.__bases__:
        # Assign ec's superclass to itself and increase
        ec = ec.__bases__[0]
        print(indent + ec.__name__)
        indent = " " + indent
```

结果应该如下所示：

```
KeyError
 +-LookupError
  +-Exception
   +-BaseException
    +-object
ZeroDivisionError
 +-ArithmeticError
  +-Exception
   +-BaseException
    +-object
AttributeError
 +-Exception
  +-BaseException
   +-object
```

我们现在已经检索了每个异常类层次结构。你可以在 Python 文档库中找到所有这些类以了解更多信息。实际上，你不会在旁边放一个异常列表。你将专注于特定的异常并处理它们，而不是试图捕获每个异常。虽然 Python 中的每个类都可以在层次结构中找到，但这也将包括自定义异常。你可以创建也可以从父类继承的异常。我们不会更多地讨论这个，因为它是一个高级主题。

### 断言错误

此错误的目的是告诉程序员存在程序无法恢复的错误。它们不显示典型的报告，这允许你以某种方式采取行动。断言就像代码中的自检。它们检查代码是否包含不可能的条件。如果发现它们，程序将崩溃，你将收到一个断言错误。它会告诉你哪个不可能的条件导致了它。

断言错误可以警告你需要修复的错误。要添加断言，你需要使用 "assert" 关键字。开发人员将 assert 语句放在函数的开头以检查输入，并在结尾检查输出。当 Python 检测到 assert 语句时，它会分析表达式。如果表达式返回 true，则可以执行函数；如果返回 false，则你会得到断言错误。

以下是语法示例：

```
assert Expression [Arguments]
```

断言错误可以像异常错误一样使用 "try / except" 语句处理。确保你捕获它们，否则程序将崩溃。

### 总结

在本章中，我们学习了基本的错误、异常以及如何处理它们。异常处理在开发过程中是一项非常有用的技术。我们还学习了断言错误，其处理方式与异常相同。
