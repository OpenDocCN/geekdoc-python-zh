## 前言

Python 是一种动态语言。

这意味着许多不同的事情。它意味着该语言更灵活。它意味着 Python 是一种更容易编程的语言。与许多其他编程语言相比，你可以快速编写一个简单的程序，而无需经历太多“仪式”。Python 脚本往往需要更少的样板代码。Python 给你更多的自由。

另一方面，这也意味着该语言安全性较低，更容易出错。这意味着用 Python 构建一个更大的系统更难。这意味着用 Python 编写的软件通常更难长期维护。

有效使用 Python 的关键是理解这种权衡。例如，Python 可以是快速原型设计或“脚本编写”的理想语言。另一方面，Python 并不常用于“企业软件”开发。

Python 是一种通用编程语言。Python 用于许多不同的应用领域，从系统管理任务到 Web 应用程序开发。Python 现在是科学家们最广泛使用的编程语言之一，他们传统上使用更高级的工具，如 Matlab。Python 为这些“非专业”程序员提供了一条更简单的“升级”路径。现在，Python 正在成为机器学习和数据科学的*首选*语言。

Python 对初学者友好。事实上，它似乎是初学程序员最喜欢的第一门语言，甚至比 JavaScript 更受欢迎。

然而，应该指出的是，Python *不是*一门简单的语言。在其大约 30 年的历史中，它经历了许多大大小小的变化。用 Python 开始编程仍然很容易。

尽管如此，一旦你达到某个阶段，比如从高级初学者到中级水平，它的复杂性可能会让人难以承受。

本书将为你提供一个关于 Python 语言的广泛且*精炼*的概述，涵盖其最新版本（例如 3.10 和 3.11）。本书对任何一直在涉猎 Python 但没有坚实基础的人都可能有用。它也可能对那些有其他编程经验并希望快速了解该语言的人有用。

本书是作为（非正式的）语言参考编写的。本书的目标不是教你如何有效地用 Python 编程，而是提供语言语法的简明总结。如果你有一些基本的编程知识，你可以或多或少地从头到尾阅读本书，无论你的具体编程背景如何，你都应该能够快速获得对 Python 编程语言的整体理解。

本书面向广泛的读者，但有一个注意事项是，与教程风格的书籍不同，本书有相当多的交叉引用。如果你之前没有用 Python 或任何类似语言编程的经验，那么你可能会发现阅读本书的某些部分有点困难。本书不适合完全的初学者。为了简洁起见，它在开头略过了一些基本概念，以便我们可以更多地关注中级和高级主题。

> 本书不是权威的语言参考。为此，我们建议读者参考官方语言规范。

## 第一章 引言

本书将为你概述 Python 语言的语法。

如今，编程语言本身与标准库之间的界限似乎正变得日益模糊。尽管我们的目标并非全面讲解 Python 标准库（这远超本书范围），但我们仍会涉及标准库模块中一些被认为或多或少属于语言本身的重要概念。

这本“参考手册”从 Python 最枯燥的部分开始。☺ 如果你打算从头到尾通读本书，而非仅仅将其用作快速参考，那么在初次阅读时可以跳过前几章，待后续需要时再回看。

“程序”一词在不同语境和不同编程语言中含义各异。我们将从对[什么是 Python 程序](what a Python program is)以及[Python 程序如何执行](how Python programs are executed)（例如在 Python 解释器中）的较为正式的解释开始。

Python 程序在逻辑上组织为[包和模块](packages and modules)，接下来我们将对此进行探讨。*模块*对应于物理文件系统中的一个文件，而模块（包括包模块）是 Python 中代码复用和共享的基本单元。

接下来，我们将简要介绍 [Python 源代码](Python source code)的一些词法元素。不同编程语言之间存在一些细微差异，但它们的词法构成相当相似，Python 也不例外。这部分内容在你的“阅读”中可以跳过。

一般来说，程序由*代码和数据*组成。代码指的是指令。Python 中的数据由*对象*表示。Python 中的对象是一个基本组件。我们在 Python 程序中处理的一切都是对象。本书的主体部分将从介绍与[对象](objects)相关的各种重要概念开始。

尽管 Python 使用动态类型系统，但类型在 Python 编程语言中仍然扮演着基础性角色。我们首先介绍 Python 中的一些[基本内置类型](basic builtin types)。Python 包含相当多的内置类型，而这本参考手册仅涉及其中一部分。如前所述，这本参考手册中讨论的所有主题普遍如此。完整性并非这本*迷你参考手册*的目标。

Python 还包括一些[内置复合类型](builtin compound types)，如列表和字典，它们是任何非平凡程序的重要组成部分。我们将在[下一章](the next chapter)中简要介绍这些类型。高级类型，特别是函数和类，将在本书后面详细讲解。

与任何命令式编程语言一样，Python 拥有表达式和语句。表达式规定了如何使用其他表达式和值来计算一个值。Python 支持其他命令式语言中常见的[大多数运算符和表达式](the common operators and expressions)。如果你有其他过程式编程语言的背景，可以略读这部分的大部分内容。

语句是指令。语句控制程序的流程以达到预期目标。在 Python 中，有两类语句：[简单语句](simple statements)和[复合语句](compound statements)。复合或复杂语句可以“包含”其他简单或复合语句。

## 第2章. Python 程序

一个完整的 Python 程序可以传递给 Python 解释器：

-   使用 -c 命令行选项传递程序文本，
-   使用 -m 命令行选项传递模块名，
-   作为第一个命令行参数传递的文件名，或
-   从标准输入。

Python 解释器在特殊模块 `__main__` 的命名空间中，将输入文件或输入文本作为完整程序执行。

当文件或标准输入是终端时，或者当 Python 解释器在没有参数的情况下被调用时，解释器会进入交互模式。在交互模式下，Python 解释器一次读取并执行一条语句（简单或复合语句），而不是执行完整的程序。

以下是一些示例：

```
$ python -c "print('Hello World!')"
```

① 使用 -c 标志执行一个完整的 Python 程序。

```
$ python main.py
```

① Python 脚本作为命令行参数传递给解释器。

```
$ python -m hello_world
```

① 通过 -m 标志执行 Python 模块 `hello_world`。

```
$ echo "print('huh?')" | python
```

① 来自 stdin（通过 Unix 管道）的 Python 代码。

```
$ python <<< "print('a')"
```

① 使用 Unix shell 的“here string”语法。

```
$ python << EOF
> print("hello")
> print("world")
> EOF
```

① “here doc”。

```
$ python
```

① 不带参数调用 python 命令会启动 REPL。

```
$ python -
```

① 同上。Python 解释器会忽略短横线 - 后面的标志。

```
$ python -i
```

① 这也会启动交互式 shell。-i 标志可以与其他标志组合使用，例如 -m、-c 等。

> 在本书中，当相关时，我们将使用 Unix shell 进行说明。根据您的平台，某些命令可能无法完全按照此处所示工作。

### 2.1. 文件/文本输入

一个完整的 Python 程序是一系列零条或多条语句，语句之间可选地包含换行符。在以下情况下需要这种形式的完整程序：

-   当从文件或输入字符串读取 Python 程序时，
-   当解析导入的模块时，以及
-   当将字符串参数传递给内置 `exec` 函数时。

如前所述，运行程序的主模块在 Python 中始终称为 `__main__`。

### 2.2. 交互模式

交互模式下的 Python 解释器（也称为 Python REPL）不执行完整的程序。相反，它一次读取并执行一条语句。每个输入都在 `__main__` 的命名空间中构成一个代码块。

在交互模式下，顶级复合语句后必须跟一个额外的换行符，解析器使用它来检测输入的结束。

```
$ python
Python 3.10.4 (main, Jun 29 2022, 12:14:53) [GCC 11.2.0] on linux
Type "help", "copyright", "credits" or "license" for more information.
>>> 
>>> for levels in ("White", "Black"):
...     print(levels)
...
White
Black
>>> 
>>> exit()
$
```

① 启动 Python REPL 的命令。在您的系统上可能有所不同。

② Python REPL 提示符，>>>。在此示例中，我们刚刚按下了 Enter。

③ for 复合语句的第一行。

④ 提示符已从 >>> 变为 ...。for “套件”中的语句需要缩进。

## 第3章. 程序执行

### 3.1. 代码块

Python 程序在一系列“执行帧”中执行。在执行帧中作为单元执行的一段 Python 程序称为“代码块”。

例如，模块和函数/类定义是代码块，以及其他东西。传递给解释器的整个程序，例如作为 -c 选项或作为文件参数，是一个代码块（更准确地说，是顶级代码块）。导入的模块也是如此。代码块可以嵌套。

### 3.2. 名称绑定

在 Python 中，所有值都是“对象”，对象通常通过其“名称”来引用。名称通过名称绑定操作引入。

例如，`import` 语句、函数和类定义以及赋值表达式和语句在其封闭代码块内绑定名称。函数参数也在函数定义块内绑定名称。如果一个名称在块中被绑定，它默认是该块的“局部变量”。

如果一个名称在模块级别被绑定，它也是一个“全局变量”。形式为“from XYZ import *”的 `import statement` 只能在模块级别使用。它绑定导入模块中定义的所有不以下划线 (_) 开头的名称。所有以此方式导入的名称都是全局的（相对于导入模块也是局部的）。

### 3.3. 作用域

一个名称在程序的特定部分内可访问，例如在名称被绑定的代码块内。这称为“作用域”。在代码块中绑定的局部变量的作用域包括该块。

当在代码块中使用名称时，会使用最小的封闭作用域来查找名称的绑定。这称为“名称解析”。如果名称无法解析，则会引发异常。根据上下文，可能会引发 `NameError`，或者更具体地说是 `UnboundLocalError`。

#### 3.3.1. 函数定义块

`function def statement` 定义一个代码块。如果一个名称在函数块中被绑定，那么它的作用域包括此函数块以及其中包含/嵌套的任何其他块。

#### 3.3.2. 类定义块

在 `class definition` 中，类块中定义的名称的作用域仅限于该类块，*不包括*封闭方法的名称。未在类块中绑定的局部变量将在全局命名空间中查找。

#### 3.3.3. `global` 语句

如果在块中对某个名称使用了 `global statement`，那么对该名称的引用将指向其在顶级命名空间中的绑定，该命名空间包括代码块模块和 `builtins` 模块的绑定，以及内置命名空间。

### 3.5. 程序启动与终止

如前所述，脚本的主模块 `__main__` 是一个代码块。

Python 解释器通过执行主代码块来启动程序，该代码块随后可以调用其他代码块，而这些代码块又可以调用其他代码块，依此类推。当一个代码块的执行完成时，它会将控制权返回给调用当前代码块的外围代码块。

当主模块代码块的执行完成时，程序终止。

### 3.6. 异常

程序的正常流程（通过代码块调用链）可以使用异常处理机制来绕过。在异常或错误条件下，可以使用 `raise` 语句显式地“引发”一个异常。当 Python 解释器检测到运行时错误时，也可以引发异常。

引发的异常可以由周围的或任何上游的代码块处理。当异常在任何代码块中都未被处理时，解释器在非交互模式下会终止程序的执行。在 REPL 模式下，它只是返回到其交互式主循环。无论哪种情况，它都会打印堆栈跟踪信息，除非异常是 `SystemExit`。

异常处理程序通过 `try` 语句指定。从 Python 3.11 开始，除了 `try - except` 之外，还支持新的 `try - except*` 语法。

Python 中有两种基本的内置异常类型：`BaseException` 和 `Exception`，它们大致分别对应于 Java 的“运行时异常”和“检查型异常”。同样，Python 现在（3.11+）包含两种内置异常组类型：`BaseExceptionGroup` 和 `ExceptionGroup`。

异常处理将在本书后面的 `try` 和 `raise` 语句上下文中进一步讨论。

## 第4章 包与模块

### 4.1. 模块

Python代码被组织成“模块”，模块与包含其他Python*对象*的命名空间相关联。模块中的Python代码可以通过“导入”过程访问不同模块中的代码。例如，可以使用`import`语句导入模块。

#### 4.1.1. `import`语句

`import`语句执行以下操作：

- 它首先搜索指定的模块，并且
- 如果找到该模块，则
    - 它创建并初始化一个模块对象，并将该对象绑定到本地作用域中的一个名称。
- 如果未找到指定的模块，则
    - 会引发`ModuleNotFoundError`异常。

例如，

```
>>> import math ①
>>> type(math) ②
<class 'module'> ③
```

① 一个`import`语句。
② 我们将在本书中了解Python的类型系统。内置的`type`函数返回给定对象的类型。
③ 模块的类型是`module`。

`import`语句以及其他与导入相关的实用函数的内部实现，由标准库模块`importlib`提供。

### 4.2. 包

为了帮助组织模块，Python引入了`包`的概念。包是一种特殊的模块，它提供了一个命名空间/命名层次结构。从技术上讲，包是包含`__path__`属性的模块。

如果使用脚本调用Python解释器，那么包对应于文件系统上的一个目录，而常规的非包模块对应于一个Python源文件。与文件系统目录类似，包是分层组织的，包可以包含其他包以及常规模块。

每个模块都有一个名称。子包名称与其父包名称用点（.）分隔。模块的`__name__`属性被设置为模块的完全限定名。模块的限定名用于在Python导入系统中唯一标识模块。

Python中有两种包：“常规包”和“命名空间包”。

#### 4.2.1. 常规包

常规包通常实现为一个包含`__init__.py`文件的目录。当导入常规包时，包及其所有父包的`__init__.py`文件会被隐式执行。每个包的`__init__.py`文件中定义的对象被绑定到包的命名空间中的名称。

#### 4.2.2. 命名空间包

隐式命名空间包是一个（虚拟的）复合包，可以包含来自文件系统不同位置的子包（例如，不受目录层次结构的限制）。

使用命名空间包时，没有包`__init__.py`文件。与常规包不同，一个命名空间包可能对应多个目录。例如，当导入包`parent/one`和`parent/two`，而没有与`parent`模块对应的物理目录时，Python会自动为顶级`parent`包创建一个命名空间包。

### 4.3. 包相对导入

相对导入使用前导点（.）。单个前导点表示相对导入，从当前包开始。两个前导点表示相对导入当前包的父包，以此类推。

绝对导入可以使用`import X.Y`或`from X import Y`语法，但相对导入只能使用第二种形式。例如，从文件夹`bbb`中的给定Python脚本/模块`xyz`，

aaa/bbb/xyz.py

```
from . import efg ①
from .hij import klm ②
from .. import mno ③
from ..opq.qrs import stu ④
```

① 模块`efg`对应于文件`aaa/bbb/efg.py`。

② 模块`klm`对应于文件`aaa/bbb/hij/klm.py`。

③ 模块`mno`对应于文件`aaa/mno.py`。

④ 模块`stu`对应于文件`aaa/opq/qrs/stu.py`。

## 第5章 Python源代码

Python解释器将程序文本读取为Unicode码点。Python源代码文件的默认编码是“UTF-8”。源文件也可以给出显式的编码声明。如果声明了编码，则编码名称必须被Python识别。如果文本无法解码，则会引发`SyntaxError`异常。

Python解释器首先将源文件分解为“词法单元”，然后将其传递给解析器。

### 5.1. 行结构

Python程序是“基于行的”。Python源代码由“逻辑行”组成。

#### 5.1.1. 逻辑行

一个语句通常包含在一个逻辑行中，其结束由NEWLINE词法单元表示。如果Python语法允许，一个`复合语句`可以跨越多个逻辑行。

一个逻辑行由一个或多个“物理行”构成。

#### 5.1.2. 物理行

物理行是以行结束序列终止的字符序列。在Python源文件和字符串中，无论平台如何，都可以使用三种标准行终止序列中的任何一种：`\n`、`\r\n`或`\r`。输入的结束也作为最后一个物理行的隐式终止符。

#### 5.1.3. 注释

注释以井号字符（#）开始，并持续到物理行的末尾。语法大多忽略注释。注释可以表示逻辑行的结束，即，除非调用隐式行连接规则，否则会生成一个NEWLINE词法单元。

#### 5.1.4. 空行

仅包含空格、制表符、换页符，可能还有注释的逻辑行会被忽略。

#### 5.1.5. 物理行连接

**显式连接**

可以使用反斜杠字符（\）将两个或多个物理行连接成一个逻辑行。

当一个物理行以反斜杠结束时，它会与下一行连接形成一个逻辑行，删除反斜杠和随后的行结束字符。

- 除了在字符串字面量或注释中，反斜杠不允许出现在行上的任何其他位置。
- 除了字符串字面量，反斜杠不能将词法单元分割到多个物理行上。
- 反斜杠不会延续注释。

例如，

```
>>> a, b = \n... 10, 20
```

这是一个逻辑行，`a, b = 10, 20`，尽管它写在两个物理行上。请注意，除了三种行终止序列之一之外，反斜杠之后不允许有其他任何字符，包括空格。

**隐式连接**

括号（`(...)`）、方括号（`[...]`）或花括号（`{...}`）中的表达式，以及三引号字符串（`"""..."""`或`'''...'''`），可以在不使用反斜杠的情况下分割成多个物理行。

隐式延续的行可以携带注释，但三引号多行字符串除外。延续行的缩进并不重要。允许空的延续行。

例如，

```
>>> fruits = [
...     "manzana",    # apple
...     "naranja",    # orange
... ]
>>> print(
...
...     fruits
... )
['manzana', 'naranja']
```

① 隐式连接的开始。
② 允许注释。
③ 逻辑行的结束。
④ 另一个隐式连接的开始。
⑤ 一个空的物理行。
⑥ 逻辑行的结束。此语句等效于在一个物理行中的`print(fruits)`。

请注意，由于括号可用于对表达式进行分组，实际上任何表达式都可以分割成多个物理行。

#### 5.1.6. 缩进

在Python中，逻辑行的缩进用于确定语句的分组。

一行的缩进级别是根据行首的前导空格计算的。第一个非空格字符之前的空格总数决定了该行的缩进。

Python词法分析器使用一个栈，根据连续逻辑行的缩进级别生成INDENT和DEDENT词法单元，然后用于将逻辑行分组为语句。

### 5.2. 词法单元

除了NEWLINE、INDENT和DEDENT，Python还有以下类别的词法单元：*标识符*、*关键字*、*字面量*、*运算符*和*分隔符*。

### 5.3. 标识符和关键字

Python中标识符（或名称）的语法基于Unicode标准。标识符的长度不受限制，并且区分大小写。在ASCII范围内（U+0001..U+007F），名称可以包含字母数字字符和下划线（_）。名称不能以数字开头。在ASCII范围之外，`unicodedata`模块对标识符中有效的字符进行分类。

#### 5.3.1. 关键字

以下名称用作保留字，或语言的关键字，不能用作普通标识符。

- False True None and as assert async await break class continue def del elif else except from finally for global if import in is lambda nonlocal not or pass raise return try while with yield

#### 5.3.2. 标识符的保留类别

除了关键字，某些类别的标识符对 Python 具有特殊含义。这些类别由前导和尾随下划线字符的模式标识：

**_**
特殊标识符 `_` 在交互式解释器中用于存储上一次求值的结果。

**_xyz**
不会被 `from module import *` 导入。

**__xyz__**
系统定义的名称，非正式地称为“双下划线名称”。这些名称由解释器及其实现（包括标准库）定义。

**__xyz**
类私有名称。在类定义中使用时，这些名称会被重写为一种修饰形式，以帮助避免基类和派生类的“私有”属性之间的名称冲突。

### 5.4. 字面量

“字面量”是词法分析器赋予特殊含义的一类特定标记，有别于标识符和其他特殊标记。通常，字面量是某些内置类型的常量值。

#### 5.4.1. 字符串和字节串字面量

字符串和字节串字面量可以用匹配的单引号 (') 或双引号 (") 括起来。也可以用匹配的三单引号 ('' '') 或三双引号 (""") 括起来。

反斜杠 (\) 字符在字符串和字节串字面量中用于“转义”那些原本具有特殊含义的字符，例如换行符、反斜杠本身或引号字符。

字节串字面量是 `bytes` 类型的实例，始终以 'b' 或 'B' 为前缀。它们只能包含 ASCII 字符。字符串和字节串字面量都可以选择性地以字母 'r'/'R' 为前缀。这样的字面量称为原始字符串，将反斜杠视为普通字符。

#### 5.4.2. 格式化字符串字面量

*格式化字符串字面量*，简称 *f-string*，是带有 'f' 或 'F' 前缀的字符串字面量。这些字符串可以包含替换字段，即用花括号 {} 括起来的变量或表达式。可以在可选的冒号 : 之后添加格式说明符。

虽然其他字符串字面量总是具有常量值，但 f-string 实际上是在运行时求值的表达式。

可以在表达式之后、冒号和格式字符串（如果有的话）之前添加等号 =。当包含它时，结果字符串将连接表达式文本、= 号和求值后的值。

例如，

```
>>> a = 10 / 7
>>> f"{a}" ①
'1.4285714285714286'
>>> f"a = {a:.5f}" ②
'a = 1.42857'
>>> f"{a=}", f"{10/7=:.5f}" ③
('a=1.4285714285714286', '10/7=1.42857')
```

① 一个简单的 f-string。

② 替换字段在 : 之后有一个格式说明符 .5f。

③ 等号打印出表达式文本，后跟 =，然后是值。

#### 5.4.3. 字符串字面量连接

我们可以使用 + 运算符连接字符串表达式。作为快捷方式，对于相邻的字符串字面量以及字节串字面量，可以在不使用 + 运算符的情况下连接它们。它们可以由（可能为空的）空格分隔，其效果与使用 + 运算符进行字符串连接相同。

字面量连接可以对每个操作数使用不同的引号风格。F-string 字面量可以与普通字符串字面量连接。例如，

```
>>> a = "hello"
>>> (
... "hi" "hi" ②
... f"{a} world" ③
... """bye bye""" ④
... )
'hihihello worldbye bye'
```

① 这里的括号表达式用于将字符串字面量放在多个物理行中，以作说明。

② "hi" "hi" 等同于 "hi" + "hi"。

③ 一个 f-string 字面量可以与其他字符串字面量连接。同样，+ 运算符是可选的。

④ 一个三引号字符串字面量，可以跨越多个物理行，与相邻的格式化字符串字面量连接，在此示例中没有使用 + 运算符。

#### 5.4.4. 数字字面量

有三种类型的数字字面量：整数、浮点数和虚数字面量。

整数必须以数字开头。以 0x/0X、0o/0O 和 0b/0B 开头的字面量分别是十六进制、八进制和二进制数。十进制数不能以数字 0 开头，数字 0 除外。

下划线 (_) 在求值整数字面量的值时被忽略。整数字面量不能以下划线开头，并且不允许连续多个下划线。例如，

```
0b_010 ①
00000_010 ②
0x0F ③
1_000_000 ④
```

① 一个二进制整数字面量，表示十进制数 2。

② 一个八进制整数字面量，表示十进制数 8。

③ 一个十六进制字面量，等同于十进制数 15。

④ 一个十进制数（基数为 10）。注意十进制整数字面量不能以 0 开头。

浮点字面量包含一个句点 (.) 和/或指数符号 (e 或 E)。整数部分和指数部分使用基数为 10 解释。

浮点数字面量示例：

```
100.
03.14
.0101
0_0_0e0_0_0
31.415_927E-10
```

虚数字面量，即带有后缀 j 或 J 的浮点字面量，生成一个实部为 0.0 的复数。表达式 1j * 1j 生成一个复数 (-1+0j)。

要创建一个具有非零实部的复数，可以使用将浮点数加到虚数字面量的表达式。例如，

```
100j ①
03.14J ②
10E10 + 0.0001E-100j ③
10 - 0.314_159e+10J ④
```

① 一个虚数字面量。

② 另一个虚数字面量。

③ 一个包含实数和虚数字面量的表达式。

④ 另一个类型为复数的表达式。

# 第 6 章. 对象

在 Python 中，数据由*对象*表示。一个对象具有*标识*、*类型*和*值*。一个对象可以通过“名称”（或“引用”）来引用。一个对象在其整个生命周期中具有唯一不变的标识，但在任何给定时刻，它可以有零个、一个或多个名称/引用。

Python 是一种动态和松散类型的编程语言。尽管类型系统是 Python 编程语言的核心，并且类型在 Python 程序中起着至关重要的作用，但它们主要在运行时才变得相关，并且只是间接相关。（Python 的类型系统通常称为“鸭子类型”。）事实上，可以在运行时创建一个新类型，甚至可以在运行时更改对象的类型（尽管这通常不是常见做法）。

某些类型的对象的值可以改变。这些对象被称为“可变的”。例如，像列表和字典这样的内置数据类型是可变类型。另一方面，对于某些其他类型，一旦创建对象，就不能*直接*修改其值。它们被称为“不可变的”。例如，数字、字符串和元组是不可变类型。（注意，不可变对象的（内部）值仍然可以改变。）

## 6.1. 标识

“标识”是一个相当抽象的概念，但 Python 包含一个内置的 `id` 函数，该函数返回一个唯一值，可用作给定对象的标识。事实上，`id` 函数“定义”了对象的标识。（在 CPython 实现中，它返回给定对象的内存地址。）

例如，

## 6.1. 标识

```python
>>> a, b, c = 3, "the ring", ['a', 'b', 'c']
>>> id(a), id(b), id(c)
(546842556784, 546841138928, 546841139008)
>>> x = 3
>>> id(a), id(x), id(3)
(546842556784, 546842556784, 546842556784)
```

① 这个提示符 `>>>` 表示我们正处于 REPL 环境中。

② 我们在本书中经常使用这种“多重赋值”语法。右侧的表达式列表会求值为一个元组。然后，通过“元组解包”，其每个元素被分别赋值给 `a`、`b`、`c`。

③ 名称 `a`、`b` 和 `c` 都有不同的 `id` 值。它们引用不同的对象。

④ 名称 `a` 和 `x` 以及对象 `3` 都具有相同的 `id` 值。`a` 和 `x` 引用同一个对象 `3`。

对象的标识也可以使用内置的 `is` 和 `is not` 运算符进行比较。`x is y` 的求值结果为 `True`，当且仅当 `x` 和 `y` 是同一个对象，即根据 `id` 函数判断。请注意，两个对象之间的标识相等（`is`）意味着它们的值相等（`==`）。

使用上面相同的例子，

```python
>>> a is a, a is b, a is c
(True, False, False)
>>> a is x, a == x
(True, True)
```

① `(a is a) == True`。名称 `a`、`b` 和 `c` 引用不同的对象。

② `(a is x) == True` 且 `(x is a) == True`。同样，`a == x` 且 `x == a`。

## 6.2. 属性

一个对象与一组“属性”相关联。属性是一个引用（其他）对象的名称。当它引用的对象是一个函数时，该属性被称为（原始对象的）*方法*。否则，它被称为数据属性（例如，在其他编程语言中通常称为“字段”）。

可以使用[属性引用](https://docs.python.org/3/reference/datamodel.html#attribute-access)语法（或“点表示法”）访问对象的属性。

```python
>>> class A: pass ①
...
>>> a = A() ②
>>> a.magic_number = 42 ③
>>> a.magic_number ④
42
>>> def f(): pass ⑤
...
>>> a.empty_method = f ⑥
>>> a.empty_method() ⑦
```

① `class` 语句创建了一个新类型 `A`。

② 类型 `A` 的一个实例对象 `a`。

③ 向 `a` 添加一个名为 `magic_number` 的数据属性，并将其绑定到对象 `42`。

④ `a.magic_number` 引用对象 `42`，其值为 `42`。

⑤ `def` 语句创建了一个新的函数对象 `f`。

⑥ 新属性 `a.empty_method` 现在引用 `f`。

⑦ 方法调用语法调用了函数 `f`。

内置的 `dir` 函数可用于

- 查找当前作用域中的所有名称，或
- 列出与给定对象关联的属性。

在当前上下文中，使用上面的例子，

```python
>>> dir() ①
['A', '__annotations__', ... '__spec__', 'a', 'f']
>>> dir(A) ②
['__class__', ... '__weakref__']
>>> dir(a) ③
['__class__', ... '__weakref__', 'empty_method',
'magic_number']
>>> type(a) ④
<class '__main__.A'>
```

① 不带任何参数调用 `dir()` 会返回当前作用域中的名称，以字符串列表的形式，按词法顺序排列。请注意，除了我们刚刚引入的三个名称 `A`、`a` 和 `f` 之外，还有一些系统定义的名称。

② 在类对象上调用 `dir(cls)` 会返回给定类 `cls` 及其所有基类（包括 `object`）的属性。

③ 在通用（非类）对象上调用 `dir(obj)` 会返回其自身属性（对于 `obj`）和其类型通过 `dir()` 返回的属性的组合列表。

④ 在此示例中，`a` 的类型是 `A`。`magic_number` 和 `empty_method` 是 `a` 自身的属性，其余来自 `A`。

请注意，`dir` 函数会调用对象的 `__dir__` 方法（如果存在）。

```python
>>> a.__dir__() ①
['magic_number', 'empty_method', '__module__', ...
'__class__']
```

① 这返回与 `dir(a)` 相同的结果（顺序可能不同）。

可以覆盖类的 `__dir__` 方法，使其从 `dir()` 函数调用返回不同的列表。

## 6.3. 类型

Python 中的每个对象除了其标识和值之外，还与一个“类型”相关联。对象的类型影响对象的行为。例如，对象的类型定义了该对象可能具有的所有值。此外，对象的类型决定了该对象支持的操作集。（请注意，Python 中的（大多数类型的）对象可以说是可变的，它们的行为可以在程序执行过程中改变，无论其指定的类型/类型名称如何。）

再举一个例子，不可变类型的对象在可能的情况下会被 Python 解释器“重用”。也就是说，对于不可变类型，返回新对象或计算新值的操作实际上可能返回对任何具有相同类型和值的现有对象的引用。相反，可变类型的对象永远不会被“重用”。

例如，

```python
>>> def f():
...     a, b = (1, 2), [3, 4]
...     print(id(a), id(b))
...     p, q = (1, 2), [3, 4]
...     print(id(p), id(q))
...
>>> f()
510220577408 510220577600
510220577408 510221371520
```

请注意，元组 `a` 和 `p`（不可变对象）的 `id` 相同，而列表 `b` 和 `q`（可变对象）的 `id` 不同。`a` 和 `p` 指向同一个对象。另一方面，`b` 和 `q` 指向两个不同的对象。

## 6.4. 内置类型函数

Python 提供了一个内置的 `type` 函数，它返回给定对象的类型/类。类型实际上是一个创建类型的“类对象”。其构造函数之一接受一个对象作为参数，并返回该对象的类型对象（例如，可能被“重用”）。

例如，

```python
>>> type, type(type), type(type(type))
(<class 'type'>, <class 'type'>, <class 'type'>)
>>> type(10), type('hello'), type([1, 2])
(<class 'int'>, <class 'str'>, <class 'list'>)
```

① `type` 的类型是 `type`。😊

② 对象 `10`、`'hello'` 和 `[1, 2]` 的类型分别是 `int`、`str` 和 `list`。

## 6.5. 可变类型与不可变类型

在 Python 中，可变性/不可变性是类型的一个特征。可变类型的对象是可变的，不可变类型的对象是不可变的。

不能直接更改不可变对象的值。然而，这并不意味着该对象是真正不可变的。不可变对象的属性所引用的对象仍然可以是可变的，并且不可变对象的整体有效值可能会改变。这对于复合类型通常是正确的。例如，对于像元组这样的不可变容器类型，它们仍然可以包含可变元素，并且元组的整体值仍然可以改变（尽管其值仍然可能无法直接更新）。

例如，

```python
>>> a, b = (1, 2), [3, 4] ①
>>> a[0] = 10 ②
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'tuple' object does not support item assignment
>>> b[0], b[1] = 30, 40 ③
>>> a, b ④
((1, 2), [30, 40])
>>> x = a, b ⑤
>>> x ⑥
((1, 2), [30, 40])
>>> x[1] = [3000, 4000] ⑦
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'tuple' object does not support item assignment
>>> x[1][0], x[1][1] = 300, 400 ⑧
>>> x ⑨
((1, 2), [300, 400])
```

① `a` 引用一个元组 `(1, 2)`，而 `b` 引用一个列表 `[3, 4]`。

② 元组是不可变的，因此不能直接修改它们的值。

③ 相反，可以轻松更改列表的值，因为列表是可变的。

④ `a` 和 `b` 引用的对象与之前相同，但 `b` 的值已经改变。

⑤ 右侧的表达式列表求值为一个元组对象。

⑥ `x` 引用一个元组，一个“不可变”对象。

⑦ 不能直接更改其值，因为 `x` 是“不可变”的。

⑧ 然而，如果集合中的某个项或对象的某个属性是可变的，就可以更改其值。在此示例中，`x` 的第二个项恰好是一个可变对象，一个列表 `[30, 40]`。因此，我们可以更改其值。

⑨ “不可变”的 `x` 的有效值确实发生了变化（尽管 `x` 的第一项和第二项在更改前后仍然是相同的对象）。

## 6.6. 构造函数

Python 的所有类型，无论是内置的还是用户定义的，都包含构造函数，或简称构造函数。类型的名称与构造函数函数的名称相同。或者，更准确地说，类型是“可调用的”。构造函数被调用时，会返回给定类型/类的实例对象。

对于某些内置类型，如元组和列表，也可以使用字面量语法来创建这些类型的实例，我们将在后面的章节“内置复合类型”中进行解释。

```python
>>> class A: pass
...
>>> type(A)
<class 'type'>
>>> a = A()
>>> type(a)
<class '__main__.A'>
```

① 类语句创建一个新的类对象，在此示例中是 `A`。

② `A` 的类型是 `type`。

③ `A`，一个类对象，是可调用的。

## 6.7. 布尔上下文

（几乎）Python中的所有对象都有“真值”，这默认取决于它们的类型和值。当一个对象在`if`或`while`条件中使用，或作为布尔运算的操作数时，会返回其真值。（`and`和`or`运算符的工作方式略有不同。）

对象的真值默认为`True`。在以下情况下为`False`：

- 如果对象/其类定义了`__bool__`方法，并且
    - 该方法返回`False`，或者
- 如果对象没有`__bool__`方法，但对象/其类定义了`__len__`方法，并且
    - 该方法返回`0`。

对于内置类型，以下对象的真值为`False`：

- `None`和`False`。
- 数值类型的“零值”对象，例如`0`、`0.0`、`0j`等。
- 空集合，例如`""`、`()`、`[]`、`{}`等。

自定义类型对象的真值默认始终为`True`，无论其属性的值如何。为了赋予不同的行为，例如根据其内部状态返回`True`或`False`，我们需要实现`__bool__`方法。此方法在自定义类型中默认不存在。

如果一个对象，或者更准确地说其类型，实现了此方法，那么它会在布尔上下文中被调用，例如由Python解释器调用。例如，

```python
class BiggerThan8:
    def __init__(self, zzz):
        self.zzz = zzz
    def __bool__(self):
        return True if self.zzz >= 8 else False
```

```python
>>> z = BiggerThan8(5)
>>> bool(z)
False
>>> z = BiggerThan8(10)
>>> bool(z)
True
```

内置的`bool`函数返回参数（对象或表达式，或名称）的布尔值。这段代码提供了“鸭子类型”的另一个例子。只要对象`z`是具有`__bool__`方法的类型，它就能按预期工作。

## 6.8. 对象的生命周期

在Python中，我们可以创建对象，例如使用构造函数，但没有办法显式地销毁它们。任何不可达的对象，例如因为没有有效的名称引用它们，都可能被Python运行时“垃圾回收”。

`del`语句可用于从给定对象中删除/解绑一个名称。此外，Python的标准库模块`gc`可用于显式控制垃圾回收过程的各个方面。

## 第7章. 简单类型

Python编程语言内置了许多类型。它们被称为“内置”类型。类型可以分为简单类型和复合类型。Python的内置复合类型将在下一章中描述。

### 7.1. None

`NoneType`简单类型只有一个值`None`，通常表示（有意义的）值的缺失。

`None`在Python中有多种用途。例如，一个没有显式返回值的函数被认为返回了`None`。也就是说，一个不返回任何具体值（如`100`或其他任何值）的函数调用表达式的值是`None`。

```python
>>> type(None)          ①
<class 'NoneType'>
>>> bool(None)          ②
False
>>> def f(): pass       ③
...
>>> print(f())          ④
None
```

① `None`是类型`NoneType`的唯一值。
② `None`的布尔/真值是`False`。
③ 一个不返回任何值的函数。*它返回none*。
④ 如果表达式的值是`None`，Python REPL不会打印该值。我们使用`print`函数来显式打印`f()`表达式的值。

### 7.2. NotImplemented

`NotImplemented`类型是另一个单例类型，它同样只有一个值`NotImplemented`。

```python
>>> type(NotImplemented)
<class 'NotImplementedType'>
>>> def f(): return NotImplemented ①
...
>>> f()
NotImplemented
>>> if NotImplemented: ②
...     print("Not implemented")
...
<stdin>:1: DeprecationWarning: NotImplemented should not be used in a boolean context
Not implemented
```

① `NotImplemented`有点像一个无效值，例如，它不能用于数值或比较运算。

② `NotImplemented`的布尔值曾经是`True`。在撰写本文时，它仍然是。但是，它的使用已被弃用，如果你在布尔上下文中使用`NotImplemented`，Python会引发警告或错误。

### 7.3. Ellipsis

一个`ellipsis`字面量`...`（或`Ellipsis`）可以在Python程序中不需要特定值的几个不同地方使用（例如作为占位符）。

```python
>>> type(...), bool(...) ①
(<class 'ellipsis'>, True)
>>> Ellipsis is ... ②
True
```

① ...的布尔值是`True`。
② `Ellipsis`和...是同义词。它们指向同一个（单例）对象。

### 7.4. 数字

所有数值类型都继承自`numbers`模块的`numbers.Number`。数值对象是不可变的。事实上，Python中简单类型的对象都是不可变的。

需要注意的一点是，`bool`在Python中是一种数值类型。

#### 7.4.1. 整数 (`numbers.Integral`)

`numbers.Integral`类型定义了一组整数。

**int**
Python中的`int`类型表示（任意精度的）整数。

**bool**
`bool`类型表示真值`False`和`True`。它们的数值分别是`0`和`1`，字符串值分别是"False"和"True"。

```python
>>> import numbers
>>> issubclass(int, numbers.Integral)
True
>>> issubclass(bool, int)
True
>>> isinstance(True, int), isinstance(10, int)
(True, True)
```

#### 7.4.2. 实数 (`numbers.Real`)

**float**
float类型表示一组双精度浮点数。

```python
>>> import numbers
>>> issubclass(float, numbers.Real)
True
>>> isinstance(1.5, float), isinstance(1.2E-5, float)
(True, True)
```

#### 7.4.3. 复数 (`numbers.Complex`)

**complex**
complex类型表示所有双精度浮点数对，即实部和虚部。

```python
>>> import numbers
>>> issubclass(complex, numbers.Complex)
True
>>> c = 1 + 2j
>>> isinstance(c, complex), isinstance(0.1 + .5j, complex)
(True, True)
>>> c.real, c.imag
(1.0, 2.0)
```

① complex类型具有real和imag属性，它们分别返回复数对象的实部和虚部。

## 第8章. 复合类型

对象可以由其他对象构建。同样，类型也可以由其他类型构建。它们被称为复杂或“复合类型”。

Python的内置复合类型，如元组和列表（我们将在本章讨论），以及用户定义的复合类型（我们将在本书后面讨论），都是使用其他（简单和复杂）类型作为构建块构建的。

Python的内置复合类型分为三类，

- 序列，如`tuple`和`list`，
- 集合，如`set`，以及
- 映射，如`dict`。

在简要讨论如何构造元组、列表、集合和字典之后，我们将在本章中逐一介绍这三类。

### 8.1. 元组、列表、集合和字典

内置的集合类型——元组、列表、集合和字典——可以通过几种不同的方式构造。

#### 8.1.1. 构造函数

首先，我们可以使用类型构造函数`tuple`、`list`、`set`和`dict`。

```python
>>> tuple()
()
>>> list()
[]
>>> set()
set()
>>> dict()
{}
```

① 调用不带任何参数的`tuple()`类型构造函数会创建一个空元组，其字面表示为`()`。
② 调用`list()`构造函数，创建一个空列表`[]`。
③ 此构造函数调用`set()`创建一个空集合。注意，空集合没有字面语法。
④ 此调用`dict()`创建一个空字典`{}`。

这些构造函数也接受可迭代类型的参数作为其初始元素。例如，

```python
>>> tuple((1, 2)), tuple([1, 2])
((1, 2), (1, 2))
>>> list((1, 2)), list([1, 2])
([1, 2], [1, 2])
>>> set((1, 2)), set([1, 2])
({1, 2}, {1, 2})
>>> dict([('a', 1), ('b', 2)])
{'a': 1, 'b': 2}
```

① 一个包含两个元素可迭代对象的可迭代对象。

此外，`dict`构造函数支持一种特殊的关键字参数语法。`dict`也可以用另一个映射对象初始化。例如，

```python
>>> x = dict(a = 1, b = 2)
>>> x
{'a': 1, 'b': 2}
>>> y = dict(x)
>>> y
{'a': 1, 'b': 2}
```

#### 8.1.2. 字面量语法

创建内置集合的另一种方式是使用字面量语法，

- 通过显式列出元素，或者
- 通过提供生成元素列表的“指令”，这被称为*推导式*。

以下是一些显式字面量语法的示例：

```
(1, 2, 3) ①
[1, 2, 3] ②
{1, 2, 3} ③
{"a": 1, "b": 2, "c": 3} ④
```

① 此字面量创建一个包含三个整数元素 `1`、`2` 和 `3` 的 `tuple`（元组），元素被一对圆括号 `((...))` 包围。对于单元素元组，需要一个尾随逗号。例如，`(1,)`。
② 一个 `list`（列表）字面量，被一对方括号 `([...])` 包围。
③ 一个 `set`（集合）字面量，被一对花括号 `({...})` 包围。
④ 此字面量创建一个包含三个键值对 `("a", 1)`、`("b", 2)` 和 `("c", 3)` 的 `dict`（字典）。字典使用与集合相同的花括号 `({...})` 作为其字面量。字面量 `{}` 代表一个空字典。

#### 8.1.3. 推导式

列表、集合和字典（但不包括元组）也可以使用推导式语法来构造。推导式由以下通用组件组成：

- 一个表达式，后跟
- 至少一个 `for` 子句，以及
- 零个或多个 `if` 或（嵌套的）`for` 子句。

`for` 子句具有以下通用语法：

- `for` 关键字，或 `async for`，
- 目标变量列表，以及
- `in` 关键字，后跟
- 一个可迭代对象，或一个 `async` 可迭代对象。

### 8.2. 序列

序列是由非负整数索引的有序集合。序列 `a` 的第 `i` 项通过 `a[i]` 选择。索引范围从 0 到 `len(a) - 1`。`len` 是一个内置函数，返回序列中的项目数量。

“切片”一个序列会产生另一个相同类型的序列，称为切片。例如，`a[i:j]` 选择索引在 `i`（包含）和 `j`（不包含）之间的所有项目。切片的索引从 0 开始，无论其如何切片。

#### 8.2.1. 序列解包

一个 `表达式列表` 会求值为一个 `tuple` 对象。当生成的元组被赋值给单个变量时，这被称为“元组打包”：

```
>>> x = 1, 'a', True
>>> x
(1, 'a', True)
```

① “元组打包”。`x` 的类型是 `tuple`。右侧的三个值已被“打包”到一个变量中。

另一方面，一个序列类型对象可以被“解包”到多个变量。

```
>>> p, q, r = ['a', 'b', 'c'] ①
>>> p, q, r
('a', 'b', 'c')
```

① 右侧包含三个项目的单个列表已被“解包”到三个变量 p、q 和 r。

通常，变量的数量必须与序列中的元素数量相同。然而，Python 支持一种用于“剩余元素”的特殊语法。例如，

```
>>> p, *q = [1, 2, 3, 4] ①
>>> p, q
(1, [2, 3, 4])
```

① 在解包语法中，最多可以有一个变量用 * 标记。此变量在每个前导或尾随元素被映射到相应的变量后，获取序列中的所有剩余元素。注意 q 的类型是 list。

### 8.3. 不可变序列

不可变序列类型的对象一旦创建就无法更改。（我们之前解释过“不可变性”在 Python 中的确切含义。）

#### 8.3.1. 字符串

字符串是表示 Unicode 码位的值序列。Python 没有字符类型。内置函数 `ord` 将码位从其字符串形式转换为范围在 0 - 0x10FFFF 内的整数。另一方面，`chr` 将范围在 0 - 0x10FFFF 内的整数转换为相应的长度为 1 的字符串对象。

字符串可以使用 `+` 运算符连接。将两个字符串相加会返回一个*新的*字符串。字符串在 Python 中是真正不可变的。

#### 8.3.2. 元组

`tuple` 是一个不可变序列，包含零个、一个或多个任意 Python 对象，位于一对圆括号内。

元组的*长度*，即元组中元素或项目的数量，可以使用 Python 内置函数 `len` 计算。

```
>>> len((False, "Hello", 1_000_000))
3
```

#### 8.3.3. 字节序列

Python 没有字节类型。相反，它有一个 `bytes` 序列类型，这是一个不可变的 8 位字节数组。

### 8.4. 可变序列

可变序列的项目在创建后可以更改，例如，使用赋值或 `del`（删除）语句。

#### 8.4.1. 列表

`list` 类型是 Python 中另一个重要的内置数据类型。列表是一个可变序列，包含零个、一个或多个任意 Python 对象。列表可以使用内置的 `list` 方法进行修改，例如 `append`、`insert`、`pop` 和 `remove` 等。列表中元素的顺序可以使用 `reverse` 和 `sort` 方法进行更新。

与元组一样，可以使用内置函数 `len` 获取列表中的元素数量。例如，

```
>>> len([1.0, 2.0, 3.0])
3
```

#### 8.4.2. 列表推导式

除了显式字面量语法外，还可以使用 `列表推导式语法` 创建 `list`。例如，

```
>>> [ x**x for x in range(5) ] ①
[0, 1, 4, 9, 16]
>>> [ x**x for x in range(5) if x % 2 == 0 ] ②
[0, 4, 16]
```

① 此表达式求值为 `[0, 1, 4, 9, 16]`。
② `if` 子句可用于过滤元素。

列表推导式以及列表字面量表达式，每次求值时都会产生一个*新的*列表对象。

#### 8.4.3. 字节数组

`bytearray` 对象类似于 `bytes` 序列，但它们是可变的。`bytearray` 对象由内置的 `bytearray` 构造函数创建。

### 8.5. 集合类型

集合类型表示唯一、不可变对象的无序集合。（集合类型的元素类似于 `映射类型` 的键。）

集合中的项目不能使用下标语法进行索引。但它们可以像其他序列类型一样被迭代。（即，集合是一个 `可迭代对象`。）内置函数 `len` 返回集合中的项目数量。

#### 8.5.1. 集合

`set` 表示一个可变集合。它们可以由内置的 `set` 构造函数创建，如前所述。

#### 8.5.2. 集合推导式

除了显式字面量语法外，还可以使用 [集合推导式语法](https://docs.python.org/3/reference/expressions.html#set-displays) 创建 `set`。例如，

```
>>> type({1, 3, 5, 7})
<class 'set'>
>>> { x*2 for x in range(5) }
{0, 2, 4, 6, 8}
```

① 一个集合字面量，`{1, 3, 5, 7}`。
② 此表达式求值为 `{0, 2, 4, 8, 6}`。

集合推导式以及包含逗号分隔元素列表的字面量表达式，每次求值时都会返回一个*新的*可变集合对象。

#### 8.5.3. 冻结集合

内置的 `frozenset` 构造函数创建一个 `frozenset` 对象，这是一个不可变集合。

### 8.6. 映射

映射类型表示由任意索引集（或键）索引的对象的有序或无序集合。内置函数 `len` 返回映射对象中的项目数量。下标表示法 `a[k]` 从映射 `a` 中选择由 `k` 索引的项目。这可以用于，例如，作为赋值或 `del` 语句的目标。

#### 8.6.1. 字典

字典是 Python 中内置的可变映射类型。它们表示由支持常量哈希值的任何类型的值索引的对象的有限集合。

字典保留插入顺序。也就是说，键将按照它们被顺序添加到字典中的相同顺序产生。

#### 8.6.2. 字典推导式

除了通过列出元素的字面量语法外，还可以使用 [字典推导式语法](https://docs.python.org/3/reference/expressions.html#dict) 创建新的 `dict` 对象。例如，

```
>>> { str(x): x**2 for x in range(4) } ①
{'0': 0, '1': 1, '2': 4, '3': 9}
```

① 注意 `for` 子句之前的表达式语法 `(k:v)`。

#### 8.6.3. 元素插入/删除

```
>>> d = {"a": 1}
>>> d['b'] = 2 ①
>>> del(d['a']) ②
>>> d
{'b': 2}
```

① 如果具有给定键的元素存在，则覆盖现有元素。否则，插入一个新元素。

② `del` 语句将在本书后面描述。

## 第9章 表达式

一个“表达式”由操作符和操作数组成，它会计算出一个值。一个值（简单来说）也是一个表达式。

在Python中，表达式的基本构建块被称为“原子”。原子包括[词法标记](lexical tokens)，如标识符和字面量，并且它们可以嵌套。用圆括号、方括号或花括号括起来的表达式在语法上也是原子。

在更高一级，以下“初等表达式”代表了结合最紧密的操作：

**属性引用**
一个初等表达式后跟一个句点和一个名称，称为属性引用，它本身也是一个初等表达式。对象的属性引用可以通过重写其 `__getattr__` 方法来自定义。

**下标**
序列（如字符串、元组或列表）或映射对象（如字典）的下标或索引，从给定的集合中选择一个项。用户定义的对象可以通过实现 `__getitem__` 方法来支持下标。

**切片**
切片从序列对象中选择一个范围内的项。切片可以用作表达式，也可以用作赋值或 `del` 语句中的目标。初等表达式使用一个由切片列表构造的键进行索引（使用与下标相同的 `__getitem__` 方法），如下所示。

- 如果切片列表包含至少一个逗号，则键是一个包含切片项转换结果的元组。
- 否则，单个切片项的转换结果就是键。

### 调用

一个“调用”使用一个（可能为空的）参数序列来调用一个可调用对象（例如，一个 `函数` 或 `类`）。所有具有 `__call__` 方法的对象都是可调用的，例如内置函数、内置对象的方法、类对象、类实例的方法以及用户定义的函数。在尝试调用之前，所有参数表达式都会被求值。

Python中有几种不同类型的表达式。一个表达式可以包含一个或多个其他（子）表达式，例如，通过操作符组合。

- 算术转换。
- 算术运算，一元和二元。
- 位运算，一元和二元。
- 移位运算。
- 比较（`==`、`!=`、`>`、`>=`、`<`、`<=`）
- 布尔运算（`not`、`and` 和 `or`）。
- 赋值表达式。
- 条件表达式。

[Lambda表达式](https://docs.python.org/3/reference/expressions.html#lambda)和[await表达式](https://docs.python.org/3/reference/expressions.html#await)将在本书后面描述。

### 9.1. 表达式列表

表达式列表是一系列由逗号分隔的一个或多个表达式。尾随逗号是可选的，除非表达式列表只包含一个表达式。表达式列表中的表达式从左到右求值。

一个或多个表达式的表达式列表会生成一个具有相应元素数量的元组。例如，

```
>>> 50 + 50, 100 * 2
(100, 200)
```

① Python解释器首先计算 `50 + 50`，其值为 `100`，然后计算 `100 * 2`，其值为 `200`。
② 这个表达式列表的值是一个元组，例如，在本例中表示为元组字面量 `(100, 200)`。

```
>>> 1 / 2
0.5
>>> 1 / 2,
(0.5,)
```

① 表达式 `1 / 2` 的值是 `0.5`。
② 表达式列表 `1 / 2,` 的值是 `(0.5,)`。对于单表达式表达式列表，尾随逗号是必需的。

“可迭代解包”语法，使用星号 `*` 操作符，可用于表达式列表中 `可迭代` 类型的表达式。

```
>>> a, b, c = 1, [2, 3, 4], 5
>>> a, b, c
(1, [2, 3, 4], 5)
>>> a, *b, c
(1, 2, 3, 4, 5)
>>> *b,
(2, 3, 4)
```

① `b` 是一个列表，一种可迭代类型。
② 解包 `b`。`b` 的元素现在是结果元组的一部分。
③ 在包含单个可迭代表达式的表达式列表中解包。注意，尾随逗号是必需的。

### 9.2. 求值顺序

作为一般规则，Python根据操作符优先级规则从左到右对表达式求值。

在对赋值求值时，右侧在左侧之前求值。然而，在“增强赋值”表达式中，左侧目标在右侧之前求值。然后，最终结果被赋回给左侧目标。

```
>>> a, b = 1 + 2, 2 * 5 ①
>>> a, b
(3, 10)
>>> a += 1 + 2 * 3 ②
>>> a
10
```

① 表达式 `1 + 2` 和 `2 * 5` 首先被求值（从左到右），生成的值（一个元组）被“解包”并分别赋给 `a` 和 `b`。

② 在此语句中，`a` 的（当前）值首先被求值，为3。接下来右侧被求值为7（因为乘法 `2 * 3` 的优先级高于加法 `+`）。然后计算 `3 + 7`，其结果被赋回给 `a`。

### 9.3. 赋值表达式

赋值表达式将赋值运算符 `:=`（非正式地称为“海象”运算符）右侧的表达式赋给左侧的名称，并返回其值（与目标的值相同）。

赋值表达式通常用于复合语句中，其中需要保留表达式求值的结果，例如，以便可以在语句体中使用。例如，

```
>>> import random
>>> def zero_or_not():
...     return random.randint(0, 4)
...
>>> while r := zero_or_not():
...     print("Not zero:", r)
...
Not zero: 1
Not zero: 1
Not zero: 4
```

① `while` 子句依赖于函数调用 `zero_or_not()` 的结果。
② 这里使用了*相同的值*。因此，该值之前已存储在变量 `r` 中。

### 9.4. 条件表达式

除了条件语句，Python还通过 `if - else` 表达式语法支持条件表达式。在表达式 `x if C else y` 中，包含三个（子）表达式 `C`、`x` 和 `y`，如果 `C` 的值为 `True`，则表达式的值为 `x`。否则，其值为 `y`。

例如，

```
family = input("What is your family name? ")
print(f"You are {'a reptile' if family == 'Python' else 'nobody'}!")
```

Python的 `if` 表达式对应于其他C风格语言中的三元运算符 `? :`。表达式 `x if C else y` 大致等同于那些语言中的 `C ? x : y`。

请注意，无论 `C` 的值如何，这三个（子）表达式中*只有两个*会被求值。条件 `C` 总是首先被求值，如果为真，则求值 `x`，但不求值 `y`。否则，求值 `y`，但不求值 `x`。

### 9.5. 算术转换

当执行二元算术运算时，

- 如果任一参数是复数，则另一个被转换为复数，并且
- 否则
    - 如果任一参数是浮点数，则另一个被转换为浮点数，并且
    - 否则，两个参数都是整数，Python不进行自动转换。

### 9.6. 算术运算

#### 9.6.1. 一元算术运算符

一元 `-`（减号）运算符生成其数值参数的相反数。一元 `+`（加号）运算符生成其数值参数本身。在这两种情况下，如果参数没有正确的类型，将引发 `TypeError` 异常。

所有一元算术和位运算具有相同的优先级。

#### 9.6.2. 二元算术运算符

`*`（乘法）运算符生成其参数的乘积。参数必须都是数字，在这种情况下它们相乘；或者一个参数必须是整数，另一个必须是序列，在这种情况下执行序列重复。负的重复因子生成一个空序列。

`/`（除法）和 `//`（整除）运算符生成其参数的商。浮点数或整数的除法生成浮点数，而整数的整除结果为整数。除以零会引发 `ZeroDivisionError` 异常。

`%`（取模）运算符生成第一个参数除以第二个参数的余数。右参数为零会引发 `ZeroDivisionError` 异常。参数可以是浮点数，例如，`3.14 % 0.7` 等于 `0.34`（因为 `3.14` 等于 `4 * 0.7 + 0.34`）。取模运算符总是生成与第二个操作数符号相同的结果（或为零）。

整除和取模运算符通过以下恒等式相关联：`x == (x // y) * y + (x % y)`。整除和取模也与内置函数 `divmod` 相关联：`divmod(x, y) == (x // y, x % y)`。

`+`（加法）运算符生成其参数的和。参数必须都是数字，在这种情况下它们相加；或者都是相同类型的序列，在这种情况下序列被连接。`-`（减法）运算符生成其参数的差。

#### 9.6.3. 幂运算符

幂运算符（`**`）接受两个参数，其工作方式与内置函数 `pow` 相同。也就是说，`x ** y` 等同于 `pow(x, y)`，它生成 `x` 的 `y` 次幂的值。例如，

```
>>> 2 ** 3
8
>>> 2 ** -3
0.125
>>> -2 ** 0.5
-1.4142135623730951
>>> (-2) ** 0.5
(8.659560562354934e-17+1.4142135623730951j)
>>> 10 ** 0
1
>>> 0.0 ** 2
0.0
>>> 0.0 ** -2
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
ZeroDivisionError: 0.0 cannot be raised to a negative power
```

① `2 ** 3` 等同于 `2 * 2 * 2`，结果是整数 `8`。

② `2 ** -3` 等同于 `1.0 / (2 * 2 * 2)`，即使两个操作数都是整数，结果也是一个浮点数。

③ `-2 ** 0.5` 等同于 `- (2 ** 0.5)`，即 -1 乘以 2 的平方根。

④ 另一方面，对负数进行分数幂运算会返回一个复数。

⑤ 无论 `x` 是什么，`x ** 0` 的结果都是 `1`。同样，`x ** 0.0` 的结果总是 `1.0`。

⑥ 对于整数和浮点数 `x`，`0 ** x` 的结果分别是 `0` 和 `0.0`。无论 `x` 的类型和值是什么，`0.0 ** x` 的结果总是 `0.0`。

⑦ 零（`0` 或 `0.0`）的负数次幂会引发 `ZeroDivisionError` 异常。

### 9.7. 按位运算

#### 9.7.1. 一元按位运算符

一元 `~`（取反）运算符仅适用于整数。它返回其整数参数的按位取反值。`x` 的按位取反定义为 `-(x+1)`。如果参数类型不正确，将引发 `TypeError` 异常。

#### 9.7.2. 二元按位运算符

- 按位与（`&`）运算符返回其整数参数的按位与结果。
- 按位异或（`^`）运算符返回其整数参数的按位异或结果。
- 按位或（`|`）运算符返回其整数参数的按位或（包含或）结果。

#### 9.7.3. 移位运算符

移位运算符接受整数作为参数。它们将第一个参数向左（`<<`）或向右（`>>`）移动，移动的位数由第二个参数指定。

- 右移 `n` 位定义为除以 `pow(2, n)` 的向下取整。
- 左移 `n` 位定义为乘以 `pow(2, n)`。

### 9.8. 布尔运算

求值结果为布尔值（`True` 或 `False`）的表达式称为布尔表达式。因此，`True` 是一个（平凡的）布尔表达式，`False` 也是。

```
>>> True
True
>>> type(False)
<class 'bool'>
```

#### 9.8.1. not

运算符 `not` 的结果是：

- 如果其参数为假，则结果为 `True`，否则
- 结果为 `False`。

#### 9.8.2. and

表达式 `x and y` 首先求值 `x`，然后：

- 如果 `x` 求值为假，则返回其值（该值可能不是布尔类型）。
- 否则，求值 `y`，并返回其结果值。

请注意，整个表达式的类型实际上是 `x` 和 `y` 类型的并集（不一定是布尔类型）。也就是说，如果 `x` 的布尔值为假，则 `and` 表达式的类型就是 `x` 的类型。否则，就是 `y` 的类型。

#### 9.8.3. or

表达式 `x or y` 首先求值 `x`，然后：

- 如果 `x` 求值为真，则返回其值（该值同样可能不是 `True`）。
- 否则，求值 `y`，并返回其结果值。

整个表达式 `x or y` 的类型同样是 `x` 和 `y` 类型的并集。也就是说，如果 `x` 的布尔值为真，则 `or` 表达式的类型就是 `x` 的类型。否则，就是 `y` 的类型。

### 9.9. 比较运算

比较运算产生布尔值：`True` 或 `False`。Python 中的比较运算可以任意链接。例如，`x < y <= z` 是一个有效的表达式，这与许多其他 C 风格语言不同，它大致等价于 `x < y and y <= z`。或者更精确地说，它等价于 `x < (t := y) and t <= z`，其中使用了一个临时变量 `t`。请注意，`y` 只被求值一次。

通常，如果 `a, b, c, ..., y, z` 是表达式，而 `op1, op2, ..., opN` 是（二元）比较运算符，那么 `a op1 b op2 c ... y opN z` 在语义上等价于 `a op1 b and b op2 c and ... y opN z`，但每个表达式最多只被求值一次。

#### 9.9.1. 同一性比较

运算符 `is` 和 `is not` 测试对象的同一性：`x is y` 为真当且仅当 `x` 和 `y` 是同一个对象。对象的同一性通过 `id` 函数确定。`x is not y` 返回相反的真值。

#### 9.9.2. 值比较

运算符 `<`、`>`、`==`、`>=`、`<=` 和 `!=` 比较两个对象的值。对象不需要具有相同的类型。

因为所有类型都是（直接或间接的）`object` 的子类型，所以它们从 `object` 继承了默认的比较行为。类型可以通过实现比较魔术方法（如 `__lt__` 和 `__gt__` 等）来自定义其比较行为。

相等比较（`==` 和 `!=`）的默认行为基于对象的同一性。因此，具有相同同一性的实例进行相等比较会得到相等，而具有不同同一性的实例进行相等比较会得到不相等，这在许多情况下可能违反直觉。通常，用户定义的类型需要自定义其比较行为。

#### 9.9.3. 成员测试运算

对于集合，运算符 `in` 和 `not in` 用于测试成员关系。如果 `x` 是 `s` 的成员，则 `x in s` 求值为 `True`，否则为 `False`。`x not in s` 返回 `x in s` 的否定。所有内置序列和 `set` 类型都支持此操作。对于 `dict`，`in` 运算符测试字典是否包含给定的键。

对于 `string` 和 `bytes` 类型，`x in y` 为真当且仅当 `x` 是 `y` 的子串。运算符 `not in` 被定义为具有与 `in` 相反的真值。例如，

```
>>> 2 in [1, 2, 3, 2]
True
>>> 'c' not in { 'a', 'b', 'd'}
True
>>> "k1" in { "k1": "v1", "k2": "v2" }
True
>>> "world" in "hello world"
True
```

对于定义了 `__contains__` 方法的用户自定义类，如果 `y.__contains__(x)` 返回真值，则 `x in y` 返回 `True`，否则返回 `False`。

## 第 10 章. 简单语句

与表达式不同，语句主要用于产生副作用，例如控制程序流程或生成输出。一个简单语句包含一个逻辑行。多个简单语句可以用分号分隔出现在同一行上。我们将在下一章讨论[复合语句](compound statements)。

简单语句包括：

- 表达式语句，
- 赋值语句，
- `assert` 语句，
- `pass` 语句，
- `del` 语句，
- `return` 语句，
- `raise` 语句，
- `break` 语句，
- `continue` 语句，
- `global` 语句，以及
- `nonlocal` 语句。

`import` 语句在参考手册的开头，[包和模块](Package and Modules)中进行了解释。`yield` 语句在协程的上下文中描述，或者更具体地说，在 [yield 表达式](the yield expressions)中描述。

### 10.1. 表达式语句

在 Python 中，一个表达式，或者通常说一个表达式列表，可以用作语句，例如，仅仅为了它的副作用。这被称为表达式语句，它对表达式列表求值并丢弃其结果。最常见的用例是调用一个返回 `None` 的函数。另一个常见用例是使用字符串表达式作为文档。

例如，

```
def a():
    "Nada"
    return "Something"

a()
```

① 函数定义 `a()` 包含一个表达式语句，它是一个字符串字面量。当调用此函数时，这个字符串字面量将被“求值”但被忽略。

② 调用函数 `a()` 是一个表达式，但这里它被写成了一个语句。函数的返回值 `"Something"` 被丢弃。

### 10.2. 赋值语句

赋值是 Python 中的简单语句之一。赋值语句主要用于将一个新名称绑定到一个对象，或将一个现有名称重新绑定到一个对象。它们也用于修改可变对象的属性或元素。

赋值语句的左侧可以是：

1. 一个新名称或一个引用对象的名称，或者
2. 一个可变对象的属性或元素。

赋值语句从左到右对右侧的表达式或表达式列表求值，并将单个

### 10.2. 赋值语句

将结果对象从左到右依次赋值给左侧对应的目标列表。如果目标列表是一个没有尾随逗号的单个目标，则将对象赋值给该目标。（如前所述，将表达式列表赋值给目标列表是[元组打包](https://docs.python.org/3/reference/expressions.html#tuple-packing)和[序列解包](https://docs.python.org/3/reference/expressions.html#sequence-unpacking)的组合。）

在 Python 中，我们也可以在一条语句中将多个名称绑定到同一个对象。例如，

```
>>> x = y = "dragon" ①
>>> p = q = ['a', 'b'] ②
>>> id(x), id(y)
(140593323269680, 140593323269680)
>>> id(p), id(q)
(140593324864960, 140593324864960)
```

① 变量 `x` 和 `y` 引用同一个对象。
② `p` 和 `q` 引用同一个对象。

#### 10.2.1. 增强赋值语句

增强赋值语句是二元运算和赋值语句的组合。增强赋值中使用的运算符有：`+=`、`-=`、`*=`、`@=`、`/=`、`//=`、`%=`、`**=`、`>>=`、`<<=`、`&=`、`^=` 和 `|=`。

与普通赋值不同，增强赋值会先计算左侧的目标，然后计算右侧的表达式列表。接着执行二元运算，并将结果写回目标。

> Python 的赋值语句不会复制源对象，这与某些其他编程语言不同。它只是将名称绑定或重新绑定到对象。要进行复制，标准库的 copy 模块提供了浅拷贝和深拷贝的函数。

### 10.3. pass 语句

pass 语句用作占位符，例如，在 Python 语法要求语句的地方。当执行该语句时，什么也不会发生。pass 是一个空操作。它主要用于开发期间，没有其他用途。

例如，

```
def find_numbers(arg):
    pass        # To be implemented
```

① 函数定义在语法上至少需要一个语句。

```
class OrdinalNumber:
    pass        # Placeholder
```

① 类定义也是如此。显然，这些注释并非必需，因为 pass 语句的使用通常意味着它们是占位符。

### 10.4. return 语句

return 语句在语法上只允许出现在函数定义内部。执行时，它会退出当前函数调用。return 语句可以包含一个可选的参数，即一个表达式列表。如果存在，该表达式列表将作为函数调用的值返回给调用者。如果不存在，则假定返回值为 None。

```
>>> def f():
...     return 1, 2, 3 ①
...
>>> f()
(1, 2, 3) ②
```

① 函数 `f` 返回一个包含三个表达式的表达式列表。
② 函数调用表达式 `f()` 的值是一个元组，例如，通过[元组打包](https://docs.python.org/3/reference/expressions.html#tuple-packing)实现。

当 `return` 将控制权从带有 `finally` 子句的 `try` 语句中传出时，该 `finally` 子句会在离开函数之前执行。

### 10.5. `raise` 语句

`raise` 语句引发或抛出一个异常或异常组：

- 如果提供了参数，
    - 如果它求值为 `BaseException` 或其子类的实例，则引发该异常/异常组对象。
    - 如果表达式是一个类，则创建一个不带参数构造的该类的实例。
- 如果未提供参数，
    - 如果当前作用域中存在一个活跃的异常，则重新引发该异常。
    - 否则，它是一个 `RuntimeError`。

当在另一个异常活跃时引发新异常时，新异常可以使用 `__cause__` 属性与现有异常关联。这被称为“异常链”。这可以通过在 `raise` 语句末尾附加 `from` 子句来实现。

```
>>> class UnusualException(Exception): pass ①
...
>>> raise UnusualException ②
Traceback (most recent call last): ③
  File "<stdin>", line 1, in <module>
__main__.UnusualException
```

① 新的异常类型应继承自 Exception(BaseException)。

② 可以引发继承自 BaseException 的异常。

③ Python REPL 捕获了该异常。通常，我们需要 try - except 语句来捕获引发的异常。否则，程序将崩溃。

```
>>> try: raise UnusualException
... except: raise ①
...
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
__main__.UnusualException
```

① 重新引发当前异常。

```
>>> try:
...     raise UnusualException("Unusual")
... except BaseException as ex: ①
...     raise UnusualException("Really unusual") from ex ②
...
Traceback (most recent call last):
  File "<stdin>", line 2, in <module>
__main__.UnusualException: Unusual

The above exception was the direct cause of the following exception:

Traceback (most recent call last):
  File "<stdin>", line 4, in <module>
__main__.UnusualException: Really unusual
```

① as 子句用于为异常命名。

② 使用 raise - from 语法进行异常链。

### 10.6. break 语句

break 语句可以在 for 或 while 循环中使用，只能直接包含，而不能作为循环内函数或类定义的一部分。它会终止最内层的封闭循环。如果循环有 else 子句，则会被跳过。

```
>>> for i in range(3): ①
...     try:
...         print(i)
...         if i == 1:
...             break ②
...         print(i)
...     finally:
...         print("finally")
... else:
...     print("huh?") ③
...
0
0
finally
1 ④
finally ⑤
```

① 一个 for 循环。

② 此 for 循环中的 break 语句。

③ 当循环因 break 语句终止时，else 子句不会执行。

④ break 语句在打印第一个 1 后执行。第二个 print 语句被跳过。

⑤ finally 子句在退出循环之前仍然执行。

### 10.7. continue 语句

与 break 类似，continue 语句只能直接在 for 或 while 循环中使用。它继续执行最内层封闭循环的下一次迭代，而不执行当前循环中剩余的语句。

```
>>> for i in range(3):
...     try:
...         print(i)
...         if i == 1:
...             continue
...         print(i)
...     finally:
...         print("finally")
... else:
...     print("huh?")
...
0
0
finally
1
finally
2
2
finally
huh
```

① 一个 for 循环。

② 此 for 循环中的 continue 语句。

③ continue 语句在打印第一个 1 后执行。第二个 print 语句被跳过。

④ finally 子句在继续下一次迭代之前仍然执行。

⑤ 继续下一次迭代。

⑥ 当循环正常终止时，else 子句执行。

### 10.8. global 语句

global 语句，关键字 global 后跟一个逗号分隔的名称列表，表示在当前代码块中声明的名称将被解释为全局变量（例如，当前模块作用域）。在 global 语句之前，不能在同一代码块中使用 global 语句中列出的名称。

注意 Python 中名称绑定和使用之间的不对称性。在非全局代码块中，无法在不使用 global 声明的情况下将值绑定到全局变量。另一方面，要访问全局变量的值，则不需要 global 声明。

```
>>> def f():
...     print(x)          ①
...     global y          ②
...     y = 4             ③
...     print(y)
...
>>> x, y = 1, 2          ④
>>> f()                  ⑤
1
4
```

① 在函数作用域中访问（后来定义的）全局变量 x 不需要 global 声明。注意变量 x 在函数 f 内部使用而没有先声明。这被称为“自由变量”。

② 这个 global 声明允许对全局变量 y 进行赋值。

③ 在 global 语句之后，我们可以为全局变量赋值。现在，在这个例子中，y 被绑定到不同的对象 4。

④ 声明了全局变量 x 和 y。

⑤ 函数调用 f() 使用了这些全局变量。

### 10.9. nonlocal 语句

nonlocal 语句的工作方式与 global 语句类似。在 nonlocal 语句中声明的名称引用最内层封闭作用域中先前绑定的变量（不包括全局变量）。

```
>>> def a():
...     x, y = 1, 2
...     def b():
...         print(x)
...         nonlocal y
...         y = 4
...         print(y)
...     b()
...
>>> a()
1
4
```

① x 和 y 是函数 a 的函数定义代码块的局部变量。

### 10.10. `del` 语句

`del` 语句会删除列出的目标名称。删除目标列表时，会从左到右递归删除每个目标。如果目标名称未绑定，将引发 `NameError` 异常。

删除名称会从局部或全局命名空间中移除该名称的绑定，具体取决于该名称是否出现在同一代码块的 `global` 语句中。

例如，

```
>>> x = 1
>>> def f():
...     global x
...     del x ①
...
>>> print(x)
1
>>> f() ②
>>> print(x) ③
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
NameError: name 'x' is not defined
```

① 此 `del` 语句删除了全局变量 `x`。

② 调用 f() 将执行 del 语句。

③ 尝试使用已删除/未绑定的名称将引发 NameError 异常。

### 10.11. assert 语句

assert 语句是一种向程序中插入调试断言的便捷方式。它有两种形式。基本形式：

```
assert <expression>
```

这等价于：

```
if __debug__:
    if not <expression>: raise AssertionError
```

请注意，内置变量（如 `__debug__`）的值在 Python 解释器启动时确定，且无法修改。变量 `__debug__` 在正常情况下为 `True`，当请求优化时（命令行选项 `-O`）为 `False`。当在编译时请求优化时，assert 语句不会生成任何代码。

扩展形式：

```
assert <expression1>, <expression2>
```

等价于

```
if __debug__:
    if not <expression1>: raise AssertionError(<expression2>)
```

例如，

```
>>> a = 1, 2, 3, ①
>>> assert a == (1, 2, 3) ②
>>> assert a == [1, 2, 3] ③
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
AssertionError
```

① 表达式列表求值为一个元组。

② 此 assert 语句成功，因此没有输出。

③ 此断言失败，抛出一个 AssertionError。

## 第 11 章. 复合语句

复合语句包含其他语句，并以某种方式影响或控制这些语句的执行。复合语句可以跨越多个（逻辑）行。

复合语句由一个或多个“子句”组成。子句由一个“头部”和一个“代码块”组成。特定复合语句的所有子句头部都处于相同的缩进级别。每个子句头部以一个唯一标识的关键字（例如 `if`、`else`、`try`、`except` 等）开始，并以冒号 (:) 结束。代码块是由子句控制的一组一个或多个语句。代码块可以是

- 与头部在同一行上、冒号之后的一个或多个用分号分隔的简单语句，或者
- 可以是后续行上缩进多一级（简单或复合）的一个或多个语句。

Python 与许多命令式编程语言一样，使用常见的控制流语句，如 `if` 语句、`while` 语句和 `for` 语句。它们在 Python 中都是复合语句。`for` 语句通常与内置的 `range` 和 `enumerate` 函数一起使用。

`try` 语句与简单的 `raise` 语句一起用于异常处理和/或提供清理代码。另一个复合语句 `with` 语句与上下文管理器密切相关，用于为一系列语句提供初始化和终结代码。

新的 `match` 语句（3.10 版本新增）将在模式匹配的更广泛上下文中稍后讨论。同样，另外两个重要的复合语句，`function def` 语句和 `class` 定义语句，将在各自的章节中讨论。最后，协程和其他相关语句将在本书的最后一章中单独解释。async/await 是 Python 的新（相对较新的）用于高级异步编程的构造。

### 11.1. if - elif - else 语句

if 语句用于条件执行。它通过从上到下依次计算表达式，直到找到一个为 True 的表达式，从而最多选择一个代码块。然后执行相应的代码块（并且 if 语句的其他部分不会被执行或计算）。如果所有表达式都为 False，则执行 else 子句的代码块（如果存在）。

例如，

```
1 from datetime import datetime
2
3 weekday = datetime.today().isoweekday() ①
4
5 if weekday == 7: ②
6     println("Today is Sunday!")
7 elif weekday == 6: ③
8     println("Today is Saturday!")
9 else: ④
10     println("Just another day. :(")
11
12 println("Regardless, let's learn some programming!")
```

① datetime 的 isoweekday 方法返回整数值，1 代表星期一，2 代表星期二，7 代表星期日，依此类推。

② 此 if 语句包含三个子句：if（第 5-6 行）、elif（第 7-8 行）和 else（第 9-10 行）。if 表达式检查今天是否是星期日。如果是，它将打印出“今天是星期日！”，然后控制权移动到 if 语句的末尾，例如第 11 行。

③ 如果今天不是星期日，则接着测试是否是星期六（6）。如果此表达式求值为 True，则执行代码块中的语句（在此示例中为单个语句，第 8 行），并且 if 语句的执行终止。

④ 在所有其他情况下，此 if 语句最终将打印“Just another day. :("。执行 if 语句后（无论执行了哪个子句），程序执行将移动到下一条语句，在此示例中为第 12 行。

请注意，与许多 C 风格语言不同，Python 的 if 语句语法不需要在 if 和 elif 行中的布尔表达式周围加括号。括号仍可用于分组目的。

### 11.2. while - else 语句

while 语句用于重复执行，类似于 for 语句。while 语句的执行取决于一个表达式。它在每次迭代开始时测试该表达式，并且只要表达式求值为 True，它就执行第一个代码块。如果表达式变为 False（可能是在第一次测试时），则执行 else 子句的代码块（如果存在），并且循环终止。

对于 while 和 for 语句，

- while/for 代码块中的 break 语句会终止循环，而不执行 else 子句的代码块。
- while/for 代码块中的 continue 语句会跳过剩余的语句，并返回到测试表达式。

Python 3.0 于 2008 年首次发布，现在（自 2020 年起？）采用年度发布计划。让我们打印出该信息：

```
>>> year = 2007
>>> while (year := year + 1) and (year < 3000): ①
...     if year < 2018: ②
...         if year == 2008:
...             print(f"Python 3 was first released in {year}.")
...         continue ③
...     elif year == 2022: ④
...         print("Python 3.11 is to be released at the end of 2022.")
...         break ⑤
...
...     version = f"3.{year - 2011}"
...     print(f"Python {version} in {year}.") ⑥
...
Python 3 was first released in 2008.
Python 3.7 in 2018.
Python 3.8 in 2019.
Python 3.9 in 2020.
Python 3.10 in 2021.
Python 3.11 is to be released at the end of 2022.
```

① 此 `while` 语句实际上是一个无限循环（`while True: ...`），其终止由 `break` 语句控制。添加条件 `year < 3000` 是作为预防措施，例如，以避免在代码中存在错误时出现失控迭代。赋值表达式 `year := year + 1` 在布尔上下文中始终产生 `True`（因为 `year > 2007 > 0`），此处用于说明。也可以使用赋值语句，例如，在 `while` 代码块的第一行中。

② 当 `year < 2018` 时，我们仅在 `year == 2008` 时执行 `continue`。

③ `continue` 语句。

④ 当 `year == 2022` 时，我们打印出一条特殊消息并终止循环。

### 11.3. for - in - else 语句

`for-in-else` 语句用于遍历序列（如字符串、元组或列表）或[其他可迭代对象](https://docs.python.org/3/glossary.html#term-iterable)的元素，包括字典。其语法如下：

```
for <target> in <expression_list>:
    <for_suite>
else:
    <else_suite>
```

首先，`<expression_list>` 会被求值，且仅求值一次。其结果是一个[迭代器](https://docs.python.org/3/glossary.html#term-iterator)。然后，对于迭代器提供的每个元素，`<for_suite>` 代码块会执行一次。每个元素依次使用[赋值](https://docs.python.org/3/reference/simple_stmts.html#assignment-statements)的标准规则赋值给 `<target>`，然后使用当前 `<target>` 的值执行 `<for_suite>`。当元素耗尽时，即迭代器引发 `StopIteration` 异常时，如果存在 `else` 子句，则执行其中的 `<else_suite>`，然后循环终止。

请注意，当 `<expression_list>` 返回一个空序列时，`<target>` 变量永远不会被赋值。

例如，

```
>>> features = {
...     "3.11": "the exception group",
...     "3.10": "the match statement",
...     "3.9": "the dictionary union operator",
...     "3.8": "the assignment expression",
... }
>>> for k in features:
...     print(f"""Python {k} includes many new features
...           such as {features[k]}.""")
...
Python 3.11 includes many new features
        such as the exception group.
Python 3.10 includes many new features
        such as the match statement.
Python 3.9 includes many new features
        such as the dictionary union operator.
Python 3.8 includes many new features
        such as the assignment expression.
```

① `dict` 是一个可迭代对象。

② 一个 `for` 语句，遍历一个字典。请注意，顺序被保留了。

#### 11.3.1. range 函数

借助内置的 `range` 函数（它生成一个整数序列），Python 的 `for in` 语句可以像 C 语言经典的 `for` 循环一样使用。

`range` 函数可以通过三种不同的方式调用：

```
range(start, end, step) ①
range(start, end) ②
range(end) ③
```

① 它指定了一个 `start`（包含）、一个 `end`（不包含）和一个 `step`，表示序列中的增量或减量。所有参数都是整数，且 `step` 不能为零。
② 如果省略 `step`，其默认值为 1。
③ 在这种情况下，`start` 和 `step` 分别使用默认值 0 和 1。

例如，让我们尝试使用 `for` 语句计算从 0 到 100 的偶数之和：

```
>>> _sum = 0
>>> for i in range(0, 100+1, 2):
...     _sum += i
...
>>> print(f"Sum of even numbers from 0 to 100 is {_sum}")
Sum of even numbers from 0 to 100 is 2550
```

实际上，有一个内置的 `sum` 函数可以对给定序列中的所有元素求和。

```
>>> sum(range(0, 101, 2))
2550
```

#### 11.3.2. enumerate 函数

上述两个用例，例如遍历任意序列中的元素和遍历整数序列，可以或多或少地与内置的 `enumerate` 函数结合使用。

`enumerate` 函数接受一个可迭代对象作为参数，并返回一个包含原始序列索引对的可迭代对象。也就是说，给定 [item1, item2, item3]，它返回一个包含三个元素的可迭代对象：(0, item1)、(1, item2) 和 (2, item3)。`enumerate` 函数不如 `range` 灵活，但它接受一个可选的第二个参数 `start`。否则，索引默认从 0 开始。

这里有一个例子：

```
>>> name = "python"
>>> for i, v in enumerate(name, 3):
...     print(i, v, end = ', ')
... else:
...     print()
...
3 p, 4 y, 5 t, 6 h, 7 o, 8 n,
>>>
```

① 字符串是一种（不可变的）序列类型，它是一个可迭代对象。
② 我们使用解包语法将 `enumerate()` 函数每次迭代的索引和值赋给两个独立的变量（`i`、`v`）。
③ `else` 子句总是在循环结束时执行，除非循环被异常终止，例如被 `break` 语句等终止。在这个例子中，我们在循环结束时添加了一个换行。
④ 否则，这一行就不会以换行结束。

### 11.4. try 语句

从 Python 3.11+ 开始，Python 的异常框架得到了扩展，允许程序同时引发和处理多个不相关的异常。首先，引入了一种新的标准异常类型 `ExceptionGroup`，它表示一组不相关的异常。其次，添加了一种新的语法 `except*` 来处理 ExceptionGroup。

经典的 `try - except` 复合语句具有以下通用语法：

```
try:
    <suite>
except <except_expression>: ①
    <suite>
except <except_expression> as <name>: ②
    <suite>
except: ③
    <suite>
else: ④
    <suite>
finally: ⑤
    <suite>
```

① 一个求值结果为 Exception 或 ExceptionGroup 类型的表达式。ExceptionGroup 类型在 Python 3.11 中引入。
② 一个 try 语句可以有零个、一个或多个 except 子句。
③ “捕获所有”子句。
④ 一个可选的 else 子句。
⑤ 一个可选的 finally 子句。请注意，当没有 except 子句时，finally 是必需的。

从 Python 3.11+ 开始，try 子句后面可以跟一个或多个 except* 子句，而不是 except。

```
try:
    <suite>
except* <exception_group>: ①
    <suite>
except* <exception_group> as <name>: ②
    <suite>
except* (<ex1>, <ex2>) as <name>: ③
    <suite>
else:
    <suite>
finally:
    <suite>
```

① 一个求值结果为 `Exception` 或 `ExceptionGroup` 类型的表达式。

② 一个 `try` 语句可以有一个或多个 `except*` 子句。请注意，与 `except` 不同，`except*` 不能用作“捕获所有”子句。也就是说，异常/异常组表达式总是必需的。

③ 一种快速简写方式，用于从现有异常类型创建“异常组”。

`except` 子句为特定的 `BaseException` 或 `BaseExceptionGroup` 类型指定异常处理程序。另一方面，`except*` 子句为包含在 `BaseExceptionGroup` 类型中的一组 `BaseException` 类型指定异常处理程序。`except/except*`、`else`、`finally` 都是可选的，但至少需要 `except/except*` 或 `finally` 中的一个。请注意，在一个 `try` 语句中不能混合使用 `except` 和 `except*`。

当 `try` 子句中没有发生异常时，不会执行任何异常处理程序。

当 `try` 代码块中发生异常时，会开始搜索异常处理程序。对于 `except` 子句，此搜索会从上到下依次检查 `except` 子句并求值其表达式，直到找到匹配该异常的子句。如果存在一个无表达式的 `except` 子句（它匹配任何异常），则它必须是最后一个。

对于 `except*` 子句，如果异常或异常组中的任何异常与给定的异常相同或是其基类型，则该子句匹配该异常。与 `except` 子句的情况不同，可以有多个 `except*` 子句匹配异常/异常组。因此，`except*` 子句中不允许使用 `break` 等控制流语句。

如果没有 `except` 子句匹配该异常，或者 `except*` 子句没有完全处理异常组中的所有异常，则会在周围的代码和调用栈中继续搜索该异常或剩余未处理异常的处理程序。

当找到匹配的 except 子句时，如果存在 `as` 关键字，则异常被赋值给该 except 子句中指定的目标，然后执行该 except 子句的代码块。所有 except 子句都必须有一个可执行块。当到达该块的末尾时，执行将在整个 try 语句之后正常继续。（这意味着，如果存在两个嵌套的处理程序处理同一个异常，并且该异常发生在内部处理程序的 try 子句中，则外部处理程序将不会处理该异常。）

如果控制流离开 `try` 代码块，没有引发异常，并且没有执行 `return`、`continue` 或 `break` 语句，则执行可选的 `else` 子句。`else` 子句中的异常不会被前面的 `except` 子句处理。

如果存在 `finally`，它指定一个“清理”处理程序。`try` 子句被执行，包括任何 `except` 和 `else` 子句。如果在任何子句中发生异常且未被处理，则该异常会被暂时保存。然后执行 `finally` 子句。如果存在保存的异常，则在 `finally` 子句结束时重新引发该异常。如果 `finally` 子句引发了另一个异常，则保存的异常将作为新异常的上下文。如果 `finally` 子句执行了 `return`、`break` 或 `continue` 语句，则保存的异常将被丢弃。

在执行 `finally` 子句期间，异常信息对程序不可用。

当在 `try...finally` 语句的 `try` 代码块中执行 `return`、`break` 或 `continue` 语句时，`finally` 子句也会在“退出时”执行。

### 11.4. try 语句

函数的返回值由最后执行的 `return` 语句决定。由于 `finally` 子句总是会执行，因此在 `finally` 子句中执行的 `return` 语句将始终是最后执行的那一个。

异常可以被“重新抛出”。例如，

```
>>> try:
...     sum = 100 + unknown_name
... except:
...     raise
...
Traceback (most recent call last):
  File "<stdin>", line 2, in <module>
NameError: name 'unknown_name' is not defined
>>>
```

① Python REPL 处理了错误，程序没有崩溃。它等待用户的下一个命令。

在存在活动 `Exception` 的上下文中，`raise` 语句会重新抛出当前的异常/错误。此示例代码与以下代码或多或少等价，后者显式地抛出当前异常。

```
>>> try:
...     sum = 100 + unknown_name
... except NameError as ex:
...     raise ex
...
Traceback (most recent call last):
  File "<stdin>", line 4, in <module>
  File "<stdin>", line 2, in <module>
NameError: name 'unknown_name' is not defined
```

### 11.5. with 语句

`with` 语句用于将一组语句的执行包装在由上下文管理器定义的方法中。例如，它可以用于封装常见的 `try...except...finally` 使用模式。其语法如下：

```
with Expression as Target:  # ①
    Suite
```

① `as` 子句是可选的。

`Expression` 必须求值为一个上下文管理器类型，该类型支持 `__enter__` 和 `__exit__` 方法。`with` 语句的执行过程如下：

- 对上下文 `Expression` 求值以获取一个上下文管理器。
- 首先加载上下文管理器的 `__enter__` 方法。
- 接着加载上下文管理器的 `__exit__` 方法。
- 调用 `__enter__` 方法。
- 如果指定了 `Target`，则将 `__enter__` 的返回值赋值给 `Target`。
- 执行 `Suite` 中的语句。
- 调用 `__exit__` 方法。

例如，

```
>>> import pathlib
>>> path = pathlib.Path("hello.py")
>>> with (c := path.open("w")) as file:  # ①
...     file.write("print('Hello World!')")  # ②
...  # ③
```

```
>>> type(c) ④
<class '_io.TextIOWrapper'>
>>> dir(c) ⑤
['_CHUNK_SIZE', '__class__', '__del__', '__delattr__',
'__dict__', '__dir__', '__doc__', '__enter__', '__eq__',
'__exit__', ...]
```

① `path.open` 函数返回一个上下文管理器。如果成功，其 `__enter__` 方法返回一个文件对象。*仅用于说明*，我们使用海象运算符将上下文管理器赋值给变量 `c`。（上下文管理器通常不能被“重用”。）

② `with` 语句套件中包含的所有语句都会被执行，例如，直到遇到错误（如果有的话）。

③ 在终止 `with` 语句之前，无论错误条件如何，都保证会调用上下文管理器的 `__exit__` 方法。

④ 内部类型 `_io.TextIOWrapper` 是一个类管理器。

⑤ 此对象包含预期的 `__enter__` 和 `__exit__` 方法。

`with` 头部可以包含多个表达式-目标项。当有多个表达式-目标对时，上下文管理器的处理方式就像多个 `with` 语句从左到右嵌套一样。

例如，

```
>>> with open("a.txt") as f1, open("b.txt") as f2:
...     print(f1.read())
...     print(f2.read())
...
```

此语句等价于以下内容：

```
>>> with open("a.txt") as f1:
...     with open("b.txt") as f2:
...         print(f1.read())
...         print(f2.read())
...
```

## 第 12 章. 模式匹配

Python 的 `match - case` 语句（首次在 3.10 中引入）现在可以在过去可能需要复杂 `if - elif - else` 语句的地方使用。模式匹配本质上是[比较表达式](https://example.com)的泛化，后者产生布尔值。

### 12.1. match - case 语句

`match` 语句具有以下通用语法：

```
match SubjectExpression:
    CaseBlock
    ...
    CaseBlock
```

与其他复合语句不同，`match` 语句不能写在一行物理行中。也就是说，冒号后总是需要换行。*SubjectExpression* 可以是命名表达式，也可以是一个或两个星号命名表达式（逗号分隔）。当使用表达式列表时，它会先被求值为一个元组，然后才开始匹配过程。

至少需要一个 *CaseBlock*。也就是说，例如，我们不能只在 `match` 语句的主体中使用像 `pass` 语句这样的占位符。每个 *CaseBlock* 具有以下形式：

```
case Pattern:
    Suite
```

或

```
case Pattern Guard:
    Suite
```

我们将在[下一节](the next section)中介绍 Python 当前支持的所有不同模式类别。*Guard* 是一个可选的布尔条件。如果包含，只有当模式匹配且条件满足时，case 子句才会匹配。

在最简单的用法中，`match` 语句可以或多或少地像 C 语言中传统的 `switch` 语句一样使用。例如，

```
>>> color = Color.BLUE
>>> match color:
...     case Color.RED:
...         print("Color red detected")
...     case Color.BLUE:
...         print("Blue wins!")
...     case _:
...         print("We don't care about other colors :)")
...
Blue wins!
```

与 C 语言的 `switch - case` 语句不同，`case` 子句不需要 `break` 语句。它们不会“贯穿”。

在此示例中，第一个 case 不匹配，因为 `color != Color.RED`。然后 `match` 语句匹配下一个 case，因为 `color == Color.BLUE`，因此执行匹配 case 中的语句。

通配符表达式 `_` 用于“捕获所有”。也就是说，（可选的）`case _` 子句或多或少等同于 C 风格 switch 语句中的 *default* case。如果没有匹配的 `case`，则 `match` 语句在不执行任何套件的情况下终止。如果指定了捕获所有 case `case _`，则执行此默认 case 中的语句。

### 12.2. 模式

#### 12.2.1. 通配符模式

如上一节末尾所示，特殊模式 `_` 匹配任何主题表达式。这是最简单的模式，因为它匹配任何表达式，所以它只能在 match 语句的最后一个 case 子句中使用。

```
>>> match 5:
...     case _:
...         print("I always match!")
...
I always match!
```

#### 12.2.2. 字面量模式

匹配字面量模式实际上等同于主题表达式和字面量模式之间的比较操作。支持大多数常量字面量表达式，包括布尔值、整数和浮点数以及字符串。复数（不是常量字面量）也属于此类。

例如，

```
>>> arr = (False, 30, 0.5 + 2j, """hello""", None)
... for val in arr:
...     match val:
...         case False: ①
...             print(f"{val} matched False")
...         case 30: ②
...             print(f"{val} matched 30")
...         case 0.5 + 2j: ③
...             print(f"{val} matched 0.5 + 2j")
...         case "hello": ④
...             print(f"{val} matched 'hello'")
...         case _: ⑤
...             print("Matched none")
...
False matched False
30 matched 30
(0.5+2j) matched 0.5 + 2j
hello matched 'hello'
Matched none
```

① 如果 `val` 是 `False`，则此 case 匹配。
② 如果 `val == 30`，则此 case 匹配。
③ 如果 `val == 0.5 + 2j`，则此 case 匹配。
④ 如果 `val == "hello"`，则此 case 匹配。
⑤ 捕获所有模式。请注意，如果 case 表达式是 `None`，则元组 `arr` 中的字面量 `None` 本可以匹配一个 case。

#### 12.2.3. 命名值模式

另一类模式是引用值的名称/变量，包括对象的属性。例如，

```
>>> class X: pass
...
>>> x = X()
>>> x.A, x.B = 5, 50
>>> match 1 + 4:
...     case x.A:
...         print("Matched x.A")
...     case x.B:
...         print("Matched x.B")
```

### 12.2. 模式

```
...
Matched x.A
```

① 对象 `x` 现在有两个（额外的）属性 `A` 和 `B`，其值分别为 `5` 和 `50`。
② 主体表达式求值为 `5`。
③ 此 case 匹配成功，因为 `x.A == 5`。

#### 12.2.4. 分组模式

括号内的模式也是一个模式，实际上，是同一个模式。例如，

```
>>> match True:
...     case (True):
...         print("True is true no matter what")
...
True is true no matter what
```

#### 12.2.5. 捕获模式

捕获模式由一个名称组成。当匹配发生时，它将主体值绑定到指定的名称。例如，

```
>>> match 1 + 2:
...     case _sum:                    ①
...         print(f"sum is {_sum}.")
...
sum is 3.
```

① 捕获模式总是成功。名称 `_sum` 被绑定到主体表达式的值 `3`。

该名称不能是下划线（`_`），并且捕获模式总是匹配主体表达式。捕获模式可以用作其他模式的一部分，例如 OR 模式或序列模式。

#### 12.2.6. OR 模式

我们可以使用竖线 `|` 在一个 case 中包含备选模式。例如，

```
>>> today = "Saturday"
>>> match today.lower():
...     case "saturday" | "sunday":     ①
...         print("It is a weekend!")
...     case _:
...         print("Just a weekday.")
...
It is a weekend!
```

① 如果主体表达式等于任一备选模式，则此 case 匹配。在此示例中，`today.lower() == "saturday"`，因此此 case 最终匹配。OR 模式中的模式从左到右进行测试。

#### 12.2.7. AS 模式

OR 模式的匹配表达式可以使用 `as` 关键字赋予一个名称。例如，

```
>>> import random
>>> match random.randint(0, 3):
...     case 0 | 2 as e:
...         print(f"Got {e}. We are even now.")
...     case 1 | 3 as o:
...         print(f"Got {o}. That is odd!")
...
Got 1. That is odd!
```

#### 12.2.8. 序列模式

序列模式匹配序列表达式的结构和值，例如元组、列表和字符串。

序列模式使用括号（`( )`）或方括号（`[ ]`），它包含一个或多个子模式，用逗号分隔，每个子模式都与主体表达式的一个或多个元素进行匹配。当括号中只使用一个子模式时，需要一个尾随逗号。否则，它被视为分组模式。

子模式可以属于任何一种模式类别。此外，最多可以使用一个“星号模式”，使用星号（`*`）后跟一个捕获模式或通配符模式。星号模式匹配“剩余元素”。如果没有星号子模式，则模式中的所有元素必须与主体序列中的相应元素匹配。（因此，它们的长度必须相同才能匹配。）

以下是使用模式匹配对内置 `len` 函数的递归实现：

```
>>> def seq_length(seq):
...     match seq:
...         case []: ①
...             return 0
...         case [_, *y]: ②
...             return 1 + seq_length(y)
...
```

① 这是一个固定长度模式。它匹配一个包含零个元素的序列。

② 这是一个可变长度模式，包含两个子模式：一个用于第一个元素的通配符模式，以及一个用于其余部分的星号捕获模式。此 case 将匹配任何至少有一个元素的序列。在此示例中，我们忽略第一个元素的具体值，并使用其余部分（一个子序列）来递归调用函数。请注意，单元素序列的长度为 1。

这是另一个示例。

```
>>> for p in [(0, 0), (0, 2), (3, 3), (2, 4)]:
...     match p:                                    ①
...         case (0, 0):                            ②
...             print("I'm the origin!")
...         case (x, 0):                            ③
...             print(f"I'm on the x-axis, x = {x}")
...         case (0, y):                            ④
...             print(f"I'm on the y-axis, y = {y}")
...         case (x, y) if x == y or x == -y:       ⑤
...             print(f"I'm on a diagonal, ({x}, {y})")
...         case (x, y):                            ⑥
...             print(f"I'm just a random point. ;(")
...
I'm the origin!
I'm on the y-axis, y = 2
I'm on a diagonal, (3, 3)
I'm just a random point. ;(
```

① 在此示例中，主体表达式是一个元组。

② 此模式匹配具有指定元素 0 和 0 的元组。请注意，模式无需使用括号（因为在此示例中主体表达式是元组）。括号和方括号可以互换，除了单元素序列模式，正如我们前面提到的。

③ 此模式包含一个捕获子模式和一个字面量模式。它将匹配任何第二个元素为 0 的两元素元组，`x` 将被绑定到元组的第一个元素。

④ 类似地，此固定长度模式包含一个用于第一个元素的字面量模式和一个用于第二个元素的捕获子模式。它将匹配任何形式为 `(0, y)` 的两元素元组，`y` 将被绑定到元组的第二个元素。

⑤ 模式 `(x, y)` 将匹配任何两元素序列，因为两个子模式都是捕获模式。此特定 case 包含“守卫”。除了匹配模式外，守卫表达式必须求值为 `True`。否则，该模式被视为不匹配。请注意，我们可以在守卫表达式中使用捕获的名称。

⑥ 由于在此示例中我们未使用捕获的名称，因此此模式等效于所有有效两元素序列的通配符模式。通配符模式 `_`，如果作为最后一个 case 包含，将捕获所有不是两元素序列的表达式。

#### 12.2.9. 映射模式

映射类型也可以用作模式。它的工作方式与序列模式类似，不同之处在于主体表达式是映射（例如字典），并且子模式涉及映射元素的键和值。

例如，

```
>>> m = {"k1": 1, "k2": 2, "k3": 3}
>>> match m:
...     case {"k1": 1, "k2": 2}:
...         print("Exactly matched the 'k1' and 'k2' elements.")
...     case {"k1": v1, "k2": v2}:
...         print(f"For key 'k1', the value is {v1}, for 'k2', the value is {v2}.")
...     case {"k1": 1}:
...         print("Exactly matched the 'k1' element.")
...     case {"k1": v1}:
...         print(f"For key 'k1', the value is {v1}.")
...     case {}:
...         print("An empty mapping pattern matches all objects of a mapping type.")
...
Exactly matched the 'k1' and 'k2' elements.
```

在此示例中，每个 case 都与给定的主体表达式（字典 `m`）“可匹配”。因此，第一个 case 匹配，`match` 语句终止。映射模式还支持“双星号”子模式，用于映射表达式中的“其余”元素。例如，`{"k1": v1, **rest}`。

序列和映射模式可以嵌套。

#### 12.2.10. 类模式

类模式是 Python 的 `match` 语句支持的另一类模式。类模式表示一个类及其位置参数和关键字参数。关键字参数使用等号（`=`），而不是映射模式中的冒号（`:`）。

这是一个示例，

```
>>> class A:
...     def __init__(self):
...         self.x, self.y = 1, 2
...
>>> a = A()
>>> match a:
...     case A(x = 1, y = 2):
...         print("Matched the attributes 'x' and 'y'.")
...     case A(x = v):
...         print(f"Matched the attribute 'x'. Its value is {v}.")
...     case A():
...         print("As long as 'a' is of type A, it always matches. In this case, the parentheses is optional.")
...
Matched the attributes 'x' and 'y'.
```

在此示例中，三个 case 的模式再次都与主体表达式（对象 `a`）兼容。因此，它匹配第一个 case，`match` 语句终止。当类型的构造函数具有位置参数时，可以使用位置参数模式。

## 第13章 函数

### 13.1. 函数定义

函数定义是一个复合语句，用于定义用户自定义函数。

- `def` 函数定义语句用于定义一个用户自定义函数。
- 当执行函数定义语句时，它会创建一个函数对象，并将函数名绑定到当前局部命名空间中的该函数对象。
- 函数定义不会执行函数体。函数体在每次调用函数对象时执行。
- 如果函数体中的第一个语句是字符串字面量表达式语句，则该字面量将用作函数对象的 `__doc__` 属性的值。因此，它成为了函数的“文档字符串”。

例如，以下语句在执行时会创建一个名为 `add` 的函数对象，该函数接受两个参数 `p1` 和 `p2`，并返回一个值。

```
>>> def add(p1, p2):
...     "Adds p1 and p2"
...     return p1 + p2
...
>>> add.__doc__
'Adds p1 and p2'
>>> add(1, 2)
3
```

① `add` 函数接受两个参数。注意，在 Python 中无法指定返回值。见下文。

② 一个文档字符串。
③ 我们只能通过阅读函数实现来推断完整的函数签名。
④ 文档字符串存储在函数对象的 `__doc__` 属性中。
⑤ 使用 1 和 2 调用此函数。它返回 3。

### 13.2. 函数参数

函数可以定义零个、一个或多个参数。Python 中有三种函数参数。

**仅位置参数**
对于此类参数，当调用函数时，对应的实参必须按照与参数列表中定义的完全相同的位置提供（例如，如 C/C++ 和其他一些 C 风格语言）。为了定义仅位置参数，我们使用分隔符 `/`（正斜杠）。位于此可选分隔符之前的任何参数都是仅位置参数。

**仅关键字参数**
对于此类参数，它们在参数列表中没有固定位置，对应的实参需要使用 `parameter=value` 语法（称为“关键字实参”）提供。另一方面，要定义*仅关键字*参数，我们使用分隔符 `*`（星号）。（或者，可变参数也用作此目的的分隔符。见下文。）位于此可选分隔符之后的参数（如果有）是仅关键字参数。

**位置或关键字参数**
默认情况下，所有其他参数都有固定位置，它们可以使用位置实参语法（在相应位置）或关键字实参语法使用。

以下是示例：

```
def fa(p2): pass ①
def fb(p1, /, p2): pass ②
def fc(p2, *, p3): pass ③
def fd(p1, /, p2, *, p3): pass ④
```

① p2 是一个“普通”函数参数。它可以用作位置实参 (fa(v2)) 或关键字实参 (fa(p2 = v2))。
② 在此示例中，p1 是一个仅位置参数。
③ 对于函数 fc，p3 只能用作关键字实参。
④ p1 和 p2 可以用作位置实参，但 p3 不行。p2 和 p3 可以用作关键字实参，但 p1 不行。

### 13.3. 可选参数

函数参数可以具有默认值，形式为 `parameter = expression`。这些参数被称为可选参数。在函数调用中，如果没有为可选参数显式提供实参值，则使用其默认值。

- 任何仅关键字参数都可以设为可选。
- 只有最后一个或多个连续的位置参数（仅位置参数或其他类型），在 `*`（如果存在）之前，可以设为可选。
- 默认参数值仅在函数定义执行时求值一次，从左到右。

### 13.4. “可变参数”函数

一个可选的（最多一个）位置可变参数可以包含在函数定义中，作为关键字参数（如果有）之前的最后一个参数。在语法上，该参数名前有一个 `*`。

当存在可变参数时，不需要使用仅关键字参数分隔符 `*`。

当调用此函数时，应指定此可变参数之前的所有实参，并且它们应使用位置语法。一个包含任何多余位置实参（可能为空）的元组将被赋值给位置可变参数。

例如，

```
def f(
    a,                     ①
    *args,                 ②
    b):                    ③
    print(a, args, b)
```

① `a` 可以使用位置或关键字实参语法。但是，当在函数调用中使用可变参数时，所有前面的实参，包括此示例中的 `a`，都应使用位置语法。

② 一个（位置）可变参数。虽然不是必需的，但在 Python 中，通常使用 `args` 作为可变参数的名称。

③ `b` 是一个仅关键字实参，因为它位于可变参数之后。

此函数 `f` 可以用多种不同的方式调用。例如，

```
f(1, b = "baby")              ①
f(b = "boy", a = 5)            ②
f(10, 20, 30, b = "girl")      ③
```

① 在函数体中，`a` 的值是 `1`。此函数调用将打印出 1 () baby。

② 此函数调用将打印出 5 () boy。

③ 这将打印出 10 (20, 30) girl。

我们也可以在函数参数列表中包含（最多一个）关键字可变参数。其名称前应有 `**`，并且它只能用作最后一个参数，要么在 `*` 或 `*args` 和其他仅关键字参数（如果有）之后，要么在没有仅关键字参数的情况下使用。

任何在函数调用中未显式指定的多余关键字实参都包含在仅关键字可变参数中，作为一个有序映射。

例如，对于定义如下的函数，

```
def f(
    a,                          ①
    /, *,
    b = "rice",                 ②
    **kwargs):                  ③
    print(a, b, kwargs)
```

① a 是一个仅位置参数。

② 在此示例中，b 是一个仅关键字参数，具有默认值 "rice"。

③ kwargs 是一个关键字可变参数，由前缀 ** 表示。在 Python 中，通常使用 kwargs 作为此类基于关键字的可变参数的名称。

```
f(1)                                ①
f(5, b = "wheat")                   ②
f(10, c = "oats", d = "hops")       ③
```

① 输出将是 1 rice {}。
② 输出将是 5 wheat {}。
③ 此函数调用将打印出 10 rice {'c': 'oats', 'd': 'hops'}。

更灵活的可变参数函数通常同时包含 `*args` 和 `**kwargs`，通常一起放在参数列表的末尾（例如，没有额外的仅关键字参数）。

### 13.5. 函数调用

#### 13.5.1. 调用

函数调用是一个表达式。调用使用零个、一个或多个（位置和/或关键字）实参调用任何 `callable` 对象，而不仅仅是函数。实参之间用逗号分隔。如果至少包含一个实参，可以选择添加一个尾随逗号。

#### 13.5.2. 可调用对象

所有具有 `__call__` 方法的对象都是*可调用的*，此外还有

- 内置和用户定义的函数，
- 内置对象的方法，
- 类对象，以及
- 类实例对象的方法。

例如，

```
>>> class X: pass ①
...
>>> x = X() ②
>>> callable(X), callable(x) ③
(True, False)
>>> x()
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'X' object is not callable
>>> X.__call__ = lambda self: 42
>>> callable(x)
True
>>> dir(x)
['__call__', ...]
>>> x()
42
```

① 使用类语句创建了一个新类型 X。
② X 是一个类对象，因此它是可调用的。它返回一个类型为 X 的实例对象，即此示例中的 x。
③ 内置的 callable 函数可用于检查对象是否可调用。x 不可调用。
④ 尝试使用括号调用语法调用不可调用对象将引发 TypeError。
⑤ 可以在其类型 X 中定义 __call__ 方法。Lambda 表达式将在后面的 Lambda 部分讨论。请注意，这个特定的 Lambda 函数接受一个名为 self 的参数。这是必需的，因为 __call__ 是一个方法。
⑥ 既然其类型 X 已经定义了 __call__ 方法，x 就是可调用的了。
⑦ 正如我们很容易验证的那样，x 现在有一个 __call__ 方法属性。
⑧ 现在调用 x 就是调用 X.__call__ 魔术方法，在此示例中它恰好简单地返回 42。

#### 13.5.3. 位置实参与关键字实参

正如我们之前所见，函数参数可以定义为仅位置、位置或关键字、或仅关键字，按此顺序，

### 13.6. Lambda 表达式

由 `*` 和 `/` 分隔，如果需要的话。

当我们调用一个函数，或任何可调用对象时，我们可以为位置参数和位置或关键字参数提供位置参数，为位置或关键字参数和仅关键字参数提供关键字参数。

此外，

-   如果有关键字参数，所有位置参数应位于所有关键字参数之前。
-   每个位置参数应放置在相应位置参数的位置上。
-   应为所有非可选参数（位置或关键字）提供有效的参数。

一些示例已在函数定义部分给出。

### 13.6. Lambda 表达式

一个 Lambda *表达式* 会求值为一个匿名函数对象。以下是通用语法：

```
lambda <parameters>: <expression>
```

上述形式或多或少等同于使用以下方式定义的（命名）函数对象：

```
def <name>(<parameters>):
    return <expression>
```

请注意，

-   使用 Lambda 表达式创建的函数对象可以“就地”调用，并且
-   Lambda 表达式不能包含语句或注解。

例如，

```
>>> (lambda x: x + 1)(10) ①
11
>>> a = [1, 2, 3]
>>> list(map(lambda x: x * 2, a)) ②
[2, 4, 6]
```

① Lambda 表达式可以赋值给一个变量，也可以在定义点直接调用，如本例所示。
② Lambda 表达式通常用作高阶函数（HOFs）的一次性函数类型参数。例如，在此例中，`map` 是一个内置函数，它接受一个函数和一个可迭代对象。

### 13.7. map、filter 和 reduce

尽管本迷你参考中不会涵盖所有内置函数，但让我们回顾一下内置的 `map` 和 `filter` 函数，以及来自 `functools` 模块的 `reduce` 函数。它们通常被称为“高阶函数”（HOFs），因为它们接受函数作为参数，和/或返回函数。

`map`、`filter` 和 `reduce` 函数（可能名称不同）是不同编程语言（无论是函数式还是命令式）中最常用的高阶函数之一。

对于这些高阶函数，使用匿名函数（Lambda 表达式）作为其函数参数是一种相当常见的做法，因为这些函数参数通常只使用一次。

#### 13.7.1. map 函数

内置的 `map` 函数接受一个函数和一个可迭代对象，并将该函数应用于给定可迭代对象中的每个元素。例如，

```
>>> list(map(lambda x: x**2, [1, 2, 3]))
[1, 4, 9]
```

① `map` 函数返回一个 map 对象。（也就是说，它是一个构造函数。）我们使用 `list` 构造函数将返回的 map 转换为列表。

#### 13.7.2. filter 函数

内置的 `filter` 函数同样接受一个函数和一个可迭代对象，并根据给定的函数返回一个过滤后的列表。例如，

```
>>> list(filter(
...     lambda x: True if x%2==0 else False,
...     [1, 2, 3, 4]))
[2, 4]
```

① 与 `filter` 相同。它是一个构造函数。
② 它将函数应用于给定可迭代对象中的每个元素，如果其布尔值为 True（例如，3，'hello' 等），则将其包含在结果中。否则（例如，None，[] 等），则将其过滤掉。

#### 13.7.3. functools.reduce 函数

`functools` 模块的 `reduce` 函数以类似的方式工作，但它将给定的函数累积地应用于给定可迭代对象中的所有元素。

以下是一个示例：

```
>>> from functools import reduce
>>> reduce(lambda s, a: s + a, [1, 2, 4], 0)  ①
7
```

① 第三个参数是可选的，如果提供了，它将用作“归约”操作中的第一个项。在此示例中，`reduce` 函数对列表 `[1, 2, 4]` 中的每个元素迭代地应用给定的 lambda 函数。也就是说，它计算 `0 + 1`（结果为 `1`），`1 + 2`（结果为 `3`），`3 + 4`，结果为 `7`，即最终结果。

### 13.8. 函数装饰器

装饰器是一个函数，或者更一般地说是一个可调用对象，它接受一个函数/可调用对象并返回一个相同类型的函数/可调用对象。在语法上，装饰器与目标函数定义一起使用，作为前缀，它转换目标函数并返回转换后的函数。换句话说，装饰器实际上成为了目标函数定义的一部分。

以下是通用语法：

```
@decorator_function
def another_function(args):
    pass
```

这在语义上等同于以下内容：

```
def another_function(args):
    pass

another_function = decorator_function(another_function)
```

`decorator_function` 是一个函数：

```
def decorator_function(f1):
    # 将 f1 转换为 f2，
    # 或者基于 f1 创建 f2。
    return f2
```

使用装饰器的关键，除了语法上的便利性，还在于

-   *decorator_function* 函数为多个不同的函数提供通用功能，并且
-   这些被装饰的函数，例如 `another_function`，在程序中将始终作为已装饰/已转换的形式使用。

#### 13.8.1. 内置装饰器

##### @property 装饰器

Python 中的“属性”是一个抽象构造，其行为类似于对象的数据属性，例如，具有 getter、setter 和 deleter，以及文档字符串等。

属性可以使用 `property` 构造函数创建。或者，可以使用 `@property` 装饰器在类定义中轻松定义新属性或修改现有属性。例如，

```
>>> class X:
...     def __init__(self):
...         self._a = 10
...     @property
...     def a(self):
...         "I am the 'a' property."
...         return f"Magical {self._a}"
...     @a.setter
...     def a(self, value):
...         self._a = value
...     @a.deleter
...     def a(self):
...         print("Urghhh, I am being deleted...")
...         del self._a
...
```

① 这创建了一个名为 `a` 的属性。被装饰的方法用作属性 `a` 的“getter”。

② 这将被装饰的方法设置为 `a` 的 setter。

③ 当对属性 `a` 使用 `del` 语句时，会调用被装饰的方法。

让我们尝试使用一个类型为 `X` 的对象：

```
>>> x = X()
>>> help(X.a)
Help on property:

    I am the 'a' property.
(END)
>>> x.a
'Magical 10'
>>> x.a = 100
>>> x.a
'Magical 100'
>>> del x.a
Urghhh, I am being deleted...
>>> x.a
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
  File "<stdin>", line 7, in a
AttributeError: 'X' object has no attribute '_a'. Did you mean: 'a'?
```

##### @staticmethod 装饰器

Python 中类的“静态方法”类似于其他面向对象语言（如 C++、Java 和 C#）中的静态方法。它是一个独立的函数，尽管是在类定义中定义的。

在语法上，静态方法可以在类或实例上调用，但它们不接收隐式的第一个参数 `self`。

`@staticmethod` 装饰器将类定义中的函数转换为静态方法。例如，

```
>>> class Y:
...     @staticmethod
...     def f1():
...         print("I am a static method")
...
>>> Y.f1()
I am a static method
>>> y = Y()
>>> y.f1()
I am a static method
```

① 此装饰器使被装饰的方法成为静态方法。
② 该方法可以在类对象上调用。
③ 如果没有 `@staticmethod` 装饰器，该方法将无法在实例对象上调用。

##### @classmethod 装饰器

Python 中类的“类方法”是在类对象上定义的方法，而不是在实例上。普通方法接收实例对象作为隐式的第一个参数。相比之下，类方法接收类对象作为隐式的第一个参数。

在语法上，类方法可以在类或实例上调用，就像静态方法一样，但它们接收类作为其第一个参数（即使是在实例上调用时）。如果为派生类调用类方法，则派生类对象将作为隐式第一个参数传递。

`@classmethod` 装饰器将函数转换为类方法。例如，

```
>>> class Z:
...     @classmethod
...     def f2(cls):
...         print(f"Class method: {cls.__name__}")
...
>>> Z.f2()
Class method: Z
>>> z = Z()
>>> z.f2()
Class method: Z
>>> class Z2(Z): pass
...
>>> Z2.f2()
Class method: Z2
```

① `@classmethod` 装饰器使被装饰的方法成为类方法。

② 如果没有装饰，这将是一个语法错误。

③ 默认情况下，在类中定义的方法是实例方法，它接收实例对象作为隐式第一个参数。对于类方法，隐式第一个参数是类对象。

④ 隐式的类对象参数现在是子类 `Z2`。

## 14.1. 类定义

一个 `class` 定义是一个复合语句，它定义了一个可调用的类对象。当执行时，它会执行类套件中的语句，并根据类定义在内存中创建一个 `type` 类型的对象。

其一般语法如下：

```
class MyType(BaseType1, BaseType2):
    Suite
```

关键字 `class` 开始该语句，后跟新的类名（例如，本例中的 `MyType`）以及括号中一个或多个基类的可选列表（例如，`BaseType1`、`BaseType2` 等）。Python 支持“多重继承”。

当未指定基类列表时，该类继承自最终基类 `object`。在这种情况下，类名后的括号可以省略。即，

```
class Orange:
    pass
```

此定义等同于以下内容：

```
class Orange(object):
    pass
```

-   一个 `class` 定义会创建一个局部命名空间，嵌套在全局命名空间中，类套件中的语句在新创建的执行帧中执行。
-   然后创建一个 `class` 对象，它继承自基类列表中（显式或隐式）指定的基类，并使用新创建的局部命名空间作为类的 `__dict__` 属性。
-   然后，在 `class` 定义语句中定义的类名被绑定到原始局部命名空间（可以是全局命名空间）中的这个类对象。

## 14.2. 类与实例

Python 的 `class` 只是一个对象（具有一些特殊功能）。但是，在 Python 编程中，它扮演着其他“基于类”的面向对象编程（OOP）语言中类的角色。至少，使用类可以帮助创建更模块化、结构更清晰的 Python 程序，其组件可以更容易地重用等。在许多问题领域，以“类”和“对象”（OOP 意义上的）来思考可能相当自然和直观。

就像函数定义一样，类定义包含一系列任何类型的语句。大多数类定义主要包含变量（例如，“类变量”）和函数（例如，“静态方法”、“类方法”和“实例方法”）的定义。

当 Python 解释器读取/执行类定义时，这些语句会被执行。例如，

```
>>> class X(object):
...     if True:
...         print(True)
...     else:
...         print(False)
...
True
```

① 这是 `if` 语句的输出。与函数定义不同，Python 在创建 `class` 对象时会执行 `class` 定义作用域中的所有语句。

一个 `class` 定义在某种程度上类似于一个“模块”。这些语句仅在 `class` 语句首次执行时（例如，当 Python 解释器读取该语句时）执行。当我们*使用*“类对象”（例如，创建该类的实例对象）时，这些语句不会被执行。例如，

```
>>> x = X()
>>>
```

① 我们“调用”类来创建该类的一个实例。以这种方式使用的类 `X` 的“函数” `X()` 是一个*构造函数*。
② 类定义中的语句没有被执行。

### 14.2.1. 类对象

Python 的 `class` 语句在内存中创建一个类对象（就像 `def` 语句创建一个函数对象一样）。一个 `class` 对象支持“[属性引用](https://docs.python.org/3/reference/datamodel.html#attribute-references)”，例如，通过点表示法，就像任何其他 Python 对象一样。

```
>>> class SoSimple:
...     one_name = "simple"
...
>>> type(SoSimple)
<class 'type'>
>>> SoSimple.one_name
'simple'
>>> SoSimple.one_number = 666
>>> SoSimple.one_number
666
>>> dir(SoSimple) ④
['__class__', ... '__weakref__', 'one_name', 'one_number']
```

① 类对象的类型是 `type`。
② 类定义中包含的属性。
③ 你可以向类对象添加任何属性，就像任何其他对象一样。
④ 注意，类对象自带许多预定义属性，所有这些属性都以双下划线（`__`）开头和结尾，也称为“dunder”。

这种类型的类属性对应于其他 OOP 编程语言中的静态变量（或静态字段）和静态方法。

### 14.2.2. 类变量

一个类对象（例如，由 `class` 语句定义的）扮演着两个主要角色。首先，正如我们之前所见，它是给定类/类型的实例对象的*构造器*。其次，它保存着该类所有实例的公共变量。事实上，正如我们刚才提到的，类变量被所有实例对象*共享*。

```
>>> class Car:
...     brand = "GM"
...
>>> car1, car2 = Car(), Car()
>>> car1.brand, car2.brand
('GM', 'GM')
>>> Car.brand = "Ford"
>>> car1.brand, car2.brand
('Ford', 'Ford')
```

### 14.2.3. 构造器

除了属性引用，一个 `class` 对象还支持“实例化操作”。Python 中的一个 `class` 对象是给定类/类型的对象的构造器。

使用之前的同一个例子，

```
>>> s = SoSimple()
>>> type(s)
<class '__main__.SoSimple'>
>>> s.one_name
'simple'
>>> s.one_number
666
```

注意，实例对象 `s` 包括临时属性 `one_number`，以及原始类定义的一部分 `one_name`。也可以向给定的实例对象添加任何额外的属性：

```
>>> s.one_address = "Playa"
>>> s.one_address
'Playa'
>>> dir(s)
['__class__', ... '__weakref__', 'one_address', 'one_name', 'one_number']
```

注意，实例 `s` 包含与原始 `SoSimple` 类中大致相同的预定义属性，以及后来添加到该类对象 `SoSimple` 的其他属性，还有实例对象 `s` 本身特有的属性。我们也可以使用 `del` 语句删除在类或实例中定义的属性。

```
>>> del s.one_number
>>> dir(s)
['__class__', ... '__weakref__', 'one_address', 'one_name']
```

执行 `del` 语句后，实例对象 `s` 不再具有属性 `one_number`。

### 14.2.4. `__init__` 函数

注意，类对象不仅在创建该类的实例对象时用作“模板”，而且它们本质上共享相同的属性。

```
>>> s = SoSimple()
>>> s.one_name
'simple'
>>> SoSimple.one_name = "not simple any more"
>>> SoSimple.one_name
'not simple any more'
>>> s.one_name
'not simple any more'
```

类的实例化（或，实例化一个类的实例对象）可以通过覆盖类的 `__init__` 方法来自定义。此方法在实例创建后自动调用，例如，由 Python 解释器调用。

```
>>> class SoEasy:
...     def __init__(self):
...         self.one_title = "programmer"
...
>>> s = SoEasy()
>>> s.one_title
'programmer'
>>> dir(s)
['__class__', ... '__weakref__', 'one_title']
```

注意函数签名。`__init__` 方法至少有一个参数。第一个参数总是指刚刚创建的实例对象，它总是被命名为 `self`。

在此示例中，类型 `SoEasy` 的一个实例对象具有一个自动附加的属性 `one_title`。这是因为我们在 `SoEasy.__init__` 方法中创建了这个名称/属性并将其附加到 `self` 对象。

### 14.2.5. 实例对象

一个类的*实例对象*包括类中定义的所有属性，并且可以包含其他实例特定的属性。属性引用可用于引用实例对象的这些属性。有两种属性：数据属性（或字段或变量）和方法。

如前所述，实例对象就像 Python 中的其他一切一样，具有属性，即数据属性和方法属性。最初，它的大部分属性来自其类型，以及在创建时在 `__init__` 方法中添加的属性。但是，与其他类型的自定义对象（包括函数、类等）一样，可以向实例对象添加新的属性。

实例对象具有与类定义中的函数相对应的方法。所有以实例对象作为第一个参数（例如 `self`）的函数，根据定义都是“方法”，Python 允许对这些函数使用对象方法调用语法。例如，

```
>>> class Ship:
...     def fly(self):
...         print("I cannot fly. Only spaceships can fly.")
```

### 14.2.6. 实例变量

虽然我们可以在 Python 中为实例对象添加任何属性，但通常是在 `__init__` 方法中完成。这样，该类的所有实例对象都将拥有相同的（非共享的）属性集。

```python
>>> class Pet:
...     def __init__(self):
...         self.kind = "dog"
...
>>> pet1, pet2 = Pet(), Pet()
>>> pet1.kind, pet2.kind
('dog', 'dog')
```

在初始化器中，参数 `self` 指的是刚刚通过*调用*类对象创建的实例对象。以 `pet1` 为例，`self` 和 `self.kind` 分别指代 `pet1` 和 `pet1.kind`。同样，对于 `pet2` 实例，`self` 和 `self.kind` 分别指代 `pet2` 和 `pet2.kind`。

属性 `kind` 是一个实例变量，它属于特定的实例对象，并且在不同实例之间不共享。此外，类对象本身没有该属性：

```python
>>> Pet.kind
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
AttributeError: type object 'Pet' has no attribute 'kind'
```

请注意在类定义中定义实例变量的语法。实例变量是在 `self` 对象上定义的。因此，作为推论，在类定义中，你只能在实例方法内定义实例变量。

> 所有用户定义的类型都是可变的。所有用户定义类型的对象都是可变的。

### 14.2.7. 实例方法

如前所述，`class` 定义中包含的最常见语句通常是用于定义函数的 `def` 语句。在类中定义的函数是类的属性，也是该类任何实例对象的属性。并且，只要其第一个参数的类型是同一个类，它就可以在对象上使用方法语法调用。

与类变量不同，类函数除了实例外，还有两种类型：

- 一种只是函数（除了点号名称语法），称为“静态方法”，以及
- 另一种是类对象的一部分，可以访问类变量。这称为“类方法”。类方法的第一个参数是类对象本身。

正如我们之前所见，内置装饰器 `@staticmethod` 和 `@classmethod` 可用于专门声明其中一种或另一种类级别的方法。否则，按照惯例，类定义中的所有其他方法都应该是实例方法，即它们的第一个函数参数必须是该类的实例，`self`。

## 14.3. 面向对象编程

Python 中的 `class` 定义语句用于为该类的对象创建一个“模板”。`class` 定义了一个自定义类型、如何创建该类型的实例对象以及如何访问对象等。

在 Python 中，一个 `class` 总是隐式地“继承”自内置类型 `object`，无论是直接还是间接。对于任何内置或自定义类型，这都是正确的。一个 `class` 可以直接继承自另一个类型（而该类型又是 `object` 的子类型）。Python 支持[从多个直接基类继承](https://docs.python.org/3/tutorial/classes.html#multiple-inheritance)。

### 14.3.1. 数据封装

在 Python 中，没有真正的“私有”属性、数据或方法。实例对象中的数据隐藏是通过约定支持的，就像 Python 中的许多其他特性一样。

以单下划线 `_` 开头的名称被视为“私有”。也就是说，按照惯例，我们不会直接访问其他类对象或实例对象的成员，如果它们的名称以一个或多个下划线开头。这样的属性被认为是实现细节，它们不是“公共 API”的一部分。

Python *确实*对[名称隐藏](https://docs.python.org/3/reference/lexical_analysis.html#private-name-mangling)有一些最小的支持。当类中变量或函数/方法的名称*以至少两个下划线开头，并以最多一个下划线结尾*时，Python 会修改属性的名称。这被称为“名称修饰”，尽管它并不真正“修饰”名称（例如，像 C++ 中那样）。无论如何，应该避免使用修饰后的名称，即使在你自己的程序中。（注意，“双下划线名称”不会被修饰或修改，因为它们以两个下划线结尾。）

### 14.3.2. 魔术方法

如前所述，`__init__` 方法是一个特殊方法。这个特定方法用于为新实例化的实例对象提供任何初始化代码。如果该方法在对象的类中被重写，Python 运行时会自动在新创建的实例对象上调用此方法。

基础类型 `object` 具有以下属性：

```python
>>> dir(object)
['__class__', '__delattr__', '__dir__', '__doc__', '__eq__',
'__format__', '__ge__', '__getattribute__', '__gt__',
'__hash__', '__init__', '__init_subclass__', '__le__',
'__lt__', '__ne__', '__new__', '__reduce__', '__reduce_ex__',
'__repr__', '__setattr__', '__sizeof__', '__str__',
'__subclasshook__']
```

由于每个类型都直接或间接/传递地继承自这个基类 `object`，因此这些属性始终对所有类型（内置或用户定义）可用。其中一些数据属性可能为空，其中一些方法属性可能具有空实现。此外，某些属性可能不会显示在 `dir()` 输出中，因为类/对象的 `__dir__` 方法可以被自定义。

`__doc__` 属性存储类型的文档字符串（如果有的话）。否则，它是一个空字符串。与函数类似，如果类定义套件中的第一条语句是字符串字面量表达式语句，则它会自动用作类的文档字符串。

（Python 的内置 `help` 函数使用对象的文档字符串作为其自动生成的帮助消息的一部分。）

`__str__` 方法在对象用于字符串上下文时被调用。当类中未实现 `__str__` 方法时，如果存在 `__repr__` 方法，则会回退到该方法。对于更底层的自定义，或显示调试信息等，可以重写 `__repr__` 方法。例如，

```python
>>> class A:
...     def __str__(self):
...         return "A from __str__()"
...
>>> class B:
...     def __repr__(self):
...         return "B from __repr__()"
...
>>> class C:
...     def __str__(self):
...         return "C from __str__()"
...     def __repr__(self):
...         return "C from __repr__()"
...
>>> a, b, c = A(), B(), C()
>>> print(f"""\n... a = {a}
... b = {b}
... c = {c}""")
a = A from __str__() ①
b = B from __repr__() ②
c = C from __str__() ③
```

① 由于 a 的类型 A 实现了 `__str__` 方法，因此在字符串上下文中调用它。

② 由于 b 的类型 B 实现了 `__repr__` 方法，但没有实现 `__str__` 方法，因此调用 `__repr__` 方法。

③ 同样，c 的 `__str__` 方法在字符串上下文中被调用。

另一方面，

```python
>>> print([a, b, c])
[<__main__.A object at 0x7093ad81f0>, B from __repr__(), C from __repr__()]
```

① 列表和其他集合类型的内部实现使用其元素的 `__repr__` 方法进行字符串表示。由于类型 A 未实现此方法，因此使用 a 的默认字符串表示（包括其类型和内存位置等）。

我们可以在许多情况下，并且应该，重写这些双下划线方法中的一些，以自定义自定义类的行为。例如，像 `__eq__`（用于 ==）、`__ne__`（用于 !=）、`__ge__`（用于 >=）、`__gt__`（用于 >）、`__le__`（用于 <=）和 `__lt__`（用于 <）这样的方法经常被重写，以自定义用户定义类型的相等性和比较相关行为。

例如，

```python
>>> from functools import total_ordering
>>> import random
>>>
>>> @total_ordering
>>> class Pt(object):
...     def __init__(self, x, y):
...         self.x, self.y = x, y
```

### 14.3.3. 继承

面向对象编程的另一个显著特性是类型继承。要定义一个继承自 `object` 子类型的类，我们在类名后的括号中指定基类（或父类、超类）。例如，

```python
>>> class Animal(object): pass
...
>>> class Pet(Animal): ①
...     def __init__(self, name = ""): ②
...         self.name = name
...     def __repr__(self): ③
...         return self.name
...     def bite(self): ④
...         print(f"{self.__class__.__name__}s do not bite.")
...
```

① `Animal` 是 `Pet` 的（直接）基类。

② 我们重写了 `__init__` 方法，该方法定义在 `object` 中（因为 `Animal` 没有自己的 `__init__` 方法），以添加一个实例变量 `self.name`。

③ `__repr__` 方法的一个简单实现，主要用于调试目的。

④ `Pet` 类包含另一个名为 `bite` 的实例方法。内置的 `object.__class__` 属性返回给定对象的类型，而 `type.__name__` 返回该类型的名称。

让我们尝试使用这个 `Pet` 类：

```python
>>> dog = Pet("puppy") ①
>>> dog.name ②
'puppy'
>>> dog.bite() ③
Pets do not bite.
>>> print(dog)
puppy
```

① 调用 `Pet` 会返回一个实例。（用户定义的类型默认是可变的，因此每次调用 `Pet()` 都会创建并返回一个*新*对象。）

② `dog` 有 `name`（一个实例变量）。

③ 并且，它可以执行 `bite`（一个实例方法）。它们按预期工作。

④ 在字符串上下文中，调用了“正确”的方法 `Pet.__repr__`。

正如我们稍后将看到的，`Pet(Animal)` 的这个 `class` 定义与下面使用[多重继承语法](https://docs.python.org/3/library/stdtypes.html#class.__bases__)的定义*完全*相同：

```python
class Pet(Animal, object): ...
```

请注意，类继承层次结构（如果你愿意的话）是从左到右的。即 `Pet` → `Animal` → `object`。因此，`class Pet(object, Animal)` 定义了一个不同的、实际上是无效的类，因为 `object` 并不继承自 `Animal`。

这是另一个类 `Python`，它是 `Pet` 的子类：

```python
>>> class Python(Pet):
...     def __init__(self, name, length):
...         super().__init__(name)
...         self.length = length
...     def __str__(self):
...         return f"Hi, I'm {self.name}. I'm {self.length} feet long."
...     def bite(self):
...         super().bite()
...         print("But we swallow our prey~~")
...
>>> python = Python("Monty Python", 10)
>>> python.name
'Monty Python'
>>> python.bite()
Pythons do not bite.
But we swallow our prey~~
>>> print(python)
Hi, I'm Monty Python. I'm 10 feet long.
```

① 这个 `class` 声明与 `class Python(Pet, Animal, object): ...` 相同，例如，使用多重继承语法。`Python` 类包含了 `Pet` 拥有的一切，以及 `Animal` 和 `object` 拥有的内容，例如通过“继承”。

② 我们重写了 `Python` 的初始化器。

③ 在派生类的 `__init__` 实现中，我们（几乎总是）调用（直接）基类的初始化器方法。`super()` 指的是类继承层次结构中最近的基类（例如 `Pet`）。在这个特定的例子中，实例变量 `name` 是在 `Pet` 的初始化器中初始化的，因此我们需要从 `Python` 的 `__init__` 中调用它。请注意，对于 `Pet` 和 `Animal`，它们的基类（包括 `object`）都有空的初始化器。

④ 实例变量 `length` 定义在 `Python` 中，而不是在 `Pet` 中。

⑤ 我们重写了 `object` 中的 `__str__` 方法。`Pet` 和 `Animal` 没有实现此方法。我们可以像引用自己的实例变量一样引用其基类之一中定义的实例变量（例如 `self.name`）。

⑥ 我们也重写了 `Pet.bite`。

⑦ 在这个特定的例子中，我们调用了超类方法。同样，`super()` 指的是直接的父基类。如果 `Pet` 没有这个 `bite` 方法，那么 Python 将从 `Animal` 开始，通过 `Pet` 的基类层次结构搜索它。

⑧ 在字符串上下文中，尽管 `Python` 有从 `Pet` 继承的 `__repr__`，但会调用 `__str__`。如果我们有一个 `Python` 列表并尝试打印该列表，则会使用 `Python` 的 `__repr__`。

⑨ 蟒蛇。😊

⑩ 由于 `Python` 没有属性 `name`，因此它从 `Pet` 开始在其某个基类中找到。

⑪ 由于 `Python` 有自己的 `bite` 方法，因此调用了它。

⑫ 在字符串上下文中，调用了 `Python.__str__`。

### 14.3.4. 多重继承

当一个给定的类有多个直接基类时，这些基类以及它们的基类、它们的基类等等，直到 `object`，都会被“线性化”，以便查找任何继承的属性。

这在 Python 中被称为 MRO（方法解析顺序），它是通过一种称为“C3 线性化算法”的方法完成的。它本质上试图找到一个与每个相关类的基类从左到右顺序一致的顺序。如果成功，此方法会在给定类的所有（直接或间接）基类（包括 `object`）中产生一个唯一的顺序。这并不总是可以实现的，在这种情况下，会引发编译错误。一个明显的例子是尝试继承两个基类顺序不兼容的类。

```python
class A: pass
class B: pass
class C(A, B): pass
class D(B, A): pass
class E(C, D): pass
```

在这个例子中，无法一致地排序 `E` 的所有基类（`A`、`B`、`C`、`D` 和 `object`），因为 `C` 要求 `A` → `B` 的顺序，而 `D` 要求 `B` → `A` 的顺序。此语句将抛出 `TypeError` 异常。

需要注意的一点是，如果没有 `E`，这些都是有效的语句，尽管*看起来* `C` 和 `D` 是冲突的。基类的线性化是针对每个类进行的。根据 `A` 和 `B` 等实现了哪些方法，`C` 和 `D` 的行为可能完全不同。但是，这仍然是一个有效的 Python 程序。

这是一个有效的多重继承示例。此示例包含了通常被称为“菱形继承”的模式。

```python
class A:
    def w(self): print("A.w")
    def x(self): print("A.x")
    def y(self): print("A.y")
    def z(self): print("A.z")
class B(A):
    def x(self): print("B.x")
class C(B):
    def w(self): print("C.w")
class D(A):
    def x(self): print("D.x")
    def y(self): print("D.y")
class E(C, D):
    pass
```

在这个例子中，例如，类 `E` 有两个（非 `object`）直接父类，`C` 和 `D`。

```
E --> C --> B --> A --> object
    --> D ---------> A -->
```

`E` 通过两条不同的“路径”间接继承自 `A`：`E` → `C` → `B` → `A` 和 `E` → `D` → `A`。因此，它是一个菱形继承模式。

在我们继续之前，让我们尝试运行以下程序。输出会是什么？

```
e = E()
print("e.w()"); e.w(); print("==")
print("e.x()"); e.x(); print("==")
print("e.y()"); e.y(); print("==")
print("z.y()"); e.z(); print("==")
```

如果你尝试运行这个程序，你会发现：

- e.w() 调用了 C.w()，
- e.x() 调用了 B.x()，
- e.y() 调用了 D.y()，
- e.z() 调用了 A.z()。

其中一个“最奇怪”的行为是，e.x() 调用了 B.x()，而不是 D.x()，尽管两者都定义了 x 方法，并且 D 是 E 的直接基类之一（C, D），正如 E 的类定义中所指定的那样。这是因为 Python 的“多重继承”使用了上述所有基类之间的线性化（无论它们是否在类定义中显式指定）。

事实上，Python 有一个内置方法 `type.mro` 来显示此信息。例如，

```
>>> E.mro()
[<class '__main__.E'>, <class '__main__.C'>, <class '__main__.B'>, <class '__main__.D'>, <class '__main__.A'>, <class 'object'>]
```

顺序是：

E --> C --> B --> D --> A --> object

因此，`e.x()` 调用 `B.x` 而不是 `D.x` 是合理的。

### 14.3.5. `super()`

在许多支持面向对象编程的语言中，像 super、base 和 parent 这样的术语具有某些相关的含义。在 Python 中，`super` 方法大致指“基类”，具有更具体的语义，尤其是在多重继承的上下文中。

例如，当在类中引用 `super().x`（无论是数据还是方法属性）时，Python 会从给定类本身开始，沿着其“线性化”的基类层次结构（MRO）从左到右（或者，从下到上，取决于你如何看待继承树）查找该属性。一旦在遍历过程中找到该属性，就会使用该属性的实现，并忽略所有其他实现（例如，在其基类中的实现）。

请注意，MRO 是为每个类定义的。例如，使用上面的例子，`B` 的 MRO 不是 `D` 的 MRO 的一个部分片段。它们可能完全不同。

```
>>> B.mro()
[<class '__main__.B'>, <class '__main__.A'>, <class 'object'>]
```

顺序是 `B` → `A` → `object`。

## 14.4. 枚举

`enum` 是一个将名称绑定到（相关的）常量值对象的集合。这些对象称为枚举的成员，它们在给定的 `enum` 中必须是唯一的。`enum` 的每个成员都有一个名称和一个值。`enum` 类型的成员可以像序列类型的对象一样进行迭代。枚举成员的值可以是任何类型，但通常使用不可变类型，如 `int` 或 `str`。

你通过继承 `enum.Enum` 类型或其子类型之一（如 `enum.IntEnum`）来创建新的 `enum` 类。这里有一个例子：

```
>>> from enum import Enum
>>>
>>> class Color(Enum):
...     RED = "red"
...     GREEN = "green"
...     BLUE = "blue"
...
```

枚举成员（例如 `Color.RED`）的类型不是其值（例如 "red"）的类型。它的类型是相同的 `enum` 类型，例如本例中的 `Color`。

```
>>> isinstance(Color.RED, Color)
True
>>> issubclass(Color, Enum)
True
```

请注意，你不能将枚举类型用作其他类型的基类，包括另一个枚举类型。如果类型继承很重要，那么你需要使用普通的 `class` 来创建新类型。

我们可以迭代枚举的 `__members__`，例如使用 `for - in` 语句。例如，

```
>>> for c in Color:
...     print(f"{c}:\t{c.value}")
...
Color.RED:      red
Color.GREEN:    green
Color.BLUE:     blue
```

# 第15章. 协程与异步编程

协程本质上是函数或子程序的泛化。函数只有一个入口点，一旦根据其逻辑执行完函数的语句，它就会将控制权一次性返回给调用者。相比之下，协程首先被进入，它们可以被多次暂停和恢复，然后才会最终终止或关闭。

大多数现代编程语言都支持某种形式的协程。在过去的几年里，Python 经历了协程的几种不同实现。在 Python 中，协程最初是作为生成器的泛化来设计和实现的。现在，应该继续使用的是基于 `async def` 的新协程（以及 `tasks` 和 `futures`）。基于生成器的协程语法在此时已被弃用（尽管内部实现仍然基于生成器）。

## 15.1. 生成器

如前所述，Python 包含许多内置类型。Python 在标准模块（如 `collections.abc`）中也定义了不少“鸭子类型”，主要用于为原本动态类型的 Python 编程语言添加一些（临时的）结构。我们在本参考文档中已经看到了一些例子，包括 `iterable` 和 `callable` 等。

> `typing` 模块是另一个持续努力为 Python 动态类型系统添加更多“结构”的例子。如前所述，类型提示可用于在 Python 软件开发过程中辅助静态类型检查。

在本节和接下来的几节中，我们将在进入本章剩余部分的主要主题——协程之前，简要地看一下迭代器和生成器。

### 15.1.1. 迭代器

像列表、元组和字典这样的集合对象，例如可以与 for 循环一起使用，它们是“可迭代的”。实现了魔术方法 `__iter__` 的类型是可迭代的。可迭代类型的对象是可迭代的，正如我们在整个参考文档中所看到的。

当使用可迭代对象（具有 `__iter__` 方法）调用内置函数 `iter()` 时，它返回一个迭代器类型的对象。迭代器实现了 `__iter__` 和 `__next__` 方法。因此，迭代器也是可迭代的。（在内部，可迭代对象和该对象的迭代器对象通常共享相同的数据流。）

使用迭代器对象（调用该对象的 `__next__` 方法）重复调用内置的 `next()` 函数会返回对象中的连续项。当没有更多项可用时，例如当到达数据流的末尾时，会引发 `StopIteration` 异常。

这是一个简单的迭代器类：

```
class ABC:
    def __init__(self):
        self.pointer = 0
        self.list = ('A', 'B', 'C')

    def __iter__(self): ①
        return self

    def __next__(self): ②
        if self.pointer >= len(self.list):
            raise StopIteration
        val = self.list[self.pointer]
        self.pointer += 1
        return val
```

① 它定义了一个 `__iter__` 方法。因此，它也是一个 `iterable`。
② 类型 `ABC` 实现了 `__iter__` 和 `__next__` 方法，因此它是一个 `iterator`。

这个类的对象可以在任何需要 `iterator` 或 `iterable` 的地方使用。例如，

```
>>> abc = ABC() ①
>>> for i in abc: ②
...     print(i)
...
A
B
C
```

① `abc` 是一个具有 `__iter__` 方法的 `iterable` 对象，该方法返回一个 `iterator`（在本例中是同一个对象）。
② `for` 语句使用迭代器的 `__next__` 方法来迭代 `abc` 中的数据。这是许多现代编程语言（不仅仅是 Python）使用的一种非常通用的模式（例如，如“设计模式”）。一旦 `for` 语句捕获到 `StopIteration`，它就会退出循环。（这隐藏在 `for` 语句的实现中。）

### 15.1.2. 生成器函数

生成器函数（同步或异步）和协程彼此非常相似。它们可以有多个入口点，可以多次 yield，并且它们的执行可以被挂起和恢复。

- 当调用一个*生成器函数*时，它返回一个*生成器对象*，该对象

## 15.2. yield 表达式

`yield` 表达式用于定义*生成器函数*或*异步生成器函数*。

> Python 语法中也有一个 `yield` 语句，它在语义上等同于（较新的）`yield` 表达式。`yield (simple)` 语句与 `yield` 表达式语句之间几乎没有区别。

函数体中包含 `yield` 表达式/语句的函数，*根据定义*，就是一个生成器函数。同样，包含 `yield` 表达式/语句的 `async def` 函数就是一个异步生成器函数。

例如，

```
>>> def a():
...     yield 1
...     yield 2
...
>>> for i in a():
...     print(i)
...
1
2
```

① `a`（自动）是一个生成器函数，因为它包含了 yield 表达式（或语句）。

② yield 表达式/语句的行为类似于 return 语句。但它不会终止函数的执行。相反，它会暂时将控制权让给调用者，当再次被调用时，它会从上次执行的 yield 之后恢复执行。请注意，`a` 函数无需显式提供，例如，`__iter__` 方法的实现。

③ 这是生成器函数 `a` 中的最后一个 yield 语句，并且由于此语句之后没有更多语句，`a` 或其生成器对象在再次恢复时将抛出 `StopIteration` 异常。

④ 调用时，`a()` 返回一个生成器对象，它是一个迭代器。请注意，返回的生成器对象隐式地实现了 `__iter__` 和 `__next__` 方法。换句话说，生成器函数隐藏了必须直接实现迭代器的复杂性（例如，正如我们在上一节中所演示的）。

这是一个 `async` 生成器函数的示例：

```
>>> async def b():
...     yield 'a'
...     yield 'b'
...
>>> async def c():
...     async for i in b():
...         print(i)
...
>>> import asyncio
>>> asyncio.run(c())
a
b
```

① Python 将 `b` 创建为一个异步生成器函数（因为它是一个包含 yield 表达式/语句的 `async def` 函数）。

② 正如我们将在本章后面讨论的，异步语句，如函数体第一行中的 `async for`，只能在异步函数中使用。我们将 `c` 声明为一个异步函数。

③ 调用 `b()` 返回一个异步生成器对象，它可以在 `async for` 循环中使用。

④ 尽管 `async` 和 `await` 是 Python 关键字，但大部分异步编程支持都包含在标准库 `asyncio` 模块中。

⑤ 异步函数可以使用 `asyncio.run` 函数来运行。

## 15.3. 生成器表达式

生成器表达式与推导式非常相似，不仅在语法上，而且在精神上。生成器表达式使用推导式语法（但使用圆括号）来产生一个新的生成器对象。例如，

```
>>> double = (
...     x * 2
...     for x in range(3)
... )
>>> for x in double:
...     print(x)
...
0
2
4
```

① 生成器表达式被括在圆括号中，因此可以写在多行上。为了说明，该表达式被求值并赋值给变量 `double`。更常见的是在需要的地方定义和使用生成器表达式。

② “下一个值”表达式，类似于推导式语法。

③ `for` 子句。

④ 生成器表达式返回一个生成器对象（本例中为 `double`），它是一个迭代器。

### 15.3.1. 异步生成器表达式

如果生成器表达式包含 `async for` 子句或 `await` 表达式，那么它将返回一个新的异步生成器对象，该对象可以被异步迭代。

```
>>> async def a():
...     yield 0; yield 1; yield 2
...
>>> adouble = (
...     x * 2
...     async for x in a()
... )
>>> async def c():
...     async for x in adouble:
...         print(x)
...
>>> import asyncio
>>> asyncio.run(c())
0
2
4
```

① 一个 `async` 生成器函数。

② `async` 生成器表达式也被括在圆括号中。这是一个 `async` 生成器表达式，因为它包含一个 `async for` 子句。

③ “下一个值”表达式，类似于推导式语法。

④ `async for` 子句。请注意，`a()` 返回一个异步生成器对象。

⑤ 我们定义了一个 `async` 函数，用于说明，因为 `async` 语句只能包含在 `async` 函数中。

⑥ `adouble` 是一个 `async` 生成器对象，它是一个 `async` 迭代器。同样，`async` 生成器表达式的结果不必赋值给一个单独的变量。我们可以在这里“就地”使用它。

⑦ 我们可以借助 `asyncio.run` 函数来运行 `c()` 函数。

迭代器类型实现了 `__iter__` 和 `__next__` 方法。同样，异步迭代器类型实现了 `__aiter__` 和 `__anext__` 方法。当它们耗尽元素时，迭代器会引发 `StopIteration` 异常，而 `async` 迭代器会引发 `StopAsyncIteration` 异常。这些细节都隐藏在（同步或异步）生成器函数和生成器表达式的实现中。

## 15.4. 协程对象

### 15.4.1. 可等待对象

实现了特殊 `__await__` 方法的类型是 `awaitable`（可等待对象）。`await 表达式`可以用于 `awaitable`。

```
object.__await__(self)
    它返回一个 `iterator`（迭代器）。它用于定义一个 `awaitable` 类型。例如，`asyncio.Future` 实现了此方法，因此 `Future` 对象可以用作 `await` 表达式的操作数。
```

### 15.4.2. 协程对象

协程对象是 `awaitable` 对象。例如，`async def` 函数返回协程对象。

协程的工作方式与可迭代对象类似。调用协程的 `__await__` 方法返回一个 `iterator`，协程通过迭代这个 `iterator` 对象来执行。当协程执行完毕并返回时，迭代器会引发 `StopIteration` 异常。

然而，与可以多次迭代的可迭代对象不同，协程对象不能被 `await`（等待）超过一次。尝试这样做会引发 `RuntimeError` 异常。

此外，协程对象也不直接支持迭代。相反，它们包含以下方法：

**`coroutine.send(value)`**
它启动或恢复协程的执行。如果参数是 `None`，这等同于推进 `__await__` 返回的迭代器。如果提供了非 `None` 值，则此方法委托给迭代器的 `generator.send` 方法。

**`coroutine.throw(type[, value[, traceback]])`**
它在协程的挂起点引发指定类型的异常。如果迭代器有 `generator.throw` 方法，则委托给此方法。如果异常在协程中未被捕获，它会传播回调用者。

**`coroutine.close()`**
它导致协程在运行任何清理代码后退出。如果迭代器有 `generator.close` 方法，则可选择先委托给该方法。然后在挂起点引发 `GeneratorExit` 异常，导致协程立即清理自身。最后，它被标记为执行完毕，结束协程的生命周期。

标准库 `asyncio` 模块提供了许多便利的类型和方法，例如 `Task` 和 `Future`，用于管理协程。

## 15.5. 协程函数

`async def` 语句定义协程函数的方式类似于 `def` 语句定义（普通）函数的方式。使用 `async def` 定义的协程*可以*，但不必，包含 `await` 表达式和 `async` 语句，如 `async for` 和 `async with`。

调用协程函数返回一个 `awaitable` 对象，更具体地说，是一个 `coroutine object`（协程对象）。协程的执行可以被挂起和恢复，如上一节所述。

```
>>> async def a():
...     print("a")
...
>>> async def b():
...     await a()
...
>>> import asyncio
>>> asyncio.run(b())
a
```

① 一个不包含任何 `async/await` 表达式/语句的协程函数定义。

② 另一个包含 `await` 表达式的协程函数定义。

③ 调用 `a()` 将返回一个协程对象，它可以被 `await`（等待）。

④ 通过调用 `b()` 返回的协程对象，可以使用 `asyncio.run` 函数来运行。调用（普通）函数会执行函数对象中定义的语句。相反，调用协程函数仅返回一个协程对象。

## 15.6. Await 表达式

`await` 表达式会挂起 *协程* 在 *可等待* 对象上的执行。它只能在 `async def 函数`（即协程函数）内部使用。

例如，

```python
>>> import asyncio ①
>>> async def a(): ②
...     print("a() called")
...     await asyncio.sleep(5.0) ③
...     print("Leaving a() after sleeping 5 seconds")
...
>>> asyncio.run(a()) ④
a() called
Leaving a() after sleeping 5 seconds
```

① 大部分的 async/await 运行时支持都包含在 `asyncio` 模块中。

② `async def`/协程函数在被调用时，会返回一个协程对象（或简称协程）。

③ `asyncio.sleep` 函数返回一个在给定时间（以秒为单位）后完成的协程。`await` 表达式可以在协程函数中与 `awaitable` 对象一起使用。

④ 该协程可以使用 `asyncio.run` 来运行。

## 15.7. 其他 async 语句

### 15.7.1. async for 语句

“异步可迭代对象”能够在其实现 `iter` 方法时调用异步代码，而“异步迭代器”则能在其 `next` 方法中调用异步代码。`async for` 语句允许方便地迭代异步迭代器。

`async for in else` 复合语句的语法与 `for in else` 语句大致相同。但是，它迭代的是 `async 迭代器` 而非 `迭代器`。

以下是一个示例：

```python
>>> async def words():
...     yield "hello"
...     yield "multiple"
...     yield "universes"
...
>>> async def greet():
...     async for word in words():
...         print(word, end=' ')
...     else:
...         print()
...
>>> import asyncio
>>> asyncio.run(greet())
hello multiple universes
>>>
```

① `async def` 函数返回一个异步迭代器对象（即协程）。

② `async` 语句只能包含在协程函数中。

③ `async for - in` 子句。`words()` 的类型是一个异步迭代器。

④ 一个可选的 `else` 子句。当 `async for` 在正常迭代完异步迭代器后退出时，会执行此子句。

⑤ 尾随的换行符由 `async for` 语句的 `else` 子句打印。

### 15.7.2. `async with` 语句

异步上下文管理器是一种能够在其 `enter` 和 `exit` 方法中挂起执行的上下文管理器。`async with` 复合语句的工作方式与 `with` 语句非常相似。但是，它使用的不是上下文管理器，而是支持 `__aenter__` 和 `__aexit__` 方法（对应于上下文管理器的 `__enter__` 和 `__exit__`）的异步上下文管理器。

其语法如下：

```python
async with Expression as Target:
    Suite
```

- `Expression` 必须求值为一个异步上下文管理器类型。
- `as` 子句是可选的，与 `with` 语句一样。
- `Suite` 可以包含 `await` 表达式和其他 `async` 语句。
