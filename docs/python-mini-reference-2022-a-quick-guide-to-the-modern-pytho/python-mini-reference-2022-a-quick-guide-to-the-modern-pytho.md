# 前言

Python是一门动态语言。

这意味着许多不同的事情。它意味着这门语言更灵活。它意味着Python是一门更容易上手的编程语言。与许多其他编程语言相比，你可以快速编写一个简单的程序，而无需经历太多"仪式"。Python脚本通常需要更少的样板代码。Python给了你更多的自由。

另一方面，这也意味着这门语言安全性较低，更容易出错。这意味着用Python构建大型系统更加困难。这意味着用Python编写的软件通常更难长期维护。

有效使用Python的关键在于理解这种权衡。例如，Python可以是快速原型设计或"脚本编写"的理想语言。另一方面，Python在"企业软件"开发中使用得不那么多。

Python是一种通用编程语言。Python被应用于许多不同的领域，从系统管理任务到Web应用开发。Python现在已成为科学家们最广泛使用的编程语言之一，而他们传统上使用的是像Matlab这样的更高级工具。Python为这些"非专业"程序员提供了一条更简单的"升级"路径。现在，Python正成为机器学习和数据科学的*首选*语言。

Python对初学者很友好。事实上，它似乎是初学编程者最喜欢的第一门语言，甚至超过了JavaScript。

然而，应该指出的是，Python*并非*一门简单的语言。在其大约30年的历史中，它经历了许多大大小小的变化。用Python开始编程仍然很容易。但是，一旦你达到某个阶段，比如说，从高级初学者到中级水平，其复杂性可能会让人难以承受。

本书将为你提供一个关于Python语言的广泛且*精炼*的概览，涵盖其最新版本（例如，3.10和3.11）。本书对于任何一直在没有坚实基础的情况下涉猎Python的人都可能有用。对于那些有其他编程语言经验并希望快速了解该语言的人，它也可能有用。

本书是作为（非正式的）语言参考编写的。本书的目标不是教你如何有效地用Python编程，而是提供该语言的简明总结。

## 1. 引言

本书将为你概述Python语言的语法。

如今，编程语言本身与标准库之间的界限似乎变得越来越模糊。虽然我们的目标并非全面介绍Python标准库（这远超本书范围），但我们将涉及标准库模块中一些被认为是语言组成部分的重要概念。

这本“参考手册”从Python最枯燥的部分开始。如果你计划从头到尾阅读本书，而不是仅仅将其用作快速参考，那么你可以在第一次阅读时跳过前几章，待需要时再回过头来阅读。

“程序”一词在不同的上下文和不同的编程语言中意味着不同的东西。我们将从对Python程序是什么以及Python程序如何执行（例如在Python解释器中）的较为正式的解释开始。

Python程序在逻辑上被组织成包和模块，接下来我们将对此进行探讨。*模块*对应于物理文件系统中的一个文件，模块（包括包模块）是Python中代码重用和共享的基本单元。

接下来，我们将简要介绍Python源代码的一些词法元素。不同编程语言之间存在一些小的差异，但它们的词法构成相当相似，Python也不例外。这部分内容可以在你的“阅读”中跳过。

一般来说，程序由*代码*和*数据*组成。代码指的是指令。Python中的数据由*对象*表示。Python中的对象是一个基本组成部分。我们在Python程序中处理的一切都是对象。本书的主体部分将从介绍与对象相关的各种重要概念开始。

尽管Python使用动态类型系统，但类型在Python编程语言中仍然扮演着基础性的角色。我们首先介绍Python中一些基本的内置类型。Python包含相当多的内置类型，而这本参考手册仅涉及其中一部分。如前所述，这本参考手册中讨论的所有主题通常都如此。完整性并非这本*迷你参考手册*的目标。

Python还包括一些内置的复合类型，如列表和字典，它们是任何非平凡程序的重要组成部分。我们将在下一章简要介绍这些类型。高级类型，特别是函数和类，将在本书后面详细解释。

与任何命令式编程语言一样，Python有表达式和语句。表达式规定了如何使用其他表达式和值来计算一个值。Python支持其他命令式语言中常见的大多数运算符和表达式。如果你来自其他过程式编程语言的背景，你可以略读这部分的大部分内容。

语句是一条指令。语句控制程序的流程以达到预期的目标。在Python中，有两类语句：简单语句和复合语句。复合语句或复杂语句可以“包含”其他简单或复合语句。

简单语句包括表达式语句、赋值语句、断言语句、空语句、删除语句、返回语句、引发语句、中断语句、继续语句、全局语句和非局部语句。Python的复合语句包括if语句、while语句、for语句、try语句、with语句、match语句、函数定义语句、类定义以及其他与协程相关的语句。

过去30多年来，Python最重大的变化之一是自Python 3.10（2021年）起，向语言中添加了结构化模式匹配。模式匹配最初由Haskell等函数式编程语言推广，现在正变得越来越广泛地应用于许多不同的编程语言，如C#、Rust、Swift、Scala，以及（是的）甚至Java。

随着我们作为一个社区更多地采用这一特性，模式匹配很可能会在未来几年改变我们使用Python编程的方式。尽管目前它仅在match语句的上下文中受支持，但考虑到其简洁性、优雅性和强大功能，可以预见，并且事实上也期望，模式匹配在不久的将来将在更广泛的语言范围内可用。

函数定义是一种复合语句。我们为函数定义专门设了一章。本章还包括其他与函数相关的主题，如lambda表达式和装饰器。

接下来解释另一种复合语句——类定义。本章也描述了其他与类相关的概念，如枚举和数据类。我们还对Python中的面向对象编程风格进行了非正式介绍，包括多重继承。

特殊类型的函数——生成器和协程——将在本书的最后一部分“协程”章节中单独讨论。我们将简要介绍使用新的async和await关键字进行（高级）异步编程的风格。在最后一节，我们提供了一个简单的异步编程示例，即生产者-消费者问题。

如前所述，我们并未在这本“迷你参考手册”中详尽涵盖Python的每一个主题，包括（低级的）多线程和多进程编程等。然而，它涵盖了任何中级到高级Python程序员都应该熟悉的所有*核心语言特性*。

## 2. PYTHON程序

一个完整的Python程序可以传递给Python解释器：

-   使用`-c`命令行选项传递程序文本，
-   使用`-m`命令行选项传递模块名，
-   作为第一个命令行参数传递的文件名，或
-   从标准输入。

Python解释器在特殊模块`__main__`的命名空间中，将输入文件或输入文本作为一个完整的程序来执行。

当文件或标准输入是终端，或者当Python解释器在没有参数的情况下被调用时，解释器会进入交互模式。在交互模式下，Python解释器一次读取并执行一条语句（简单或复合），而不是执行一个完整的程序。

以下是几个示例：

### 2.1. 文件/文本输入

一个完整的 Python 程序由零条或多条语句组成，语句之间可以有可选的换行符。在以下情况下，需要这种形式的完整程序：

-   当从文件或输入字符串读取 Python 程序时，
-   当解析导入的模块时，以及
-   当将字符串参数传递给内置的 `exec` 函数时。

如前所述，运行程序的主模块在 Python 中始终被称为 `__main__`。

### 2.2. 交互模式

交互模式下的 Python 解释器（也称为 Python REPL）不会执行完整的程序。相反，它一次读取并执行一条语句。每次输入都构成 `__main__` 命名空间中的一个代码块。

在交互模式下，顶层复合语句后必须跟一个额外的换行符，解析器用它来检测输入的结束。

```
$ python
Python 3.10.4 (main, Jun 29 2022, 12:14:53) [GCC 11.2.0] on linux
Type "help", "copyright", "credits" or "license" for more information.
>>> 
>>> for levels in ("White", "Black"):
...     print(levels)
...
White
Black
>>> 
>>> exit()
$
```

-   启动 Python REPL 的命令。在你的系统上可能有所不同。
-   Python REPL 提示符 `>>>`。在此示例中，我们刚刚按下了 Enter 键。
-   for 复合语句的第一行。
-   提示符已从 `>>>` 变为 `...`。for "套件"中的语句需要缩进。
-   我们再次按 Enter 键，向 Python 解释器发出复合语句结束的信号。
-   输出，分两行显示。请注意，输出行前面没有 REPL 提示符。
-   Python REPL 等待另一个命令/语句。在此示例中，我们再次按 Enter 键。
-   你可以通过调用内置的 `exit` 或 `quit` 函数，或发出 EOF 信号（例如，CTRL+D 或 CMD+D）来退出 Python REPL。
-   返回 Unix shell。

## 3. 程序执行

### 3.1. 代码块

Python 程序在一系列“执行帧”中执行。在执行帧中作为一个单元执行的 Python 程序片段称为“代码块”。

例如，模块和函数/类定义等都是代码块。传递给解释器的整个程序，例如作为 `-c` 选项或作为文件参数，是一个代码块（更准确地说，是顶层代码块）。导入的模块也是如此。

代码块可以嵌套。

### 3.2. 名称绑定

在 Python 中，所有值都是“对象”，对象通常通过其“名称”来引用。名称通过名称绑定操作引入。

例如，导入语句、函数和类定义以及赋值表达式和语句在其封闭的代码块内绑定名称。函数参数也在函数定义块内绑定名称。如果一个名称在块中被绑定，它默认是该块的“局部变量”。

如果一个名称在模块级别绑定，它也是一个“全局变量”。形式为 `from XYZ import *` 的导入语句只能在模块级别使用。它绑定导入模块中所有不以下划线 (`_`) 开头的名称。所有以此方式导入的名称都是全局的（对于导入模块来说也是局部的）。

### 3.3. 作用域

一个名称在程序的特定部分内是可访问的，例如，在名称被绑定的代码块内。这称为“作用域”。在代码块中绑定的局部变量的作用域包括该块。

当在代码块中使用一个名称时，会使用最小的封闭作用域来查找该名称的绑定。这称为“名称解析”。如果无法解析该名称，则会引发异常。根据上下文，可能会引发 `NameError`，或者更具体地说是 `UnboundLocalError`。

#### 3.3.1. 函数定义块

函数 `def` 语句定义一个代码块。如果一个名称在函数块中被绑定，那么它的作用域包括此函数块以及其中包含/嵌套的任何其他块。

#### 3.3.2. 类定义块

在类定义中，类块中定义的名称的作用域仅限于该类块，*不包括*封闭方法中的名称。未在类块中绑定的局部变量将在全局命名空间中查找。

#### 3.3.3. global 声明

如果在块中对一个名称使用了 `global` 语句，那么对该名称的引用将指向其在顶层命名空间中的绑定，该命名空间包括代码块模块和内置模块的绑定，以及内置命名空间。

#### 3.3.4. nonlocal 声明

如果在代码块中对一个名称使用了 `nonlocal` 语句，该名称将引用在最小封闭函数作用域中绑定的名称。如果在编译时未找到该名称绑定，则会引发 `SyntaxError` 异常。

这是一个单一模块（例如，名为 *my_module.py* 的 Python 源文件），演示了函数和类块以及这些块中定义的变量的作用域。（注意：由于格式原因，程序/模块被分成三个部分。）

-   [一个函数定义](https://example.com)。它创建一个块，第 4-12 行。

## 3.5. 程序启动与终止

如前所述，脚本的主模块 `__main__` 是一个代码块。

Python 解释器通过执行主代码块来启动程序，该代码块随后可以调用其他代码块，而这些代码块又可以调用其他代码块，依此类推。当一个代码块执行完毕时，它会将控制权返回给调用它的外围代码块。

当主模块代码块执行完毕时，程序终止。

## 3.6. 异常

程序的正常流程（通过代码块调用链）可以使用异常处理机制来绕过。在异常或错误条件下，可以使用 `raise` 语句显式地“引发”一个异常。当 Python 解释器检测到运行时错误时，也可以引发异常。

一个被引发的异常可以由外围的或任何上游的代码块处理。当异常在任何代码块中都未被处理时，解释器会终止程序的执行（在非交互模式下）。在 REPL 模式下，它只是返回到其交互式主循环。无论哪种情况，它都会打印一个堆栈跟踪信息，除非异常是 `SystemExit`。

异常处理程序通过 `try` 语句指定。从 Python 3.11 开始，除了 `try - except` 之外，还支持新的 `try - except*` 语法。

Python 中有两种基本的内置异常类型：`BaseException` 和 `Exception`，它们大致分别对应于 Java 中的“运行时异常”和“受检异常”。同样，Python 现在（3.11+）包含两种内置异常组类型：`BaseExceptionGroup` 和 `ExceptionGroup`。

异常处理将在本书后续关于 `try` 和 `raise` 语句的章节中进一步讨论。

## 4. 包与模块

### 4.1. 模块

Python 代码被组织成“模块”，模块与包含其他 Python *对象* 的命名空间相关联。模块中的 Python 代码可以通过“导入”过程访问不同模块中的代码。例如，可以使用 `import` 语句导入一个模块。

#### 4.1.1. import 语句

`import` 语句执行以下操作：

-   它首先搜索指定的模块，并且
-   如果找到该模块，则
    -   它创建并初始化一个模块对象，并将该对象绑定到本地作用域中的一个名称。
-   如果未找到指定的模块，则
    -   引发一个 `ModuleNotFoundError` 异常。

例如，

```python
>>> import math
>>> type(math)
<class 'module'>
```

`import` 一个导入语句。

`type` 我们将在本书中探讨 Python 的类型系统。内置的 `type` 函数返回给定对象的类型。

`module` 模块对象的类型是 `module`。

`import` 语句的内部实现以及其他与导入相关的实用函数，都由标准库模块 `importlib` 提供。

### 4.2. 包

为了帮助组织模块，Python 引入了包的概念。包是一种特殊的模块，它提供了一个命名空间/命名层次结构。从技术上讲，包是包含 `__path__` 属性的模块。

如果使用脚本调用 Python 解释器，那么一个包对应于文件系统上的一个目录，而一个常规的非包模块对应于一个 Python 源文件。像文件系统目录一样，包是分层组织的，包可以包含其他包以及常规模块。

每个模块都有一个名称。子包名称与其父包名称用点（.）分隔。模块的 `__name__` 属性被设置为模块的完全限定名。模块的限定名用于在 Python 导入系统中唯一标识模块。

Python 中有两种包：“常规包”和“命名空间包”。

#### 4.2.1. 常规包

常规包通常实现为一个包含 `__init__.py` 文件的目录。当导入一个常规包时，该包及其所有父包的 `__init__.py` 文件会被隐式执行。在每个包的 `__init__.py` 文件中定义的对象被绑定到该包命名空间中的名称。

#### 4.2.2. 命名空间包

隐式命名空间包是一个（虚拟的）复合包，它可以包含来自文件系统不同位置的子包（例如，不受目录层次结构的限制）。

使用命名空间包时，没有包 `__init__.py` 文件。与常规包不同，一个命名空间包可能对应于多个目录。例如，当导入包 `parent/one` 和 `parent/two`，而没有与父模块对应的物理目录时，Python 会自动为顶级父包创建一个命名空间包。

## 4.3. 包相对导入

相对导入使用前导点（`.`）。单个前导点表示从当前包开始的相对导入。两个前导点表示从当前包的父包开始的相对导入，以此类推。

绝对导入可以使用 `import X.Y` 或 `from X import Y` 语法，但相对导入只能使用第二种形式。例如，对于文件夹 `bbb` 中给定的 Python 脚本/模块 `xyz`，

清单 2. aaa/bbb/xyz.py

```
from . import efg
from .hij import klm
from .. import mno
from ..opq.qrs import stu
```

- 模块 `efg` 对应文件 `aaa/bbb/efg.py`。
- 模块 `klm` 对应文件 `aaa/bbb/hij/klm.py`。
- 模块 `mno` 对应文件 `aaa/mno.py`。
- 模块 `stu` 对应文件 `aaa/opq/qrs/stu.py`。

## 5. PYTHON 源代码

Python 解释器将程序文本读取为 Unicode 码点。Python 源代码文件的默认编码是 "UTF-8"。源文件也可以给出显式的编码声明。如果声明了编码，则编码名称必须能被 Python 识别。如果文本无法解码，则会引发 SyntaxError 异常。

Python 解释器首先将源文件分解为“词法单元”，然后将其输入解析器。

### 5.1. 行结构

Python 程序是“基于行”的。Python 源代码由“逻辑行”组成。

#### 5.1.1. 逻辑行

一条语句通常包含在一个逻辑行中，其结尾由 NEWLINE 词法单元表示。如果 Python 语法允许，复合语句可以跨越多个逻辑行。

一个逻辑行由一个或多个“物理行”构成。

#### 5.1.2. 物理行

物理行是以行结束序列终止的字符序列。在 Python 源文件和字符串中，无论平台如何，都可以使用三种标准行终止序列中的任何一种：\n、\r\n 或 \r。输入的结束也作为最后一个物理行的隐式终止符。

#### 5.1.3. 注释

注释以井号字符（#）开头，并持续到物理行的末尾。语法通常忽略注释。注释可以表示逻辑行的结束，即，除非调用了隐式行连接规则，否则会生成一个 NEWLINE 词法单元。

#### 5.1.4. 空行

仅包含空格、制表符、换页符，可能还有注释的逻辑行会被忽略。

#### 5.1.5. 物理行连接

显式连接

可以使用反斜杠字符（\）将两个或多个物理行连接成一个逻辑行。

当一个物理行以反斜杠结尾时，它会与下一行连接，形成一个单一的逻辑行，同时删除反斜杠和随后的行结束字符。

- 除了在字符串字面量或注释中，反斜杠不允许出现在行上的任何其他位置。
- 除了字符串字面量，反斜杠不能将一个词法单元分割到多个物理行上。
- 反斜杠不会延续注释。

例如，

```
>>> a, b = \
... 10, 20
```

这是一个逻辑行，a, b = 10, 20，尽管它写在两个物理行上。请注意，除了三种行终止序列之一外，反斜杠后不允许有其他任何字符，包括空格。

隐式连接

括号（(...)）、方括号（[...]）或花括号（{...}）中的表达式，以及三引号字符串（"""...""" 或 '''...'''），可以分割在多个物理行上，而无需使用反斜杠。

隐式延续的行可以携带注释，但三引号多行字符串除外。延续行的缩进并不重要。允许空的延续行。

例如，

```
>>> fruits = [
...     "manzana",    # apple
...     "naranja",    # orange
... ]
>>> print(
...     fruits
... )
['manzana', 'naranja']
```

- 隐式连接的开始。
- 允许注释。
- 逻辑行的结束。
- 另一个隐式连接的开始。
- 一个空的物理行。
- 逻辑行的结束。此语句等效于在一个物理行中的 print(fruits)。

请注意，由于括号可用于对表达式进行分组，实际上任何表达式都可以分割成多个物理行。

#### 5.1.6. 缩进

在 Python 中，逻辑行的缩进用于确定语句的分组。

一行的缩进级别是根据行首的前导空白计算的。第一个非空白字符之前的空格总数决定了该行的缩进。

Python 词法分析器使用一个栈，根据连续逻辑行的缩进级别生成 INDENT 和 DEDENT 词法单元，然后用于将逻辑行分组为语句。

### 5.2. 词法单元

除了 NEWLINE、INDENT 和 DEDENT，Python 还有以下类别的词法单元：*标识符*、*关键字*、*字面量*、*运算符*和*分隔符*。

### 5.3. 标识符和关键字

Python 中标识符（或名称）的语法基于 Unicode 标准。标识符长度不受限制，并且区分大小写。在 ASCII 范围内（U+0001..U+007F），名称可以包含字母数字字符和下划线（_）。名称不能以数字开头。在 ASCII 范围之外，unicodedata 模块对标识符中有效的字符进行分类。

#### 5.3.1. 关键字

以下名称用作保留字或语言的关键字，不能用作普通标识符。

- False True None and as assert async await break class continue def del elif else except from finally for global if import in is lambda nonlocal not or pass raise return try while with yield

#### 5.3.2. 保留的标识符类别

除了关键字，某些类别的标识符对 Python 具有特殊含义。这些类别通过前导和尾随下划线字符的模式来识别：

- `_`
    特殊标识符 _ 在交互式解释器中用于存储上次评估的结果。
- `_xyz`
    不会被 `from module import *` 导入。
- `__xyz__`
    系统定义的名称，非正式地称为“双下划线名称”。这些名称由解释器及其实现（包括标准库）定义。
- `__xyz`
    类私有名称。在类定义中使用时，这些名称会被重写为一种修饰形式，以帮助避免基类和派生类的“私有”属性之间的名称冲突。

### 5.4. 字面量

“字面量”是词法分析器具有特殊含义的一类特定词法单元，不同于标识符和其他特殊词法单元。通常，字面量是某些内置类型的常量值。

#### 5.4.1. 字符串和字节字面量

字符串和字节字面量可以用匹配的单引号（'）或双引号（"""）括起来。它们也可以用匹配的三单引号（'''）或三双引号（""")括起来。

反斜杠（\）字符在字符串和字节字面量中用于“转义”那些原本具有特殊含义的字符，例如换行符、反斜杠本身或引号字符。

字节字面量是 bytes 类型的实例，始终以 'b' 或 'B' 为前缀。它们只能包含 ASCII 字符。字符串和字节字面量都可以选择性地以字母 'r'/'R' 为前缀。这样的字面量称为原始字符串，将反斜杠视为字面字符。

#### 5.4.2. 格式化字符串字面量

*格式化字符串字面量*，简称 *f-string*，是带有前缀 'f' 或 'F' 的字符串字面量。这些字符串可以包含替换字段，即用花括号 {} 括起来的变量或表达式。可以添加格式说明符，位于可选的冒号 : 之后。

虽然其他字符串字面量总是具有常量值，但 f-string 实际上是在运行时求值的表达式。

可以在表达式之后、冒号和格式字符串（如果有）之前添加等号 =。当包含它时，结果字符串将连接表达式文本、= 号和求值后的值。

例如，

```
>>> a = 10 / 7
>>> f"{a}"
'1.4285714285714286'
>>> f"a = {a:.5f}"
'a = 1.42857'
>>> f"{a=}", f"{10/7=:.5f}"
('a=1.4285714285714286', '10/7=1.42857')
```

- 一个简单的 f-string。
- 替换字段在 : 之后有格式说明符 .5f。
- 等号打印出表达式文本，后跟 =，然后是值。

#### 5.4.3. 字符串字面量连接

我们可以使用 `+` 运算符来连接字符串表达式。作为快捷方式，对于相邻的字符串字面量以及字节字面量，可以在不使用 `+` 运算符的情况下进行连接。它们可以由（可能为空的）空格分隔，并且其效果与使用 `+` 运算符进行字符串连接相同。

字面量连接可以对每个操作数使用不同的引号风格。F-string 字面量可以与普通字符串字面量连接。例如，

```
>>> a = "hello"
>>> (
...     "hi" "hi"
...     f"{a} world"
...     """bye bye"""
... )
'hihihello worldbye bye'
```

此处使用带括号的表达式是为了将字符串字面量放在多个物理行中，以作说明。

"hi" "hi" 等同于 "hi" + "hi"。

F-string 字面量可以与其他字符串字面量连接。同样，`+` 运算符是可选的。

一个可以跨越多个物理行的三引号字符串字面量，在此示例中与相邻的格式化字符串字面量连接，无需使用 `+` 运算符。

#### 5.4.4. 数值字面量

数值字面量有三种类型：整数、浮点数和虚数。

整数字面量必须以数字开头。以 `0x/0X`、`0o/0O` 和 `0b/0B` 开头的字面量分别是十六进制、八进制和二进制数。十进制数不能以数字 `0` 开头，数字 `0` 除外。

在计算整数字面量的值时，下划线 (`_`) 会被忽略。整数字面量不能以下划线开头，并且不允许连续多个下划线。例如，

```
0b_010                # 表示十进制数 2 的二进制整数字面量。
00000_010             # 一个八进制整数字面量，表示十进制数 8。
0x0F                  # 一个等同于十进制数 15 的十六进制字面量。
1_000_000             # 一个十进制数（基数为 10）。注意十进制整数字面量不能以 0 开头。
```

浮点字面量包含一个句点 (`.`) 和/或指数符号 (`e` 或 `E`)。整数部分和指数部分均使用基数 10 解释。

浮点数字面量示例：

```
100.
03.14
.0101
0_0_0e0_0_0
31.415_927E-10
```

虚数字面量，即带有后缀 `j` 或 `J` 的浮点字面量，会生成一个实部为 `0.0` 的复数。表达式 `1j * 1j` 会生成一个复数 `(-1+0j)`。

要创建一个具有非零实部的复数，可以使用将浮点数与虚数字面量相加的表达式。例如，

```
100j                # 一个虚数字面量。
03.14J              # 另一个虚数字面量。
10E10 + 0.0001E-100j # 一个包含实数和虚数字面量的表达式。
10 - 0.314_159e+10J  # 另一个类型为复数的表达式。
```

## 6. 对象

在 Python 中，数据由 *对象* 表示。一个对象具有 *标识*、*类型* 和 *值*。一个对象可以通过一个“名称”（或“引用”）来引用。一个对象在其整个生命周期中具有唯一不变的标识，但在任何给定时刻，它可以有零个、一个或多个名称/引用。

Python 是一种动态和松散类型的编程语言。尽管类型系统是 Python 编程语言的核心，并且类型在 Python 程序中起着至关重要的作用，但它们主要在运行时才变得相关，并且只是间接相关。（Python 的类型系统通常被称为“[鸭子类型](https://en.wikipedia.org/wiki/Duck_typing)”。）事实上，可以在运行时创建一个新类型，甚至可以在运行时更改对象的类型（尽管这通常不是常见做法）。

某些类型的对象的值可以更改。这些对象被称为“[可变](https://en.wikipedia.org/wiki/Immutable_object)”。例如，像列表和字典这样的内置数据类型是可变类型。另一方面，对于其他一些类型，一旦对象创建，就不能*直接*修改其值。它们被称为“[不可变](https://en.wikipedia.org/wiki/Immutable_object)”。例如，数字、字符串和元组是不可变类型。（注意，不可变对象的（有效）值仍然可以更改。）

### 6.1. 标识

“标识”是一个相当抽象的概念，但 Python 包含一个内置的 `id` 函数，该函数返回一个唯一值，可用作给定对象的标识。事实上，`id` 函数“定义”了对象的标识。（在 CPython 实现中，它返回给定对象的内存地址。）

例如，

```
>>> a, b, c = 3, "the ring", ['a', 'b', 'c']
>>> id(a), id(b), id(c)
(546842556784, 546841138928, 546841139008)
>>> x = 3
>>> id(a), id(x), id(3)
(546842556784, 546842556784, 546842556784)
```

此提示符 `>>>` 表示我们处于 REPL 环境中。

我们经常在本书中使用这种“多重赋值”语法。右侧的[表达式列表](https://docs.python.org/3/reference/expressions.html#expression-lists)会计算为一个元组。然后通过“元组解包”，其每个元素被分别赋值给 `a`、`b`、`c`。

名称 `a`、`b` 和 `c` 都有不同的 `id` 值。它们引用不同的对象。

名称 `a` 和 `x` 以及对象 `3` 都具有相同的 `id` 值。`a` 和 `x` 引用同一个对象 `3`。

对象的标识也可以使用内置的 `is` 和 `is not` 运算符进行比较。`x is y` 的计算结果为 `True`，当且仅当 `x` 和 `y` 是同一个对象，即根据 `id` 函数。注意，两个对象之间的标识相等 (`is`) 意味着它们的值相等 (`==`)。

使用上面相同的例子，

```
>>> a is a, a is b, a is c
(True, False, False)
>>> a is x, a == x
(True, True)
```

`(a is a) == True`。名称 `a`、`b` 和 `c` 引用不同的对象。

`(a is x) == True` 且 `(x is a) == True`。同样，`a == x` 且 `x == a`。

### 6.2. 属性

一个对象与一组“属性”相关联。属性是一个引用（其他）对象的名称。当它引用的对象是一个函数时，该属性被称为（原始对象的）*方法*。否则，它被称为数据属性（例如，在其他编程语言中通常称为“字段”或数据成员）。

对象的属性可以使用[属性引用](https://docs.python.org/3/reference/datamodel.html#attribute-access)语法（或“点表示法”）来访问。

```
>>> class A: pass
...
>>> a = A()
>>> a.magic_number = 42
>>> a.magic_number
42
>>> def f(): pass
...
>>> a.empty_method = f
>>> a.empty_method()
```

`class` 语句创建一个新类型 `A`。

调用 `A()`，这被称为`构造函数`，返回类型 `A` 的一个实例对象。在此示例中，该对象被`绑定到名称` `a`。

向 `a` 添加一个名为 `magic_number` 的数据属性，并将其绑定到一个（不可变的）数字对象 `42`。

`a.magic_number` 引用对象 `42`，其值为 `42`。（注意，对于`简单类型`，对象的标识和值紧密相关。）

`def` 语句创建一个新的函数对象 `f`。

新属性 `a.empty_method` 现在引用对象 `f`。

方法调用语法调用函数 `f`。

内置的 `dir` 函数可用于

- 查找当前作用域中的所有名称，或
- 列出与给定对象关联的属性。

在当前上下文中，使用上面的例子，

```
>>> dir()
['A', '__annotations__', ... '__spec__', 'a', 'f']
>>> dir(A)
['__class__', ... '__weakref__']
>>> dir(a)
```

### 6.3. 类型

Python 中的每个对象，除了其标识和值之外，都关联着一个“类型”。对象的类型会影响其行为。例如，对象的类型定义了该对象可能具有的所有值。此外，对象的类型还决定了该对象支持的操作集合。（请注意，Python 中（大多数类型的）对象可以说是可变的，其行为在程序执行过程中可能会发生变化，无论其指定的类型/类型名称如何。参见 [鸭子类型](https://docs.python.org/3/glossary.html#term-duck-typing)。）

再举一个例子，对于不可变类型的对象，Python 解释器会在可能的情况下“重用”它们。也就是说，对于不可变类型，某些本应返回新对象或计算新值的操作，实际上可能会返回一个指向任何具有*相同类型和值*的现有对象的引用。相反，对于可变类型，在这些情况下对象永远不会被“重用”。例如，

```
>>> def f():
...     a, b = (1, 2), [3, 4]
...     print(id(a), id(b))
...     p, q = (1, 2), [3, 4]
...     print(id(p), id(q))
...
>>> f()
510220577408 510220577600
510220577408 510221371520
```

请注意，元组 `a` 和 `p`（不可变对象）的 id 相同，而列表 `b` 和 `q`（可变对象）的 id 不同。`a` 和 `p` 指向同一个对象。另一方面，`b` 和 `q` 指向两个不同的对象。

### 6.4. 内置 type 函数

Python 提供了一个内置的 `type` 函数，它返回给定对象的类型/类。类型实际上是一个创建类型的“类对象”。其构造函数之一接受一个对象作为参数，并返回该对象的类型对象（例如，可能是“重用”的）。

例如，

```
>>> type, type(type), type(type(type))
(<class 'type'>, <class 'type'>, <class 'type'>)
>>> type(10), type('hello'), type([1, 2])
(<class 'int'>, <class 'str'>, <class 'list'>)
```

- `type` 的类型是 `type`。
- 对象 `10`、`'hello'` 和 `[1, 2]` 的类型分别是 `int`、`str` 和 `list`。

### 6.5. 可变类型与不可变类型

在 Python 中，可变性/不可变性是类型的一个特征。可变类型的对象是可变的，不可变类型的对象是不可变的。

不能直接更改不可变对象的值。然而，这并不意味着该对象是真正不可变的。例如，一个不可变对象的属性引用了一个对象，而该对象可能仍然是可变的。并且，不可变对象的整体有效值可能会改变。

这对于复合类型通常是成立的。例如，对于像元组这样的不可变容器类型，它们仍然可以包含可变元素，并且元组的整体值仍然可以改变（尽管其值可能仍然无法直接更新）。

例如，

```
>>> a, b = (1, 2), [3, 4]
>>> a[0] = 10
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'tuple' object does not support item assignment
>>> b[0], b[1] = 30, 40
>>> a, b
((1, 2), [30, 40])
>>> x = a, b
>>> x
((1, 2), [30, 40])
>>> x[1] = [3000, 4000]
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'tuple' object does not support item assignment
>>> x[1][0], x[1][1] = 300, 400
>>> x
((1, 2), [300, 400])
```

- `a` 引用一个元组 `(1, 2)`，而 `b` 引用一个列表 `[3, 4]`。
- 元组是不可变的，因此不能更改其元素。
- 相反，可以更改列表的项，因为列表是可变的。
- `a` 和 `b` 引用的对象与之前相同，但 `b` 的值已经改变。
- 右侧的逗号分隔的[表达式列表](https://docs.python.org/3/reference/expressions.html#expression-lists)求值为一个元组对象。
- `x` 指向一个元组，一个不可变对象。
- 不能直接更改其值，因为 `x` 是“不可变的”。
- 然而，如果集合的项或对象的属性是可变的，就可以更改其值。在这个例子中，`x` 的第二个项恰好引用了一个可变对象，一个列表 `[30, 40]`。因此，我们可以更改其值。
- “不可变的” `x` 的有效值确实改变了（尽管 `x` 的项仍然是*相同的*对象）。

### 6.6. 构造函数

Python 中的所有类型，无论是内置的还是用户定义的，都包含[构造函数](https://www.google.com/search?q=constructor+functions)，或简称*构造器*。类型的名称与构造函数的名称相同。或者更准确地说，类型/类是“可调用的”。构造函数被调用时，会返回给定类型/类的实例对象。

对于某些内置类型，如元组和列表，也可以使用字面量语法来创建这些类型的实例，我们将在后面的章节 [内置复合类型](https://www.google.com/search?q=Builtin+Compound+Types) 中解释。

```
>>> class A: pass
...
>>> type(A)
<class 'type'>
>>> a = A()
>>> type(a)
<class '__main__.A'>
```

- [class 语句](https://www.google.com/search?q=class+statement)创建一个新的类对象，在本例中是 `A`。
- 类型 `A` 的类型是 `type`。
- `A`，一个类对象，是可调用的。
- 调用构造函数返回一个类型为 `A` 的实例对象。

### 6.7. 布尔上下文

（几乎）Python 中的所有对象都有“真值”，默认情况下取决于它们的类型和值。当一个对象在 `if` 或 `while` 条件中使用，或作为布尔运算的操作数时，会返回其真值。（`and` 和 `or` 运算符的工作方式略有不同。）

对象的真值默认为 `True`。在以下情况下为 `False`：

- 如果对象/其类定义了 `__bool__` 方法，并且
  - 此方法返回 `False`，或者
- 如果对象没有 `__bool__` 方法，但对象/其类有 `__len__` 方法，并且
  - 此方法返回 `0`。

对于内置类型，以下对象的真值为 `False`：

- `None` 和 `False`。
- 所有数值类型的“零值”对象，例如 `0`、`0.0`、`0j` 等。
- 空集合，例如 `""`、`()`、`[]`、`{}` 等。

自定义类型对象的真值默认始终为 `True`，无论其属性的值如何。为了赋予不同的行为，例如根据其内部状态返回 `True` 或 `False`，我们需要实现 `__bool__` 方法。此方法在自定义类型中默认不存在。

如果一个对象，或者更准确地说其类，实现了此方法，那么它将在布尔上下文中被调用。例如，

```
class BiggerThan8:
    def __init__(self, zzz):
        self.zzz = zzz
    def __bool__(self):
        return True if self.zzz >= 8 else False
```

```
>>> z1, z2 = BiggerThan8(5), BiggerThan8(10)
>>> bool(z1), bool(z2)
(False, True)
```

内置的 `bool` 函数返回参数（对象或表达式，或名称）的布尔值。这段代码展示了“鸭子类型”的另一个（微妙的）强大之处。只要对象 `z` 的类型具有 `__bool__(self)` 方法，它就能按预期工作。在其他静态类型编程语言中，这种功能是通过额外的构造提供的，如接口、特征和原型等。

### 6.8. 对象的生命周期

在Python中，我们可以创建对象，例如使用[构造函数](constructor functions)，但没有办法显式地销毁它们。任何不可达的对象，例如因为没有有效的名称引用它们，都可能被Python运行时进行“垃圾回收”。

[del语句](del statement)可用于从给定对象中移除/解绑名称。此外，Python的标准库模块`gc`可用于显式控制垃圾回收过程的各个方面。

请注意，某些对象可能引用系统资源，如文件和窗口，这些资源由操作系统控制。当这样的对象被垃圾回收时，这些系统资源会自动释放。要显式释放这些资源，可以在[try语句](the try statement)的`finally`块中添加清理逻辑（例如调用`file.close()`）。或者，可以将此类逻辑包含在上下文管理器的`__exit__`方法中，以便与[with语句](the with statement)一起使用。

## 8. 复合类型

### 8.2. 序列

序列是由非负整数索引的有序集合。序列 `a` 的第 `i` 项通过 `a[i]` 选取。索引范围从 `0` 到 `len(a) - 1`。`len` 是一个内置函数，返回序列中的项数。

对序列进行“切片”会产生另一个相同类型的序列，称为切片。例如，`a[i:j]` 选取所有索引在 `i`（包含）和 `j`（不包含）之间的项。切片的索引始终从 `0` 开始，无论其如何切片。

#### 8.2.1. 序列解包

一个表达式列表会求值为一个元组对象。当生成的元组被赋值给单个变量时，这被称为“元组打包”：

```
>>> x = 1, 'a', True
>>> x
(1, 'a', True)
```

“元组打包”。`x` 的类型是 `tuple`。右侧的三个值已被“打包”到一个变量中。

另一方面，一个序列类型对象可以被“解包”到多个变量。

```
>>> p, q, r = ['a', 'b', 'c']
>>> p, q, r
('a', 'b', 'c')
```

右侧一个包含三个项的列表已被“解包”到三个变量 `p`、`q` 和 `r` 中。

通常，变量的数量必须与序列中的元素数量相同。然而，Python 支持一种用于“剩余元素”的特殊语法。例如，

```
>>> p, *q = [1, 2, 3, 4]
>>> p, q
(1, [2, 3, 4])
```

在解包语法中，最多可以有一个变量被标记为 `*`。在将每个前导或尾随元素映射到相应的变量后，此变量会获取序列中的所有剩余元素。请注意，`q` 的类型是 `list`。

### 8.3. 不可变序列

不可变序列类型的对象一旦创建就无法更改。

#### 8.3.1. 字符串

字符串是表示 Unicode 码位的值序列。Python 没有字符类型。内置函数 `ord` 将码位从其字符串形式转换为范围在 `0` 到 `0x10FFFF` 之间的整数。另一方面，`chr` 将范围在 `0` 到 `0x10FFFF` 之间的整数转换为相应的长度为 1 的字符串对象。

字符串可以使用 `+` 运算符进行连接。将两个字符串相加会返回一个*新的*字符串。字符串在 Python 中是真正不可变的。字符串类型包含许多内置方法，如 `count`、`index`、`find`、`replace`、`format`、`join`、`split`、`startswith`、`endswith`、`upper`、`lower`、`isalnum`、`isalpha`、`islower`、`isupper` 和 `isspace`。

#### 8.3.2. 元组

元组是零个、一个或多个任意 Python 对象在一对括号内的不可变序列。元组的*长度*，即元组中元素或项的数量，可以使用 Python 内置函数 `len` 计算。

```
>>> len((False, "Hello", 1_000_000))
3
```

#### 8.3.3. 字节序列

Python 没有字节类型。相反，它有一个字节序列类型，这是一个不可变的 8 位字节数组。

### 8.4. 可变序列

可变序列的项在创建后可以更改，例如，使用赋值或 `del`（删除）语句。

#### 8.4.1. 列表

列表类型是 Python 中另一个重要的内置数据类型。列表是零个、一个或多个对象的可变序列。列表可以使用内置列表方法（如 `append`、`insert`、`pop` 和 `remove` 等）进行修改。列表中元素的顺序可以使用 `reverse` 和 `sort` 方法进行更新。

与元组一样，可以使用内置函数 `len` 获取列表中的元素数量。例如，

```
>>> len([1.0, 2.0, 3.0])
3
```

#### 8.4.2. 列表推导式

除了显式的字面量语法外，还可以使用列表推导式语法创建列表。例如，

```
>>> [ x*x for x in range(5) ]
[0, 1, 4, 9, 16]
>>> [ x*x for x in range(5) if x % 2 == 0 ]
[0, 4, 16]
```

此表达式求值为 `[0, 1, 4, 9, 16]`。`if` 子句可用于过滤元素。

列表推导式以及列表字面量表达式，每次求值时都会产生一个*新的*列表对象。

#### 8.4.3. 字节数组

`bytearray` 对象类似于字节序列，但它们是可变的。`bytearray` 对象由内置的 `bytearray` 构造函数创建。

### 8.5. 集合类型

集合类型表示唯一、不可变对象的无序集合。（集合类型的元素类似于映射类型的键。）集合中的项不能使用下标语法进行索引。但它们可以像其他序列类型一样被迭代。（也就是说，集合是可迭代的。）`len` 函数返回集合中的项数。

#### 8.5.1. 集合

集合表示一个可变集合。它们可以由内置的 `set` 构造函数创建，如前所述。

#### 8.5.2. 集合推导式

除了显式的集合字面量语法外，还可以使用集合推导式语法创建集合。例如，

```
>>> type({1, 3, 5, 7})
<class 'set'>
>>> { x*2 for x in range(5) }
{0, 2, 4, 6, 8}
```

一个集合字面量，`{1, 3, 5, 7}`。此推导式表达式求值为 `{0, 2, 4, 8, 6}`。

集合推导式以及包含逗号分隔元素列表的字面量表达式，每次求值时都会返回一个*新的*可变集合对象。

#### 8.5.3. 冻结集合

内置的 `frozenset` 构造函数创建一个 `frozenset` 对象，这是一个不可变集合。

### 8.6. 映射

映射类型表示由任意索引集（或键）索引的对象的有序或无序集合。内置函数 `len` 返回给定映射对象中的项数。下标表示法 `a[k]` 从映射 `a` 中选取由 `k` 索引的项。这可以用作赋值或 `del` 语句的目标。

#### 8.6.1. 字典

字典是 Python 中内置的可变映射类型。它们表示由支持常量哈希值的任何类型的值索引的对象的有限集合。字典保留插入顺序。也就是说，键将按照它们被依次添加到字典中的相同顺序产生。

#### 8.6.2. 字典推导式

除了通过列出元素的字典字面量语法外，还可以使用[字典推导式语法](https://docs.python.org/3/reference/expressions.html#dict)创建新的 `dict` 对象。例如，

```
>>> { str(x): x**2 for x in range(4) }
{'0': 0, '1': 1, '2': 4, '3': 9}
```

请注意 `for` 子句之前的表达式语法 `(k:v)`。

#### 8.6.3. 元素插入/删除

```
>>> d = {"a": 1}
>>> d['b'] = 2
>>> del(d['a'])
>>> d
{'b': 2}
```

如果存在具有给定键的元素，则覆盖现有元素。否则，插入一个新元素。

## 9. 表达式

“表达式”由运算符和操作数组成，它求值为一个值。一个值（平凡地）就是一个表达式。

在 Python 中，表达式的构建块被称为“原子”。原子包括[词法标记](https://docs.python.org/3/reference/lexical_analysis.html#tokens)，如标识符和字面量，并且它们可以嵌套。用括号、方括号或花括号括起来的表达式在语法上也是原子。

在下一级，以下“初等表达式”表示绑定最紧密的操作：

*属性引用*

一个初等表达式后跟一个句点和一个名称，称为属性引用，也是一个初等表达式。对象的属性引用可以通过重写其 `__getattr__` 方法来自定义。

*下标*

序列（例如，字符串、元组或列表）或映射对象（例如，字典）的下标或索引从给定集合中选取一个项。用户定义的对象可以通过实现 `__getitem__` 方法来支持下标。

### 切片

切片操作选取序列对象中的一系列项。切片可以用作表达式，也可以用作[赋值中的目标](https://docs.python.org/3/reference/simple_stmts.html#assignment-statements)或 [del 语句](https://docs.python.org/3/reference/simple_stmts.html#the-del-statement)中的目标。初等表达式使用从切片列表构造的键进行索引（使用与下标相同的 `__getitem__` 方法），如下所示。

- 如果切片列表包含至少一个逗号，则键是一个包含切片项转换的元组。
- 否则，单个切片项的转换就是键。

### 调用

“调用”使用一个（可能为空的）参数序列调用一个[可调用对象](https://docs.python.org/3/glossary.html#term-callable)（例如，函数或类）。所有具有 `__call__` 方法的对象都是可调用的，例如内置函数、内置对象的方法、类对象、类实例的方法和用户定义的函数。所有参数表达式在尝试调用之前都会被求值。

Python 中有几种不同类型的表达式。一个表达式可以包含一个或多个其他（子）表达式，例如，与运算符组合。

- 算术转换。
- 算术运算，一元和二元。

### 9.1. 表达式列表

表达式列表是一系列由一个或多个表达式组成，以逗号分隔的序列。尾随逗号是可选的，但由单个表达式组成的表达式列表除外。表达式列表中的表达式从左到右依次求值。

一个或多个表达式的表达式列表会生成一个具有相应元素数量的元组对象。例如，

```
>>> 50 + 50, 100 * 2
(100, 200)
```

Python 解释器首先计算 `50 + 50`，其值为 100，然后计算 `100 * 2`，其值为 200。该表达式列表的值是一个元组，例如，在此例中表示为元组字面量 `(100, 200)`。

```
>>> 1 / 2
0.5
>>> 1 / 2,
(0.5,)
```

表达式 `1 / 2` 的值是 `0.5`。表达式列表 `1 / 2,` 的值是 `(0.5,)`。对于单表达式的表达式列表，尾随逗号是必需的。

“可迭代对象解包”语法，使用星号 `*` 运算符，可用于表达式列表中可迭代类型表达式的元素。

```
>>> a, b, c = 1, [2, 3, 4], 5
>>> a, b, c
(1, [2, 3, 4], 5)
>>> a, *b, c
(1, 2, 3, 4, 5)
>>> *b,
(2, 3, 4)
```

`b` 是一个列表，一种可迭代类型。解包 `b`。`b` 的元素现在成为结果元组的一部分。在由单个可迭代表达式组成的表达式列表中进行解包。请注意，尾随逗号是必需的。

### 9.2. 求值顺序

一般而言，Python 根据运算符优先级规则从左到右对表达式求值。

在对赋值语句求值时，右侧表达式先于左侧表达式求值。然而，在“增强赋值”表达式中，左侧目标先于右侧表达式求值。然后，最终结果被赋值回左侧目标。

```
>>> a, b = 1 + 2, 2 * 5
>>> a, b
(3, 10)
>>> a += 1 + 2 * 3
>>> a
10
```

表达式 `1 + 2` 和 `2 * 5` 首先被求值（例如，从左到右），生成的值（一个元组）被“解包”并分别赋值给 `a` 和 `b`。在这个增强赋值中，`a` 的（当前）值首先被求值，即 3。然后右侧表达式被求值为 7（因为乘法 `2 * 3` 的优先级高于加法 `+`）。接着计算 `3 + 7`，其结果被赋值回 `a`。

### 9.3. 赋值表达式

赋值表达式将赋值运算符 `:=`（非正式地称为“海象”运算符）右侧的表达式赋值给左侧的名称，并返回其值（与目标的值相同）。

赋值表达式通常用于复合语句中，当需要保留表达式求值的结果时，例如，以便在语句体中使用。例如，

```
>>> import random
>>> def zero_or_not():
...     return random.randint(0, 4)
...
>>> while r := zero_or_not():
...     print("Not zero:", r)
...
Not zero: 1
Not zero: 1
Not zero: 4
```

`while` 子句依赖于函数调用 `zero_or_not()` 的结果。在每次迭代中使用的是*相同的值*。因此，该值之前已存储在变量 `r` 中。

### 9.4. 条件表达式

除了条件语句外，Python 还通过 `if - else` 表达式语法支持条件表达式。在表达式 `x if C else y` 中，包含三个（子）表达式 `C`、`x` 和 `y`，如果 `C` 求值为 `True`，则表达式的值为 `x`。否则，其值为 `y`。

例如，

```
family = input("What is your family name? ")
print(f"You are {'a reptile' if family == 'Python' else 'nobody'}!")
```

Python 的 `if` 表达式对应于其他 C 风格语言中的三元运算符 `? :`。表达式 `x if C else y` 大致等同于那些语言中的 `C ? x : y`。

请注意，无论 `C` 的值如何，这三个表达式中*只有两个*会被求值。条件 `C` 总是首先被求值，如果为真，则求值 `x`，但不求值 `y`。否则，求值 `y`，但不求值 `x`。

### 9.5. 算术类型转换

当执行二元算术运算时，

- 如果任一参数是复数，则另一个参数被转换为复数，且
- 否则
    - 如果任一参数是浮点数，则另一个参数被转换为浮点数，且
    - 否则，两个参数都是整数，Python 不进行自动转换。

### 9.6. 算术运算

#### 9.6.1. 一元算术运算符

一元 `-`（取负）运算符生成其数值参数的相反数。一元 `+`（取正）运算符生成其数值参数本身。在这两种情况下，如果参数没有正确的类型，将引发 `TypeError` 异常。

所有一元算术和位运算具有相同的优先级。

#### 9.6.2. 二元算术运算符

`*`（乘法）运算符生成其参数的乘积。参数必须都是数字，在这种情况下它们相乘；或者一个参数必须是整数，另一个必须是序列，在这种情况下执行序列重复。负的重复因子生成一个空序列。

`/`（除法）和 `//`（整除）运算符生成其参数的商。浮点数或整数的除法生成浮点数，而整数的整除结果为整数。除以零会引发 `ZeroDivisionError` 异常。

`%`（取模）运算符生成第一个参数除以第二个参数的余数。右参数为零会引发 `ZeroDivisionError` 异常。参数可以是浮点数，例如，`3.14 % 0.7` 等于 `0.34`（因为 `3.14` 等于 `4 * 0.7 + 0.34`）。取模运算符总是生成与第二个操作数符号相同的结果。

整除和取模运算符通过以下恒等式相关联：`x == (x // y) * y + (x % y)`。整除和取模也与内置函数 `divmod` 相关联：`divmod(x, y) == (x // y, x % y)`。

`+`（加法）运算符生成其参数的和。参数必须都是数字，在这种情况下它们相加；或者都是相同类型的序列，在这种情况下序列被连接。`-`（减法）运算符生成其参数的差。

#### 9.6.3. 幂运算符

幂运算符 (`**`) 接受两个参数，其工作方式与内置函数 `pow` 相同。也就是说，`x ** y` 等同于 `pow(x, y)`，生成 `x` 的 `y` 次幂的值。

例如，

```
>>> 2 ** 3
8
>>> 2 ** -3
0.125
>>> -2 ** 0.5
-1.4142135623730951
>>> (-2) ** 0.5
(8.659560562354934e-17+1.4142135623730951j)
>>> 10 ** 0
1
>>> 0.0 ** 2
0.0
>>> 0.0 ** -2
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
ZeroDivisionError: 0.0 cannot be raised to a negative power
```

`2 ** 3` 与 `2 * 2 * 2` 相同，结果是整数 `8`。`2 ** -3` 与 `1.0 / (2 * 2 * 2)` 相同，即使两个操作数都是整数，结果也是浮点数。`-2 ** 0.5` 与 `- (2 ** 0.5)` 相同，即 -1 乘以 2 的平方根。另一方面，负数的分数次幂返回一个复数。无论 `x` 是什么，`x ** 0` 都生成 `1`。同样，`x ** 0.0` 总是 `1.0`。对于整数和浮点数 `x`，`0 ** x` 分别生成 `0` 和 `0.0`。无论 `x` 的类型和值如何，`0.0 ** x` 总是返回 `0.0`。零（`0` 或 `0.0`）的负数次幂会引发 `ZeroDivisionError` 异常。

### 9.7. 位运算

#### 9.7.1. 一元位运算符

一元 `~`（按位取反）运算符仅适用于整数。它生成其整数参数的按位反转。`x` 的按位反转定义为 `-(x+1)`。如果参数没有正确的类型，将引发 `TypeError` 异常。

#### 9.7.2. 二元位运算符

- 按位与 (`&`) 运算符生成其整数参数的按位与。
- 按位异或 (`^`) 运算符生成其整数参数的按位异或。
- 按位或 (`|`) 运算符生成其整数参数的按位或（包含或）。

#### 9.7.3. 移位运算符

左移（`<<`）和右移（`>>`）运算符都接受整数作为参数。它们将第一个参数向左（`<<`）或向右（`>>`）移动第二个参数所指定的位数。

-   右移 n 位定义为除以 `pow(2,n)` 的向下取整。
-   左移 n 位定义为乘以 `pow(2,n)`。

### 9.8. 布尔运算

求值结果为布尔值（`True` 或 `False`）的表达式称为布尔表达式。因此，`True` 是一个（平凡的）布尔表达式，`False` 也是。

#### 9.8.1. `not` 运算符

`not` 运算符产生：

-   如果其参数为假，则结果为 `True`，否则
-   结果为 `False`。

#### 9.8.2. `and` 运算符

`and` 运算 `x` and `y` 首先求值 `x`，然后：

-   如果 `x` 求值为假，则返回其值（该值可能不是布尔类型）。
-   否则，求值 `y`，并返回其结果值。

请注意，整个表达式的类型实际上是 `x` 和 `y` 类型的并集（不一定是布尔类型）。也就是说，如果 `x` 的布尔值为假，则 `and` 表达式的类型就是 `x` 的类型。否则，就是 `y` 的类型。

#### 9.8.3. `or` 运算符

`or` 运算 `x` or `y` 首先求值 `x`，然后：

-   如果 `x` 求值为真，则返回其值（同样，该值可能不是 `True`）。
-   否则，求值 `y`，并返回其结果值。

整个表达式 `x` or `y` 的类型同样是 `x` 和 `y` 类型的并集。也就是说，如果 `x` 的布尔值为真，则 `or` 表达式的类型就是 `x` 的类型。否则，就是 `y` 的类型。

### 9.9. 比较

比较产生布尔值：`True` 或 `False`。Python 中的比较可以任意链式使用。例如，`x < y <= z` 是一个有效的表达式，这与许多其他 C 风格语言不同，它大致等价于 `x < y and y <= z`。或者更精确地说，它等价于 `x < (t := y) and t <= z`，其中使用了一个临时变量 `t`。请注意，`y` 只被求值一次。

通常，如果 `a`, `b`, `c`, ..., `y`, `z` 是表达式，而 `op1`, `op2`, ..., `opN` 是二元比较运算符，那么 `a op1 b op2 c ... y opN z` 在语义上等价于 `a op1 b and b op2 c and ... y opN z`，但每个表达式最多只被求值一次。

#### 9.9.1. 同一性比较

运算符 `is` 和 `is not` 测试对象的同一性：`x is y` 为真当且仅当 `x` 和 `y` 是同一个对象。对象的同一性通过 `id()` 函数确定。`x is not y` 产生相反的真值。

#### 9.9.2. 值比较

运算符 `<`、`>`、`==`、`>=`、`<=` 和 `!=` 比较两个对象的值。对象不需要具有相同的类型。

因为所有类型都是（直接或间接的）`object` 的子类型，所以它们从 `object` 继承了默认的比较行为。类型可以通过实现比较魔术方法（如 `__lt__` 和 `__gt__` 等）来自定义其比较行为。

相等性比较（`==` 和 `!=`）的默认行为基于对象的同一性。因此，具有相同同一性的实例进行相等性比较会得到相等，而具有不同同一性的实例进行相等性比较会得到不相等，这在许多情况下可能违反直觉。通常，用户定义的类型需要自定义其比较行为。

#### 9.9.3. 成员测试操作

对于集合，运算符 `in` 和 `not in` 测试成员资格。`x in s` 求值为 `True` 如果 `x` 是 `s` 的成员，否则为 `False`。`x not in s` 返回 `x in s` 的否定。所有内置序列和集合类型都支持此操作。对于字典，`in` 运算符测试字典是否具有给定的键。

对于字符串和字节类型，`x in y` 为 `True` 当且仅当 `x` 是 `y` 的子串。运算符 `not in` 被定义为具有与 `in` 相反的真值。例如：

```
>>> 2 in [1, 2, 3, 2]
True
>>> 'c' not in { 'a', 'b', 'd'}
True
>>> "k1" in { "k1": "v1", "k2": "v2" }
True
>>> "world" in "hello world"
True
```

对于定义了 `__contains__` 方法的用户自定义类，`x in y` 返回 `True` 如果 `y.__contains__(x)` 返回一个真值，否则返回 `False`。

例如：

```

    "Nada"
```

函数定义 `a()` 包含一个表达式语句，即一个字符串字面量。

调用函数 `a()` 是一个表达式，但这里它被写成了一个语句。函数的返回值 `"Something"` 被丢弃了。

### 10.2. 赋值语句

赋值是 Python 中的简单语句之一。赋值语句主要用于将一个新名称绑定到一个对象，或将一个现有名称重新绑定到一个对象。它们也用于修改可变对象的属性或元素。

赋值语句的左侧可以是：

1.  一个新名称或一个引用对象的名称，或者
2.  一个可变对象的属性或元素。

赋值语句从左到右对右侧的表达式或表达式列表求值，并将得到的单个对象从左到右分配给左侧对应的目标列表。如果目标列表是一个没有尾随逗号的单个目标，则将该对象分配给该目标。（如前所述，将表达式列表赋值给目标列表是元组打包和序列解包的组合。）

在 Python 中，我们也可以在一条语句中将多个名称绑定到同一个对象。例如：

```
>>> x = y = "dragon"
>>> p = q = ['a', 'b']
>>> id(x), id(y)
(140593323269680, 140593323269680)
>>> id(p), id(q)
(140593324864960, 140593324864960)
```

变量 `x` 和 `y` 引用同一个对象。

`p` 和 `q` 引用同一个对象。

#### 10.2.1. 增强赋值语句

增强赋值语句是二元运算和赋值语句的组合。增强赋值中使用以下运算符：`+=`、`-=`、`*=`、`@=`、`/=`、`//=`、`%=`、`**=`、`>>=`、`<<=`、`&=`、`^=` 和 `|=`。

与普通赋值不同，增强赋值首先对左侧的目标求值，然后对右侧的表达式列表求值。然后执行二元运算，并将结果写回目标。

> Python 的赋值语句不会复制源对象，这与某些其他编程语言不同。它只是将名称绑定或重新绑定到对象。要进行复制，标准库的 `copy` 模块提供了浅拷贝和深拷贝的函数。

### 10.4. return 语句

从语法上讲，`return` 语句只允许出现在函数定义内部。当执行时，它会退出当前的函数调用。`return` 语句可以包含一个可选的参数，即一个表达式列表。如果提供了该表达式列表，它将作为函数调用的值返回给调用者。如果未提供，则假定返回值为 `None`。

### 10.5. raise 语句

`raise` 语句会引发或抛出一个异常或异常组：

- 如果提供了参数，
  - 如果该参数求值为 `BaseException` 或其子类的实例，则引发该异常/异常组对象。
  - 如果该表达式是一个类，则会创建一个不带参数构造的该类的实例。
- 如果未提供参数，
  - 如果当前作用域中存在一个活跃的异常，则重新引发此异常。
  - 否则，它是一个 `RuntimeError`。

当在另一个异常当前活跃时引发新异常时，可以使用 `__cause__` 属性将新异常与现有异常关联起来。这被称为“异常链”。这可以通过在 `raise` 语句末尾附加 `from` 子句来实现。

```python
>>> class UnusualException(Exception): pass
...
>>> raise UnusualException
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
__main__.UnusualException
```

- 新的异常类型应继承自 `Exception`（或 `BaseException`）。
- 继承自 `BaseException` 的异常可以被*引发*。
- Python REPL 会捕获该异常。通常，我们需要使用 `try - except` 语句来捕获引发的异常。否则，程序将会崩溃。

```python
>>> try: raise UnusualException
... except: raise
...
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
__main__.UnusualException
```

- 重新引发当前异常。

```python
>>> try:
...     raise UnusualException("Unusual")
... except BaseException as ex:
...     raise UnusualException("Really unusual") from ex
...
Traceback (most recent call last):
  File "<stdin>", line 2, in <module>
__main__.UnusualException: Unusual
```

```
The above exception was the direct cause of the following exception:

Traceback (most recent call last):
  File "<stdin>", line 4, in <module>
__main__.UnusualException: Really unusual
```

- `as` 子句用于为异常命名。
- 使用 `raise - from` 语法进行异常链。

### 10.6. break 语句

`break` 语句可以在 `for` 或 `while` 循环中使用，但只能直接包含在其中，而不能作为循环内函数或类定义的一部分。它会终止最内层的封闭循环。如果循环有 `else` 子句，则会被跳过。

```python
>>> for i in range(3):          # 1
...     try:
...         print(i)
...         if i == 1:
...             break          # 2
...         print(i)
...     finally:
...         print("finally")
... else:
...     print("huh?")           # 3
...
0
0
finally
1                          # 4
finally                    # 5
```

- 一个使用 `range` 函数的 `for` 循环。
- 此 `for` 循环中的一个 `break` 语句。
- 当循环因 `break` 语句而终止时，`else` 子句不会被执行。
- `break` 语句在打印第一个 1 之后执行。第二个 `print` 语句被跳过。
- 在退出循环之前，`finally` 子句仍然会被执行。

### 10.7. continue 语句

与 `break` 类似，`continue` 语句也只能直接在 `for` 或 `while` 循环中使用。它会继续执行最内层封闭循环的下一次迭代，而不执行当前循环中剩余的语句。

```python
>>> for i in range(3):          # 1
...     try:
...         print(i)
...         if i == 1:
...             continue      # 2
...         print(i)
...     finally:
...         print("finally")
...     else:
...         print("huh?")
...

0
finally
1                          # 3
finally                    # 4
2                          # 5
2
finally
```

- 一个 `for` 复合语句。
- 此 `for` 循环中的一个 `continue` 语句。
- `continue` 语句在打印第一个 1 之后执行。第二次 `print` 函数调用被跳过。
- 在继续下一次迭代之前，`finally` 子句仍然会被执行。
- 继续下一次迭代。
- 当循环正常终止时，`else` 子句会被执行。

### 10.8. global 语句

`global` 和 `nonlocal` 语句之前在代码块和作用域的上下文中已简要讨论过。

`global` 语句，即关键字 `global` 后跟一个逗号分隔的名称列表，表示在当前代码块中声明的名称将被解释为全局变量（例如，当前模块作用域）。在 `global` 语句之前，不能在同一代码块中使用该 `global` 语句中列出的名称。

注意 Python 中名称绑定和使用之间的不对称性。在非全局代码块中，如果不使用 `global` 声明，就无法将值绑定到全局变量。另一方面，要访问/读取全局变量的值，则不需要 `global` 声明。

```python
>>> def f():
...     print(x)           # 1
...     global y           # 2
...     y = 4              # 3
...     print(y)
...
>>> x, y = 1, 2           # 4
>>> f()                   # 5
1
4
```

- 在函数作用域中访问（后定义的）全局变量 `x` 不需要 `global` 声明。注意变量 `x` 在函数 `f` 内部使用而无需事先声明。这被称为“自由变量”。
- 此 `global` 声明允许对全局变量 `y` 进行赋值。
- 在 `global` 语句之后，我们可以为全局变量赋值。现在，在这个例子中，`y` 被绑定到一个不同的对象 `4`。
- 全局变量 `x` 和 `y` 被声明。
- 函数调用 `f()` 使用了这些全局变量。

### 10.9. nonlocal 语句

`nonlocal` 语句的工作方式与 `global` 语句类似。在 `nonlocal` 语句中声明的名称引用的是最内层封闭作用域（不包括全局作用域）中*先前绑定*的变量。

之前在“作用域示例”部分给出了一些例子。这里是另一个例子：

```python
>>> def a():
...     x, y = 1, 2
...     def b():
...         print(x)
...         nonlocal y
...         y = 4
...         print(y)
...     b()
...
>>> a()
1
4
```

- `x` 和 `y` 是函数 `a` 的函数定义代码块的局部变量。
- 一个新的（嵌套的）代码块的开始。
- `x` 引用在局部作用域（对于 `b`）之外声明的变量 `x`。
- 为了能够为非局部变量 `y` 赋予新值，需要将该变量声明为 `nonlocal`。
- 变量 `y` 引用的是在 `a` 的作用域中定义但在 `b` 的作用域之外的那个变量。

### 10.10. del 语句

`del` 语句删除列出的目标名称。删除目标列表会从左到右递归地删除每个目标。如果目标名称未绑定，将引发 `NameError` 异常。

删除名称会从本地或全局命名空间中移除该名称的绑定，具体取决于该名称是否出现在同一代码块中的 `global` 语句中。

例如，

```python
>>> x = 1
>>> def f():
...     global x
...     del x
...
>>> print(x)
1
>>> f()
>>> print(x)
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
NameError: name 'x' is not defined
```

- 此 `del` 语句删除了全局变量 `x`。
- 调用 `f()` 将执行 `del` 语句。
- 尝试使用已删除/未绑定的名称将引发 `NameError` 异常。

### 10.11. assert 语句

`assert` 语句是一种在程序中插入调试断言的便捷方式。它有两种形式。

#### 10.11.1. 基本形式

```python
assert <expression>
```

这等价于：

```python
if __debug__:
    if not <expression>: raise AssertionError
```

请注意，内置变量（如 `__debug__`）的值在 Python 解释器启动时确定，并且不能被修改。变量 `__debug__` 在正常情况下为 `True`，在请求优化时（命令行选项 `-O`）为 `False`。当在编译时请求优化时，`assert` 语句不会生成任何代码。

#### 10.11.2. 扩展形式

```python
assert <expression1>, <expression2>
```

等价于

```python
if __debug__:
    if not <expression1>: raise AssertionError(<expression2>)
```

例如，

```python
>>> a = 1, 2, 3,
>>> assert a == (1, 2, 3)
>>> assert a == [1, 2, 3]
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
AssertionError
```

- 表达式列表求值结果为一个元组。
- 此 `assert` 语句成功，因此没有输出。
- 此断言失败，它抛出一个 `AssertionError` 异常。

## 11. 复合语句

复合语句包含其他语句，并以某种方式影响或控制这些语句的执行。复合语句可以跨越多个（逻辑）行。

复合语句由一个或多个“子句”组成。子句由一个“头部”和一个“代码块”组成。特定复合语句的所有子句头部都处于相同的缩进级别。每个子句头部以一个唯一标识的关键字（例如，`if`、`else`、`try`、`except` 等）开始，并以冒号（`:`）结束。代码块是由子句控制的一组一个或多个语句。代码块可以是：

- 与头部在同一行上、冒号之后的一个或多个用分号分隔的简单语句，或者
- 在后续行上缩进一级的一个或多个简单或复合语句。

Python 与许多命令式编程语言一样，使用常见的控制流语句，例如 [if 语句](https://docs.python.org/3/reference/compound_stmts.html#if)、[while 语句](https://docs.python.org/3/reference/compound_stmts.html#while) 和 [for 语句](https://docs.python.org/3/reference/compound_stmts.html#for)。它们在 Python 中都是复合语句。`for` 语句通常与内置的 `range` 和 `enumerate` 函数一起使用。

`try` 语句与简单的 `raise` 语句一起用于异常处理和/或提供清理代码。另一个复合语句 `with` 语句与上下文管理器密切相关，用于为一系列语句提供初始化和终结代码。

新的 `match` 语句（3.10 版本新增）将在模式匹配的更广泛上下文中稍后讨论。同样，另外两个重要的复合语句，函数 `def` 语句和类定义语句，将在各自的章节中讨论。最后，协程和其他相关语句将在本书的最后一章中单独解释。`async/await` 是 Python（相对较新的）用于高级异步编程的构造。

### 11.1. if-elif-else 语句

`if` 语句用于条件执行。它通过从上到下依次计算 `if`/`elif` 表达式，直到找到一个为 `True` 的表达式，从而最多选择一个代码块。然后执行相应的代码块（并且不执行或计算 `if` 复合语句的任何其他部分）。如果所有表达式都为 `False`，则执行 `else` 子句的代码块（如果存在）。

例如，

```python
from datetime import datetime

weekday = datetime.today().isoweekday()

if weekday == 7:
    println("Today is Sunday!")
elif weekday == 6:
    println("Today is Saturday!")
else:
    println("Just another day. :(")

println("Regardless, let's learn some programming!")
```

- `datetime` 的 `isoweekday` 方法返回整数值，1 代表星期一，2 代表星期二，7 代表星期日，依此类推。
- 此 `if` 语句包含三个子句：`if`（第 5-6 行）、`elif`（第 7-8 行）和 `else`（第 9-10 行）。`if` 表达式检查今天是否是星期日。如果是，它将打印出 *今天是星期日！*，然后控制权移动到 `if` 语句的末尾，例如第 11 行。
- 如果今天不是星期日，则接着测试是否是星期六（6）。如果此表达式求值为 `True`，则执行代码块中的语句（在此示例中为单个语句，第 8 行），并且 `if` 语句的执行终止。
- 在所有其他情况下，此 `if` 语句最终将打印 *只是普通的一天。:(*。执行 `if` 语句后（无论执行了哪个子句），程序执行将移动到下一条语句，在此示例中为第 12 行。

请注意，与许多 C 风格语言不同，Python 的 `if` 语句语法不需要在 `if` 和 `elif` 行中的布尔表达式周围加括号。括号仍可用于表达式分组目的。

### 11.2. while - else 语句

`while` 语句用于重复执行，类似于 [for 语句](https://docs.python.org/3/reference/compound_stmts.html#the-for-statement)。`while` 语句的执行取决于一个表达式。它在每次迭代开始时测试该表达式，并且只要表达式求值为 `True`，它就执行第一个代码块。如果表达式变为 `False`（可能是在第一次测试时），则执行 `else` 子句的代码块（如果存在），并且循环终止。

对于 `while` 和 `for` 语句，

- `while`/`for` 代码块中的 [break 语句](https://docs.python.org/3/reference/simple_stmts.html#break) 会终止循环，而不执行 `else` 子句的代码块。
- `while`/`for` 代码块中的 [continue 语句](https://docs.python.org/3/reference/simple_stmts.html#continue) 会跳过复合语句的其余部分，并返回到测试表达式。

Python 3.0 于 2008 年首次发布，现在（自 2020 年起？）采用年度发布计划。让我们打印出该信息：

```python
>>> year = 2007
>>> while (year := year + 1) and (year < 3000):
...     if year < 2018:
...         if year == 2008:
...             print(f"Python 3 was first released in {year}.")
...         continue
...     elif year == 2022:
...         print("Python 3.11 is to be released at the end of 2022.")
...         break
...
...     version = f"3.{year - 2011}"
...     print(f"Python {version} in {year}.")
...
Python 3 was first released in 2008.
Python 3.7 in 2018.
Python 3.8 in 2019.
Python 3.9 in 2020.
Python 3.10 in 2021.
Python 3.11 is to be released at the end of 2022.
```

- 此 `while` 语句实际上是一个无限循环，其终止由 `break` 语句控制。条件 `year < 3000` 是作为预防措施添加的，例如，以避免在代码中存在错误时发生失控迭代等。此处使用了 [赋值表达式](https://docs.python.org/3/reference/expressions.html#assignment-expressions) `year := year + 1`（在布尔上下文中始终产生 `True`，因为 `year > 2007 > 0`），仅用于说明。
- 当 `year < 2018` 时，除非 `year == 2008`，否则我们只是 `continue`。
- `continue` 语句。
- 当 `year == 2022` 时，我们终止循环。
- 此 `break` 语句在此示例中终止了 `while` 循环。
- 请注意，此消息仅在 `2018 <= year < 2022` 时打印。

### 11.3. for - in - else 语句

`for-in-else` 语句用于遍历序列（如字符串、元组或列表）或[其他可迭代对象](https://docs.python.org/3/glossary.html#term-iterable)（包括字典）的元素。它具有以下语法：

#### 11.3.1. range 函数

借助内置的 `range` 函数（该函数生成一个整数序列），Python 的 `for in` 语句可以像 C 语言经典的 `for` 循环一样使用。

`range` 函数可以通过三种不同的方式调用：

```
range(start, end, step)
range(start, end)
range(end)
```

- 它指定了起始值（包含）、结束值（不包含）和步长，其中步长表示序列的增量或减量。所有参数都是整数，且步长不能为零。
- 如果省略步长，其默认值为 1。
- 在这种情况下，`start`/`step` 分别使用默认值 0/1。

例如，让我们尝试使用 `for` 语句计算从 0 到 100 的偶数之和：

```
>>> _sum = 0
>>> for i in range(0, 100+1, 2):
...     _sum += i
...
>>> print(f"Sum of even numbers from 0 to 100 is {_sum}")
Sum of even numbers from 0 to 100 is 2550
```

实际上，有一个内置的 `sum` 函数可以对给定序列中的所有元素求和。

```
>>> sum(range(0, 101, 2))
2550
```

#### 11.3.2. enumerate 函数

上述两种用例，例如，遍历任意序列中的元素和遍历整数序列，可以或多或少地与内置的 `enumerate` 函数结合使用。

`enumerate` 函数接受一个可迭代对象作为参数，并返回一个包含原始序列索引对的可迭代对象。也就是说，给定 `[item1, item2, item3]`，它返回一个包含三个元素的可迭代对象：`(0, item1)`、`(1, item2)` 和 `(2, item3)`。`enumerate` 函数不如 `range` 灵活，但它接受一个可选的第二个参数 `start`。否则，索引默认从 0 开始。

这里有一个例子：

```
>>> name = "python"
>>> for i, v in enumerate(name, 3):
...     print(i, v, end = ', ')
... else:
...     print()
...
3 p, 4 y, 5 t, 6 h, 7 o, 8 n,
>>>
```

字符串是一种不可变的序列类型，它是一个可迭代对象。

我们使用解包语法，将 `enumerate()` 函数调用的每次迭代中的索引和值分配给两个独立的变量 `(i, v)`。

`else` 子句总是在循环结束时执行，除非循环被异常终止，例如被 `break` 语句等终止。在这个例子中，我们在循环结束时添加了一个换行符。

否则，这一行就不会以换行符结束。

### 11.4. try 语句

从 Python 3.11+ 开始，Python 的异常框架得到了扩展，允许程序同时引发和处理多个不相关的异常。首先，引入了一种新的标准异常类型 `ExceptionGroup`，它表示一组不相关的异常。其次，添加了一种新的语法 `except*` 用于处理 `ExceptionGroup`。

经典的 `try - except` 复合语句具有以下通用语法：

```
try:
    <suite>
except <except_expression>:
    <suite>
except <except_expression> as <name>:
    <suite>
except:
    <suite>
else:
    <suite>
finally:
    <suite>
```

一个求值结果为 `Exception` 或 `ExceptionGroup` 类型的表达式。`ExceptionGroup` 类型在 Python 3.11 中引入。

一个 `try` 语句可以有零个、一个或多个 `except` 子句。

“捕获所有”子句。

一个可选的 `else` 子句。

一个可选的 `finally` 子句。请注意，当没有 `except` 子句时，`finally` 是必需的。

从 Python 3.11+ 开始，`try` 子句后面可以跟一个或多个 `except*` 子句，而不是 `except`。

```
try:
    <suite>
except* <exception_group>:
    <suite>
except* <exception_group> as <name>:
    <suite>
except* (<ex1>, <ex2>) as <name>:
    <suite>
else:
    <suite>
finally:
    <suite>
```

一个求值结果为 `Exception` 或 `ExceptionGroup` 的表达式。

- 一个 `try` 语句可以有一个或多个 `except*` 子句。请注意，与 `except` 不同，`except*` 不能用作“捕获所有”子句。也就是说，异常/异常组表达式始终是必需的。
- 一种从现有异常类型快速创建“异常组”的简写方式。

`except` 子句为特定的 `BaseException` 或 `BaseExceptionGroup` 类型指定异常处理程序。另一方面，`except*` 子句为 `BaseExceptionGroup` 类型中包含的一组 `BaseException` 类型指定异常处理程序。`except`/`except*`、`else`、`finally` 都是可选的，但至少需要 `except`/`except*` 或 `finally` 中的一个。请注意，`except` 和 `except*` 不能在同一个 `try` 语句中混合使用。

当 `try` 子句中没有发生异常时，不会执行任何异常处理程序。

当 `try` 套件中发生异常时，将开始搜索异常处理程序。对于 `except` 子句的情况，此搜索会按从上到下的顺序检查 `except` 子句并计算表达式，直到找到与异常匹配的子句。如果存在无表达式的 `except` 子句（它匹配任何异常），则必须放在最后。

对于 `except*` 子句的情况，如果异常或异常组中的任何异常与给定的异常相同或是其基类型，则该子句与异常匹配。与 `except` 子句的情况不同，多个 `except*` 子句可以匹配异常/异常组。因此，`except*` 子句中不允许使用 `break` 等控制流语句。

如果没有 `except` 子句匹配异常，或者 `except*` 子句没有完全处理异常组中的所有异常，则将继续在周围代码和调用栈中搜索异常或剩余未处理异常的处理程序。

当找到匹配的 `except` 子句时，如果存在 `as` 关键字，则异常将被分配给该 `except` 子句中 `as` 关键字后指定的目标，然后执行该 `except` 子句的套件。所有 `except` 子句都必须有一个可执行块。当到达此块的末尾时，执行将在整个 `try` 语句之后正常继续。（这意味着，如果存在两个针对同一异常的嵌套处理程序，并且异常发生在内部处理程序的 `try` 子句中，则外部处理程序将不会处理该异常。）

如果控制流离开 `try` 套件，没有引发异常，并且没有执行 `return`、`continue` 或 `break` 语句，则执行可选的 `else` 子句。`else` 子句中的异常不会被前面的 `except` 子句处理。

如果存在 `finally`，则它指定一个“清理”处理程序。执行 `try` 子句，包括任何 `except` 和 `else` 子句。如果在任何子句中发生异常且未被处理，则该异常将被暂时保存。执行 `finally` 子句。如果存在已保存的异常，它将在`finally`子句结束时被重新引发。如果`finally`子句引发了另一个异常，已保存的异常将被设置为新异常的上下文。如果`finally`子句执行了`return`、`break`或`continue`语句，已保存的异常将被丢弃。

在`finally`子句执行期间，异常信息对程序不可用。

当在`try...finally`语句的`try`套件中执行`return`、`break`或`continue`语句时，`finally`子句也会在“退出时”执行。

函数的返回值由最后执行的`return`语句决定。由于`finally`子句总是会执行，因此在`finally`子句中执行的`return`语句将始终是最后执行的那个。

异常可以被“重新引发”。例如，

```python
>>> try:
...     sum = 100 + unknown_name
... except:
...     raise
...
Traceback (most recent call last):
  File "<stdin>", line 2, in <module>
NameError: name 'unknown_name' is not defined
>>>
```

Python REPL 处理了错误，它没有崩溃。它等待下一个用户命令。

在存在活动异常的上下文中，`raise`语句会重新引发当前的异常/错误。此示例代码或多或少等同于以下显式引发当前异常的代码。

```python
>>> try:
...     sum = 100 + unknown_name
... except NameError as ex:
...     raise ex
...
Traceback (most recent call last):
  File "<stdin>", line 4, in <module>
  File "<stdin>", line 2, in <module>
NameError: name 'unknown_name' is not defined
```

### 11.5. with 语句

`with`复合语句用于将一组语句的执行包装在由上下文管理器定义的方法中。例如，它可以用于封装常见的`try...except...finally`使用模式。它具有以下语法：

```
with Expression as Target:
    Suite
```

`as`子句是可选的。

`Expression`必须求值为一个上下文管理器类型，该类型支持`__enter__`和`__exit__`方法。`with`语句的执行过程如下：

- 对上下文`Expression`求值以获取上下文管理器。
- 首先加载上下文管理器的`__enter__`。
- 接着加载上下文管理器的`__exit__`。
- 调用`__enter__`方法。
- 如果指定了`Target`，则将`__enter__`的返回值赋给`Target`。
- 执行`Suite`中的语句。
- 最后，调用`__exit__`方法。

例如，

```python
>>> import pathlib
>>> path = pathlib.Path("hello.py")
>>> with (c := path.open("w")) as file:
...     file.write("print('Hello World!')")
...
21
>>> type(c)
<class '_io.TextIOWrapper'>
>>> dir(c)
['_CHUNK_SIZE', '__class__', '__del__', '__delattr__', '__dict__',
'__dir__', '__doc__', '__enter__', '__eq__', '__exit__', ...]
```

`path.open`函数返回一个上下文管理器。如果成功，其`__enter__`方法返回一个文件对象。*仅用于说明*，我们使用[海象运算符](https://docs.python.org/3/whatsnew/3.8.html#the-walrus-operator)将上下文管理器赋值给变量`c`。（上下文管理器通常不能被“重用”。）

`with`语句套件中包含的所有语句都会被执行，例如，直到遇到错误（如果有的话）。

在终止`with`语句之前，无论错误条件如何，都保证会调用上下文管理器的`__exit__`。通常，“清理代码”在`__exit__`方法中实现。在文件操作的情况下，如果文件已（成功）打开，则会调用`file.close()`，以及其他操作。

一个内部类型`_io.TextIOWrapper`是一个类管理器。

正如预期的那样，此对象包含`__enter__`和`__exit__`方法。

`with`头可以包含多个表达式-目标项。对于多个表达式-目标对，上下文管理器的处理方式就像多个`with`语句从左到右嵌套一样。

例如，

```python
>>> with open("a.txt") as f1, open("b.txt") as f2:
...     print(f1.read())
...     print(f2.read())
...
```

此语句等同于以下内容：

```python
>>> with open("a.txt") as f1:
...     with open("b.txt") as f2:
...         print(f1.read())
...         print(f2.read())
...
```

## 12. 模式匹配

Python 的 `match - case` 语句（首次在 3.10 中引入）现在可以在可能需要复杂的 `if - elif - else` 语句的地方使用。模式匹配本质上是[比较表达式](the comparison expressions)的泛化，后者产生布尔值。

### 12.1. match - case 语句

`match`语句具有以下通用语法：

```python
match SubjectExpression:
    CaseBlock
    ...
    CaseBlock
```

与其他复合语句不同，`match`语句不能写在一行物理行中。也就是说，冒号后总是需要换行。*SubjectExpression*可以是命名表达式，也可以是一个或多个星号命名表达式（逗号分隔）。当使用表达式列表时，它会在匹配过程开始前先被求值为一个元组。

至少需要一个*CaseBlock*。也就是说，例如，我们不能只在`match`语句的主体中使用像`pass`语句这样的占位符。每个*CaseBlock*具有以下形式：

```
case Pattern:
    Suite
```

或

```
case Pattern Guard:
    Suite
```

我们将在[下一节](the next section)中介绍 Python 当前支持的所有不同模式类别。*Guard*是可选的布尔条件。如果包含，只有当模式匹配且条件也满足时，case子句才会匹配。

在最简单的用法中，`match`语句可以或多或少地像 C 语言中的传统 `switch` 语句一样使用。例如，

```python
>>> color = Color.BLUE
>>> match color:
...     case Color.RED:
...         print("Color red detected")
...     case Color.BLUE:
...         print("Blue wins!")
...     case _:
...         print("We don't care about other colors :)")
...
Blue wins!
```

与 C 的 `switch - case` 语句不同，case子句不需要`break`语句。它们不会“贯穿”。

在此示例中，第一个case不匹配，因为`color != Color.RED`。然后`match`语句匹配下一个case，因为`color == Color.BLUE`，因此执行匹配case中的语句。

通配符表达式`_`用于“捕获所有”。也就是说，`case _`子句或多或少等同于 C 风格 `switch` 语句中的*default* case。如果没有匹配的case，则`match`语句在不执行任何套件的情况下终止。如果指定了捕获所有case `case _`，则执行此默认case中的语句。

### 12.2. 模式

#### 12.2.1. 通配符模式

如上一节末尾所示，特殊模式`_`匹配任何主题表达式。这是最简单的模式，因为它匹配任何表达式，所以除非使用守卫，否则它只能在`match`语句的最后一个case子句中使用。

```python
>>> match 5:
...     case _:
...         print("I always match!")
...
I always match!
```

#### 12.2.2. 字面量模式

匹配字面量模式实际上等同于主题表达式和字面量模式之间的比较操作。支持大多数常量字面量表达式，包括布尔值、整数和浮点数以及字符串。复数（不是常量字面量）也属于此类。

例如，

```python
>>> arr = (False, 30, 0.5 + 2j, """hello""", None)
... for val in arr:
...     match val:
...         case False:
...             print(f"{val} matched False")
...         case 30:
...             print(f"{val} matched 30")
...         case 0.5 + 2j:
...             print(f"{val} matched 0.5 + 2j")
...         case "hello":
...             print(f"{val} matched 'hello'")
...         case _:
...             print("Matched none")
...
False matched False
30 matched 30
(0.5+2j) matched 0.5 + 2j
hello matched 'hello'
Matched none
```

如果`val`是`False`，则此case匹配。

#### 12.2.3. 值模式

另一类模式是引用值的名称/变量，包括对象的属性。例如，

```
>>> class X: pass
...
>>> x = X()
>>> x.A, x.B = 5, 50
>>> match 1 + 4:
...     case x.A:
...         print("Matched x.A")
...     case x.B:
...         print("Matched x.B")
...
Matched x.A
```

- 对象 x 现在有两个（额外的）属性 A 和 B，其值分别为 5 和 50。
- 主体表达式求值为 5。
- 此 case 匹配成功，因为 x.A == 5。

#### 12.2.4. 分组模式

用括号括起来的模式也是一个模式，实际上是同一个模式。例如，

```
>>> match True:
...     case (True):
...         print("True is true no matter what")
...
True is true no matter what
```

#### 12.2.5. 捕获模式

捕获模式由一个名称组成。当匹配发生时，它将主体值绑定到指定的名称。例如，

```
>>> match 1 + 2:
...     case _sum:
...         print(f"sum is {_sum}.")
...
sum is 3.
```

- 捕获模式总是成功。名称 `_sum` 被绑定到主体表达式的值，即 3。

该名称不能是下划线（_），并且捕获模式总是匹配主体表达式。捕获模式可以用作其他模式的一部分，例如或模式或序列模式。

#### 12.2.6. 或模式

我们可以使用竖线 | 在一个 case 中包含备选模式。例如，

```
>>> today = "Saturday"
>>> match today.lower():
...     case "saturday" | "sunday":
...         print("It is a weekend!")
...     case _:
...         print("Just a weekday.")
...
It is a weekend!
```

- 如果主体表达式等于任一备选模式，则此 case 匹配成功。在此示例中，today.lower() == "saturday"，因此此 case 最终匹配。或模式中的模式从左到右进行测试。

#### 12.2.7. AS 模式

或模式的匹配表达式可以使用 as 关键字赋予一个名称。例如，

```
>>> import random
>>> match random.randint(0, 3):
...     case 0 | 2 as e:
...         print(f"Got {e}. We are even now.")
...     case 1 | 3 as o:
...         print(f"Got {o}. That is odd!")
...
Got 1. That is odd!
```

#### 12.2.8. 序列模式

序列模式匹配序列表达式的结构和值，例如元组、列表和字符串。

序列模式使用圆括号（()）或方括号（[]），它包含一个或多个子模式，用逗号分隔，每个子模式都与主体表达式的一个或多个元素进行匹配。当在圆括号中只使用一个子模式时，需要一个尾随逗号。否则，它被视为分组模式。

子模式可以属于任何一种模式类别。此外，最多可以使用一个“星号模式”，使用星号（*）后跟一个捕获模式或通配符模式。星号模式匹配“剩余元素”。如果没有星号子模式，则模式中的所有元素必须与主体序列中的相应元素匹配。（因此，它们的长度应该相同才能匹配。）

以下是使用模式匹配对内置 len 函数的递归实现：

```
>>> def seq_length(seq):
...     match seq:
...         case []:            # a
...             return 0
...         case [_, *y]:       # b
...             return 1 + seq_length(y)
...
```

- a 这是一个固定长度模式。它匹配一个包含零个元素的序列。
- b 这是一个可变长度模式，有两个子模式：第一个元素的通配符模式和其余部分的星号捕获模式。此 case 将匹配任何至少有一个元素的序列。在此示例中，我们忽略第一个元素的具体值，并使用其余部分（一个子序列）递归调用函数。请注意，单元素序列的长度为 1。

这是另一个示例。

```
>>> for p in [(0, 0), (0, 2), (3, 3), (2, 4)]:
...     match p:
...         case (0, 0):  # ①
...             print("I'm the origin!")
...         case (x, 0):  # ②
...             print(f"I'm on the x-axis, x = {x}")
...         case (0, y):  # ③
...             print(f"I'm on the y-axis, y = {y}")
...         case (x, y) if x == y or x == -y:  # ④
...             print(f"I'm on a diagonal, ({x}, {y})")
...         case (x, y):  # ⑤
...             print(f"I'm just a random point. ;(")
...
I'm the origin!
I'm on the y-axis, y = 2
I'm on a diagonal, (3, 3)
I'm just a random point. ;(
```

- ① 在此示例中，主体表达式是一个元组。
- ② 此模式匹配具有指定元素 0 和 0 的元组。请注意，模式不需要使用圆括号（仅因为在此示例中主体表达式是元组）。圆括号和方括号可以互换，除了单元素序列模式，正如我们前面提到的。
- ③ 此模式包含一个捕获子模式和一个字面量模式。它将匹配任何第二个元素为 0 的二元组，x 将被绑定到元组的第一个元素。
- ④ 类似地，此固定长度模式包含第一个元素的字面量模式和第二个元素的捕获子模式。它将匹配任何形式为 (0, y) 的二元组，y 将被绑定到元组的第二个元素。
- ⑤ 模式 (x, y) 将匹配任何二元序列，因为两个子模式都是捕获模式。此特定 case 包含一个守卫条件表达式。除了匹配模式外，守卫表达式必须求值为 True。否则，整个模式被视为不匹配。请注意，我们可以在守卫表达式中使用捕获的名称。

由于我们在此示例中未使用捕获的名称，此模式等效于所有有效二元序列的通配符模式。通配符模式 _，如果作为最后一个 case 包含，将捕获所有不是二元序列的表达式。

#### 12.2.9. 映射模式

映射类型也可以用作模式。它的工作方式与序列模式类似，不同之处在于主体表达式是一个映射（例如字典），子模式涉及映射元素的键和值。

例如，

```
>>> m = {"k1": 1, "k2": 2, "k3": 3}
>>> match m:
...     case {"k1": 1, "k2": 2}:
...         print("Exactly matched the 'k1' and 'k2' elements.")
...     case {"k1": v1, "k2": v2}:
...         print(f"For key 'k1', the value is {v1}, for 'k2', the value is {v2}.")
...     case {"k1": 1}:
...         print("Exactly matched the 'k1' element.")
...     case {"k1": v1}:
...         print(f"For key 'k1', the value is {v1}.")
...     case {}:
...         print("An empty mapping pattern matches all objects of a mapping type.")
...
Exactly matched the 'k1' and 'k2' elements.
```

在此示例中，每个 case 都“可匹配”给定的主体表达式（字典 m）。因此，第一个 case 最终匹配，match 语句终止。映射模式还支持“双星号子模式”，用于映射表达式中的“其余”元素。例如，{"k1": v1, **rest}。

序列和映射模式可以嵌套。

#### 12.2.10. 类模式

类模式是 Python 的 match 语句支持的另一类模式。类模式表示一个类及其位置参数和关键字参数。关键字参数使用等号（=）而不是冒号（:），冒号用于映射模式。

```
...
>>> a = A()

...         print("Exactly matched the attributes 'x' and 'y'.")
```

### 13.1. 函数定义

函数定义是一个复合语句，用于定义一个新函数。

例如，以下语句在执行时会创建一个名为 `add` 的函数对象，该函数接受两个参数 `p1` 和 `p2`，并返回一个值。

`add` 函数接受两个参数。注意，在 Python 中无法指定返回值。请参见下文。

一个文档字符串。注意，f-string 表达式不能用作文档字符串。

我们只能通过阅读函数实现来推断完整的函数签名。

文档字符串存储在函数对象的 `__doc__` 属性中。

使用 1 和 2 调用此函数。它返回 3。

### 13.2. 函数参数

函数可以定义零个、一个或多个参数。Python 中有三种函数参数。

*仅位置参数*

对于此类参数，当调用函数时，相应的实参必须以与参数列表中定义的完全相同的位置提供（例如，如 C/C++ 和其他一些 C 风格语言）。为了定义仅位置参数，我们使用分隔符 `/`（正斜杠）。在此可选分隔符之前的任何参数都是仅位置参数。

*仅关键字参数*

对于此类参数，它们在参数列表中没有固定位置，相应的实参需要使用 `parameter=value` 语法（称为“关键字实参”）提供。要定义 *仅关键字* 参数，我们使用分隔符 `*`（星号）。（或者，可变参数也用作此目的的分隔符。请参见下文。）此可选分隔符之后的参数（如果有）是仅关键字参数。

*位置或关键字参数*

默认情况下，对于所有其他参数，它们具有固定位置，可以使用位置实参语法（在相应位置）或关键字实参语法。

以下是一些示例：

```
def fa(p2): pass

def fb(p1, /, p2): pass

def fc(p2, *, p3): pass

def fd(p1, /, p2, *, p3): pass
```

`p2` 是一个“普通”函数参数。它既可以作为位置实参 (`fa(v2)`) 使用，也可以作为关键字实参 (`fa(p2 = v2)`) 使用。

在此示例中，`p1` 是一个仅位置参数。

对于函数 `fc`，`p3` 只能作为关键字实参使用。

`p1` 和 `p2`（但不是 `p3`）可以用作位置实参。`p2` 和 `p3`（但不是 `p1`）可以用作关键字实参。

### 13.3. 可选参数

函数参数可以具有默认值，形式为 `parameter = expression`。这些参数被称为可选参数。在函数调用中，如果没有为可选参数显式提供实参值，则使用其默认值。

-   任何仅关键字参数都可以设为可选。
-   只有在 `*`（如果存在）之前的最后一个或多个连续的位置参数（仅位置或其他）也可以设为可选。
-   默认参数值在函数定义执行时 *仅计算一次*，从左到右。然后在后续的任何函数调用中使用相同的对象。当默认值对象是可变对象时，这可能导致意外结果，因为 *默认值* 在不同的函数调用中可能实际上是不同的。

例如，

```
>>> def f(arr = []):
...     arr.append(1)
...     return list(arr)
...
>>> f(), f(), f()
([1], [1, 1], [1, 1, 1])
```

在此示例中，`arr` 的默认值设置为 `[]`，这是一个可变的列表对象。

表达式列表中的表达式从左到右求值。每次调用 `f()` 都会导致不同的结果。

每次我们调用同一个函数 `f`，依赖于可选参数 `arr` 的默认值时，它的行为都不同。一种解决方法是在函数内部设置 *真正的* 默认值。例如，

```
>>> def f(arr = None):
...     if arr == None:
...         arr = []
...     arr.append(1)
...     return list(arr)
...
>>> f(), f(), f()
([1], [1], [1])
```

这是惯用法。当需要在多次函数调用中为可变参数使用相同的默认值时（通常如此），我们将默认值设置为 `None`。

然后，在函数体中，如果实参值是 `None`，则我们使用真正的默认值（在此示例中为 `[]`）设置该参数。

现在每次调用 `f()` 都返回相同的结果。

### 13.7. map、filter 和 reduce

尽管我们不会在这个迷你参考中介绍所有内置函数，但让我们回顾一下内置的 map 和 filter 函数，以及 functools 模块中的 reduce 函数。它们通常被称为*高阶函数*（HOFs），因为它们接受函数作为参数，和/或返回函数。

#### 13.7.2. filter 函数

内置的 filter 函数同样接受一个函数和一个可迭代对象，并根据给定的函数返回一个过滤后的列表。例如，

```

...     lambda x: True if x%2==0 else False,
...     [1, 2, 3, 4]))

```

â` filter 与 map 类似。它是一个构造函数。

â`i 它将函数应用于给定可迭代对象中的每个元素，如果其布尔值为 True（例如，3、'hello' 等），则将其包含在结果中。否则（例如，None、[] 等），则将其过滤掉。

#### 13.7.3. functools.reduce 函数

functools 模块的 reduce 函数以类似的方式工作，但它将给定的函数*累积地*应用于给定可迭代对象中的所有元素。以下是一个示例：

### 13.8. 函数装饰器

装饰器是一个函数，或者更一般地说是一个可调用对象，它接受一个函数/可调用对象并返回相同类型的函数/可调用对象。在语法上，装饰器与目标函数定义一起使用，作为前缀，它转换目标函数并返回转换后的函数。换句话说，函数装饰器实际上成为目标函数定义的一部分。

以下是通用语法：

```
@decorator_function
def another_function(args):
    pass
```

这在语义上等同于以下内容：

```
def another_function(args):
    pass

another_function = decorator_function(another_function)
```

decorator_function 是一个函数：

```
def decorator_function(f1):
    # 将 f1 转换为 f2，
    # 或者基于 f1 创建 f2。
    return f2
```

使用装饰器的关键，除了语法上的便利性，是

#### 13.8.1. 内置装饰器

##### @property 装饰器

Python 中的“属性”是一种抽象构造，其行为类似于对象的数据属性，例如，具有 getter、setter、deleter 以及文档字符串等。

属性可以使用 `property` 构造函数创建。或者，也可以使用 `@property` 装饰器在类定义中轻松地定义新属性或修改现有属性。例如，

```python
>>> class X:
...     def __init__(self):
...         self._a = 10
...     @property
...     def a(self):
...         "I am the 'a' property."
...         return f"Magical {self._a}"
...     @a.setter
...     def a(self, value):
...         self._a = value
...     @a.deleter
...     def a(self):
...         print("Urghhh, I am being deleted...")
...         del self._a
```

这将创建一个名为 `a` 的属性。被装饰的方法用作属性 `a` 的“getter”。

此方法的文档字符串用作属性 `a` 的文档字符串。

这将被装饰的方法设置为 `a` 的 setter。也就是说，`a` 现在可以在[赋值语句](https://docs.python.org/3/reference/simple_stmts.html#assignment-statements)或[赋值表达式](https://docs.python.org/3/reference/expressions.html#assignment-expressions)的左侧使用。

当对属性 `a` 使用 [del 语句](https://docs.python.org/3/reference/simple_stmts.html#the-del-statement)时，将调用被装饰的方法。

让我们尝试使用一个 `X` 类型的对象：

```python
>>> x = X()
>>> help(X.a)                       # 
Help on property:

    I am the 'a' property.
(END)
>>> x.a                             # 
'Magical 10'
>>> x.a = 100                       # 
>>> x.a
'Magical 100'
>>> del x.a                         # 
Urghhh, I am being deleted...
>>> x.a                             # 
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
  File "<stdin>", line 7, in a
AttributeError: 'X' object has no attribute '_a'. Did you mean: 'a'?
```

`X` 有一个属性 `a`，其文档字符串在内置的 `help` 函数中使用。

此表达式调用被 `@property` 装饰的 `a(self)` 方法。

此赋值语句使用被 `@a.setter` 装饰的 `a(self, value)` 方法。

此 `del` 语句调用被 `@a.deleter` 装饰的 `a(self)` 方法。

此时，属性（或虚拟数据属性）`a` 不再存在。

##### @staticmethod 装饰器

Python 中类的“静态方法”类似于其他面向对象编程语言（如 C++、Java 和 C#）中的静态方法。它本质上是一个独立的函数，尽管它是在类定义中定义的。

从语法上讲，静态方法可以在类或实例上调用，但它们不接收隐式的第一个参数，例如 `self`。

`@staticmethod` 装饰器将类定义中的函数转换为静态方法。例如，

```python
>>> class Y:
...     @staticmethod
...     def f1():
...         print("I am a static method")
...
>>> Y.f1()
I am a static method
>>> y = Y()
>>> y.f1()
I am a static method
```

此装饰器使被装饰的方法成为静态方法。

从语法上讲，静态方法可以在类对象上调用。

或者，在实例对象上调用。如果没有 `@staticmethod` 装饰器，在类定义中定义的普通函数不能在实例对象上使用方法调用语法。

##### @classmethod 装饰器

Python 中类的“类方法”是在类对象上定义的方法，而不是在实例上定义的。普通方法将实例对象作为隐式的第一个参数接收。相反，类方法将类对象作为隐式的第一个参数接收。

从语法上讲，类方法可以在类或实例上调用，就像静态方法一样，但它们接收类作为其第一个参数（即使它们是在实例对象上调用的）。如果为派生类调用类方法，则派生类对象将作为隐式第一个参数传递。

`@classmethod` 装饰器将函数转换为类方法。例如，

```python
>>> class Z:
...     @classmethod
...     def f2(cls):
...         print(f"Class method: {cls.__name__}")
...
>>> Z.f2()                       # 
Class method: Z
>>> z = Z()
>>> z.f2()                       # 
Class method: Z
>>> class Z2(Z): pass
...
>>> Z2.f2()                      # 
Class method: Z2
```

`@classmethod` 装饰器使被装饰的方法成为类方法。

如果没有装饰，这将是一个类型错误。

默认情况下，在类中定义的方法是[实例方法](instance method)，它将实例对象作为隐式的第一个参数。对于类方法，隐式的第一个参数是类对象。

隐式的类对象参数现在是子类 `Z2`。

## 14. 类

### 14.1. 类定义

`class` 定义是一个复合语句，用于定义一个可调用的类对象。执行时，它会执行类套件中的语句，并在内存中创建一个 `type` 类型的对象，该对象由类定义指定。

一般语法如下：

```python
class MyType(BaseType1, BaseType2):
    Suite
```

关键字 `class` 开始语句，后跟新类名（例如，本例中的 `MyType`）以及括号中一个或多个基类的可选列表（例如，`BaseType1`、`BaseType2` 等）。Python 支持“[多重继承](https://docs.python.org/3/tutorial/classes.html#multiple-inheritance)”。

当未指定基类列表时，类从最终基类 `object` 继承。在这种情况下，类名后的括号可以省略。即，

```python
class Orange:
    pass
```

此定义等效于以下内容：

```python
class Orange(object):
    pass
```

- 类定义创建一个局部命名空间，嵌套在全局命名空间中，类套件中的语句在新创建的执行帧中执行。
- 然后创建一个类对象，该对象从基类列表中（显式或隐式）指定的基类继承，并使用新创建的局部命名空间作为类的 `__dict__` 属性。
- 然后，类定义语句中定义的类名在原始局部命名空间（可以是全局命名空间）中绑定到此 class 对象。

### 14.2. 类与实例

Python 的类只是一个对象（具有一些特殊功能）。但是，在 Python 编程中，它扮演着其他“基于类”的面向对象编程（OOP）语言中类的角色。至少，使用类可以帮助创建更模块化、更结构化的 Python 程序，其组件可以更容易地重用等。在许多问题领域中，从“类”和“对象”（在 OOP 的意义上）的角度思考可能相当自然和直观。

就像函数定义一样，类定义包含一系列语句。大多数类定义主要包含变量（例如，“类变量”）和函数（例如，“静态方法”、“类方法”和“实例方法”）的定义。

当 Python 解释器读取/执行类定义时，这些语句将被执行。例如，

```python
>>> class X(object):
...     if True:
...         print(True)
...     else:
...         print(False)
...
True
```

这是 `if` 语句的输出。与函数定义不同，Python 在创建类对象时会执行类定义作用域中的所有语句。

类定义在某种程度上类似于“模块”。这些类套件语句仅在类语句首次执行时执行（例如，当 Python 解释器读取该语句时）。当我们*使用*“类对象”（例如，创建该类的实例对象）时，这些语句不会被执行。例如，

```python
>>> x = X()
```

#### 14.2.1. 类对象

Python 的 `class` 语句会在内存中创建一个类对象（就像 `def` 语句创建一个函数对象一样）。类对象支持“[属性引用](https://docs.python.org/3/glossary.html#term-attribute-reference)”语法，例如通过点号表示法，就像任何其他 Python 对象一样。

```python
>>> class SoSimple:
...     one_name = "simple"
...
>>> type(SoSimple)
<class 'type'>
>>> SoSimple.one_name
'simple'
>>> SoSimple.one_number = 666
>>> SoSimple.one_number
666
>>> dir(SoSimple)
['__class__', ... '__weakref__', 'one_name', 'one_number']
```

- 类对象的类型是 `type`。
- 类定义中包含的属性。
- 我们可以向类对象添加任何额外的属性，就像对任何其他对象一样。
- 请注意，类对象自带许多预定义属性，所有这些属性都以双下划线（`__`）开头和结尾，也称为“dunder”。

这种类型的类属性对应于其他面向对象编程语言中的静态变量（或静态字段）和静态方法。

#### 14.2.2. 类变量

一个类对象（例如，由 `class` 语句定义的）扮演着多种角色。首先，正如我们之前所见，它是给定类/类型的实例对象的*构造器*。其次，它保存着该类所有实例共享的变量。事实上，正如我们刚才提到的，类变量是被所有实例对象*共享*的。

```python
>>> class Car:
...     brand = "GM"
...
>>> car1, car2 = Car(), Car()
>>> car1.brand, car2.brand
('GM', 'GM')
>>> Car.brand = "Ford"
>>> car1.brand, car2.brand
('Ford', 'Ford')
```

- `Car.brand` 是一个类变量。它属于类对象。
- 你可以从类的实例访问它。
- 但是，该变量指向同一个对象，即 `Car.brand`。

#### 14.2.3. 构造器

除了属性引用，类对象还支持“实例化操作”。Python 中的类对象是给定类/类型的对象的构造器。

使用之前的同一个例子，

```python
>>> s = SoSimple()
>>> type(s)
<class '__main__.SoSimple'>
>>> s.one_name
'simple'
>>> s.one_number
666
```

请注意，实例对象 `s` 包含了临时属性 `one_number`，以及原始类定义中的一部分 `one_name`。我们也可以向给定的实例对象添加任何额外的属性：

```python
>>> s.one_address = "Playa"
>>> s.one_address
'Playa'
>>> dir(s)
['__class__', ... '__weakref__', 'one_address', 'one_name', 'one_number']
```

请注意，实例 `s` 包含了原始 `SoSimple` 类中或多或少相同的预定义属性，以及后来添加到该类对象 `SoSimple` 的其他属性，还有特定于实例对象 `s` 本身的属性。我们也可以使用 `del` 语句删除在类或实例中定义的属性。

```python
>>> del s.one_number
>>> dir(s)
['__class__', ... '__weakref__', 'one_address', 'one_name']
```

执行 `del` 语句后，实例对象 `s` 不再拥有属性 `one_number`。

#### 14.2.4. `__init__` 函数

请注意，类对象不仅在创建该类的实例对象时用作“模板”，而且它们本质上共享相同的属性。

```python
>>> s = SoSimple()
>>> s.one_name
'simple'
>>> SoSimple.one_name = "not simple any more"
>>> SoSimple.one_name
'not simple any more'
>>> s.one_name
'not simple any more'
```

类的实例化（或，实例化一个类的实例对象）可以通过覆盖类的 `__init__` 方法来自定义。此方法在实例创建后会自动被调用，例如，由 Python 解释器调用。

```python
>>> class SoEasy:
...     def __init__(self):
...         self.one_title = "programmer"
...
>>> s = SoEasy()
>>> s.one_title
'programmer'
>>> dir(s)
['__class__', ... '__weakref__', 'one_title']
```

注意函数签名。`__init__` 方法至少有一个参数。第一个参数总是指刚刚创建的实例对象，它总是被命名为 `self`。

在此示例中，`SoEasy` 类型的一个实例对象有一个属性 `one_title` 自动附加到它上面。这是因为我们在 `SoEasy.__init__` 方法中创建了这个名称/属性并将其附加到 `self` 对象上。

#### 14.2.5. 实例对象

一个类的*实例对象*包含类中定义的所有属性，并且可以包含其他特定于实例的属性。属性引用可用于引用实例对象的这些属性。属性有两种类型：数据属性（或字段、变量）和方法。

如前所述，实例对象与 Python 中的其他所有事物一样，具有属性，即数据属性和方法属性。最初，它的大部分属性来自其类型，以及在创建时在 `__init__` 方法中添加的属性。但是，与其他类型的自定义对象（包括函数、类等）一样，可以向实例对象添加新的属性。

实例对象具有与类定义中的函数相对应的方法。所有以实例对象作为第一个参数（例如 `self`）的函数，根据定义都是“方法”，Python 允许使用对象方法调用语法来调用这些函数。例如，

```python
>>> class Ship:
...     def fly(self):
...         print("I cannot fly. Only spaceships can fly.")
...
>>> s = Ship()
>>> s.fly()
I cannot fly. Only spaceships can fly.
```

这里，我们在实例对象 `s` 上调用了方法 `fly()`。这等同于函数调用：

```python
>>> Ship.fly(s)
I cannot fly. Only spaceships can fly.
```

- 注意此调用中的函数参数。

事实上，`<instance>.f(...)` 只是更常规的函数调用语法 `<class_name>.f(self, ...)` 的“语法糖”。（注意参数列表的差异。）只要函数的第一个参数 `self` 是给定类型/类的对象，这就可以工作。

#### 14.2.6. 实例变量

虽然我们可以在 Python 中向实例对象添加任何属性，但通常是在 `__init__` 方法中完成的。然后，该类的所有实例对象都将拥有相同的（非共享的）属性集。

```python
>>> class Pet:
...     def __init__(self):
...         self.kind = "dog"
...
>>> pet1, pet2 = Pet(), Pet()
>>> pet1.kind, pet2.kind
('dog', 'dog')
```

在初始化器中，参数 `self` 指的是刚刚通过调用类对象创建的实例对象。以 `pet1` 为例，`self` 和 `self.kind` 分别指代 `pet1` 和 `pet1.kind`。同样，对于 `pet2` 实例，`self` 和 `self.kind` 分别指代 `pet2` 和 `pet2.kind`。

属性 `kind` 是一个实例变量，它属于特定的实例对象，并且在不同实例之间不共享。此外，类对象没有该属性：

```python
>>> Pet.kind
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
AttributeError: type object 'Pet' has no attribute 'kind'
```

请注意在类定义中定义实例变量的语法。实例变量是在 `self` 对象上定义的。因此，作为推论，在类定义中，你只能在实例方法内定义实例变量。

> 所有用户定义的类型都是可变的。所有用户定义类型的对象都是可变的。

#### 14.2.7. 实例方法

如前所述，类定义中包含的最常见语句通常是用于定义函数的 `def` 语句。在类中定义的函数是类的属性，也是该类任何实例对象的属性。并且，只要其第一个参数的类型是同一个类，就可以在对象上使用方法语法来使用它。

与类变量不同，类函数除了实例外，还有两种类型：

- 一种只是函数（除了点号名称语法），称为“静态方法”，以及
- 另一种是类对象的一部分，可以访问类变量。这称为“类方法”。类方法的第一个参数是类对象本身。

正如我们之前所见，内置装饰器 `@staticmethod` 和 `@classmethod` 可用于专门声明其中一种或另一种类型的类级方法。否则，按照惯例，类定义中的所有其他方法都应该是实例方法。

### 14.3. 面向对象编程

在 Python 中，类定义语句用于为该类的对象创建一个“模板”。类定义了一个自定义类型、如何创建该类型的实例对象以及如何访问该对象等。

在 Python 中，一个类总是隐式地“继承”自内置类型 `object`，无论是直接还是间接继承。这对于任何内置类型或自定义类型都是如此。一个类可以直接继承自另一个类型，而该类型本身又是 `object` 的子类型。Python 支持[从多个直接基类继承](inheritance from more than one direct base class)。

#### 14.3.1. 数据封装

在 Python 中，不存在真正的“私有属性”、数据或方法。实例对象中的数据隐藏是通过约定来支持的，就像 Python 中的许多其他特性一样。

以单下划线 `_` 为前缀的名称被视为“私有”。也就是说，按照约定，我们不会直接访问其他类对象或实例对象的成员，如果它们的名称以一个或多个下划线开头。这样的属性被认为是实现细节，它们不属于“公共 API”的一部分。

然而，Python *确实*对[名称隐藏](https://docs.python.org/3/reference/lexical_analysis.html#private-name-mangling)有一些基本的支持。当一个类中的变量或函数/方法的名称*以至少两个下划线开头，并以至多一个下划线结尾*时，Python 会修改该属性的名称。这被称为“名称修饰”，尽管它并不真正“修饰”名称（例如，不像 C++ 那样）。无论如何，即使在你自己的程序中，也应该避免使用修饰后的名称。（注意，“双下划线名称”不会被修饰或修改，因为它们以两个下划线结尾。）

#### 14.3.2. 魔术方法

`__init__` 方法是一个特殊方法，如前所述。这个特定的方法用于为新实例化的实例对象提供任何初始化代码。如果该方法在对象的类中被重写，Python 运行时会自动在新创建的实例对象上调用此方法。

基础类型 `object` 具有以下属性：

```
>>> dir(object)
['__class__', '__delattr__', '__dir__', '__doc__', '__eq__',
'__format__', '__ge__', '__getattribute__', '__gt__', '__hash__',
'__init__', '__init_subclass__', '__le__', '__lt__', '__ne__',
'__new__', '__reduce__', '__reduce_ex__', '__repr__', '__setattr__',
'__sizeof__', '__str__', '__subclasshook__']
```

由于每个类型都直接或间接/传递地继承自这个基类 `object`，因此这些属性始终对所有类型（内置或用户定义）可用。其中一些数据属性可能为空，其中一些方法属性可能具有空实现。此外，某些属性可能不会在 `dir()` 输出中显示，因为类/对象的 `__dir__` 方法可以被自定义。

`__doc__` 属性存储类型的文档字符串（如果有的话）。否则，它是一个空字符串。与[函数](functions)一样，如果类定义套件中的第一条语句是常量字符串字面量表达式，则它会自动用作类的文档字符串。

（Python 的内置 `help` 函数使用对象的文档字符串作为其自动生成的帮助消息的一部分。）

`__str__` 方法在对象用于字符串上下文时被调用。当类中没有实现 `__str__` 方法时，如果存在 `__repr__` 方法，则会回退到该方法。对于更底层的自定义，或显示调试信息等，可以重写 `__repr__` 方法。例如，

```python
>>> class A:
...     def __str__(self):
...         return "A from __str__()"
...
>>> class B:
...     def __repr__(self):
...         return "B from __repr__()"
...
>>> class C:
...     def __str__(self):
...         return "C from __str__()"
...     def __repr__(self):
...         return "C from __repr__()"
...
>>> a, b, c = A(), B(), C()
>>> print(f"""\n... a = {a}
... b = {b}
... c = {c}""")
a = A from __str__()
b = B from __repr__()
c = C from __str__()
```

- 由于 `a` 的类型 `A` 实现了 `__str__` 方法，因此在字符串上下文中调用它。
- 由于 `b` 的类型 `B` 实现了 `__repr__` 方法，但没有实现 `__str__` 方法，因此调用 `__repr__` 方法。
- 同样，`c` 的 `__str__` 方法在字符串上下文中被调用。

另一方面，

```python
>>> print([a, b, c])
[<__main__.A object at 0x7093ad81f0>, B from __repr__(), C from __repr__()]
```

- 列表和其他集合类型的内部实现使用其元素的 `__repr__` 方法进行字符串表示。由于类型 `A` 没有实现此方法，因此使用 `a` 的默认字符串表示（包括其类型和内存位置等）。

我们可以在许多情况下，并且应该重写这些双下划线方法来自定义自定义类的行为。例如，像 `__eq__`（用于 `==`）、`__ne__`（用于 `!=`）、`__ge__`（用于 `>=`）、`__gt__`（用于 `>`）、`__le__`（用于 `<=`）和 `__lt__`（用于 `<`）这样的方法经常被重写，以自定义用户定义类型的相等性和比较相关行为。

例如，

```python
>>> from functools import total_ordering
>>> import random
>>>
>>> @total_ordering
... class Pt(object):
...     def __init__(self, x, y):
...         self.x, self.y = x, y
...     def __eq__(self, other):
...         return (self.x == other.x) and (self.y == other.y)
...     def __lt__(self, other):
...         return (self.x + self.y) < (other.x + other.y)
...     def __repr__(self):
...         return f"({self.x}, {self.y})"
...
>>> points = [Pt(-2, 0), Pt(-1, 0), Pt(0, 1), Pt(0, 2)]
>>>
>>> random.shuffle(points)
>>> print("Shuffled:", points)
Shuffled: [(-2, 0), (0, 2), (0, 1), (-1, 0)]
>>>
>>> result = sorted(points)
>>> print("Sorted:  ", result)
Sorted:   [(-2, 0), (-1, 0), (0, 1), (0, 2)]
```

- 通过使用 `@total_ordering` 装饰器，我们只需要实现 `__eq__` 和 `__ge__`、`__gt__`、`__le__`、`__lt__` 中的一个。其余的相关双下划线方法由该装饰器自动生成。
- 这个类表示一个二维点。
- `__eq__` 方法的一个典型实现。它使用逐成员比较。
- 我们为二维空间中的点赋予一个（任意的）顺序。
- 我们为此类实现了 `__repr__` 方法，但没有实现 `__str__` 方法。
- `random.shuffle` 函数就地打乱序列。
- 当点列表用于字符串上下文时，例如作为 `print` 函数调用的参数，它会调用其元素类型 `Pt` 的 `__repr__` 方法。
- 内置的 `sorted` 函数依赖于序列中项目的顺序。

#### 14.3.4. 多重继承

当一个类有多个直接基类时，这些基类及其基类，以及它们的基类，等等，一直到 `object`，都会被“线性化”或排序，以便查找任何继承的属性。

在 Python 中，这被称为 MRO，即方法解析顺序，它通过一种名为“C3 线性化算法”的方法来完成。该算法本质上试图找到一个顺序，使其与每个相关类的基类从左到右的顺序保持一致。如果成功，该方法会在给定类的所有（直接或间接）基类（包括 `object`）中产生一个唯一的排序。这并非总是可以实现，在这种情况下，会引发编译错误。一个明显的例子是尝试继承两个基类顺序不兼容的类。

```python
class A: pass
class B: pass
class C(A, B): pass
class D(B, A): pass
class E(C, D): pass
```

在这个例子中，无法一致地排序 E 的所有基类（A、B、C、D 和 `object`），因为 C 要求 A → B 的顺序，而 D 要求 B → A 的顺序。此语句将抛出 `TypeError` 异常。

需要注意的是，如果没有 E，这些语句都是有效的，尽管 C 和 D *看起来* 存在冲突。基类的线性化是针对每个类分别进行的。根据 A 和 B 等类中实现了哪些方法，C 和 D 的行为可能大相径庭。但是，这仍然是一个有效的 Python 程序（没有 E 的情况下）。

下面是一个有效的多重继承示例。这个例子包含了通常被称为“菱形继承模式”的内容。

```python
class A:
    def w(self): print("A.w")
    def x(self): print("A.x")
    def y(self): print("A.y")
    def z(self): print("A.z")
class B(A):
    def x(self): print("B.x")
class C(B):
    def w(self): print("C.w")
class D(A):
    def x(self): print("D.x")
    def y(self): print("D.y")
class E(C, D):
    pass
```

在这个例子中，例如类 E 有两个（非 `object` 的）直接父类，C 和 D。

```
E --> C --> B --> A --> object
    --> D ---------> A -->
```

E 通过两条不同的“路径”间接继承自 A：E → C → B → A 和 E → D → A。因此，它是一个菱形继承模式。

在继续之前，让我们尝试运行以下程序。输出会是什么？

```python
e = E()
print("e.w()"); e.w(); print("==")
print("e.x()"); e.x(); print("==")
print("e.y()"); e.y(); print("==")
print("z.y()"); e.z(); print("==")
```

如果你尝试运行这个程序，你会意识到：

- e.w() 最终调用 C.w，
- e.x() 最终调用 B.x，
- e.y() 最终调用 D.y，以及
- e.z() 最终调用 A.z。

其中一个“最奇怪”的行为是 e.x() 调用了方法 B.x，而不是 D.x，尽管两者都定义了 x 方法，并且 D 是 E 的直接基类之一，如 E(C, D) 的类定义中所指定。这是因为 Python 的“多重继承”使用了上述所有基类（无论它们是否在类定义中明确指定）之间的线性化顺序。

事实上，Python 有一个内置方法 `type.mro` 来显示此信息。例如，

```python
>>> E.mro()
[<class '__main__.E'>, <class '__main__.C'>, <class '__main__.B'>,
<class '__main__.D'>, <class '__main__.A'>, <class 'object'>]
```

顺序是，

```
E --> C --> B --> D --> A --> object
```

因此，e.x() 最终调用 B.x 而不是 D.x（它在链中更靠后）是合理的。

#### 14.3.5. super()

在许多支持面向对象编程（OOP）的语言中，像 `super`、`base` 和 `parent` 这样的术语具有某些相关的含义。在 Python 中，`super` 方法大致指代“基类”，具有更具体的语义，尤其是在多重继承的上下文中。

例如，当在类中引用 `super().x`（无论是数据还是方法属性）时，Python 会从给定类本身开始，沿着其线性化/排序的基类层次结构（MRO）从左到右（或从下到上，取决于你如何看待继承树）查找该属性。一旦在遍历过程中找到该属性，就会使用该属性的实现，并忽略所有其他（例如，其上游基类中的）实现。

请注意，MRO 是针对每个类定义的。例如，使用上面的例子，B 的 MRO 不是 D 的 MRO 的一个部分片段。它们可能完全不同。

```python
>>> B.mro()
[<class '__main__.B'>, <class '__main__.A'>, <class 'object'>]
```

顺序是 B → A → `object`。

#### 14.3.6. 鸭子类型

Python 从根本上说是一种基于对象的编程语言，不同于其他基于类的 OOP 语言，如 C++、Java 或 C#。这些传统的 OOP 语言使用类和继承来支持运行时的*多态性*，以及其他特性。在编译时声明为一种类型的变量，可以在运行时被赋予不同的类型，并且其行为可能不同。

Python 不需要这种多态性。Python 是一种动态类型编程语言。尽管 Python 多年来从 OOP 语言中借鉴了许多特性，但类（及其支持特性，如继承）的主要目的是将其用作对象的模板或原型，即创建多个“结构等效”的对象。许多支持某种“类”的动态编程语言，如 JavaScript、Perl、Lua 等，都属于同一类别。

为了演示 Python 中的“伪多态性”，让我们创建几个新类型：

```python
class Frog:
    def __repr__(self):
        return "Frog"
    def jump(self):
        print("Big jump")
```

### 14.4. 数据类

一个类可以仅用于数据的组织和访问，而不用于定义行为。这类类通常被称为“记录”或“结构体类型”。Python 在标准库中提供了一个辅助模块 `dataclasses`，用于创建类似记录的类，称为数据类。

可以使用 `@dataclasses.dataclass` 装饰器来创建数据类，而无需手动实现许多必需的方法。例如，

```python
>>> from dataclasses import dataclass
>>>
>>> @dataclass
... class Point:
...     x: int = 0
...     y: int = 0
...
>>> p1 = Point(1, 2)
>>> p2, p3 = Point(x=1, y=2), Point(2)
>>> p1, p2, p3
(Point(x=1, y=2), Point(x=1, y=2), Point(x=2, y=0))
>>> p1 is p2, p1 == p2
(False, True)
>>> p1 is p3, p1 == p3
(False, False)
```

默认情况下，`@dataclass` 装饰器会自动为被装饰的类添加 `__init__`、`__repr__` 和 `__eq__` 等方法的实现。

- 你通过类型注解来指定数据属性或“字段”，例如本例中的 `x: int` 和 `y: int`。可以添加可选的默认值，例如 `= 0`。
- 包含指定字段的构造函数会自动创建。因为两个字段都包含了默认值，所以在这个例子中，两个构造函数参数也都是可选的。
- 自动生成的 `__repr__` 实现使用类及其字段的名称/值。
- 自动生成的 `__eq__` 方法确保了数据类的值相等语义。也就是说，两个具有相同值的不同对象被视为相等。
- 同样，两个具有不同值的对象不可能是同一个对象。

通常，`@dataclass` 装饰器接受以下参数（均为可选）：

```python
@dataclass(
    init=True,
    repr=True,
    eq=True,
    order=False,
    unsafe_hash=False,
    frozen=False,
    match_args=True,
    kw_only=False,
    slots=False
)
class DataClass:
    pass
```

- 如果 `init=False` 或者该方法已存在于被装饰的类中，则不会自动生成 `__init__` 方法。
- 如果 `repr=False` 或者该方法已存在，则不会自动生成 `__repr__` 方法。
- 如果 `eq=False` 或者该方法已实现，则不会自动生成 `__eq__` 方法。
- 如果 `order=True` 且这些方法都不存在，则会自动生成 `__lt__`、`__le__`、`__gt__` 和 `__ge__` 方法。
- 默认情况下，`@dataclass` 仅在“安全”时才会添加 `__hash__` 方法。（更多信息请查阅官方参考文档。）设置 `unsafe_hash=True` 将始终自动生成 `__hash__`，无论是否“安全”（只要它尚未实现）。
- 如果 `frozen=True` 且被装饰的类没有 `__setattr__` 和 `__delattr__` 方法，则该类将被设为“不可变”。尝试对不可变数据类的字段进行赋值将引发异常。
- 如果 `match_args=False`，或者被装饰的类中已包含 `__match_args__` 属性，则不会自动生成此属性。`__match_args__` 属性（一个元组类型）用于支持[用户定义类](https://docs.python.org/3/reference/datamodel.html#customizing-instance-and-subclass-checks)的[结构化模式匹配](https://docs.python.org/3/reference/compound_stmts.html#the-match-case-statement)。
- 如果 `kw_only=True`，则所有字段在自动生成的 `__init__` 方法中都将是[仅关键字](https://docs.python.org/3/glossary.html#keyword-only-parameter)参数。
- 如果 `slots=True`，则会生成 `__slots__` 属性，并且将返回一个新类而不是原始类。

### 14.5. 枚举

枚举是一组绑定到（相关的）常量值对象的名称集合。这些对象被称为枚举的成员，它们在给定的枚举中必须是唯一的。枚举的每个成员都有一个名称和一个值。枚举类型的成员可以像序列类型的对象一样进行迭代。枚举成员的值可以是任何类型，但通常使用不可变类型，如 `int` 或 `str`。

通过继承 `enum.Enum` 类型或其子类型（如 `enum.IntEnum`）来创建新的枚举类。以下是一个示例：

```python
>>> from enum import Enum
>>>
>>> class Color(Enum):
...     RED = "red"
...     GREEN = "green"
...     BLUE = "blue"
...
```

枚举成员（例如 `Color.RED`）的类型不是其值（例如 `"red"`）的类型。它的类型是同一个枚举类型，例如本例中的 `Color`。

```python
>>> isinstance(Color.RED, Color)
True
>>> issubclass(Color, Enum)
True
```

请注意，你不能将枚举类型用作其他类型的基类，包括另一个枚举类型。如果类型继承很重要，那么你需要使用普通类来创建新类型。

我们可以迭代枚举的 `__members__`，例如使用 [for - in 语句](https://docs.python.org/3/reference/compound_stmts.html#the-for-statement)。（也就是说，所有枚举成员的集合是一个可迭代对象。）例如，

```python
>>> for c in Color:
...     print(f"{c}:\t{c.value}")
...
Color.RED:     red
Color.GREEN:   green
Color.BLUE:    blue
```

我们还可以向 `Color` 枚举类添加一个 `__str__` 方法来自定义其字符串表示。例如，

```python
>>> class Color(Enum):
...     RED = "red"
...     GREEN = "green"
...     BLUE = "blue"
...     def __str__(self):
...         match self.value:
...             case "red":
...                 return "Red"
...             case "green":
...                 return "Green"
...             case "blue":
...                 return "Blue"
...             case _:
...                 raise ValueError
...
>>> for c in Color:
...     print(c)
...
Red
Green
Blue
```

`Frog` 和 `Flea` 这两种类型彼此几乎无关，除了它们都像 Python 中的每种类型一样继承自 `object`。特别是，`Frog` 类型有一个 `jump` 方法，而 `Flea` 没有。另一方面，`Bullfrog` 是 `Frog` 的子类，它重写了 `__repr__` 方法以及 `jump` 方法。

现在，假设我们有一个对象列表，我们需要在 `for` 循环中对每个对象调用 `jump` 方法。在传统的面向对象语言中，列表中的所有对象都需要是 `Frog` 类型或其子类。例如，通常的工作方式如下：

```python
>>> frog = Frog()
>>> bull = Bullfrog()
>>>
>>> for jumper in (frog, bull):
...     print(jumper, end=": ")
...     jumper.jump()
...
Frog: Big jump
Bull: Huge jump
```

在这个例子中，`jumper` 变量需要是 `Frog` 类型。（我们忽略了值与引用等问题。）即使我们给 `Flea` 添加相同的 `jump` 方法，在强类型的面向对象语言中，`Flea` 对象也不能在这个 `for` 循环中使用。另一方面，在 Python 中，类型并不重要。任何具有 `jump` 方法的对象都可以在这个 `for` 循环中工作，无论其具体类型如何。

例如，让我们尝试一下：

```python
>>> flea = Flea()
>>> flea.jump = lambda:print("Small jump")
>>>
>>> for jumper in (frog, flea):
...     print(jumper, end=": ")
...     jumper.jump()
...
Frog: Big jump
Flea: Small jump
```

`flea` 的类型是 `Flea`，然而我们可以在 `for` 循环中混合使用 `Flea` 和 `Frog`。`jumper` 的类型并不重要。重要的是迭代中的每个 `jumper` 都定义了 `jump` 方法。

如前所述，这类类型系统通常被称为“鸭子类型”（就像“如果它看起来像鸭子，游泳像鸭子，叫声像鸭子，那么它很可能就是鸭子”）。Python 对类型的支持，例如 `object.__class__` 属性、`issubclass` 和 `isinstance` 函数，以及类和类继承等，提供了（很多）便利，但它们并非必不可少。最终，Python 在运行时使用鸭子类型。

```python
class Bullfrog(Frog):
    def __repr__(self):
        return "Bull"
    def jump(self):
        print("Huge jump")
```

## 15. 协程与异步编程

协程本质上是函数或子程序的泛化。一个函数只有一个入口点，一旦根据其逻辑执行完函数语句，它就会将控制权一次性地返回给调用者。相比之下，协程首先被进入，它们可以被多次暂停和恢复，然后才会最终终止或关闭。

大多数现代编程语言都支持某种形式的协程。在过去几年里，Python 中出现了几种不同的协程实现形式。在 Python 中，协程最初是作为生成器的泛化来设计和实现的。现在，应该使用的是基于 `async def` 的新式协程（以及任务和未来对象）。基于生成器的协程语法在此时已被弃用（尽管其内部实现仍然基于生成器）。

### 15.1. 生成器

如前所述，Python 包含许多内置类型。Python 在标准模块（如 `collections.abc`）中也定义了不少“鸭子类型”，主要用于为原本动态类型的 Python 编程语言添加一些（临时的）结构。我们在本参考手册中已经看到了一些例子，包括可迭代对象和可调用对象等。

> `typing` 模块是另一个持续努力为 Python 动态类型系统添加更多“结构”的例子。类型提示可用于在 Python 软件开发过程中辅助静态类型检查。

#### 15.1.1. 迭代器

像列表、元组和字典这样的集合对象，例如，可以与 `for` 循环一起使用，它们是“可迭代的”。实现了魔术方法 `__iter__` 的类型是可迭代的。可迭代类型的对象是可迭代的，正如我们在本参考手册中所见。

当使用一个可迭代对象（具有 `__iter__` 方法）调用内置函数 `iter()` 时，它返回一个迭代器类型的对象。迭代器实现了 `__iter__` 和 `__next__` 方法。因此，迭代器也是可迭代的。（在内部，可迭代对象和该对象的迭代器对象通常共享相同的数据流。）

使用迭代器对象（调用该对象的 `__next__` 方法）重复调用内置的 `next()` 函数，会返回对象中的连续项。当没有更多项可用时，例如，当到达数据流末尾时，会引发 `StopIteration` 异常。

这是一个简单的迭代器类：

```
class ABC:
    def __init__(self):
        self.pointer = 0
        self.list = ('A', 'B', 'C')

    def __iter__(self):
        return self

    def __next__(self):
        if self.pointer >= len(self.list):
            raise StopIteration

        val = self.list[self.pointer]
        self.pointer += 1
        return val
```

它定义了一个 `__iter__` 方法。因此，它也是一个可迭代的。

类型 ABC 实现了 `__iter__` 和 `__next__` 方法，因此它是一个迭代器。

这个类的对象可以在任何需要迭代器或可迭代对象的地方使用。例如，

```
>>> abc = ABC()
>>> for i in abc:
...     print(i)
...
A
B
C
```

`abc` 是一个具有 `__iter__` 方法的可迭代对象，该方法返回一个迭代器（在本例中是同一个对象）。

`for` 语句使用迭代器的 `__next__` 方法来遍历 `abc` 中的数据。这是许多现代编程语言（不仅仅是 Python）使用的一种非常通用的模式（例如，如“设计模式”中所示）。一旦 `for` 语句捕获到 `StopIteration`，它就会退出循环。（这隐藏在 `for` 语句的实现中。）

#### 15.1.2. 生成器函数

生成器函数（同步或异步）和协程彼此非常相似。它们可以有多个入口点，可以多次产出，并且它们的执行可以被挂起和恢复。

- 当调用一个*生成器函数*时，它返回一个迭代器类型的*生成器对象*，该对象随后控制生成器函数的执行。
- 当调用一个异步生成器函数时，它返回一个异步迭代器，称为*异步生成器对象*，该对象随后控制异步生成器函数的执行。

### 15.2. yield 表达式

在定义*生成器函数*或*异步生成器函数*时使用 yield 表达式。

> Python 还有一个 yield 语句，它在语义上等同于（较新的）yield 表达式。yield 简单语句和 yield 表达式语句之间几乎没有区别。

在函数体中包含 yield 表达式/语句的函数，*根据定义*，是一个生成器函数。同样，包含 yield 表达式/语句的 `async def` 函数是一个异步生成器函数。

例如，

```
>>> def a():
...     yield 1
...     yield 2
...
>>> for i in a():
...     print(i)
...
1
2
```

`a`（自动）是一个生成器函数，因为它包含 yield 表达式（或语句）。

yield 表达式/语句的行为类似于 return 语句。但它不会终止函数执行。相反，它暂时将控制权让给调用者，当再次被调用时，它会从*上次执行的* yield 之后恢复执行。请注意，`a` 函数无需显式提供，例如，`__iter__` 方法的实现。

这是生成器函数 `a` 中的最后一个 yield 语句，并且由于此语句之后没有更多语句，`a` 或其生成器对象在再次恢复时将抛出 `StopIteration` 异常。

调用时，`a()` 返回一个*生成器对象*，它是一个迭代器。请注意，返回的生成器对象隐式地实现了 `__iter__` 和 `__next__` 方法。换句话说，生成器函数隐藏了必须直接实现迭代器的复杂性（例如，正如我们在上一节中所演示的）。

这是一个 `async` 生成器函数的示例：

```
>>> async def b():
...     yield 'a'
...     yield 'b'
...
>>> async def c():
...     async for i in b():
...         print(i)
...
>>> import asyncio
>>> asyncio.run(c())
a
b
```

Python 将 `b` 创建为一个异步生成器函数（因为它是一个包含 yield 表达式/语句的 `async def` 函数）。

正如我们将在本章后面讨论的，`async` 语句，如函数体第一行中的 `async for`，只能在异步函数中使用。我们将 `c` 声明为一个异步函数。

调用 `b()` 返回一个异步生成器对象，该对象可以在 `async for` 循环中使用。

尽管 `async` 和 `await` 是 Python 关键字，但大部分异步编程支持都包含在标准库 `asyncio` 模块中。

异步函数可以使用 `asyncio.run` 函数运行。

### 15.3. 生成器表达式

生成器表达式与*推导式*非常相似，不仅在语法上，而且在精神上。生成器*表达式*使用推导式语法产出一个新的生成器对象，只是它使用圆括号。例如，

```
>>> double = (
...     x * 2
...     for x in range(3)
... )
>>> for x in double:
...     print(x)
...
0
2
4
```

#### 15.3.1. 异步生成器表达式

生成器表达式被括号包围，因此可以写在多行上。该表达式被求值并赋值给一个变量 `double`，以作说明。更常见的是在需要的地方定义和使用生成器表达式，例如 [Lambda 函数](https://docs.python.org/3/reference/expressions.html#lambda)。

- “下一个值”表达式，类似于推导式语法。
- `for` 子句。
- 生成器表达式返回一个生成器对象（在本例中是绑定到 `double` 的对象），它是一个迭代器。

如果生成器表达式包含 `async for` 子句或 `await` 表达式，则它返回一个新的异步生成器对象，该对象可以被异步迭代。

```
>>> async def a():
...     yield 0; yield 1; yield 2
...
>>> adouble = (
...     x * 2
...     async for x in a()
... )
>>> async def c():
...     async for x in adouble:
...         print(x)
...
>>> import asyncio
>>> asyncio.run(c())
0
2
4
```

- 一个异步生成器函数。
- 异步生成器表达式也被括号包围。这是一个异步生成器表达式，因为它包含一个 `async for` 子句。
- “下一个值”表达式，类似于推导式语法。
- `async for` 子句。注意 `a()` 返回一个异步生成器对象。
- 我们在这里定义了一个异步函数，以作说明，因为 `async` 语句只能包含在异步函数中。
- `adouble` 引用一个异步生成器对象，它是一个异步迭代器。同样，异步生成器表达式的结果不必赋值给一个单独的变量。我们本可以直接在这里“就地”使用它。
- 我们可以借助 `asyncio.run` 函数来运行 `c()` 函数。

迭代器类型实现了 `__iter__` 和 `__next__` 方法。同样，异步迭代器类型实现了 `__aiter__` 和 `__anext__` 方法。

当它们耗尽元素时，迭代器会引发 `StopIteration` 异常，而异步迭代器会引发 `StopAsyncIteration` 异常。这些细节都隐藏在（同步或异步）生成器函数和生成器表达式的实现中。

### 15.4. 协程对象

#### 15.4.1. 可等待对象

实现了特殊 `__await__` 方法的类型是可等待的。[await 表达式](https://docs.python.org/3/reference/expressions.html#await)可以与可等待对象一起使用。

```
object.__await__(self)
```

它返回一个迭代器。用于定义一个可等待类型。例如，`asyncio.Future` 实现了此方法，因此 `Future` 对象可以用作 `await` 表达式的操作数。

#### 15.4.2. 协程对象

协程对象是可等待对象。例如，`async def` 函数返回协程对象。

协程的工作方式与可迭代对象类似。调用协程的 `__await__` 方法返回一个迭代器，协程通过迭代该迭代器对象来执行。当协程执行完毕并返回时，迭代器会引发 `StopIteration` 异常。

然而，与可以多次迭代的可迭代对象不同，协程对象不能被 `await` 多次。尝试这样做会引发 `RuntimeError` 异常。

此外，协程对象不直接使用迭代器方法来支持迭代。相反，它们包含以下方法：

```
coroutine.send(value)
    它启动或恢复协程的执行。如果参数是 None，则等同于推进 __await__() 返回的迭代器。如果 value 不是 None，则此方法
```

委托给迭代器的 `generator.send` 方法。

```
coroutine.throw(type, value, traceback)
```

它在协程的挂起点引发指定类型的异常。如果迭代器有 `generator.throw` 方法，则委托给此方法。如果异常在协程中未被捕获，则会传播回调用者。

```
coroutine.close()
```

它导致协程退出，如果有的话，在运行清理代码之后。如果迭代器有 `generator.close` 方法，则可选择先委托给该方法。然后在挂起点引发 `GeneratorExit` 异常，导致协程立即清理自身。最后，它被标记为执行完毕，结束协程的生命周期。

标准库 `asyncio` 模块提供了许多便利的类型和方法，例如 `Task` 和 `Future`，用于管理协程。

### 15.5. 协程函数

`async def` 语句定义协程函数的方式与 `def` 语句定义（普通）函数的方式类似。使用 `async def` 定义的协程*可以*，但不是必须，包含 [await 表达式](https://docs.python.org/3/reference/expressions.html#await) 和 [async 语句](https://docs.python.org/3/reference/simple_stmts.html#async)，例如 `async for` 和 `async with`。

调用协程函数对象返回一个可等待对象，更具体地说，是一个[协程对象](https://docs.python.org/3/glossary.html#term-coroutine-object)。协程的执行可以被挂起和恢复，如[上一节](https://docs.python.org/3/reference/compound_stmts.html#async-def)所述。

```
>>> async def a():
...     print("a")
...
>>> async def b():
...     await a()
...
>>> import asyncio
>>> asyncio.run(b())
a
```

- 一个不包含任何 `async`/`await` 表达式/语句的协程函数定义。
- 另一个包含 `await` 表达式的协程函数定义。
- 调用 `a()` 将返回一个协程对象，该对象可以被 `await`。
- 一个协程对象，由调用 `b()` 返回，可以使用 `asyncio.run` 函数运行。调用一个（普通）函数会执行函数对象中定义的语句。相反，调用协程函数对象仅仅返回[一个协程对象](https://docs.python.org/3/glossary.html#term-coroutine-object)。

### 15.6. Await 表达式

`await` 表达式在*可等待*对象上挂起协程的执行。它只能在[协程函数](https://docs.python.org/3/glossary.html#term-coroutine-function)内部使用，例如 `async def` 函数。

例如，

```
>>> import asyncio
>>> async def a():
...     print("a() called")
...     await asyncio.sleep(5.0)
...     print("Leaving a() after sleeping 5 seconds")
...
>>> asyncio.run(a())
a() called
Leaving a() after sleeping 5 seconds
```

大部分 async/await 运行时支持都包含在 `asyncio` 模块中。

`async def`/协程函数被调用时，返回一个[协程对象](https://docs.python.org/3/glossary.html#term-coroutine-object)（或简称协程）。

`asyncio.sleep` 函数返回一个协程，该协程在给定时间（以秒为单位）后完成。`await` 表达式可以在协程函数中与[可等待对象](https://docs.python.org/3/glossary.html#term-awaitable)一起使用。

协程可以使用 `asyncio.run` 运行。

### 15.7. 其他 async 语句

#### 15.7.1. async for 语句

“异步可迭代对象”可以在其 `__aiter__` 实现中调用异步代码，“异步迭代器”可以在其 `__anext__` 方法中调用异步代码。`async for` 语句允许方便地迭代异步迭代器。

`async for in else` 复合语句的语法与 [for in else 语句](https://docs.python.org/3/reference/compound_stmts.html#the-for-statement)大致相同。但是，它迭代的是一个 `async` 迭代器，而不是同步迭代器。

这是一个例子：

```
>>> async def words():
...     yield "hello"
...     yield "parallel"
...     yield "universe"
...
>>> async def greet():
...     async for word in words():
...         print(word, end=' ')
...     else:
...         print()
...
>>> import asyncio
>>> asyncio.run(greet())
hello parallel universe
>>>
```

- 一个带有 `yield` 语句的 `async def` 函数（隐式地）返回一个异步迭代器对象（更具体地说，是一个协程）。
- `async` 语句只能包含在协程函数中。
- `async for` 子句。`words()` 的类型是一个异步迭代器。
- 一个可选的 `else` 子句。当 `async for` 在正常迭代异步迭代器后退出时，执行此子句。
- 尾随换行符是从 `greet` 函数体内的 `async for` 语句的 `else` 子句打印的。

#### 15.7.2. `async with` 语句

异步上下文管理器是一种能够在 `enter` 和 `exit` 方法中暂停执行的上下文管理器。`async with` 复合语句的工作方式与 [with 语句](https://docs.python.org/3/reference/compound_stmts.html#the-with-statement)非常相似。但是，它不使用上下文管理器，而是使用异步上下文管理器，该管理器支持 `__aenter__` 和 `__aexit__` 方法（分别对应上下文管理器的 `__enter__` 和 `__exit__`）。

以下是语法：

```
async with Expression as Target:
    Suite
```

- `Expression` 必须求值为异步上下文管理器类型。
- `as` 子句是可选的，与 `with` 语句一样。
- `Suite` 可以包含 `await` 表达式和其他异步语句。

### 15.8. 生产者消费者问题

作为最后一个示例和练习，这里是一个使用 Python 协程实现的经典（单）生产者 - （单）消费者问题的简单实现。（更多信息，请参阅 [维基百科文档](https://en.wikipedia.org/wiki/Producer%E2%80%93consumer_problem)。）

此代码示例按原样提供，*没有任何*注释或说明。鼓励读者通读代码，或尝试自己实现一个类似的异步程序。

该程序由三个文件组成：*pc* 子文件夹下的 *producer.py* 和 *consumer.py*，以及项目文件夹中的 *main.py*。你可以尝试为生产者和消费者模块中定义的四个常量设置不同的值，看看它们如何影响生产者和消费者之间的协调。（注意：此示例程序使用了 *asyncio* 模块中的一些 API，本书未明确涵盖这些 API。）

*清单 3. pc/producer.py*

```
import asyncio

_PRODUCE_ITEMS = 3
_PRODUCE_IVAL = 0.5

async def produce(queue: asyncio.Queue,
                 items: int = _PRODUCE_ITEMS,
                 interval: float = _PRODUCE_IVAL):
    for i in range(items):
        print("Producing:", i)
        await asyncio.sleep(interval)
        await queue.put(i)
        print("Produced:", i)

    print("Producer done!")
```

*清单 4. pc/consumer.py*

```
import asyncio

_CONSUME_ITEMS = 5
_CONSUME_IVAL = 1.0

async def consume(queue: asyncio.Queue,
                  items: int = _CONSUME_ITEMS,
                  interval: float = _CONSUME_IVAL):
    for _ in range(items):
        item = await queue.get()
        print("Got item:", item)
        await asyncio.sleep(interval)
        queue.task_done()
        print("Consumed", item)

    print("Consumer done!")
```

*清单 5. main.py*

```
import asyncio
from pc.consumer import consume
from pc.producer import produce

async def _main():
    try:
        queue = asyncio.Queue()
        p = asyncio.create_task(produce(queue))
        c = asyncio.create_task(consume(queue))

        await asyncio.gather(p)
        await queue.join()
        c.cancel()
        await asyncio.gather(c)
    except asyncio.CancelledError:
        return
    except RuntimeError as err:
        print(err)

if __name__ == "__main__":
    asyncio.run(_main())
```
